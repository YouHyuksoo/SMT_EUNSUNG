# CONTEXT — 은성 MES 도메인 용어집 (ubiquitous language)

이 문서는 은성 MES(`eunsung-mes`)의 **공용 언어**다. 사람과 코드가 같은 뜻으로 쓰는
용어·경계·불변식을 여기 적는다. **결정의 근거(왜 이렇게 정했나)는 여기가 아니라
`docs/adr/`에 남긴다.**

- 살아있는 문서다 — 용어가 확정되는 순간 `/grill-with-docs`·`/wayfinder`가 갱신한다.
- 아래 용어집은 **이미 확립·문서화된 것만** 담은 seed다. 도메인 엔티티의 정밀한 의미는
  아직 비어 있고, grilling 세션에서 하나씩 확정해 채운다 (추측으로 채우지 않는다).

## 시스템 개요 (검증된 사실)

- **Site**: 은성전장 (EUNSUNG).
- **Shape**: pnpm + turborepo 모노레포 — NestJS 백엔드(`@eunsung/backend`, :3003) +
  Next.js 프론트엔드(`@eunsung/frontend`, :3100) + 공유 패키지(`@smt/shared`).
- **DB**: Oracle. 백엔드는 TypeORM으로 접속(스키마 `INFINITY21_JSMES`, 서비스 `XE`).
- **언어(UI)**: 한국어/영어/스페인어/베트남어.
- **도메인**: 생산 · 품질 · 재고 · 자재 · 설비 · OEE · 출하 · 외주 · 소모품 ·
  메뉴/권한 · 인터페이스 · AI.

## 용어집 — 확립된 것

| 용어 | 정의 | 근거 |
|---|---|---|
| **은성전장 / EUNSUNG** | 이 MES가 운영되는 사이트. 모노레포 이름 `eunsung-mes`. | CLAUDE.md |
| **Legacy Display** | 프론트엔드 안의 3D 메뉴 + `/display/[screenId]` 모니터링 화면 서브시스템. 백엔드 API가 아니라 Next API Route + `apps/frontend/src/lib/db.ts`로 Oracle에 직접 접속한다. | CLAUDE.md, screens.ts |
| **display screen** | `apps/frontend/src/lib/screens.ts`의 `SCREENS`에 등록된 모니터링 화면(예: `21` 제품생산현황, `42` FCT). | screens.ts |
| **업무일 (business day)** | 캘린더 자정이 아니라 **07:30 ~ 다음날 07:30**. FCT(`/display/42`) 집계 기준: `TRUNC(SYSDATE - 7.5/24) + 7.5/24`. | CLAUDE.md (FCT screen rules) |
| **MENU_CODE** | 화면/기능 단위 식별 코드. `docs/business-logics/<MENU_CODE>.md` 문서 단위이자 화면 로직 분석의 기준. | docs/README.md |
| **ESDB** | 은성 내부 Oracle 접속(내부 IP). | 접속 프로파일 |
| **ESDBEXT** | 은성 외부 Oracle 접속(외부 IP, `oracle-db` 스킬 프로파일 `ESDBext`). 내부와 포트·계정·서비스(XE) 동일. | 접속 프로파일 |
| **품목구분** | 품목을 **원자재·반제품·제품** 중 하나로 구분하는 분류축. 조달·제작 유형이나 세트 구성 여부와는 다른 개념이다. | 업무 합의, ID_ITEM 실데이터 |
| **제조유형** | 품목이 자작인지, 무상 또는 유상으로 조달되는지를 구분하는 분류축. 품목구분이나 상세 구입경로와는 다른 개념이다. | 업무 합의, ID_ITEM 실데이터, PKG_PLANNING |
| **구입유형** | 품목의 국내구매·무상구매·사급·내부거래·OEM·자작 등 상세 구입 또는 공급 경로를 나타내는 분류축. 제조유형과는 구분한다. | 업무 합의, ID_ITEM 실데이터 |
| **품목분류** | 품목을 PCB·저항·커패시터·IC·ASSY 등 기술적 종류로 세분화하는 분류축. 원자재·반제품·제품을 나누는 품목구분과는 다른 개념이다. | 업무 합의, ID_ITEM 실데이터 |
| **세트구성여부** | 품목이 세트를 구성하는 품목인지 단품인지를 구분한다. `Y`는 세트 구성 품목, `N`은 단품이다. | 업무 합의, ID_ITEM 실데이터, 세트 조회 함수 |
| **품목 마스터** | MES가 단독으로 소유·관리하는 품목 기준정보. ERP 수신본이 아니며 품목 등록·변경의 기준이다. BOM 등에 편의상 복제된 품목 속성보다 우선한다. | 업무 합의, ID_ITEM 및 트리거 |
| **품목별 공급처 마스터** | 품목과 공급처의 조합별 정보를 관리한다. 현행 MES에서는 공급처별 IQC 검사 대상 여부를 구분하는 데 주로 사용하며, 이 MES는 발주를 내는 시스템이 아니다. | 업무 합의, IM_ITEM_MASTER 및 ID_ITEM 트리거 |
| **미지정 공급처** | 신규 품목 등록 시 실제 공급처를 아직 모를 때 사용하는 임시 공급처 값 `*`. 입력 직후 공식 공급처 코드로 정리하는 것이 원칙이지만 시스템이 강제하지는 않는다. | 업무 합의, TRG_ID_ITEM_INS |
| **공급처 마스터** | MES에서 사용하는 공식 공급처 기준정보. 현재 운영 데이터는 협력사 유형이며 품목별 공급처 마스터가 이 코드를 참조한다. | 업무 합의, ICOM_SUPPLIER 실데이터 |
| **IQC 검사 실적** | 입하 품목의 공급처·검사수량·불량수량·검사방법·검사결과를 기록하는 정보. 현행 `IQ_ITEM_IQC` 데이터 흐름은 중단된 휴면 상태다. | 업무 합의, IQ_ITEM_IQC 및 입하 트리거 |
| **생산 BOM** | MES가 단독으로 소유·관리하며 엑셀 업로드로 등록하는 생산·자재 소요용 품목 구성정보. 구성 수량·투입 공정·라인·로스율·스크랩율·하위 반제품 전개 여부를 관리한다. | 업무 합의, ID_ENG_BOM 및 생산/자재 서비스 |
| **상위품목 / 하위품목** | 생산 BOM에서 만들어지는 쪽을 상위품목, 그 생산에 투입되는 구성 품목을 하위품목이라 한다. 모품목·자품목, 부모·자식은 같은 관계를 가리키는 기존 동의어다. | 업무 합의, ID_ENG_BOM |
| **BOM 적용기간** | 생산 BOM 구성관계가 유효한 시작일과 종료일. 동일 구성관계의 과거·현재·미래 버전이 적용기간을 키로 동시에 존재한다. | 업무 합의, ID_ENG_BOM |
| **BOM 리비전** | 생산 BOM에 참고로 기록하는 리비전 정보. 현재는 사용하지 않으며 BOM의 실제 버전은 적용기간으로 관리한다. | 업무 합의, ID_ENG_BOM 실데이터 |
| **하위 BOM 전개여부** | 생산 BOM 전개 시 하위 반제품의 BOM을 계속 재귀적으로 펼칠지 결정한다. `Y`는 계속 전개하고 `N`은 해당 하위품목에서 중지한다. | 업무 합의, ID_ENG_BOM 및 PKG_DESIGN |
| **BOM 투입공정** | 생산 BOM에서 하위품목이 투입되는 공정을 참고로 기록하는 정보. 현재는 사용하지 않으며 `*`만 저장되어 있다. | 업무 합의, ID_ENG_BOM 실데이터 |
| **생산라인 마스터** | 생산라인의 코드·명칭·구분·생산능력 등 기준정보를 관리한다. 현행 테이블에는 현재 운영상태·장착상태·최근 이력도 함께 저장되어 있다. | 업무 합의, IP_PRODUCT_LINE 실데이터 |
| **라인운영상태** | 생산라인이 정상·고장·수리·정지·모델변경·자재교환·샘플생산 등 현재 어떤 운영 상태인지 나타낸다. | 업무 합의, IP_PRODUCT_LINE.LINE_STATUS |
| **생산활성여부** | 생산라인이 현재 생산 활성 상태인지 대기 상태인지 나타낸다. 라인 마스터의 사용·미사용 여부가 아니다. | 업무 합의, IP_PRODUCT_LINE.ACTIVE_YN |
| **라인 현재생산정보** | 생산라인에 현재 투입된 모델·품목·RUN을 나타내는 상태 정보. 생산계획이나 실적 원장이 아니라 라인 화면·인터록을 위한 현재 상태다. | 업무 합의, IP_PRODUCT_LINE 실데이터 |
| **설비 마스터** | 생산에 사용하는 개별 설비의 코드·명칭·유형·모델·능력·사용상태를 관리한다. 설비는 생산라인에 소속되며 현행 테이블에는 통신설정과 현재상태도 함께 저장된다. | 업무 합의, IMCN_MACHINE 실데이터 |
| **공정 마스터** | 제품이 거치는 작업 단계의 코드·명칭·순서·유형·표준시간·능력을 관리한다. 생산라인은 물리적 생산 구역, 설비는 개별 장비, 공정은 작업 단계다. | 업무 합의, IP_PRODUCT_WORKSTAGE 실데이터 |
| **생산 라우팅** | 현재 유효한 생산 BOM을 가진 반제품 또는 완제품의 고정 공정 순서와 공정별 하위품목 배정을 묶은 생산 기준정보다. | 업무 합의, IP_ROUTING_* 및 라우팅 서비스 |
| **라우팅 그룹** | 반제품 또는 완제품 품목 하나에 고정된 단일 생산 라우팅으로, 라우팅 코드는 품목코드와 같다. | 업무 합의, IP_ROUTING_GROUPS |
