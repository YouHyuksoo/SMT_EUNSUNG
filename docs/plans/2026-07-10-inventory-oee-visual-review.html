01 · AS-IS · TO-BE · GAP

현황 분석 — AS-IS / TO-BE / GAP

본 문서는 분석보고서 구조를 따릅니다. 현재 운영 상태(AS-IS), 개발 완료 후의 운영 형태(TO-BE), 그 사이의 격차(GAP)를 먼저 정리하고, 이후 절은 GAP 해소에 필요한 설계 내용을 항목 순서대로 다룹니다.

01
AS-IS현재 — 파워빌더 MES 부분 운영원자재 입출고·LOT 분할까지만 원장 존재. 공정 이후 재고 추적 불가, 재고금액 산출 불가, 자재 소비 미기록, OEE 지표 없음.
→
이번 개발GAP 항목별 해소용어 표준 → 수불 테이블 신설 → 수불 흐름 정의 → 재고평가 → 백플러시 → 실사·마감 → 기준정보 → OEE → 화면 개발 순서로 진행.
→
TO-BE개발 완료 후 운영 형태6개 재고구분 수불 원장 운영, 월마감 시 재고금액 산출, 실적 등록 시 자재 소비 자동 기록, OEE·직행율 집계.

AS-IS — 현재 운영의 4가지 문제점

현재 공백 1

공정 이후 재고 원장이 없다

원자재창고 출고 이후 자재가 어느 공정창고에 얼마나 있는지, 반제품·완제품이 창고 입고 전 어디에 머무는지 장부로 추적할 수 없습니다.

결과: 실물은 현장에 있는데 시스템 재고는 0이거나, 반대로 소비된 자재가 장부에 남습니다.

현재 공백 2

재고금액을 계산할 수 없다

구매단가(IM_ITEM_UNIT_PRICE)와 판가(IS_PRODUCT_SALE_PRICE) 테이블은 있으나 재고평가에 연결돼 있지 않습니다.

결과: 월말에 "재고가 얼마어치인가"라는 질문에 시스템이 답하지 못합니다.

현재 공백 3

백플러시가 없다

생산실적을 등록해도 BOM/라우팅 기준 자재 소비(출고)가 자동으로 일어나지 않습니다.

결과: 공정창고 재고가 실제 소비와 무관하게 쌓이고, 수불부의 출고 열이 비게 됩니다.

현재 공백 4

OEE를 측정할 수 없다

무작업/비가공 사유, CT 기준 성능, 직행율이 어디에도 기록되지 않아 설비·라인이 왜 멈췄는지, 얼마나 느리게 돌았는지 데이터로 답할 수 없습니다.

결과: 손실 개선이 데이터가 아니라 감(感)에 의존하고, 라인 간·기간 간 비교가 불가능합니다.

WHY

왜 수불을 먼저 잡고 OEE를 그 위에 올리는가?

OEE의 분자·분모(생산수량, 가동시간, 양품수량)는 결국 생산실적과 자재 흐름에서 나옵니다. 수불 원장이 신뢰되지 않으면 OEE 수치도 신뢰될 수 없습니다. 그래서 이번 범위는 수불 기반 안정화 → 마감으로 데이터 잠금 → OEE 집계 순서로 쌓습니다. 같은 이유로 ERP 인터페이스는 수불/마감 기준이 안정된 뒤 후속 과제로 미룹니다 — 흔들리는 원장을 ERP로 보내면 정합성 문제가 양쪽으로 번지기 때문입니다.

TO-BE — 개발 완료 후 운영 형태

재고 원장 운영

6개 재고 구분 전체에 수불 원장이 생성되고, 바코드/UID 단위로 보유 위치·수량·거래 이력을 조회할 수 있습니다.

재고금액 산출

월마감 시 총평균법으로 원자재 재고금액·출고금액이 산출되고, 마감 스냅샷으로 확정됩니다.

자재 소비 자동 기록

공정실적 등록 시 생성유형(BOM/라우팅) 기준 백플러시로 소비가 원장에 기록됩니다. 기준정보 미등록 모델은 보류 처리되어 잘못된 소비가 발생하지 않습니다.

OEE 지표 운영

OEE(가동율×성능율×양품율)와 직행율이 공정/라인/셀/설비 단위로 집계되고, 무작업 사유별 손실 시간을 확인할 수 있습니다.

재고 구분 용어 사전 — 6종 표준

수불 논의에서 가장 먼저 사고가 나는 곳이 용어입니다. 실제 협의 중에도 "완제품재공", "완제품대기재고" 같은 표현이 혼용되다가 "공정창고 완제품재고"로 통일하자는 결론이 났습니다. 아래 6종이 문서·화면·코드 전체에서 쓰는 유일한 표준입니다. 규칙은 단순합니다: 용어 = 보관 장소(창고) + 물건 종류(재고).

06 · Inventory Valuation

재고평가 — 총평균법 기본, 이동평균법 대안

평가 대상은 원자재로 제한합니다(원자재창고 재고금액 · 공정창고 원자재 재고금액 · 원자재창고 출고금액 · 백플러시 출고금액). 고객이 "원가관리는 필요 없고 재료비만"이라고 확인했기 때문에 반제품/완제품 원가, 재료비 전이, 노무비·간접비·설비비는 전부 제외합니다.

06

총평균법 기본 채택

월평균단가 (월마감 시 1회 산출)
월평균단가 = (기초재고금액 + 당월입고금액)
    ÷ (기초재고수량 + 당월입고수량)
해당 월의 재고금액·출고금액 평가에 동일하게 적용
  • 장점: 월 1회 계산으로 검증이 쉽고, 마감 스냅샷과 자연스럽게 맞물림. 월중 입고 순서·시점 오류에 둔감.
  • 단점: 월중에는 확정 단가를 알 수 없음 (마감 후 확정).
  • 어울리는 조건: 월 단위 마감 체계를 이번에 새로 세우는 은성 상황.

이동평균법 대안

이동평균단가 (입고 때마다 재계산)
새 단가 = (직전 재고금액 + 입고금액)
    ÷ (직전 재고수량 + 입고수량)
출고 시점마다 그 시점의 단가로 출고금액 계산
  • 장점: 실시간으로 재고금액을 알 수 있음.
  • 단점: 입고 전표의 순서·시점·수정에 민감 — 소급 수정 하나가 이후 모든 단가를 다시 흔듦. 검증 난이도 높음.
  • 결론: 수불 체계가 안정된 뒤 필요 시 전환을 검토하는 대안으로만 유지.
EX

계산 사례 — 메탈 PCB 원판, 2026년 7월. 기초재고 1,000장 × 1,200원 = 1,200,000원. 당월 입고 ① 7/3 2,000장 × 1,260원 = 2,520,000원, ② 7/18 1,000장 × 1,320원 = 1,320,000원. 당월 출고 2,500장.

총평균법: 월평균단가 = (1,200,000 + 2,520,000 + 1,320,000) ÷ (1,000 + 2,000 + 1,000) = 5,040,000 ÷ 4,000 = 1,260원. → 출고금액 2,500 × 1,260 = 3,150,000원, 월말 재고금액 1,500 × 1,260 = 1,890,000원. 언제 출고했는지와 무관하게 동일.

이동평균법이라면: 7/3 입고 후 단가 1,240원 → 7/10 출고분은 1,240원, 7/18 입고 후 단가는 다시 재계산 — 출고가 입고 앞뒤 어디에 있느냐에 따라 금액이 달라지고, 7/3 전표를 나중에 수정하면 이후 모든 단가가 재계산 대상이 됩니다.

금지

품목별 평가법 혼용 금지. 같은 기간에 A품목은 총평균, B품목은 이동평균을 쓰면 두 품목의 재고금액을 더한 합계가 회계적으로 아무 의미가 없어집니다. 평가법은 기간 단위로 하나만 적용합니다. 판매단가(IS_PRODUCT_SALE_PRICE)는 재고평가에 사용하지 않고 판매/분석용으로만 분리합니다.

실사와 마감의 공통 원칙은 오류를 차단하고 끝내는 것이 아니라 해소까지 이어지게 하는 것입니다 (고객 요구사항: "일은 항상 마무리가 되어야 하며 제한하고 끝나면 안 된다"). 실사는 현재고를 덮어쓰지 않고 조정 거래를 생성하며, 마감은 오류 발견 시 오류 목록 → 담당자 조치 → 재실행 절차를 거쳐 마감 완료까지 진행합니다.

09

재고실사 — 4개 창고 영역

원자재창고 · 공정창고 · 반제품창고 · 완제품창고 실사를 각각 제공합니다.

1. 실사수량 입력

완제품은 포장/박스/매거진 스캔 — 시스템 반영은 PACK_SERIAL의 바코드/UID 낱개 단위.

2. 차이 계산

장부수량 − 실사수량. 현재고를 직접 덮어쓰는 기능은 존재하지 않습니다.

3. 조정 거래 생성

차이가 양수면 ADJ_IN, 음수면 ADJ_OUT. 모든 차이가 수불 원장에 흔적을 남깁니다.

4. 실사 확정

확정 후 수정 제한. 필요 시 실사취소/재실사 이력으로만 관리.

WHY

왜 덮어쓰기 금지인가? 현재고를 직접 수정하면 "왜 줄었는가"가 시스템 어디에도 남지 않습니다. 조정 거래로 반영하면 수불부의 조정 열에 나타나고, 반복되는 조정 패턴(특정 품목·특정 창고)이 그대로 개선 대상 데이터가 됩니다.

창고마감 — 순서와 루프

마감 순서: 원자재창고 → 공정창고 → 반제품창고 → 완제품창고. 상류 원장이 확정돼야 하류 기초가 성립하기 때문입니다.

1. 마감 실행 → 사전 검증

검증 규칙 전체를 자동 수행.

2. 오류 목록 자동 생성

무엇이, 어느 전표에서, 왜 — 담당자가 바로 조치 가능한 수준으로.

3. 담당자 조치 → 완료 확인

조치 후 재실행. 통과할 때까지 루프.

4. 마감 스냅샷 생성

해당 월 거래 수정 제한. 다음 달 수불부의 "기초"가 됨. 필요 시 마감취소/재마감 이력 관리.

EX

사례. 7월 원자재창고 마감 실행 → 오류 3건: 미확정 입고 1건(7/29 전표), 단가 미등록 1품목, 음수재고 1건. 구매담당이 전표 확정, 단가담당이 단가 등록, 창고담당이 음수 원인(이중 출고)을 조정 처리 → 재실행 → 통과 → 스냅샷 생성. 오류가 "마감 불가"라는 메시지로 끝나지 않고 조치 가능한 작업 목록으로 변환되는 것이 핵심입니다.

운영

실사 없이 마감만 하는 달 — 정상 운영 시나리오입니다. 마감의 전제 조건은 실사가 아니라 원장 정합성 검증입니다. 실사를 못 한 달에도 마감 오류(미확정 전표, 음수재고, 단가 미등록 등)만 해소되면 마감 스냅샷이 생성되고, 차월 기초재고는 장부(원장) 기준으로 이월되어 운영은 그대로 계속됩니다.

· 실사를 한 달: 조정 거래(ADJ_IN/ADJ_OUT)까지 반영된 기초로 이월 — 장부와 실물이 일치한 상태에서 새 달 시작.
· 실사를 건너뛴 달: 장부 기준 이월 — 마감·수불부·재고평가는 정상 동작하지만, 장부-실물 차이는 다음 실사까지 누적된 채 이월됩니다.

리스크 관리: 실사 공백이 길어질수록 차이 누적 위험이 커지므로, 재고 연령 분석(16절 ⑥)과 이동 중 재고 경보(16절 ①)를 실사 없는 달의 보조 신호로 활용하고, 실사 주기(매월 / 분기 순환실사 / 연 1회 전수)는 창고별로 협의해 확정합니다(19절).

12 · Time Concepts

시간 개념 사전 — CT · ST · Takt · Neck · UPH

협의 과정에서 실제로 혼동이 있었던 부분입니다. "공정에서 가장 시간이 많이 걸리는 곳"을 Takt Time으로 알고 계셨지만 그것은 Neck(병목) Time이고, Takt Time은 고객 요구를 맞추기 위한 목표 간격입니다. 또 "왜 ST가 아니라 CT로 성능율을 계산하는가"라는 질문에 대한 답도 여기에 있습니다.

12

CT — Cycle Time 성능율의 기준

1개를 생산하는 데 필요한 기준 시간(실제 또는 표준 사이클 타임).

사례: SMT-01에서 모델 X의 CT가 12초라면, 1시간 정상 가동 시 이론상 300개가 나와야 합니다. 240개가 나왔다면 성능율 80%.

ST — Standard Time 계획의 기준

표준작업시간. 작업 표준·공수·생산계획 수립에 쓰는 시간. 준비·부수 작업이 포함된 계획용 개념입니다.

사례: 수납땜 셀의 ST가 45초/개라면, 500개 지시에는 약 6.25시간의 공수를 계획합니다.

Takt Time 수요의 속도

가용시간 ÷ 필요수량. 고객 요구수량을 맞추기 위한 목표 생산 간격.

사례: 하루 가용 460분에 고객 요구 920개면 Takt = 30초. CT가 34초인 공정은 수요를 못 따라가는 공정으로 즉시 식별됩니다.

Neck Time 병목

라우팅 중 가장 시간이 긴 공정의 시간. 라인 전체 산출량은 이 공정이 결정합니다.

협의 정정: "가장 오래 걸리는 공정 시간"은 Takt가 아니라 Neck(Bottleneck) Time입니다.

UPH / UPD 사후 분석

시간당/일당 생산량. 결과를 되돌아보는 지표이며, 성능율의 직접 입력값으로 쓰지 않습니다.

협의 중 "UPH는 시간이 지난 후 분석하는 것 아니냐"는 질문 그대로 — 맞습니다. 사후 지표로만 사용합니다.

왜 성능율은 ST가 아니라 CT인가

성능율은 "실제 가동한 시간 동안 기준 속도만큼 뽑았는가"를 보는 지표입니다. 계획·공수용 개념인 ST를 쓰면 계획 여유분이 성능처럼 보입니다. 기계의 기준 속도인 CT가 맞습니다.

그 외 SET UP 표준시간, 모델변경 표준시간, 근무/휴식 시간도 전부 기준정보 관리 대상입니다.

계산 사례 — SMT-01, 주간 근무조. 근무 480분, 계획정지(조회·PM) 30분 → 부하시간 450분. 무작업(모델변경 40분 + 자재대기 20분) 60분 → 실제 가동 390분 → 가동율 86.7%. 모델 X의 CT 12초 × 생산 1,700개 = 340분 → 340 ÷ 390 = 성능율 87.2% (부족분 50분이 순간정지·속도저하 = 시스템이 계산한 성능손실). 양품 1,666개 ÷ 1,700개 = 양품율 98.0%. → OEE = 86.7% × 87.2% × 98.0% = 74.1%.

직행율 — 양품율과 별도 지표

공정 직행율 = 재작업/재검 없이 1회 통과 수량 ÷ 공정 투입수량
라우팅 직행율 = 전체 라우팅 무재작업 통과 수량 ÷ 최초 투입수량

양품율은 "최종적으로 양품이었나", 직행율은 "한 번에 통과했나"를 봅니다. 재검 2번 만에 합격한 제품은 양품율에는 양품이지만 직행율에는 손실입니다 — 검사 재시도 낭비는 직행율에서만 보입니다.

실시간: 현재 생산실적·작업지시·무작업 입력·설비/라인 상태 기준의 임시 계산. 모니터링 화면(15절)이 사용.
