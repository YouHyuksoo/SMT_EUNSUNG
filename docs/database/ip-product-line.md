# IP_PRODUCT_LINE — 생산라인 마스터

## 확인된 역할

`IP_PRODUCT_LINE`은 생산라인 기준정보를 관리한다. 기본키는
`LINE_CODE + ORGANIZATION_ID`다.

2026-07-12 라이브 데이터는 56개 라인이며 라인구분은 SMT 15개, 조립 16개,
가공 16개, 검사 7개, 수리 1개, 기타 1개다.

## 기준정보

- 라인코드·라인명·라인구분
- 라인 생산유형
- 생산능력·UPH·사용률·용량단위
- MES 표시여부·그룹·순서
- 주간·야간 시작시간

## 레거시 책임 혼합

동일 행에 다음 운영 정보도 함께 저장한다.

- 현재 라인상태와 활성여부
- 현재 모델·품목·RUN
- 마스크·스퀴지·솔더·리플로우 등 장착 또는 확인 상태
- ICT·샘플·프로파일 등 검사일과 LOT
- 이전 생산·장착 정보

생산라인 마스터라는 기준 역할과 현재 상태·최근 이력 역할이 한 테이블에 혼합된
레거시 구조다. 컬럼이 존재한다는 이유로 모두 별도 기준정보로 해석하지 않는다.

## 상태 구분

- `LINE_STATUS`: 라인운영상태. 정상·고장·수리·정지·모델변경·자재교환·샘플생산
  등의 현재 상태를 나타낸다.
- `ACTIVE_YN`: 생산활성여부. `Y`는 생산 활성, `N`은 대기이며 마스터 사용여부가
  아니다.

2026-07-12 라이브 데이터는 56개 라인 모두 `LINE_STATUS='N'(정상)`이고,
`ACTIVE_YN`은 `N` 또는 NULL이다.

## 라인 현재생산정보

`MODEL_NAME`, `ITEM_CODE`, `RUN_NO`는 라인에 현재 투입된 모델·품목·RUN을 나타낸다.
생산계획이나 생산실적의 원장이 아니라 라인 화면·인터록을 위한 현재 상태 정보다.
`LAST_*` 컬럼은 직전 생산 또는 장착 정보를 보존한다.

라이브 데이터 56개 라인 중 모델은 12개, 품목과 RUN은 각각 11개 라인에 설정돼
있다. 모든 라인의 고정 모델 매핑으로 해석하지 않는다.
