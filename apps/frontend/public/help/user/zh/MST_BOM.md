# BOM管理

## 画面目的
管理定义成品·半成品由哪些**子物料（物料）构成**的**物料清单(BOM)**。此处登记的构成为生产时物料需求量计算（展开）和投入的依据。

## 画面构成
- **左侧**：选择父物料（成品/半成品）。
- **右侧**：所选父物料的**BOM树**。子物料再拥有下级子物料时按**层级(Lv)**展开（递归结构）。可折叠/展开行（全部折叠·展开）。
- **Excel上传**：一次性批量登记多个BOM行。
- 行添加/修改在BOM表单中进行，也可同时查看子物料的工序路线。

## 概念关系
```
父物料(成品) ─┬─ 子物料A (需求量)
             ├─ 子物料B (需求量)
             └─ 半成品C (需求量) ─┬─ 子物料C1
                                   └─ 子物料C2
```

---

## BOM列 (BOM_MASTERS)

| 列 | 作用 / 含义 |
|------|------|
| **Lv（层级）** | BOM树中的深度。父级正下方为1，其子物料又作为父级时为2…以此类推（非存储值，为树计算值）。 |
| **子物料代码(childItemCode)** | 构成父物料的**子物料代码**。必须是已在物料主数据中登记的物料。（父+子物料+版本为一行键） |
| **子物料名称(childPartName)** | 子物料的名称（从物料主数据获取，显示用）。 |
| **类型(type)** | 子物料的物料类型（原材料/半成品等，物料主数据值）。半成品时将进一步展开为下级BOM。 |
| **工序(oper)** | 该子物料**投入的工序**。指定在哪个工序使用。 |
| **需求量(qtyPer)** | 制造1个父物料所需的该子物料**数量**。总需求量 = 生产数量 × 需求量。 |
| **版本(revision)** | BOM构成的版本（默认`A`）。即使同一父-子物料也可通过版本区分（键的一部分）。 |
| **面(side)** | 双面板等情况下区分应用面（例如：TOP/BOT）的值。 |
| **有效开始(validFrom)** | 该BOM行开始应用的日期。 |
| **有效结束(validTo)** | 该BOM行应用结束的日期。 |

### 表单中同时输入的项目
| 列 | 作用 / 含义 |
|------|------|
| **序号(seq)** | 同一父级中子物料的显示/处理顺序。 |
| **BOM组(bomGrp)** | 将BOM分组管理时使用的分类值。 |
| **ECO编号(ecoNo)** | 设计变更(ECO)追踪编号。变更依据管理。 |
| **备注(remark)** | 备注。 |
| **使用与否(useYn)** | 仅`Y`时为有效构成。 |

## 使用顺序
1. 在左侧选择**父物料**。
2. 在右侧树中选择行并**添加子物料**（输入子物料代码·需求量·工序·有效期）。
3. 如果子物料是半成品，则以该半成品为父级继续构建下级BOM。
4. 多条记录可通过**Excel上传**一次性批量登记。

## 输入规则
- 子物料代码必须存在于物料主数据中。
- 需求量必须大于0。
- 同一父物料+子物料+版本组合不可重复（一行键）。

## 常见问题
- **问：** 层级(Lv)如何决定？
  **答：** 非直接输入，当子物料拥有自己的BOM时自动加深一层（递归展开）。
- **问：** 需求量以什么为基准？
  **答：** 制造1个父物料所需的子物料数量。总需求量 = 生产数量 × 需求量。
- **问：** 版本为何包含在键中？
  **答：** 因为同一父-子物料可根据规格变更（版本）拥有不同构成。
