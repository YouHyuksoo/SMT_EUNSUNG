# 消耗品入库

## 画面目的
将入库的**消耗品确认为仓库库存**的画面。可扫描标签发行时生成的消耗品UID条码逐件入库，也可不使用UID按数量手动入库。入库处理后消耗品主数据的持有库存增加，并留下入库记录。

> 消耗品入库有两个途径：**① 条码扫描入库**是在[消耗品标签发行](/consumables/label)中预先发行单个UID后扫描确认的方式；**② 手动入库**是不使用UID仅凭消耗品·数量增加库存的方式。

## 画面构成
- **上方（条码扫描面板）**：切换入库/退回模式并扫描UID条码。入库模式下还显示尚未入库的**未入库等待列表**。
- **下方（入库历史）**：入库·退回历史网格。使用上方工具栏按**期间**、**搜索词**、**类型过滤**缩小查询范围。
- **右侧（滑动面板）**：点击上方**入库登记**按钮时打开的手动入库表单。

---

## ① 条码扫描面板

| 项目 | 作用 / 含义 |
|------|------|
| **入库 / 退回模式** | 左侧切换处理类型。**入库**确认未入库UID入库；**退回（退回入库）**撤回已入库的UID并扣减库存。 |
| **UID条码输入** | 扫描（或输入后按Enter）消耗品UID条码后立即处理。扫描后输入框自动清空等待下次扫描。 |
| **保管位置（入库模式）** | 选择入库消耗品的存放保管位置（可选）。入库确认时记录到该UID的保管位置。 |
| **退货原因（退回模式）** | 输入退回原因（可选）。将作为退回历史的原因留存。 |
| **入库确认 / 退回确认按钮** | 处理输入的UID。如使用条码扫描器输入Enter，无需点击按钮即可自动处理。 |

### 未入库等待列表（仅在入库模式下显示）
显示仅发行标签但尚未入库（等待中）的UID。扫描此处的UID即可确认入库。

| 列 | 作用 / 含义 |
|------|------|
| **UID(conUid)** | 等待入库的消耗品单个标识号。 |
| **消耗品代码(consumableCode)** | 该UID的消耗品代码。 |
| **消耗品名称(consumableName)** | 消耗品名称。 |
| **分类(category)** | 模具·夹具·工具等消耗品分类。 |
| **标签打印日期(labelPrintedAt)** | 该UID标签的发行时间。 |
| **供应商名称(vendorName)** | 标签发行时输入的供应商。 |

## ② 入库历史网格

| 列 | 作用 / 含义 |
|------|------|
| **日期时间(createdAt)** | 入库·退回处理的时间。 |
| **消耗品代码(consumableCode)** | 已入库的消耗品代码。 |
| **消耗品名称(consumableName)** | 消耗品名称。 |
| **UID(conUid)** | 条码扫描入库时的单个UID。手动入库无UID，显示为`-`。 |
| **类型(logType)** | **入库(IN)**或**入库退货(IN_RETURN)**。通过颜色标签区分。 |
| **数量(qty)** | 入库/退回数量。入库显示`+`，退回显示`-`。 |
| **供应商代码(vendorCode)** | 供应商代码。 |
| **供应商名称(vendorName)** | 供应商名称。 |
| **单价(unitPrice)** | 入库单价（元）。 |
| **入库区分(incomingType)** | **新规(NEW)**或**更换(REPLACEMENT)**。 |
| **备注(remark)** | 入库/退回时留下的备注。 |

### 工具栏过滤
| 项目 | 作用 / 含义 |
|------|------|
| **开始日期 / 结束日期** | 查询的入库历史期间。 |
| **搜索词** | 按消耗品代码·消耗品名称搜索历史。 |
| **类型过滤** | 选择显示全部/入库/入库退货类型。 |

## ③ 手动入库表单（右侧面板）

通过上方**入库登记**按钮打开。不使用UID，按消耗品·数量单位增加库存。

| 项目 | 作用 / 含义 |
|------|------|
| **消耗品** | 通过放大镜按钮搜索·选择消耗品。为入库对象，必填。 |
| **数量(qty)** | 入库数量。持有库存将增加此数量（默认1）。 |
| **入库区分(incomingType)** | 选择**新规**或**更换**。 |
| **供应商代码 / 供应商名称** | 入库消耗品的供应商信息。 |
| **单价(unitPrice)** | 消耗品的单个入库单价（可选）。 |
| **备注(remark)** | 入库备注（可选）。 |

## 使用顺序
1. **条码扫描入库**：上方确认**入库**模式 → （如需）选择保管位置 → 扫描UID条码 → 立即入库确认。可在未入库等待列表中确认目标UID。
2. **退回入库**：切换模式为**退回** → （如需）输入退货原因 → 扫描已入库的UID → 确认退回（库存扣减）。
3. **手动入库**：上方**入库登记** → 搜索·选择消耗品 → 输入数量·入库区分·供应商·单价 → **登记**。
4. 处理结果在下方**入库历史**网格中确认。

## 输入规则 / 校验
- 手动入库需**选择消耗品**才能激活登记按钮。
- 扫描入库仅可确认**未入库（等待）状态的UID**。已入库的UID再次扫描会报错。
- 退回入库仅可处理**当前入库（激活）状态的UID**。

## 常见问题
- **问：** 未入库等待列表中没有UID。
  **答：** 入库前需先在[消耗品标签发行](/consumables/label)中发行UID。标签发行后才会出现在等待列表中。
- **问：** 扫描时显示"已入库状态"。
  **答：** 该UID已是入库确认完成的。请在入库历史中确认。
- **问：** 手动入库和扫描入库有何不同？
  **答：** 扫描入库是按UID单个追踪的每件入库；手动入库是不使用UID仅按数量增加库存的方式。两者都会增加持有库存。
- **问：** 入库数量如何反映到库存？
  **答：** 入库(IN)增加消耗品持有库存，退回(IN_RETURN)减少持有库存。
