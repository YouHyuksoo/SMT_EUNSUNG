# 进货检验(IQC)

## 画面目的
对到货的物料LOT进行品质检验，判定**合格(PASS)/不合格(FAIL)**的画面。
在物料主数据中`IQC与否=Y`的物料到货后变为待检(PENDING)状态，须在此画面完成检验后方可进行入库处理。

> 检验对象按到货编号+物料单位汇总显示。同一到货批次的同一物料有多个序列号时合计为1行。

## 画面构成
- **上方过滤栏**：搜索词（到货编号·物料代码·物料名称）、检验区分(FULL/SKIP)过滤、检验状态过滤
- **列表网格**：以表格形式显示待检·进行中·已完成记录。每行有**检验**按钮
- **检验模态框**：点击行的检验按钮时打开。序列号扫描→项目测量/判定→结果登记

---

## ① 列表网格列

| 列 | 作用 / 含义 |
|------|------|
| **检验（按钮）** | 打开所选到货批次的检验模态框。仅检验状态为`待检`或`检验中`的记录激活。 |
| **到货编号(arrivalNo)** | 标识到货的编号。通过此编号追踪到货批次。 |
| **到货日期(arrivalDate)** | 物料到货的日期。 |
| **供应商(supplierName)** | 交付物料的协作公司（供应商）名称。 |
| **物料代码(itemCode)** | 检验目标物料的MES内部代码。 |
| **物料名称(itemName)** | 检验目标物料的名称。 |
| **检验区分(inspectMethod)** | 物料主数据设置的检验方式。`检验(FULL)`=正式进货检验，`免检(SKIP)`=省略检验。 |
| **序列号数(serialCount)** | 该到货批次包含的物料序列号(LOT)数量。表示需扫描的对象数。 |
| **总数量(totalQty)** | 所有序列号的数量合计（含单位）。 |
| **状态(status)** | 检验进行状态。参见下面状态值。 |
| **检验员(inspector)** | 执行检验的负责人名称。检验完成后显示。 |

### 检验状态值

| 状态代码 | 显示 | 含义 |
|------|------|------|
| `PENDING` | 待检 | 到货完成后检验前的状态。此状态可开始检验。 |
| `IQC_IN_PROGRESS` | 检验中 | 检验已开始但尚未完成。检验按钮仍可激活。 |
| `PASSED` | 合格 | 检验判定结果为合格。可进行入库处理。 |
| `FAILED` | 不合格 | 检验判定结果为不合格。序列号自动移至废品仓库。 |

## ② 检验模态框 — 到货信息显示

模态框上方固定显示到货批次的基本信息。

| 项目 | 作用 / 含义 |
|------|------|
| **到货编号** | 当前检验中的到货批次编号。 |
| **供应商** | 交货协作公司名称。 |
| **序列号数** | 该到货批次所属的全部序列号数。 |
| **总数量** | 全部序列号的数量合计。 |
| **物料** | 检验目标物料名称与物料代码。 |

---

## ③ 检验模态框 — 左侧输入面板

| 项目 | 作用 / 含义 |
|------|------|
| **序列号扫描（输入）** | 扫描检验的序列号条码或直接输入后按Enter。待检列表中无此序列号时显示错误消息。可通过**查询待检序列号**按钮确认等待列表，也可个别点击或通过**全部添加**批量登记。 |
| **扫描进度计数器** | 以`扫描数 / 全部待检数`格式显示进度。 |
| **检验员** | 输入执行检验的人员姓名。 |
| **备注** | 输入检验相关备注。 |
| **默认样品数** | 物料主数据中登记的默认样品数量。检验员可修改。 |
| **AQL样本数量** | 基于物料关联的AQL政策自动计算的建议样品数。同时显示检验水准和检验模式（例如：`Normal II`）。可预先查看项目Ac/Re（合格/不合格判定个数）。 |
| **检验成绩书** | 上传PDF·图片·Excel文件。结果登记后保存至服务器。 |
| **不良代码** | 发生FAIL判定时输入不良类型（公共代码`DEFECT_TYPE`）和数量。可通过添加行按钮同时登记多个不良类型。无FAIL时不可输入。 |
| **破坏/全数检验** | 物料中已登记破坏检验(DESTRUCTIVE)或全数检验(FULL)类型检验项目时显示。按项目直接输入**检验数量**和**不良数**。不良数≥1时该项目判定为FAIL。 |

## ④ 检验模态框 — 中间面板（扫描序列号列表）

扫描或添加的序列号按顺序排列。

| 项目 | 作用 / 含义 |
|------|------|
| **序号** | 扫描顺序号。 |
| **序列号(matUid)** | 该物料LOT的唯一标识符（序列号）。 |
| **判定状态** | 该序列号的当前判定结果。全部检验项目判定完成时显示PASS/FAIL，未完成时显示`等待`。 |
| **移除按钮** | 从列表中移除误扫描的序列号。 |

---

## ⑤ 检验模态框 — 右侧测量面板

在序列号列表中选择序列号时，右侧显示该序列号的检验项目表。

| 列 | 作用 / 含义 |
|------|------|
| **#** | 检验项目序号。 |
| **检验项目** | 要检验的项目名称。可能显示不良等级(Critical/Major/Minor)标签和AQL值。 |
| **规格(spec)** | 该项目的规格说明。 |
| **下限(LSL)** | 测量值的下限合格标准。输入值小于此值时自动判定为FAIL。 |
| **上限(USL)** | 测量值的上限合格标准。输入值大于此值时自动判定为FAIL。 |
| **判定基准** | 目视检验等非数值的合格标准以文字说明。 |
| **测量值** | 有LSL/USL的项目直接输入数字。在范围内则自动判定为PASS。无LSL/USL的目视检验项目通过PASS/FAIL按钮直接选择。 |
| **判定** | 根据测量/输入结果的合格(✓)/不合格(✗)图标。 |

> **无检验项目的物料**：该物料未登记IQC检验项目时，按序列号手动选择PASS/FAIL。

---

## 使用顺序

1. 在列表中找到要检验的到货批次行，点击**检验**按钮。
2. 模态框打开后，在左侧扫描输入栏中通过条码扫描或直接输入物料序列号后按Enter（或**查询待检序列号→全部添加**）。
3. 在中间面板中选择序列号，右侧显示检验项目表。
4. 按项目输入测量值或选择PASS/FAIL。有LSL/USL时自动判定。
5. 有破坏/全数检验项目时，在左侧面板下方输入检验数量和不良数。
6. 有FAIL判定时选择**不良代码**并输入数量（必填）。
7. 如有需要上传**检验成绩书**文件。
8. 点击模态框下方的**登记检验结果**按钮保存结果。

---

## 输入规则 / 校验

- 未扫描的序列号（尚未检验的）不可无判定直接登记。
- 已扫描的序列号中有未完成判定的，登记按钮禁用。
- 有FAIL判定（序列号或破坏检验）时**必须输入至少1条不良代码**。
- 无FAIL但输入不良代码时阻止登记。
- 扫描的序列号不在待检列表中时显示错误消息并不予登记。

## 常见问题

- **问：** 检验按钮已禁用。
  **答：** 该到货批次的状态已为`PASSED`或`FAILED`，判定已完成。如需取消判定请咨询运营人员。
- **问：** AQL样本数量不显示。
  **答：** 该物料未关联AQL政策。请在物料主数据中设置AQL政策。
- **问：** 扫描序列号时出现"非待检序列号"错误。
  **答：** 扫描的序列号当前不是PENDING状态或属于其他到货编号。请通过**查询待检序列号**确认正确的序列号列表。
- **问：** 不合格处理后库存消失了。
  **答：** IQC不合格时该序列号自动移至废品仓库。如需复检请向运营人员申请取消。
- **问：** 无检验项目时如何判定？
  **答：** 物料未登记IQC检验项目时，在模态框右侧区域按序列号直接点击PASS/FAIL按钮判定。
- **问：** 仅有破坏检验项目时是否不需要扫描序列号？
  **答：** 是的。如无AQL序列号检验项目而仅有破坏/全数检验项目时，可不扫描序列号，仅输入破坏检验数量后登记结果。
