# 消耗品标签发行

## 画面目的
为了**按单个实例追踪**消耗品（模具·夹具·工具等），为每个物品发行唯一的**消耗品UID(conUid)**，并打印带有该UID的标签（条码）。即使同一消耗品代码，入库的每个单品都有独立的UID，贴上标签后可对后续入库·出库·安装·更换进行个别管理。

> UID发行即**创建未入库(PENDING)实例**。打印标签贴到实物上后，在消耗品入库画面扫描条码完成入库确认，即变为可用(ACTIVE)状态并增加库存数量。

## 画面构成
- **左侧（列表）**：可发行标签的消耗品主数据网格。上方可搜索**消耗品代码/消耗品名称**、按**分类过滤**、**刷新**。每行有**选择复选框**和**发行数量**输入框。
- **右上侧（发行工具）**：**状态消息**（发行/打印进度提示）、**刷新**、**标签模板选择**、**UID发行**按钮。
- **右侧（详情面板）**：点击列表中的行时展开该消耗品的**未入库(PENDING)实例列表**。可**预览**或**重新发行**（重新打印标签）各实例。

## 发行到打印流程
```
选择消耗品 + 输入发行数量
        │
        ▼
点击[UID发行] → 按数量生成conUid（创建未入库实例）
        │
        ▼
按所选标签模板绘制标签并调用浏览器打印窗口
        │
        ▼
将标签粘贴到实物消耗品 → 在消耗品入库画面扫描·入库确认
```

## ① 列表列（左侧网格）

| 列 | 作用 / 含义 |
|------|------|
| **选择（复选框）** | 选择要发行标签的消耗品。表头复选框可**全选/取消全选**。需选择1件以上才能激活UID发行。 |
| **图片** | 消耗品照片缩略图。点击可放大，打印标签时也可包含（取决于模板设置）。无照片时显示`-`。 |
| **消耗品代码(consumableCode)** | 发行目标消耗品的唯一代码。发行的所有UID都归属于此代码。 |
| **消耗品名称(consumableName)** | 消耗品名称。用于搜索和标签显示。 |
| **分类(category)** | 模具(MOLD)·夹具(JIG)·工具(TOOL)分类（公共代码CONSUMABLE_CATEGORY）。上方分类过滤的依据。 |
| **当前库存(stockQty)** | 消耗品主数据累计的持有库存数量。每次入库确认后增加。 |
| **实例(instanceCount)** | 该消耗品**迄今为止发行的UID(物品)数量**。旁边如有黄色显示**(未入库: n)**，表示其中尚未入库确认的PENDING数量。 |
| **发行数量(qtyInput)** | 本次要发行的UID数量（1~99）。例如输入`5`则生成5个conUid。默认值为1。 |

## ② 标签发行工具（右上侧）

| 项目 | 作用 / 含义 |
|------|------|
| **状态消息** | 显示发行·打印进度的单行消息（例如"已完成{{count}}件发行"，错误时显示红色）。有选中数量且未进行中时显示"已选择{{count}}件"。 |
| **刷新** | 重新加载列表和发行数·未入库数至最新状态。 |
| **标签模板选择** | 选择打印使用的标签设计。可选用**默认设计**或已注册的模板（消耗品/夹具分类）。不同模板具有不同的标签尺寸·条码·显示项·打印方式。 |
| **UID发行** | 按发行数量生成所选消耗品的UID，并立即打开标签打印窗口。发行中显示"发行中..."，打印准备中显示"输出中"。 |

## ③ 未入库实例面板（右侧）
在列表中点击消耗品行时展开。仅显示该消耗品的**未入库(PENDING)UID**（已入库的除外）。

| 项目 | 作用 / 含义 |
|------|------|
| **摘要（当前库存·未入库·保管位置）** | 面板顶部摘要显示所选消耗品的当前库存、未入库数量、默认保管位置。 |
| **消耗品UID(conUid)** | 已发行的单个UID。是标签·条码中的追踪键。 |
| **状态** | 实例状态标签。此面板仅显示**未入库(PENDING)**（公共代码CON_STOCK_STATUS）。 |
| **使用次数(currentCount / expectedLife)** | 累计次数与预期寿命。未入库阶段通常为`0`。 |
| **入库日期(recvDate)** | 入库确认的日期。未入库的条目为空(`-`)。 |
| **预览** | 在模态框中查看该UID标签的条码·布局（不打印）。 |
| **重新发行** | **重新打印**该UID标签。标签损坏或丢失时使用，发送到标签打印机(agent)。 |

## 使用顺序
1. 在左侧列表中**勾选**要发行标签的消耗品（通过搜索·分类过滤快速查找）。
2. 在每行输入**发行数量**（按单品数量）。
3. 确认/选择上方的**标签模板**。
4. 点击**UID发行**，生成UID并打开打印窗口，在打印对话框中输出。
5. 将打印的标签粘贴到实物消耗品上。
6. （需要重新打印时）在列表中点击该消耗品，在右侧面板中对相应UID进行**预览/重新发行**。
7. 贴标后在**消耗品入库**画面扫描条码完成入库确认。

## 输入规则 / 校验
- 必须**选择至少1件消耗品**才能激活UID发行按钮。
- **发行数量**范围为1~99（超出时自动修正）。
- 打印通过**新窗口(弹出窗口)**打开。浏览器阻止弹窗会中断发行，请允许此站点的弹窗。
- 发行数量较多或条码生成未完成时，建议先在预览中确认标签后再打印（条码未完成时会阻止打印）。

## 常见问题
- **问：** 发行UID后库存会立即增加吗？
  **答：** 不会。发行仅创建**未入库(PENDING)**实例。需贴标后在**消耗品入库**中扫描·入库确认才能变为可用(ACTIVE)并增加当前库存。
- **问：** 标签丢失了怎么办？
  **答：** 在列表中点击该消耗品，在右侧未入库面板中点击该UID的**重新发行**按钮，将使用同一UID重新打印。
- **问：** 打印窗口不出现。
  **答：** 这是由于浏览器弹窗阻止。请在地址栏中允许此站点的弹窗后重新发行。
- **问：** 发行数量不能设为100以上。
  **答：** 每次仅可发行1~99个。数量多时请分批发行。
- **问：** 实例旁边的"(未入库: n)"是什么意思？
  **答：** 表示已发行但尚未入库确认的UID数量。提示贴标后还需完成入库处理。
