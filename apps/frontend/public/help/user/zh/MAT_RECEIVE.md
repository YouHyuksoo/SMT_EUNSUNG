# 物料入库管理

## 画面目的
**将进货检验(IQC)合格的物料LOT入库（放置）到仓库**的画面。采用**条码扫描方式**替代全量手工输入，将客户条码与自有粘贴条码进行映射，准确确认入库。

> 流程：到货（采购入库）→ 进货检验(IQC) → **仅合格件待入库** → 扫描入库 → 库存反映

## 画面构成
- **上方统计**：待入库件数/数量、当日入库件数/数量。
- **过滤**：到货日期（默认当天）、供应商、物料搜索。
- **待入库网格**：IQC合格 + 未入库（有剩余）的LOT列表。用于确认入库对象。
- **入库处理（扫描）模态框**：循环扫描客户条码→自有粘贴条码。**仅扫描到的映射确认入库**。

---

## ① 待入库列（可入库LOT）

| 列 | 作用 / 含义 |
|------|------|
| **物料序列号(matUid)** | 入库目标物料LOT的唯一序列号。到货时发放的物料标识值。 |
| **物料代码/物料名称(part.itemCode / itemName)** | 入库目标物料（基于物料主数据）。 |
| **物料类型(itemType)** | 原材料/消耗品等分类。 |
| **单位(part.unit)** | 数量单位。 |
| **到货数量(initQty)** | 到货时点的LOT初始数量。 |
| **已入库数量(receivedQty)** | 已入库处理的数量（部分入库累计）。 |
| **剩余数量(remainingQty)** | 尚未入库的数量（= 可入库数量）。 |
| **到货日期(recvDate)** | 物料到货的日期。 |
| **制造日期(manufactureDate)** | 物料制造日期。 |
| **有效期(expireDate / expiryDays)** | 物料的有效期及剩余天数。用于管理期限临近/过期。 |
| **采购单编号(poNo)** | 关联的采购单(PO)编号。 |
| **供应处(vendor)** | 交付物料的客户。 |
| **检验状态(iqcStatus)** | 进货检验结果。待入库仅显示**合格**的记录。 |
| **到货仓库(arrivalWarehouse)** | 物料所在的到货仓库。 |
| **成绩书(certRequired / certUploaded)** | 是否需要成绩书及是否已上传。需提供但未上传时可能阻止入库。 |
| **入库阻止原因(receivingBlockedReason)** | 入库被阻止时的原因（例如：成绩书未提交）。 |

## ② 扫描入库方法
1. 在待入库中确认对象后点击**入库处理**按钮。
2. 扫描**客户条码**（供应商粘贴的条码）。
3. 接着扫描**自有粘贴条码（物料序列号）**。
4. 仅两个条码完成映射的记录确认入库。可循环扫描多个记录。

> 客户条码通过[制造商条码映射]中登记的规则与MES品番/序列号关联。

## ③ 入库历史列
| 列 | 作用 / 含义 |
|------|------|
| **入库编号/交易编号(receiveNo / transNo)** | 入库处理标识号。 |
| **入库日期(transDate)** | 入库处理日期。 |
| **数量(qty)** | 入库的数量。 |
| **入库仓库(toWarehouse)** | 物料放置的仓库。 |
| **供应处/制造商(vendor / manufacturer)** | 交货客户与制造商。 |
| **区分(materialClass)** | 量产(PROD)/消耗品(MRO)区分。 |
| **状态(status)** | 入库状态。 |

## 常见问题
- **问：** 待入库中看不到物料。
  **答：** 需IQC合格且剩余数量>0才显示。检验未完成/不合格或已全部入库时不会显示。
- **问：** 入库被阻止。
  **答：** 请确认"入库阻止原因"，例如需要成绩书的物料未上传等情况。
- **问：** 能否部分入库？
  **答：** 可以，可在剩余数量范围内累计部分入库。
