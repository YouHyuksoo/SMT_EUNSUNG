# 物料库存盘点反映

## 画面目的
库存盘点会话完成后，审查PDA扫描的盘点数量，必要时直接修改并反映(Apply)到系统库存。是将盘点产生的差异实际反映到库存数量的最终阶段。

## 画面构成
- **上方 — 4个统计卡片**: 显示总数 / 已盘点 / 一致 / 不一致件数汇总
- **筛选区域**: 基准年月 + 仓库选择 + 搜索词输入
- **DataGrid列表**: 按物料显示系统数量、盘点数量、差异。盘点数量列**可直接修改**。
- **盘点反映按钮**: 存在不一致项时激活。

---

## ① DataGrid列表列

| 列 | 作用 / 含义 |
|------|------|
| **仓库(warehouseName)** | 库存所在的仓库。 |
| **物料代码(itemCode)** | 盘点目标物料代码。 |
| **物料名称(itemName)** | 盘点目标物料名称。 |
| **物料序列号(matUid)** | LOT/序列号。 |
| **系统数量(qty)** | 当前系统上的库存数量。 |
| **盘点数量(countedQty)** | PDA扫描的盘点数量。**可点击直接修改**。 |
| **差异(diffQty)** | 盘点数量 - 系统数量。正数(蓝色) / 负数(红色) / 0(绿色)。 |
| **盘点时间(countedAt)** | 最后PDA扫描时间。 |

---

## 使用顺序
1. 在上方筛选中选择基准年月和仓库查询盘点结果。
2. 在列表中对比各物料的**盘点数量**与PDA扫描结果。
3. 必要时**直接点击修改盘点数量**(更正错误)。
4. 点击**盘点反映**按钮在确认弹窗中最终确认。
5. 反映完成后在[物料库存现状查询]中确认变更后的库存数量。

## 输入规则 / 验证
- 盘点数量必须为**0以上**的整数。
- 反映前请务必再次确认不一致项。反映后难以恢复。
