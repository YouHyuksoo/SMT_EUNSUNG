# AQL基准管理

## 画面目的
在进货检验(IQC)中为入库LOT的**抽样检验**判定合格/不合格登记·管理基准。
因不检验全部LOT而仅检验部分样本，需预先规定"抽取几个样本（样本数），其中有多少个不良品则合格(Ac)、多少个起不合格(Re)"。这些基准的集合称为**AQL政策**和**AQL基准**。

> AQL(Acceptable Quality Limit, 合格品质界限)：可判定LOT合格的允许不良水平。**数值越小越严格**（例如1.0比2.5严格）。

## 画面构成
- **左侧 — AQL政策管理**：直接关联物料的单位。按Major（重缺陷）·Minor（轻缺陷）分别绑定使用的AQL基准。
- **右上 — AQL基准列表**：以AQL值为单位的基准。政策引用此基准。
- **右下 — LOT数量判定基准**：针对所选AQL基准，按LOT大小区间定义样本数·Ac·Re。

## 概念关系
```
物料主数据 ──关联──▶ AQL政策 ──(Major/Minor)──▶ AQL基准 ──▶ LOT数量判定基准(样本数·Ac·Re)
```

---

## ① AQL政策列 (IQC_AQL_POLICIES)

| 列 | 作用 / 含义 |
|------|------|
| **政策代码(policyCode)** | 标识政策的唯一代码。**物料主数据的'AQL政策'字段引用此代码**。例如：`AQLP-II-1.0-2.5`。注册后不可更改。 |
| **政策名称(policyName)** | 供人识别的名称。例如：`II Major 1.0 Minor 2.5`。 |
| **检验水准(inspectionLevel)** | 决定样本大小的水准。通常使用**II（普通检验）**。I为更少（放宽），III为更多（严格）。LOT大小与检验水准共同决定样本大小。 |
| **Major AQL(majorAqlCode)** | **重缺陷**（对功能·安全影响大的缺陷）适用的AQL基准。从右侧'AQL基准列表'中选择。通常选择比Minor严格（数值小）的基准。 |
| **Minor AQL(minorAqlCode)** | **轻缺陷**（外观等轻微缺陷）适用的AQL基准。通常选择比Major宽松的基准。 |
| **致命缺陷处理(criticalMode)** | **致命缺陷(Critical)**发生时的处理方式。`IMMEDIATE_FAIL`表示即使出现1个致命缺陷，无论样本数·Ac如何**立即判定LOT不合格**。 |
| **使用与否(useYn)** | 仅`Y`时可被物料关联·检验使用。`N`时为停用。 |
| **备注(remark)** | 管理用备注。 |

---

## ② AQL基准列 (AQL_STANDARDS)

| 列 | 作用 / 含义 |
|------|------|
| **AQL代码(aqlCode)** | 标识AQL基准的唯一代码。政策的Major/Minor引用此代码。例如：`AQL-1.0`。注册后不可更改。 |
| **AQL名称(aqlName)** | 基准的名称。例如：`普通检验 AQL 1.0`。 |
| **检验水准(inspectionLevel)** | 该基准前提的检验水准（通常II）。 |
| **AQL值(aqlValue)** | **合格品质界限数值**（例如：0.65，1.0，2.5）。表示允许不良率的上限，**数值越小越严格**。此值与样本数共同决定Ac/Re。 |
| **使用与否(useYn)** | 仅`Y`时可在政策中选择。 |
| **备注(remark)** | 管理用备注。 |

---

## ③ LOT数量判定基准 (rules)

即使同一AQL值，**LOT越大则需要抽取更多样本**，因此按LOT大小区间分别定义样本数·Ac·Re。

| 列 | 作用 / 含义 |
|------|------|
| **lotQtyFrom** | 该行适用的LOT数量区间**起始值（下限）**。 |
| **lotQtyTo** | 该行适用的LOT数量区间**结束值（上限）**。 |
| **sampleSize (n)** | 该区间LOT中**抽取检验的样本个数**。 |
| **acceptQty (Ac)** | **合格判定个数**。检验不良数**≤Ac**时LOT合格。 |
| **rejectQty (Re)** | **不合格判定个数**。检验不良数**≥Re**时LOT不合格。（通常Re = Ac + 1） |
| **排序顺序(sortOrder)** | 行显示顺序。保存时按LOT数量自动整理。 |

> 判定示例：样本数n=13，Ac=1，Re=2 → 13个样本中不良**≤1个合格**，**≥2个不合格**。

---

## 使用顺序
1. 先在右侧**AQL基准**登记（例如：`AQL-1.0`、`AQL-2.5`）。为各基准填写**LOT数量判定基准**（样本数·Ac·Re）。
2. 在左侧创建**AQL政策**，在**Major/Minor**中关联上述基准。
3. 在[物料主数据]中将物料的**AQL政策**字段指定为此政策。
4. 此后该物料入库LOT的进货检验将按此基准自动判定。

## 输入规则 / 校验
- LOT数量From不能大于To。
- Re数量必须大于Ac数量。
- LOT数量区间不可重叠（前区间的To必须小于后区间的From）。

## 常见问题
- **问：** 为什么Major和Minor要分开？
  **答：** 为了根据缺陷严重度应用不同的合格基准。通常重缺陷(Major)用严格（小AQL），轻缺陷(Minor)用宽松值。
- **问：** 为什么需要LOT数量判定基准？
  **答：** LOT越大，需要检验更多样本才能获得相同置信水平（基于KS Q ISO 2859-1抽样标准）。
- **问：** 政策留空（未关联物料）会怎样？
  **答：** AQL自动判定不适用（免检或手动判定）。
