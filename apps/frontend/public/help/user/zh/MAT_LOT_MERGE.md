# 物料合并

## 画面目的
将同一物料且**同一到货批次**入库的多个物料LOT（序列号）**合并为一个新的统一序列号**的画面。用于将零散小批量LOT合并以简化管理·发放。合并后原LOT全部废弃(MERGED)，**新发放1个**具有合计数量的物料序列号，并可打印标签。

> 物料合并与物料分割(LOT分割)是对称概念。分割是将1个分成多个，合并是将多个合并为1个。

## 画面构成
- **条码扫描区域（上方）**：通过条码扫描或直接输入累积要合并的序列号。累积的序列号以标签形式列出，同时显示所选LOT数·合计数量统计卡片和[选择合并]按钮。
- **可合并LOT列表（下方网格）**：显示当前所有可合并的LOT。每行的**＋**按钮可将其累积到扫描区域。通过上方搜索可按LOT编号·物料代码过滤。
- **合并确认模态框**：最终确认合并对象和合计数量。
- **标签预览模态框**：合并成功后预览并输出新序列号的物料标签。

---

## ① 可合并LOT列表网格列

| 列 | 作用 / 含义 |
|------|------|
| **（添加）** | 将行累积到扫描区域的**＋**按钮。已添加的序列号将被禁用。 |
| **物料序列号(matUid)** | 物料LOT的唯一标识符（标签条码）。合并对象的标识依据。 |
| **物料代码(itemCode)** | 物料的物料代码。合并仅适用于**同一物料代码**之间。 |
| **物料名称(itemName)** | 物料名称。 |
| **数量(qty)** | 该LOT的当前库存数量。合并时这些数量将全部合计。 |
| **到货编号(arrivalNo)** | LOT到货的到货编号。合并仅允许**同一到货编号**之间（为继承到货标签）。 |
| **供应处(vendor)** | 交付LOT的客户。有客户名称时显示名称，无则显示代码。 |

> 此列表仅显示同时满足**正常(NORMAL)·持有库存·入库完成·无预订·无出库历史**条件的LOT。

---

## ② 条码扫描区域

| 项目 | 作用 / 含义 |
|------|------|
| **扫描输入框** | 扫描物料条码或直接输入序列号后按Enter即可累积。扫描后自动重新获取焦点。 |
| **添加按钮** | 将输入框的值添加到累积列表。 |
| **累积序列号标签** | 当前累积的序列号连同数量以标签形式显示。每个标签的**✕**可单独移除。 |
| **所选LOT（卡片）** | 累积的序列号数量。 |
| **合计数量（卡片）** | 累积序列号数量的合计。合并后将成为新序列号的数量。 |
| **取消选择按钮** | 清空所有累积的序列号。 |
| **选择合并按钮** | 打开合并确认模态框。累积少于2个时禁用。 |

---

## ③ 合并确认模态框

| 项目 | 作用 / 含义 |
|------|------|
| **合并LOT** | 显示合并目标序列号列表（只读）。 |
| **合计数量** | 新统一序列号中将包含的总数量。 |
| **合并执行按钮** | 执行实际合并。原LOT全部废弃，新发放1个新序列号。 |

> 合并后原LOT全部废弃(MERGED)，创建**1个新序列号**持有合计数量。此操作不可撤销，请谨慎确认。

---

## ④ 标签预览模态框

合并成功后新序列号的物料标签以预览方式打开。标签上印有新物料UID、物料代码·物料名称、合计数量、到货编号、制造商等信息。通过[打印标签]发送到标签打印机，贴于新序列号箱上。

---

## 使用顺序

1. 在下方列表中查找要合并的LOT，或通过上方搜索过滤。
2. 扫描要合并的物料条码，或通过列表行的**＋**按钮累积序列号（需2个以上）。
3. 在统计卡片中确认累积序列号和合计数量。
4. 点击**选择合并**按钮。
5. 在确认模态框中确认合并LOT和合计数量后点击**合并执行**。
6. 在标签预览中确认新序列号标签后**打印标签**，贴于新箱上。

---

## 输入规则 / 校验

- 需累积**2个以上**不同的序列号才能合并。
- 仅可合并**同一物料代码**的LOT（添加不同物料时显示错误提示）。
- 仅可合并**同一到货编号**的LOT。无到货编号的LOT无法合并。
- 以下LOT不可合并：HOLD状态、非正常(NORMAL)状态、库存为0、持有预订数量、入库未完成（需入库确认）、有物料出库历史。
- 已添加的序列号不会重复添加。

---

## 常见问题

- **问：** 扫描了不同物料时出现错误。
  **答：** 合并仅允许同一物料代码之间。与已添加LOT不同的物料不会被添加。

- **问：** 为什么到货编号不同的LOT不能合并？
  **答：** 新序列号需继承原LOT的到货编号和到货信息以保持标签·历史追踪的一致性，因此仅允许同一到货批次的LOT合并。

- **问：** 合并后原序列号会怎样？
  **答：** 原LOT全部废弃(MERGED)，库存变为0。将新发放1个持有合计数量的新序列号。收发历史保留，可进行追溯。

- **问：** 合并错误能否撤销？
  **答：** 合并没有撤销画面。如有需要请通过[物料分割](/material/lot-split)重新分割使用。

- **问：** 列表中看不到LOT。
  **答：** 入库未完成、库存为0、持有预订数量、有出库历史、HOLD/异常状态的LOT将从列表中排除。请先确认入库确认·预订解除等状态。
