# 托盘出货

## 画面目的
通过条码扫描或输入查询已装载完毕且可出货(CLOSED)的托盘，执行最终出货确认(SHIPPED)。可查看每个托盘包含的箱子清单。

## 画面构成
- **左侧 — 出货指令列表**：显示出货确认(CONFIRMED)状态的出货指令列表。
- **中间 — 托盘列表**：以DataGrid显示所选出货指令的托盘列表。
- **右侧 — 箱子明细**：显示所选托盘中包含的箱子列表及状态。

---

## 使用顺序
1. 从左侧出货指令列表中选择要处理的出货指令。
2. 在中间托盘列表中确认要出货的托盘。
3. 在**托盘扫描**输入框中扫描或手动输入托盘条码。
4. 如果输入的托盘有效，将添加到列表中，并激活**出货确认**按钮。
5. 点击出货确认按钮完成最终出货处理。

## 中间 — 托盘DataGrid列

| 列 | 作用/说明 |
|------|------|
| **托盘编号(PALLET_NO)** | 托盘的唯一标识编号。 |
| **箱数(BOX_COUNT)** | 托盘上装载的箱子数量。 |
| **总数量(TOTAL_QTY)** | 托盘的产品总数量。 |
| **状态(STATUS)** | 托盘当前状态(OPEN / CLOSED / LOADED / SHIPPED)。 |
| **装载完成时间(CLOSE_TIME)** | 托盘装载完成的时间。 |
| **出货时间(SHIPPED_TIME)** | 出货确认时间。 |

## 右侧 — 箱子明细列

| 列 | 作用/说明 |
|------|------|
| **箱子编号(BOX_NO)** | 箱子的唯一标识编号。 |
| **品目代码** | 箱子中所含产品的品目代码。 |
| **品目名称** | 产品名称。 |
| **数量(QTY)** | 每箱的产品数量。 |
| **状态(STATUS)** | 箱子状态。 |

## 输入规则
- 托盘编号可通过条码扫描器扫描或直接输入。
- 只有**CLOSED**(装载完成)状态的托盘才可出货。
- 出货确认时，托盘及包含的所有箱子状态变为**SHIPPED**，并扣除库存。

## 状态流程
```
OPEN(托盘创建) → CLOSED(装载完成,可出货) → SHIPPED(出货确认)
```
