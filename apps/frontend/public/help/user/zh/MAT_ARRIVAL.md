# 到货管理

## 画面目的
采购单(PO)中登记的物料实际交货时**登记到货**的画面。按行单位记录到货数量后自动生成物料LOT序列号，可打印LOT标签贴于现场。到货后的物料转为进货检验(IQC)等待状态。

> 到货 = 交货确认阶段。入库（库存反映）在IQC通过后另行处理。

## 流程图
```
采购单(PO)确认 → 到货管理(本画面)确认交货
  → 自动发放物料LOT序列号 + 打印标签
  → IQC进货检验等待(PENDING)
  → IQC合格 → 到货库存 → 入库(物料当前库存)
```

## 画面构成
- **主网格(PO行列表)**：显示所有可到货的PO行。上方提供状态过滤·物料代码·PO编号搜索。
- **物料到货按钮（单元格或行点击）**：打开该行的到货登记模态框。
- **手动到货按钮（右上角）**：不通过PO，直接指定物料/数量进行到货。
- **标签模板选择（右上角）**：预先选择标签输出使用的模板。

## ① PO行网格列

| 列 | 作用 / 含义 |
|------|------|
| **（操作）** | **物料到货**按钮。余量为0或CLOSE状态时禁用。 |
| **采购单编号(poNo)** | 采购单(PO)编号。仅显示在采购管理中已变为CONFIRMED状态的PO。 |
| **行号(lineNo)** | 采购单行号。一个PO有多个物料时区分各物料。 |
| **修订号(revNo)** | 行的修订版本号。区分同一行的修改历史。 |
| **物料代码(itemCode)** | 物料的MES内部物料代码。 |
| **物料名称(itemName)** | 物料名称。 |
| **采购数量(orderQty)** | 采购单中记载的总采购数量。 |
| **累计到货(receivedQty)** | 迄今为止该行登记的到货数量合计。 |
| **余量(remainingQty)** | 尚未到货的数量（采购数量 − 累计到货）。以蓝色粗体显示。 |
| **采购日期(orderDate)** | 采购单创建的日期。 |
| **客户(partnerName)** | 交货供应商（客户）名称。 |
| **用途区分(useType)** | 采购的用途区分（例如：生产用PROD、非生产等）。公共代码`PO_USE_TYPE`。 |
| **状态(lineStatus)** | 行的到货进度状态。`OPEN`（未到货）/ `PARTIAL`（部分到货，黄色背景）/ `CLOSE`（完成，灰色背景·禁用）。 |

> 行背景色含义：白色=未到货，黄色=部分到货，蓝色=余量0(CLOSE前)，灰色=CLOSE(到货完成)。

## ② 到货登记模态框（点击物料到货时）

输入所选PO行的到货信息。

| 输入项目 | 作用 / 含义 |
|------|------|
| **采购信息（上方摘要）** | 摘要显示所选采购单编号/行号/修订号和物料代码·物料名称·客户（只读）。 |
| **累计到货 / 采购数量 / 余量** | 摘要显示至今的到货情况（只读）。余量为可到货的最大值。 |
| **到货数量 \*** | 本次实际收到的数量。不得超过余量。 |
| **到货日期 \*** | 物料实际到厂的日期。默认值为当天，不可输入未来日期。 |
| **制造商 \*** | 实际制造物料的公司。从`PARTNER_MASTERS`中的制造商(MFG)类型中选择。打印在标签上。 |
| **仓库 \*** | 存放到货物料的原材料仓库。仅可选择原材料(RAW)类型仓库。 |
| **序列号构成单位(lotUnitQty)** | 物料主数据中登记的LOT构成单位（一个序列号可容纳的数量）。只读。未设置时显示"1 LOT"。 |
| **预计序列号数量** | 到货数量÷序列号构成单位自动计算出的将发放序列号数量。仅供确认，不可修改。 |
| **备注** | 自由输入到货备注（可选）。 |

保存后将弹出**序列号发放最终确认模态框**。确认后在服务器完成登记，并打开**标签预览模态框**。

## ③ 标签预览模态框（到货登记成功后）

可预览已发放的物料LOT序列号标签。

| 项目 | 作用 / 含义 |
|------|------|
| **标签模板选择** | 选择输出的标签设计。右上角预先选择的模板将默认应用。 |
| **标签预览区域** | 按发放的序列号数量排列标签。各标签印有物料UID、物料代码、物料名称、数量、制造商、到货编号、到货日期等。 |
| **标签输出（打印按钮）** | 通过本地Print Agent发送到标签打印机。 |

## ④ 手动到货模态框（无PO到货）

用于无PO的紧急交货·库存调整等例外情况。

| 输入项目 | 作用 / 含义 |
|------|------|
| **物料代码 \*** | 通过物料搜索选择要到货的物料。 |
| **仓库 \*** | 存放到货物料的仓库。 |
| **数量 \*** | 到货数量。 |
| **供应商UID(supUid)** | 供应商自行赋予的物料序列号（可选）。 |
| **制造日期(manufactureDate)** | 物料的制造（生产）日期（可选）。 |
| **客户(vendor)** | 交货的供应商（可选）。从`SUPPLIER`类型客户中选择。 |
| **备注** | 到货原因·备注（可选）。 |

## 使用顺序

**基于PO到货（常规）**
1. 在上方搜索中以物料代码或采购单编号查找目标行。
2. 确认状态过滤中`OPEN`/`PARTIAL`已勾选。
3. 点击该行的**物料到货**按钮。
4. 输入到货数量、到货日期、制造商、仓库后点击**保存**。
5. 确认预计序列号数量后点击**确认**。
6. 在标签预览中确认标签内容并**打印标签**。
7. 将打印的标签贴在物料箱上。

**无PO紧急到货**
1. 点击右上角**手动到货**按钮。
2. 输入物料、仓库、数量（必填）后点击**登记**。

---

## 输入规则 / 校验

- 到货数量必须≥1且不得超过余量。
- 到货日期仅允许当天及之前的日期（含过去和当天）。
- 制造商和仓库为必填项。
- CLOSE状态或余量为0的行**物料到货**按钮将被禁用。

## 常见问题

- **问：** 有余量为0但不是CLOSE的行。
  **答：** 即使余量为0，PO尚未CLOSE处理时会出现此情况。物料到货按钮为禁用状态，无法追加到货。

- **问：** 序列号构成单位为空（1 LOT）。
  **答：** 物料主数据中未设置LOT构成单位(lotUnitQty)。此时到货数量整体以1个序列号发放。

- **问：** 到货后的IQC检验在哪里进行？
  **答：** 在到货管理登记完成后，于[进货检验(IQC)]画面按LOT进行检验。

- **问：** 到货错误能否取消？
  **答：** 有操作员权限时，可在[到货记录/收发台账]画面取消DONE状态的到货记录。取消采用反向分录方式而非删除。

- **问：** 标签无法打印。
  **答：** 请确认PC上Print Agent是否正在运行。如果没有Agent，可在标签预览画面手动保存标签后打印。
