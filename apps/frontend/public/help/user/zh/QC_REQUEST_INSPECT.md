# 委托检验输入

## 画面目的
生产现场的Kiosk中，操作员**难以直接判定的检验项目**（例如：需要测量仪器·试验设备的项目、破坏检验）不由操作员输入结果，而是**委托(DELEGATE)给品质负责人**。这样转交的检验记录以"等待(PENDING)"状态堆积在此画面，由品质负责人实际测量·确认后判定**合格(PASS)/不合格(FAIL)**。

> 自检：生产工序中由操作员或负责人直接执行的工序检验。检验方式为**直接检验(DIRECT)**时在Kiosk中直接判定，为**委托检验(DELEGATE)**时转入本画面。

> Kiosk委托的检验以**等待(PENDING)状态留存时，该工单的生产实绩输入将被阻止**。必须在本画面完成判定后，操作员才能继续输入实绩。

## 画面构成
- **左侧 — 委托检验等待列表**：尚未判定的委托检验(PENDING)项目按请求顺序（从最旧开始）显示。点击行时右侧打开输入区域。
- **右侧 — 结果输入**：确认所选项目的检验基准(LSL/USL·单位·基准)后，输入测量值·备注，然后点击合格/不合格。选择前仅显示"请在左侧选择项目"提示。

## 处理流程
```
Kiosk自检(委托检验项目) ──委托──▶ 等待(PENDING)列表 ──品质负责人判定──▶ 合格(PASS)/不合格(FAIL)
                                                                        │
                                                               等待解除 → Kiosk实绩输入可进行
```

---

## ① 等待列表列

| 列 | 作用 / 含义 |
|------|------|
| **工单(orderNo)** | 产生该检验的生产工单编号。标识在哪个指示的生产中进行的检验。 |
| **工序(processCode)** | 检验发生的工序代码。区分是哪个工序阶段的检验。 |
| **项目名称(itemName)** | 检验项目名称（例如：外观、尺寸、压接力等）。表示检验的内容。 |
| **时机(timing)** | 生产中检验的时机。**首件(FIRST)**=作业开始后首件产品、**中件(MID)**=作业中间、**末件(LAST)**=作业结束时。同一工单中可按时点分别产生检验。 |
| **请求日期时间(createdAt)** | Kiosk中此检验被委托的时间。旧请求在上方显示，可优先处理。 |

---

## ② 结果输入区域

选择行时，随项目信息显示以下项目供输入。

| 项目 | 作用 / 含义 |
|------|------|
| **检验基准 — LSL** | 规格下限值(Lower Spec Limit)。测量值**小于此值则规格不符**。仅计量型项目显示。 |
| **检验基准 — USL** | 规格上限值(Upper Spec Limit)。测量值**大于此值则规格超限**。LSL~USL之间为正常范围。 |
| **单位(unit)** | 测量值的单位（例如：mm、N、kgf）。提示按何种单位测量·输入。 |
| **基准/规格(standard)** | 项目设置的检验基准说明文字（当无数字测量规格时的判断标准等）。 |
| **测量值(measureValue)** | 输入实际测量的数值（可选输入）。判定型(VISUAL)项目可留空。输入后一并记录到检验结果。 |
| **备注(remark)** | 记录特记事项·判定原因等备注（可选输入）。 |
| **合格 (PASS)** | 确定检验结果为**合格**。与输入的测量值·备注一同保存，并从等待列表中移除。 |
| **不合格 (FAIL)** | 确定检验结果为**不合格**。不合格处理后后续品质措施（不良处理等）另行按程序进行。 |

> 判定基准设定为LSL/USL的项目，测量值**≥LSL且≤USL**时通常视为合格。但本画面的合格/不合格由负责人直接点击**手动判定**，因此参考规格后由负责人做最终判断。

## 使用顺序
1. 在左侧**等待列表**中点击要处理的项目（旧请求在上方）。
2. 在右侧确认**检验基准(LSL/USL·单位)**。
3. 实际测量·确认后输入**测量值**（如需写**备注**），根据结果点击**合格(PASS)**或**不合格(FAIL)**。
4. 处理完成的项目从等待列表中移除，该工单的Kiosk生产实绩输入阻止解除。
5. 列表未刷新时点击右上方**刷新**。

## 输入规则 / 校验
- 必须选择项目才能激活结果输入·判定按钮。
- 测量值和备注均为**可选输入**。判定型（外观等）项目可不输入测量值直接点击合格/不合格。
- 测量值仅输入数字。

## 常见问题
- **问：** 直接检验与委托检验有何不同？
  **答：** 直接检验(DIRECT)由操作员在Kiosk中直接判定。委托检验(DELEGATE)由操作员不判定而转交品质负责人，在本画面进行判定。
- **问：** 不在本画面处理会怎样？
  **答：** 该工单留有等待(PENDING)委托检验时，Kiosk生产实绩输入被阻止。需尽快判定以继续生产。
- **问：** LSL/USL不显示。
  **答：** 该项目为判定型(VISUAL)或未设置规格(LSL/USL)时。此时显示"判定型项目（无规格）"或"未设置规格"，由负责人根据目视·基准判定。
