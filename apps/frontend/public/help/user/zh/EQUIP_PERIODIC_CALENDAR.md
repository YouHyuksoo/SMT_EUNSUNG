# 定期点检日历

## 画面目的
本画面以月度日历管理设备的**定期点检**（周期性预防保全点检）。
根据各设备点检项目主数据中定义的**周期（cycle）**，自动计算"哪天需要点检哪台设备"并显示在日历上，选择日期后按设备输入点检结果（合格/不合格）。

> 与日常点检日历的画面结构和操作完全相同，仅**点检类型为定期点检（PERIODIC）**。点检项目和结果与日常点检分开管理。

## 画面构成
- **顶部标题栏**：工序筛选、当月/次月生成按钮、刷新
- **月度统计卡片（4种）**：计划总数 / 完成 / 不合格 / 延迟
- **左侧 — 日历**：以颜色和进度条显示各日期点检进度
- **右侧 — 日期面板**：所选日期需要点检的设备列表 + 执行点检/单独添加
- **点检执行弹窗**：按项目输入合/不合并保存

## 概念关系
```
各设备点检项目（周期 cycle）──自动排程──▶ 日历（各日期点检对象）──选择日期──▶ 按设备执行点检（按项目合/不合）──▶ 综合判定（PASS/FAIL）
```

---

## ① 顶部标题栏

| 项目 | 功能 / 含义 |
|------|------|
| **工序筛选** | 日历和日期面板中仅显示特定工序的设备。留空则显示所有工序。 |
| **生成当月** | 跳转至当月，重新计算并显示当月点检计划。 |
| **生成次月** | 跳转至下月，提前查看下月的点检计划。 |
| **刷新** | 重新加载当前显示月份的数据。 |

## ② 月度统计卡片

| 卡片 | 功能 / 含义 |
|------|------|
| **计划总数** | 本月计划点检的（设备×日期）件数合计，根据周期（cycle）自动计算。 |
| **完成** | 已输入点检结果并完成的件数。 |
| **不合格** | 综合判定为**不合格（FAIL）**的件数。 |
| **延迟** | 点检预定日已过但尚未点检的**延迟（OVERDUE）**天数。 |

## ③ 日历

每个日期格展示当天的点检进度一览。

| 显示 | 功能 / 含义 |
|------|------|
| **比率文本（N/M）** | 完成件数 / 计划总件数。 |
| **进度条** | 以条状显示完成比率。 |
| **状态颜色** | 按日期以颜色区分状态（见下方）。 |

> 日期状态（status）
> - **全部合格（ALL_PASS）**：所有计划点检全部完成且无不合格 — 绿色
> - **存在不合格（HAS_FAIL）**：有不合格件 — 红色
> - **进行中（IN_PROGRESS）**：部分完成 — 黄色
> - **延迟（OVERDUE）**：预定日已过但尚未开始 — 红色边框
> - **计划中（NOT_STARTED）**：尚未到来的未来预定日
> - **无（NONE）**：当天无点检计划

## ④ 日期面板 — 设备卡片

所选日期需要点检的设备以卡片形式列出。

| 项目 | 功能 / 含义 |
|------|------|
| **设备编码 / 设备名称** | 点检对象设备。 |
| **产线 / 设备类型** | 设备的产线编码和类型（参考信息）。 |
| **状态图标** | 表示合格（绿色√）/ 不合格（红色×）/ 未点检（灰色时钟）。 |
| **综合判定** | 点检完成后以合格（PASS）/不合格（FAIL）标签显示。 |
| **点检项目标签** | 该设备的点检项目。各项目显示 ✅合格 / ❌不合格 / ⏳未点检。 |
| **点检人** | 执行点检的作业员姓名。 |
| **执行/编辑按钮** | 未点检时为**执行点检**，完成后为**编辑**以修改结果。 |
| **单独添加点检** | 面板右上角按钮。即使是当天自动排程中没有的设备，也可以直接选择添加点检。 |

## ⑤ 点检执行弹窗

| 项目 | 功能 / 含义 |
|------|------|
| **点检日期** | 自动设为所选日期（不可修改）。 |
| **点检人** | 选择执行点检的作业员。 |
| **点检项目（顺序·项目名·判定标准·标准图片）** | 显示主数据中定义的项目和判定标准。有标准图片时一并显示。 |
| **结果（合格/不合格）** | 按项目选择合格（PASS）/不合格（FAIL）。 |
| **不合格原因（备注）** | 将项目设为不合格时输入原因。 |
| **综合判定** | 根据输入结果自动计算。**只要有1项不合格即综合不合格（FAIL）**，全部合格则为合格（PASS）。 |
| **整体备注** | 关于整体点检的备注（选填）。 |

> ⚠️ 综合判定以**不合格（FAIL）**保存后，该设备将自动设为**互锁（INTERLOCK，使用锁定）**状态。处置后需将设备状态恢复为正常。

## ⑥ 单独添加点检弹窗

| 项目 | 功能 / 含义 |
|------|------|
| **选择设备** | 选择要点检的设备。选择后将从主数据自动加载该设备的**定期点检项目**。 |
| **下一步** | 以加载的项目打开点检执行弹窗。 |

## 使用顺序
1. 通过**工序筛选**（如有需要）和**生成当月/次月**按钮显示要点检的月份。
2. 在日历中**点击日期**，右侧将显示当天的点检对象设备。
3. 点击设备卡片的**执行点检**，按项目输入**合格/不合格**（不合格需输入原因）。
4. **保存**后综合判定自动计算，日历进度随之更新。
5. 需要对自动排程之外的设备进行点检时，使用**单独添加点检**。

## 输入规则 / 验证
- 点检项目须**全部选择合格/不合格**后综合判定才能确定（有未输入项目时，保存前将弹出确认提示）。
- **不合格项目**原则上需要输入原因（备注）。
- 点检日期固定为日历中所选的日期。
- 同一设备、同一日期的定期点检以1件管理，重新点检时通过**编辑**修改。

## 常见问题
- **Q.** 日历中看不到点检对象。
  **A.** 需要在各设备点检项目主数据中注册**定期点检（PERIODIC）项目**和**周期（cycle）**。根据周期，仅在相应日期显示（周间=周一，月间=每月1日）。
- **Q.** 与日常点检有什么区别？
  **A.** 画面操作相同，但点检类型为定期点检，项目和结果与日常点检分开统计。
- **Q.** 以不合格保存后设备被锁定了。
  **A.** 不合格时设备将进入互锁（使用锁定）状态。原因处置后，请将设备状态切换为正常。
- **Q.** 延迟（OVERDUE）什么时候显示？
  **A.** 点检预定日已过但完全没有点检记录的日期将显示延迟。
