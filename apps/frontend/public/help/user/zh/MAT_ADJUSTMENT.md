# 库存调整

## 画面目的
手动**调整(Adjust)**实际仓库库存数量与系统记录库存数量之间的差异，并将该明细作为历史进行管理。用于反映库存盘点结果、更正出入库错误、处理破损·丢失等。

## 画面构成
- **上方 — 3个统计卡片**：一目了然查看总调整件数 / 增加调整件数 / 减少调整件数。
- **下方 — DataGrid列表**：按日期·物料·数量变动·原因等显示调整历史。可通过期间筛选和搜索词缩小查询范围。
- **弹窗 — 调整注册**：点击右上角**调整注册**按钮打开弹窗注册新调整。注册后立即反映到库存(无需审批)。

---

## ① DataGrid列表列

| 列 | 作用 / 含义 |
|------|------|
| **处理日(createdAt)** | 调整注册的日期时间。仅显示年月日，可通过期间筛选指定查询范围。 |
| **仓库(warehouseCode)** | 发生调整的仓库代码。 |
| **物料代码(itemCode)** | 调整目标物料代码。 |
| **物料名称(itemName)** | 调整目标物料名称。 |
| **调整前(beforeQty)** | 调整前的系统库存数量。同时显示单位。 |
| **调整后(afterQty)** | 调整后的系统库存数量。 |
| **差异(diffQty)** | 调整前后的数量差异。**正数(蓝色)** = 库存增加，**负数(红色)** = 库存减少。 |
| **原因(reason)** | 执行调整的原因。 |
| **处理人(createdBy)** | 注册调整的用户。 |

---

## ② 调整注册弹窗字段

| 字段 | 作用 / 含义 |
|------|------|
| **仓库(warehouseCode)** | 选择要调整库存的仓库。 |
| **物料搜索(partSearch)** | 搜索并选择要调整的物料。输入2个以上字符的物料代码或物料名称，搜索结果将显示在下拉列表中。 |
| **调整后(afterQty)** | 输入调整应用后的**最终数量**。与调整前数量的差异自动计算并记录于差异(diffQty)。 |
| **原因(reason)** | 输入调整原因。例如："盘点差异调整"、"漏入库更正"、"破损废弃"等。 |

---

## 使用顺序
1. 点击上方**调整注册**按钮打开弹窗。
2. 选择仓库，搜索并选择物料。
3. 输入调整后的最终数量和原因并保存(注册后立即反映库存)。
4. 在列表中确认结果。差异(diffQty)为正数表示库存增加，为负数表示库存减少。

## 输入规则 / 验证
- 仓库·物料·数量·原因均为**必填项**。任意一项为空则保存按钮将停用。
- 调整后数量(afterQty)必须为**0以上**的整数。

## 常见问题
- **问：** 调整注册错误可以取消吗？**答：** 注册后无取消(删除)功能。需重新注册反向调整(如增加则再次减少)以恢复原状。
- **问：** 调整与入库·出库有何区别？**答：** 入库/出库为正常库存变动(采购入库、生产出库)，调整是为使系统数量与实际数量一致而进行的**例外调整**。
- **问：** 调整历史保存多久？**答：** 如无单独删除政策则永久保存(定期销毁政策按运营政策执行)。
