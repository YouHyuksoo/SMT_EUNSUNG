# 物料出库管理（量产）

## 画面目的
根据生产计划审核·批准·出库现场请求的**物料出库请求**，或仓库人员通过**LOT条码扫描**即时全量出库的画面。出库账户固定为**量产(PRODUCTION)**，量产以外的账户（不良·样品·外协·报废等）在[其他出库管理](/material/issue-other)中处理。

> 流程：创建出库请求（现场）→ 审批（仓库/管理员）→ 出库处理 → 库存扣减(MatStock)

## 画面构成
画面由三个标签页组成：

| 标签 | 作用 |
|---|---|
| **出库请求处理** | 查询现场提交的出库请求列表。执行审批·驳回·出库处理操作 |
| **条码扫描** | 直接扫描LOT条码进行即时全量出库处理 |
| **出库历史** | 查询包含已完成·已取消的全部出库历史及出库取消 |

---

## ① 出库请求处理标签

### 统计卡片

| 卡片 | 含义 |
|---|---|
| **请求** | 尚未审批的待处理出库请求数量 |
| **审批** | 已批准等待出库处理的数量 |
| **完成** | 出库处理已完成的数量 |
| **驳回** | 已驳回处理的数量 |

### 请求列表列

| 列 | 作用 / 含义 |
|---|---|
| **请求编号(requestNo)** | 出库请求的唯一标识号。编号格式为`REQ-YYYYMMDD-NNN`。 |
| **请求日期(requestDate)** | 出库请求登记的日期。 |
| **工单(workOrderNo)** | 与该出库请求关联的生产工单编号。基于工单的请求以蓝色突出显示。 |
| **物料数(itemCount)** | 该请求包含的物料行数。 |
| **总数量(totalQty)** | 请求的所有物料数量合计。 |
| **出库账户(issueType)** | 出库类型。本画面固定为**量产(PRODUCTION)**。 |
| **状态(status)** | 当前处理状态（参见下面状态值）。 |
| **请求人(requester)** | 提出出库请求的人员。 |
| **处理(actions)** | 根据状态显示审批·驳回·出库处理按钮。 |

### 出库请求状态值

| 状态代码 | 显示 | 含义 |
|---|---|---|
| `REQUESTED` | 请求 | 出库请求已提交，等待审批。 |
| `APPROVED` | 审批 | 审批完成，等待出库处理。 |
| `COMPLETED` | 完成 | 所有物料已出库处理完成。 |
| `REJECTED` | 驳回 | 请求已被驳回。 |

### 出库处理模态框列（审批后出库处理时）

| 列 | 作用 / 含义 |
|---|---|
| **物料代码(itemCode)** | 出库物料的代码（基于物料主数据）。 |
| **物料名称(itemName)** | 出库物料的名称。 |
| **请求数量(requestQty)** | 现场请求的数量。 |
| **已出库(issuedQty)** | 已出库处理的数量（部分出库累计）。 |
| **剩余(remainQty)** | 尚未出库的剩余数量（= 请求数量 − 已出库）。 |
| **出库LOT(matUidSelect)** | 选择要出库的物料LOT。选择时以`LOT序列号 / 仓库名 / 可用数量`格式显示。 |
| **出库数量(issueQty)** | 输入实际出库数量。不得超过剩余数量。 |
| **单位(unit)** | 数量单位（基于物料主数据）。 |

## ② 条码扫描标签

扫描LOT条码时将该LOT的**全部剩余库存即时出库**。无需调整个别数量，适用于需要快速出库的场合。

### 扫描结果信息

| 项目 | 作用 / 含义 |
|---|---|
| **物料代码** | 扫描LOT的物料代码。 |
| **物料名称** | 扫描LOT的物料名称。 |
| **物料序列号(matUid)** | 扫描LOT的唯一序列号(LOT标识值)。 |
| **当前数量** | 该LOT的当前库存数量。全量出库时将扣减此数量。 |
| **IQC** | 进货检验结果。非PASS状态或保留(HOLD)的LOT将被阻止出库。 |
| **供应商** | 交付物料的供应商名称。 |

### 当日扫描出库历史

| 列 | 作用 / 含义 |
|---|---|
| **时间(issuedAt)** | 扫描出库处理时间。 |
| **物料代码** | 出库物料的代码。 |
| **物料名称** | 出库物料的名称。 |
| **物料序列号(matUid)** | 出库LOT的序列号。 |
| **出库数量(issueQty)** | 出库的数量。 |
| **单位(unit)** | 数量单位。 |

| 过滤项 | 作用 / 含义 |
|---|---|
| **出库日期（期间）** | 查询的出库日期范围。默认值为当天。 |

| **状态(statusFilter)** | 完成/取消过滤。 |
| **出库账户(issueTypeFilter)** | 按出库类型过滤（公共代码ISSUE_TYPE）。 |

### 历史列

| 列 | 作用 / 含义 |
|---|---|
| **出库编号(issueNo)** | 出库处理记录的唯一标识号。 |
| **出库日期(issueDate)** | 出库处理的日期。 |
| **物料代码(itemCode)** | 出库物料的代码。 |
| **物料名称(itemName)** | 出库物料的名称。 |
| **物料序列号(matUid)** | 出库LOT的序列号。 |
| **数量(issueQty)** | 出库数量（含单位显示）。 |
| **出库账户(issueType)** | 出库类型（公共代码ISSUE_TYPE）。 |

## 使用顺序

### 出库请求处理方式
1. 点击**出库请求处理**标签。
2. 在请求列表中确认目标。可使用状态过滤或文本搜索。
3. 对状态为**请求(REQUESTED)**的记录，点击勾选（审批）图标进行审批。如需驳回则点击X图标并输入原因。
4. 对状态为**审批(APPROVED)**的记录，点击循环（出库处理）图标。
5. 在出库处理模态框中为每个物料选择**出库LOT**，确认**出库数量**（必要时修改）后点击**出库**按钮。
6. 所有物料数量完全出库后，请求状态变为**完成(COMPLETED)**。

### 条码扫描方式
1. 点击**条码扫描**标签。
2. 在输入框中扫描LOT条码或直接输入物料序列号后按Enter。
3. 确认LOT信息（物料名称、当前数量、IQC状态等）。
4. 点击**全量出库**按钮。
5. 下方"当日扫描出库历史"中将添加处理记录，输入框自动获取焦点可继续下次扫描。

---

## 输入规则 / 校验

- 出库处理时**未选择出库LOT**将阻止出库。
- 出库数量**不得超过剩余数量**。
- 条码扫描出库仅允许**IQC合格(PASS)**状态且**非保留(HOLD)**的LOT。
- 条码扫描时库存已耗尽的LOT将被拒绝出库。
- 出库取消仅在**完成(DONE)**状态下可用，取消原因必须输入。
- 若后续工序（生产实绩）已进行，出库将被阻止取消。

## 常见问题

- **问：** 出库请求列表中看不到记录。
  **答：** 请确认状态过滤是否设置了特定状态。尝试改为"全部状态"或清除搜索词。

- **问：** 出库处理模态框中LOT列表为空。
  **答：** 该物料无可用库存（IQC合格、可用数量>0）时会出现此情况。请确认入库情况。

- **问：** 条码扫描时出现"IQC不合格LOT"错误。
  **答：** 该LOT未完成进货检验(IQC)合格处理。请向IQC负责人请求完成检验。

- **问：** 无法取消出库。
  **答：** 后续工序（生产实绩）已进行时取消会被阻止。请按生产实绩→FG标签→箱/OQC→托盘→出货顺序逆向处理后重试。

- **问：** 需按量产以外账户（样品、不良等）出库？
  **答：** 请使用[其他出库管理](/material/issue-other)画面。
