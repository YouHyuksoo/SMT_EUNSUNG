# 仓库/位置管理

## 画面目的
注册和管理库存存放·移动的**仓库**以及仓库内的**详细装载位置（位置）**。此处登记的仓库/位置是入库·发放·库存·生产实绩·库存移动的基础。

## 画面构成
上方标签分为2个区域。右上角按钮（刷新/登记）根据所选标签切换。

- **仓库管理标签**：仓库主数据列表（左侧网格）+ 搜索·仓库类型过滤。行✏️（修改）/🗑️（删除），右上角**新建登记**。点击时打开登记/修改模态框。
- **位置标签**：仓库内详细位置列表 + 仓库过滤。位置的登记/修改/删除。

两个区域的关系：
```
仓库(WAREHOUSES)
  └─ 位置(WAREHOUSE_LOCATIONS)  : 一个仓库内有多个详细位置
```

---

## ① 仓库管理标签 — 列/表单字段

| 列/字段 | 作用 / 含义 |
|------|------|
| **仓库代码(warehouseCode)** | 标识仓库的**唯一代码**。注册后不可更改（库存·实绩通过此代码关联）。 |
| **仓库名称(warehouseName)** | 现场识别的仓库名称。 |
| **仓库类型(warehouseType)** | 仓库的用途分类。入库·库存·生产处理方式据此值变化。值：原材料仓库(RAW)·半成品仓库(WIP)·成品仓库(FG)·工序在制品(FLOOR)·不良仓库(DEFECT)·报废仓库(SCRAP)·外协处(SUBCON)。 |
| **线别(lineCode)** | **工序在制品(FLOOR)**仓库所属的生产线。仅在类型为FLOOR时输入。 |
| **工序(processCode)** | **工序在制品(FLOOR)**仓库所属的工序。仅在类型为FLOOR时输入。 |
| **默认(isDefault)** | 是否为同一类型中默认使用的仓库。自动入库/放置时按类型优先选择默认仓库。 |
| **使用(useYn)** | 仅`Y`时为查询·选择·使用对象。停用为`N`。 |

> 仅仓库类型为**工序在制品(FLOOR)**时显示线别·工序输入框。其他类型不使用线别/工序。

## ② 位置标签 — 列/表单字段

一个仓库可设置多个位置（bin/rack/shelf等详细位置）。

| 列/字段 | 作用 / 含义 |
|------|------|
| **仓库(warehouseCode)** | 该位置所属的仓库。注册后不可更改。 |
| **位置代码(locationCode)** | 标识仓库内详细位置的代码。注册后不可更改（仓库+位置代码组合唯一）。 |
| **位置名称(locationName)** | 现场识别的位置名称。 |
| **区域(zone)** | 仓库内区域划分值。 |
| **行(rowNo)** | 货架/搁板的行编号。 |
| **列(colNo)** | 货架/搁板的列编号。 |
| **层(levelNo)** | 货架/搁板的层编号。 |
| **使用(useYn)** | 仅`Y`时为使用对象（列表中以●绿色点显示）。 |
| **备注(remark)** | 位置管理参考事项。 |

## 使用顺序
1. 在**仓库管理标签**中通过右上角**新建登记**创建仓库（仓库代码·仓库名称·仓库类型必填，FLOOR时还需线别·工序）。
2. 如需在**位置标签**中为该仓库登记详细位置（区域/行/列/层）。
3. 按类型指定用于自动放置的仓库时勾选**默认**复选框。

## 输入规则 / 校验
- 仓库代码·位置代码注册后不可更改（修改时代码框禁用）。
- 同一仓库代码、同一（仓库+位置）组合不可重复登记。
- **有库存残留的仓库不可删除。**

## 常见问题
- **问：** 线别/工序输入框不显示。
  **答：** 需将仓库类型选为**工序在制品(FLOOR)**后才显示线别·工序输入框。
- **问：** 删除仓库时失败。
  **答：** 该仓库有残留库存时删除被拒绝。请清空库存后重试，或将使用与否改为`N`停用。
