# 日常点检

## 画面目的
登记和管理每日作业前后进行的设备日常点检(DAILY)结果。在左侧设备列表中选择目标设备，在右侧面板中对各点检项目进行合格/不合格(PASS/FAIL)判定并保存。

## 画面构成
- **左侧 — 设备列表(EquipListPanel)**：按设备类型分组显示当日的点检对象设备列表。提供搜索筛选和状态筛选(未点检/完成OK/完成NG)功能。
- **右侧 — 点检输入(InspectEntryPanel)**：显示所选设备的点检项目列表和判定输入表单。

---

## ① 左侧设备列表列

| 项目 | 作用/含义 |
|------|------|
| **设备代码** | 各设备的唯一标识代码。 |
| **设备名称** | 设备名称。 |
| **点检人** | 对该设备进行点检的操作人员姓名。 |
| **结果** | 点检综合结果(未点检/合格/不合格)。 |
| **项目数** | 该设备已登记的点检项目数量。 |

## ② 点检输入面板构成

| 项目 | 作用/含义 |
|------|------|
| **点检日期** | 点检执行日期(默认值：今天)。 |
| **点检人（必填）** | 选择执行点检的操作人员。从操作人员主数据查询。 |
| **开始时间** | 点检开始时间。 |
| **点检项目列表** | 显示关联到所选设备的点检项目。 |
| **判定输入** | 各项目选择OK(合格)或NG(不合格)，测量型则输入数值。 |
| **综合判定** | 根据所有项目的判定结果自动显示。 |

## 点检项目列表列

| 列 | 作用/含义 |
|------|------|
| **图片** | 点检参考图片(缩略图)。 |
| **点检项目** | 点检项目主数据的项目名称。 |
| **类型** | 判定型(VISUAL)或测量型(MEASURE)区分。 |
| **基准** | 合格/不合格判断基准或测量基准范围。 |
| **测量值/输入** | 测量型输入测量值，判定型选择OK/NG。 |
| **判定** | OK(合格)或NG(不合格)结果。 |

## 使用顺序
1. 在左侧设备列表中选择要点检的设备。
2. 在右侧面板中确认/选择点检日期和点检人。
3. 输入各点检项目的结果：
   - **判定型(VISUAL)**：通过下拉菜单或按钮选择OK/NG
   - **测量型(MEASURE)**：以数字形式输入实际测量值
4. 如果有NG(不合格)项目，输入不良内容。
5. 点击**保存（PASS）**或**保存（NG）**按钮保存结果。

## 输入规则
- 点检人为**必填项**。
- 必须对所有点检项目输入判定结果才能保存。
- 测量型若超出LSL(下限)和USL(上限)范围，将自动判定为NG。
- 若为NG(不合格)，需要输入原因/备注。
