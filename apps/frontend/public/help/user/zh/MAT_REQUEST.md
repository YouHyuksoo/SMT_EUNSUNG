# 出库请求管理

## 画面目的

向**仓库人员请求出库**工单所需原物料的画面。基于BOM自动计算需求量，减轻数量输入负担，并在一个画面中追踪请求·审批·出库的完整流程。

> 流程：出库请求(REQUESTED) → 审批(APPROVED) → 实际出库(COMPLETED) / 驳回(REJECTED)

以关联工单为默认方式，也可不通过工单直接搜索物料进行**手动出库请求**。

## 画面构成

- **上方标题**：画面标题 + [手动出库请求]按钮。
- **过滤栏**：输入工单编号、型号（物料代码/物料名称）、工单状态以缩小左侧列表范围。
- **左侧 — 工单列表**：以卡片形式列出当前存在的工单。点击后右侧显示该工单的详情。
- **右侧 — 出库请求详情 / 新建**：选择工单后按请求记录分组显示现有出库请求，点击[新建]切换为BOM基准物料网格。未选择工单时默认显示最近出库请求列表。

---

## ① 过滤栏项目

| 项目 | 作用 / 含义 |
|------|------|
| **工单编号** | 输入特定工单编号过滤左侧列表。输入即搜索（300ms延迟）。 |
| **型号（物料代码/物料名称）** | 按工单关联产品的物料代码或物料名称过滤。 |
| **工单状态** | 按公共代码`JOB_ORDER_STATUS`选择状态（WAITING/RUNNING/HOLD/DONE等）。仅查看进行中的工单时使用。 |
| **重置** | 清除所有已输入的过滤条件。至少输入一个过滤条件时显示。 |

---

## ② 工单列表（左侧）列

| 项目 | 作用 / 含义 |
|------|------|
| **工单编号(orderNo)** | 工单标识号。点击选择该工单，在右侧加载详情。 |
| **物料名称(itemName)** | 生产产品的物料名称。 |
| **物料代码(itemCode)** | 生产产品的物料代码（显示在物料名称下方小字）。 |
| **计划数量(planQty)** | 工单的生产目标数量。BOM需求量计算的基础。 |
| **状态(status)** | 按公共代码`JOB_ORDER_STATUS`的标签显示（例如：等待/进行/完成）。 |

---

## ③ 出库请求详情（右侧 — 详情模式）

选择工单后显示关联该工单的出库请求记录，按组显示。

### 请求组标题

| 项目 | 作用 / 含义 |
|------|------|
| **请求编号(requestNo)** | 出库请求标识号。点击打开详情模态框。 |
| **状态(status)** | 请求的当前状态标签（参见下面状态定义）。 |
| **请求日期(requestDate)** | 出库请求登记的日期·时间。 |
| **物料数** | 该出库请求包含的物料记录数。 |
| **请求数量合计(totalRequestQty)** | 该请求中所有物料的请求数量总和。 |
| **请求人(requester)** | 提出出库请求的人员。 |

### 请求组物料表

| 列 | 作用 / 含义 |
|------|------|
| **物料代码(itemCode)** | 出库请求物料的代码。 |
| **物料名称(itemName)** | 出库请求物料的名称。 |
| **单位(unit)** | 数量单位（EA、M、KG等）。 |
| **请求数量(requestQty)** | 向仓库请求的出库数量。 |
| **已出库(issuedQty)** | 该请求中实际已出库的累计数量。与请求数量相等时表示完成。 |

---

## ④ 新建 — BOM物料网格（右侧 — 新建模式）

点击[新建]按钮后基于工单的BOM自动计算并显示所需原材料列表。

| 列 | 作用 / 含义 |
|------|------|
| **物料代码(itemCode)** | 所需原材料的代码。 |
| **物料名称(itemName)** | 所需原材料的名称。 |
| **单位(unit)** | 数量单位。 |
| **BOM需求(bomReqQty)** | 基于工单计划数量的BOM需求量（= BOM单位需求量 × 计划数量）。 |
| **已发放(prevIssueQty)** | 该工单中已发放的累计数量。从BOM需求中减去此值可得实际还需的数量。 |
| **现场库存(floorStockQty)** | 现场（工艺线）已持有的数量。现场库存充足时可减少出库请求数量。 |
| **当前库存(currentStock)** | 仓库当前库存数量。请求数量超过当前库存时显示红色警告。 |
| **请求数量(requestQty)** | 直接输入出库请求数量。数量为0的物料将从请求中排除。 |

输入请求数量后，在上方选择**[请求原因]**（生产投入/试制品/样品/其他）并点击**[请求登记]**。

## ⑤ 出库请求详情模态框

点击请求编号时打开详情模态框。

| 项目 | 作用 / 含义 |
|------|------|
| **请求编号(requestNo)** | 出库请求唯一标识号。 |
| **工单(workOrderNo)** | 关联的工单编号。无关联显示'-'。 |
| **状态(status)** | 当前请求状态标签。 |
| **出库账户(issueType)** | 按公共代码`ISSUE_TYPE`的出库账户区分（例如：生产投入、样品等）。 |
| **请求日期时间(requestDate)** | 出库请求登记的日期·时间。 |
| **请求人(requester)** | 登记请求的人员。 |
| **审批人(approver)** | 审批请求的人员。未审批显示'-'。 |
| **审批日期时间(approvedAt)** | 审批处理的日期·时间。 |
| **请求数量 / 出库数量 / 剩余数量** | 请求整体的汇总值。剩余 = 请求 − 出库。 |
| **备注(remark)** | 请求时输入的原因或备注。 |
| **驳回原因(rejectReason)** | 被驳回时的原因显示。 |

### 详情模态框物料表

| 列 | 作用 / 含义 |
|------|------|
| **物料代码/物料名称** | 请求物料标识信息。 |
| **请求(requestQty)** | 请求的数量。 |
| **出库(issuedQty)** | 实际出库的数量。 |
| **剩余** | 尚未出库的数量（= 请求 − 出库）。 |
| **当前库存(currentStock)** | 请求时的仓库库存数量。 |
| **BOM需求(bomReqQty)** | BOM基准需求量。 |
| **已发放(prevIssueQty)** | 该工单中已发放的既有数量。 |
| **现场库存(floorStockQty)** | 现场持有的数量。 |
| **单位(unit)** | 数量单位。 |

## ⑥ 手动出库请求模态框

无工单但需要原材料时使用上方[手动出库请求]按钮。

1. **工单**：选择后将基于BOM自动填充物料。也可不选择。
2. **请求原因**：选择生产投入/试制品/样品/其他。
3. **物料搜索**：输入物料代码或物料名称进行搜索。确认搜索到的物料当前库存后，通过[+]添加到请求列表。
4. **输入请求数量**：输入各物料的数量。超过当前库存时显示警告。
5. 点击**请求登记**按钮。

---

## 出库请求状态值

| 状态代码 | 画面显示 | 含义 |
|------|------|------|
| **REQUESTED** | 等待（黄色） | 出库请求已登记，等待仓库人员审批。 |
| **APPROVED** | 审批（蓝色） | 已批准，可进行实际出库处理。 |
| **REJECTED** | 驳回（红色） | 请求被驳回。确认驳回原因后重新请求。 |
| **COMPLETED** | 出库完成（绿色） | 批准后实际出库已完成。 |

---

## 使用顺序（基于工单请求）

1. 在过滤栏中输入工单编号或型号名称查找目标工单。
2. 在左侧点击选择工单。
3. 在右侧详情模式中确认是否有现有请求。
4. 点击[新建]切换为BOM基准物料网格。
5. 参考各物料的BOM需求·已发放·现场库存·当前库存，输入请求数量。
6. 选择请求原因后点击[请求登记]。
7. 在右侧详情模式中确认登记的请求记录和状态。

> 请求数量为0的物料将自动排除。至少1个物料的数量≥1才能登记。

---

## 输入规则

- 请求数量必须为正整数（≥1）。
- 手动出库请求模态框中，所有物料均需输入数量才能登记（有数量为0的物料则不可）。
- BOM新建模式下，数量为0的物料将被排除后登记。
- 请求数量超过当前库存时显示警告，但仍可登记（不阻止）。

---

## 常见问题

- **问：** BOM基准数量显示为0。
  **答：** 可能是物料主数据未登记BOM，或工单计划数量为0。请手动输入数量或检查BOM。

- **问：** 能否请求超过当前库存的数量？
  **答：** 会显示警告但可登记。实际出库仅在仓库库存范围内进行。

- **问：** 出库请求被驳回了。
  **答：** 在详情模态框中确认驳回原因，解决问题后通过新建重新请求。

- **问：** 状态为APPROVED但库存未反映。
  **答：** 审批仅表示同意出库请求，实际库存变动需在物料出库管理画面进行实际出库处理后才会发生。

- **问：** 手动出库请求中搜索不到原材料以外的物料。
  **答：** 出库请求物料搜索仅针对原材料(RAW_MATERIAL)物料。
