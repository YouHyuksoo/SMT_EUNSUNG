# 不良代码管理

## 画面目的
登记·管理在检验和生产实绩中记录不良时选择的**不良代码**。不良代码不直接使用，而是先归属于**3级分类**（检验阶段→型号区分→不良类型）之下，再附加**不良等级**和**适用范围**进行管理。

> 3级分类为：**1级 = 检验阶段**（IQC进货检验 / LQC工序检验 / OQC出货检验）、**2级 = 型号区分**（低电压LV / 高电压HV）、**3级 = 不良类型**（功能 / 外观 / 其他）。不良代码必须关联到**3级分类**。

## 画面构成
- **左侧 — 已登记的不良全部**：以表格显示所有不良代码。点击行时在右侧修改。上方搜索框可按不良代码/不良名称查找。
- **右上 — 不良代码添加/修改**：输入一条不良代码的所有项目。必须依次选择1·2·3级分类才能保存。
- **右下 — 分类快速添加**：即时创建新分类（检验阶段/型号区分/不良类型）的区域。

## 概念关系
```
检验阶段(1级) ──▶ 型号区分(2级) ──▶ 不良类型(3级) ──关联──▶ 不良代码 ──(等级/适用范围)
   IQC/LQC/OQC        低电压LV/高电压HV     功能/外观/其他         例如: SOLDER-01
```
- 型号区分(LV/HV)自动反映到不良代码的**型号区分应用列表**中。即根据选择的2级确定该不良代码用于哪些型号。

---

## ① 不良代码列表列（左侧表）

| 列 | 作用 / 含义 |
|------|------|
| **不良代码(defectCode)** | 标识不良的唯一代码。在检验·实绩画面中使用此代码选择不良。例如：`SOLDER-01`。注册后不可更改（防止关联中断）。 |
| **不良名称(defectName)** | 供人阅读的不良名称。例如：`焊接不良`。在检验画面列表中同时显示。 |
| **1级** | 该不良代码所属的**检验阶段**分类名称（IQC/LQC/OQC）。跟随3级分类自动显示。 |
| **2级** | **型号区分**分类名称（低电压/高电压）。 |
| **3级** | **不良类型**分类名称（功能/外观/其他）。不良代码实际关联的分类。 |
| **等级(defectGrade)** | 不良的严重度。**致命(CRITICAL)**=对安全·功能致命、**重(MAJOR)**=对功能影响大的重缺陷、**轻(MINOR)**=外观等轻微缺陷。是合格·不合格判定和统计汇总的依据。 |
| **适用范围(defectScope)** | 该不良的使用范围。**通用(COMMON)**=任何场合、**原材料(RAW_MATERIAL)**=进货检验等物料、**产品(PRODUCT)**=成品、**工序(PROCESS)**=生产工序。 |
| **使用与否(useYn)** | `使用(Y)`时可在检验·实绩中选择。`停用(N)`时从列表中排除。 |

---

## ② 不良代码添加/修改字段（右上）

| 字段 | 作用 / 含义 |
|------|------|
| **不良代码(defectCode)** | 新登记时输入的唯一代码。自动转换为大写。**修改时锁定不可更改**。 |
| **不良名称(defectName)** | 不良名称。必填项。 |
| **1级（检验阶段）** | 选择检验阶段分类（IQC/LQC/OQC）。必须先选择才能激活2级。 |
| **2级（型号区分）** | 选择1级下的型号区分（低电压/高电压）。此处选择的型号区分将成为该不良代码的**适用型号**。 |
| **3级（不良类型）** | 选择2级下的不良类型（功能/外观/其他）。**不良代码实际关联到此3级分类**，因此必须选择才能保存。 |
| **等级(defectGrade)** | 选择致命/重/轻之一。默认值为**重**。 |
| **适用范围(defectScope)** | 选择通用/原材料/产品/工序之一。默认值为**通用**。 |
| **使用与否(useYn)** | 选择使用/停用。 |
| **说明(description)** | 不良的补充说明·基准备注（可选）。 |

> 保存条件：**不良代码·不良名称·3级分类**均需填写才能保存。需按1·2·3级从上到下依次选择。

---

## ③ 分类快速添加字段（右下）

用于创建新的检验阶段·型号区分·不良类型分类。

| 字段 | 作用 / 含义 |
|------|------|
| **分类级别(levelNo)** | 要创建分类的级别：1级（检验阶段）/ 2级（型号区分）/ 3级（不良类型）。 |
| **上级分类(parentCategoryCode)** | 2·3级分类所属的上一级分类。**1级时留空**（无上级）。2级时选择1级，3级时选择2级。 |
| **分类代码(categoryCode)** | 分类唯一代码。转换为大写。例如：`IQC`、`IQC_LV`、`IQC_LV_FUNCTION`。 |
| **分类名称(categoryName)** | 分类名称。例如：`IQC`、`低电压`、`功能`。 |

> 分类的级别必须精确匹配。例如：2级的上级必须是1级，1级不可有上级。

---

## 使用顺序
1. （如无分类）通过**分类快速添加**按1级（检验阶段）→2级（型号区分）→3级（不良类型）顺序创建。
2. 在**不良代码添加**中依次选择1·2·3级分类，输入不良代码·不良名称·等级·适用范围后保存。
3. 登记的不良代码在[不良管理]·检验·生产实绩Kiosk的不良输入中被选择使用。

## 输入规则 / 校验
- 不良代码·不良名称·3级分类为必填项。任一为空则不保存。
- 不良代码注册后不可更改（修改时代码输入框锁定）。
- 不良代码仅关联到**3级（不良类型）分类**。不可直接关联到1·2级。
- 仅可关联到使用中的(`使用`)分类。

## 常见问题
- **问：** 等级（致命/重/轻）如何确定？
  **答：** 对安全·核心功能致命的为**致命**、对功能影响大的缺陷为**重**、外观等轻微为**轻**。等级是检验合·不合格判定和不良统计的基准。
- **问：** 为什么要选择2级（型号区分）？
  **答：** 即使同一不良，在低电压/高电压产品中管理方式可能不同，因此按型号区分分别登记·统计不良代码。所选型号区分即为该不良代码的适用型号。
- **问：** 不再使用的不良代码是否删除？
  **答：** 建议将**使用与否设为停用(N)**，避免与过去不良记录的关联中断。
