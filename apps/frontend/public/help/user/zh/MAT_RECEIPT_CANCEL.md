# 物料入库取消

## 画面目的
选择**已完成入库(DONE)的物料交易**进行取消并管理其明细。取消时原交易状态变更为`CANCELED`，在`STOCK_TRANSACTIONS`中自动生成反向符号的逆事务(Reverse Transaction)，并扣减`MatStock`库存。

> 用于退回错误入库或发现质量异常物料的入库。

## 画面构成
- **上方 — 筛选区域**: 通过**期间筛选(DateRangeFilter)**指定查询的入库期间，通过**搜索词输入**可搜索物料代码·物料名称·供应商·交易编号。**刷新(Refresh)**按钮更新列表。
- **下方 — DataGrid列表**: 显示可取消的入库(DONE)项和已取消(CANCELED)项。可通过状态筛选区分查看**DONE(正常)**/**CANCELED(已取消)**。
- **弹窗 — 取消处理**: 在列表中点击**取消**按钮即打开取消确认弹窗。确认交易信息后输入**取消原因**以确认取消。

## ① DataGrid列表列

| 列 | 作用 / 含义 |
|------|------|
| **交易日(transDate)** | 入库交易发生的日期。可通过期间筛选指定查询范围。 |
| **交易编号(transNo)** | 入库交易的唯一标识编号(PK)。 |
| **交易类型(transType)** | 交易类型。入库取消画面仅显示`RECEIVE`类型。 |
| **物料代码(itemCode)** | 入库物料的代码。 |
| **物料名称(itemName)** | 入库物料的名称。 |
| **物料序列号(matUid)** | 入库物料LOT的唯一序列号。 |
| **供应商(vendorName)** | 供货物料的客户名称。 |
| **仓库(warehouseName)** | 物料入库的仓库名称。 |
| **数量(qty)** | 入库数量。 |
| **状态(status)** | 显示为`DONE`(正常，绿色)/`CANCELED`(已取消，红色)。 |
| **操作(actions)** | **取消**按钮。`CANCELED`状态的项按钮停用。 |

## ② 取消处理弹窗

| 项目 | 作用 / 含义 |
|------|------|
| **交易编号(transNo)** | 显示取消目标原交易编号。 |
| **交易日(transDate)** | 显示原交易日期。 |
| **物料代码 / 物料名称** | 显示取消目标物料。 |
| **物料序列号(matUid)** | 显示取消目标LOT的序列号。 |
| **供应商 / 仓库** | 显示客户和仓库信息。 |
| **取消数量** | 显示原入库数量。 |

| **确认取消** | 输入原因后点击**取消**按钮确认。 |

## 使用顺序
1. 通过上方**期间筛选**选择查询的入库期间并点击**搜索**按钮。
2. 在列表中找到要取消的入库项并点击**取消**按钮。
3. 在取消弹窗中确认交易信息并输入**取消原因**。
4. 点击**确认取消**按钮后生成反向事务，该项状态变更为`CANCELED`。
5. 在列表中通过状态筛选确认取消明细。

## 输入规则 / 验证
- 取消原因为**必填项**。为空时取消按钮将停用。
- **已取消的项**不可再次取消(按钮停用)。
- 非入库类型(`RECEIVE`)的交易不显示在此画面，不属于取消对象。
- 与取消目标入库项关联的**后续作业(progress)**存在时，取消可能被阻止(例如：该LOT已投入生产的情况)。

## 常见问题
- **问：** 已取消的明细可以恢复吗？**答：** 取消后无法恢复。如需重新入库，需通过正常入库流程(MAT_RECEIVE)再次入库处理。
- **问：** 列表中没有显示入库项。**答：** 请确认期间筛选是否适当。默认为当天基准，过去的入库需扩大期间查询。此外仅显示`RECEIVE`类型。
- **问：** 取消按钮已停用。**答：** 该交易已是`CANCELED`状态，或存在后续作业(生产投入等)导致无法取消。
- **问：** 取消时库存如何处理？**答：** 按原入库数量扣减`MatStock`，并在`STOCK_TRANSACTIONS`中记录反向符号事务。
