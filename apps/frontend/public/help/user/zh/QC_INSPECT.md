# 外观检查

## 画面目的
对已完工并已发放FG标签的产品进行目视外观检查，判定合格(PASS)或不合格(FAIL)，并登记检验结果。

## 画面构成
- **左侧 — 作业指令列表**：外观检查对象的作业指令（已完成指令）列表。可按作业指令编号、品目代码进行搜索。
- **右侧 — 外观检查面板**：显示所选作业指令的FG条码扫描、合格判定及检验记录。

---

## 使用顺序
1. 从左侧列表中选择要检验的作业指令。
2. 在FG条码输入框中扫描产品条码。
3. 确认扫描到的产品信息。
4. 点击**合格(PASS)**或**不合格(FAIL)**按钮进行判定。
   - 不合格时必须输入不良代码和详细原因。
5. 检验结果将立即反映在右下方的记录网格中。

## 输入规则
- 仅可检验**ISSUED**状态的FG标签（已完成检验、包装、出货的产品不可检验）。
- 扫描的条码必须属于所选作业指令的产品。
- 不合格(FAIL)判定时必须输入不良代码（`VISUAL_DEFECT`组代码）和详细原因。
- 已检验完成的产品不可重新检验。

## 检验记录列

| 列 | 作用 / 含义 |
|------|------|
| **检验时间** | 外观检查执行的日期和时间。 |
| **判定** | 合格(PASS/绿色)或不合格(FAIL/红色)结果。 |
| **FG条码** | 被检验产品的FG条码。 |
| **不良代码** | 不合格时选择的不良代码。 |
| **详细原因** | 不合格时输入的详细不良内容。 |
