# 재고보정

## 화면 목적
실제 창고 재고 수량과 시스템에 기록된 재고 수량 간의 차이를 **수동으로 보정(Adjust)**하고 그 내역을 이력으로 관리합니다. 재고 실사(Physical Inventory) 결과 반영이나 입출고 오류 정정, 파손·분실 처리 등에 사용합니다.

## 화면 구성
- **상단 — 통계 카드 3개**: 전체 보정 건수 / 증가 보정 건수 / 감소 보정 건수를 한눈에 확인합니다.
- **하단 — DataGrid 목록**: 보정 이력을 날짜·품목·수량 변동·사유 등과 함께 표시합니다. 기간 필터와 검색어로 조회 범위를 좁힐 수 있습니다.
- **모달 — 보정 등록**: 우측 상단 **보정등록** 버튼으로 모달을 열어 새 보정을 등록합니다. 등록 즉시 재고에 반영됩니다(승인 생략).

---

## ① DataGrid 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **처리일(createdAt)** | 보정이 등록된 일시입니다. 연월일만 표시되며, 기간 필터로 조회 범위를 지정할 수 있습니다. |
| **창고(warehouseCode)** | 보정이 발생한 창고 코드입니다. |
| **품목코드(itemCode)** | 보정 대상 품목의 코드입니다. |
| **품목명(itemName)** | 보정 대상 품목의 이름입니다. |
| **보정전(beforeQty)** | 보정 전 시스템 재고 수량입니다. 단위가 함께 표시됩니다. |
| **보정후(afterQty)** | 보정 후 시스템 재고 수량입니다. |
| **차이(diffQty)** | 보정 전후 수량 차이입니다. **양수(파랑)** = 재고 증가, **음수(빨강)** = 재고 감소. |
| **사유(reason)** | 보정을 수행한 사유입니다. |
| **처리자(createdBy)** | 보정을 등록한 사용자입니다. |

---

## ② 보정 등록 모달 필드

| 필드 | 역할 / 의미 |
|------|------|
| **창고(warehouseCode)** | 보정할 재고가 있는 창고를 선택합니다. |
| **품목 검색(partSearch)** | 보정할 품목을 검색하여 선택합니다. 품목코드 또는 품목명을 2자 이상 입력하면 검색 결과가 드롭다운에 표시됩니다. |
| **보정후(afterQty)** | 보정 적용 후의 **최종 수량**을 입력합니다. 보정전 수량과의 차이가 자동으로 계산되어 차이(diffQty)에 기록됩니다. |
| **사유(reason)** | 보정을 하는 이유를 입력합니다. 예: "실사 차이 조정", "입고 누락 정정", "파손 폐기" 등. |

---

## 사용 순서
1. 상단 **보정등록** 버튼을 클릭하여 모달을 엽니다.
2. 창고를 선택하고, 품목을 검색하여 선택합니다.
3. 보정 후 최종 수량과 사유를 입력하고 저장합니다(등록 즉시 재고 반영).
4. 목록에서 결과를 확인합니다. 차이(diffQty)가 양수이면 재고 증가, 음수이면 재고 감소를 의미합니다.

## 입력 규칙 / 검증
- 창고·품목·수량·사유는 **모두 필수 입력**입니다. 하나라도 비어 있으면 저장 버튼이 비활성화됩니다.
- 보정후 수량(afterQty)은 **0 이상**의 정수여야 합니다.

## 자주 묻는 질문
- **Q.** 보정을 잘못 등록했는데 취소할 수 있나요? **A.** 등록 후에는 취소(삭제) 기능이 없습니다. 원상복구를 위해 반대 보정(예: 증가했으면 다시 감소)을 새로 등록해야 합니다.
- **Q.** 보정과 입고·출고의 차이는 무엇인가요? **A.** 입고/출고는 정규 재고 변동(구매입고, 생산출고)이며, 보정은 시스템 수량과 실제 수량의 차이를 맞추기 위한 **예외적 조정**입니다.
- **Q.** 보정 이력은 얼마나 보관되나요? **A.** 별도 삭제 정책이 없으면 영구 보관됩니다(정기 파기 정책은 운영 정책에 따름).
