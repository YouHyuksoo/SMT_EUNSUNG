# 입하실적조회

## 화면 목적
입하관리에서 등록된 입하 실적을 조회하고, 선택 입하의 LOT 시리얼 확인·라벨 재출력·제조사 변경·입하 취소를 처리하는 화면입니다.

## 화면 구성
- 상단 필터에서 상태, 품목코드, 입하번호, 공급사 조건으로 입하 실적을 조회합니다.
- 그리드 행을 선택하면 해당 입하번호의 LOT 시리얼 목록을 불러옵니다.
- 선택한 LOT 시리얼은 라벨 템플릿을 골라 재출력할 수 있습니다.

## ① 컬럼·필드

| 컬럼 | 역할 / 의미 |
|------|------|
| **작업(actions)** | 입하 상세 선택, 라벨 재출력, 취소 같은 후속 업무 진입점입니다. |
| **입하번호(arrivalNo)** | 입하 배치 식별번호입니다. LOT 시리얼, 입하재고, 입하 원장을 묶는 기준입니다. |
| **PO번호(poNo)** | 발주 기반 입하일 때 연결되는 구매발주 번호입니다. |
| **L/N(lineNo)** | PO 내 품목 라인 번호입니다. |
| **R/N(relNo)** | PO 라인 릴리즈/개정 번호입니다. |
| **입하일(arrivalDate)** | 자재가 실제 납품된 날짜입니다. |
| **등록일자(createdAt)** | MES에 입하 실적이 생성된 날짜입니다. |
| **공급사(vendorName)** | 납품 거래처입니다. |
| **제조사(mfgPartnerCode/mfgPartnerName)** | 실제 자재 제조사입니다. 공급사와 별도이며 라벨·추적성에 사용됩니다. |
| **품목코드(itemCode)** | 입하된 자재의 품목마스터 코드입니다. |
| **품목명(itemName)** | 품목 표시명입니다. |
| **입하수량(qty)** | 이번 입하 실적의 총 수량입니다. |
| **시리얼(serialCount)** | 입하에서 발급된 자재 LOT 시리얼 수입니다. |
| **구분(poType)** | 발주 기반/수동 등 입하 출처 구분입니다. |
| **상태(status)** | DONE이면 정상 입하, CANCELED이면 취소된 입하입니다. |
| **라벨 템플릿(templateKey)** | 재출력 시 사용할 mat_lot 라벨 디자인입니다. |

## 사용 순서
1. 조건을 입력하고 입하 실적을 조회합니다.
2. 행을 선택해 LOT 시리얼 상세를 확인합니다.
3. 라벨이 필요한 LOT을 체크하고 템플릿을 선택해 재출력합니다.
4. 제조사 보정이 필요하면 제조사 선택 모달에서 저장합니다.
5. 오입하 건은 DONE 상태에서 취소 처리합니다.

## 입력 규칙·검증
- 취소는 삭제가 아니라 입하 상태와 원장에 취소 이력을 남기는 방식입니다.
- 제조사 변경은 선택 입하에 연결된 MAT_LOTS의 제조사 코드를 일괄 보정합니다.
- 라벨 재출력은 선택된 LOT 시리얼이 있어야 실행됩니다.

## 자주 묻는 질문
- **Q.** 이 화면의 데이터가 다른 자재 화면과 다르게 보입니다.
  **A.** 입하재고, 창고재고, 수불원장은 서로 다른 단계입니다. 같은 자재UID(matUid)로 관련 화면을 함께 확인하세요.
- **Q.** 검색해도 데이터가 안 보입니다.
  **A.** 기간, 상태, 품목, 창고 필터를 초기화한 뒤 원본 업무 단계에서 데이터가 생성되었는지 확인하세요.
