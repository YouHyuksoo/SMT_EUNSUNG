# 제품입고

## 화면 목적

포장 공정에서 박스 번호가 부여된 완제품(FG) 박스를 창고에 입고합니다.
좌측에서 입고 이력을 확인하고, 우측 패널에서 바코드 스캔 또는 목록 체크로 박스를 선택한 뒤 창고를 지정해 일괄 입고합니다.

> 입고 처리 후 재고(제품현재고 화면)에 수량이 즉시 반영됩니다.

## 화면 구성

```
┌──────────────────────────────┬──────────────────────┐
│  ① 입고 이력 그리드           │  ② 스캔·입고 패널     │
│  (기간 필터 + 검색 + 내보내기) │  • 박스 스캔 입력     │
│                              │  • 입고창고 선택      │
│                              │  • 입고 가능 박스 목록 │
└──────────────────────────────┴──────────────────────┘
```

- **① 입고 이력 그리드**: 당일(기본) 또는 지정 기간의 제품입고(FG_IN)·입고취소(FG_IN_CANCEL) 거래 이력.
- **② 스캔·입고 패널**: 포장 완료 후 아직 창고 입고가 되지 않은 박스 목록과 스캔 입력 영역.

## ① 입고 이력 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **거래일자** | 입고 또는 입고취소가 처리된 날짜(YYYY-MM-DD). |
| **거래번호** | 거래 고유번호. 접두어 형식의 고정값이며 입고취소 시 원 번호와 연결 확인에 사용합니다. |
| **유형** | `FG_IN`(입고) 또는 `FG_IN_CANCEL`(취소). 취소는 빨간색, 입고는 파란색 배지로 구분합니다. |
| **품목코드** | 입고된 완제품 코드. |
| **품목명** | 품목 명칭. |
| **창고** | 입고된 창고명(입고 시) 또는 출고된 창고명(취소 시). |
| **수량** | 입고 수량은 `+녹색`, 취소는 `-빨간색`으로 표시합니다. 단위도 함께 표시됩니다. |
| **작업지시번호** | 해당 박스를 생산한 작업지시 번호. 없으면 `-`로 표시됩니다. |
| **상태** | 거래 처리 상태 배지(PROD_RESULT_STATUS 코드 기준). |

### 그리드 도구 모음

| 도구 | 역할 |
|------|------|
| **검색** | 텍스트를 입력해 전체 컬럼에서 빠른 필터. |
| **기간 필터** | 기본값은 당일. 기간을 변경해 이전 입고 이력을 조회합니다. |
| **새로 고침** | 현재 기간 조건으로 이력을 다시 불러옵니다. |
| **내보내기** | 조회된 이력을 엑셀로 내려받습니다. |

## ② 스캔·입고 패널 항목

| 항목 | 역할 / 의미 |
|------|------|
| **박스 스캔 입력** | 바코드 스캐너로 박스 번호를 스캔하거나 직접 입력합니다. Enter 키로 확인합니다. |
| **입고창고** | 박스를 입고할 완제품 창고를 선택합니다. 입고창고를 선택해야 입고 버튼이 활성화됩니다. |
| **선택 수량 요약** | 선택된 박스 수와 합계 수량을 실시간으로 보여줍니다. |
| **입고 버튼** | 선택된 박스를 지정 창고로 일괄 입고합니다. |
| **새로 고침(아이콘)** | 입고 가능 박스 목록을 서버에서 다시 조회합니다. |

### 입고 가능 박스 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **체크박스** | 개별 박스를 선택하거나 전체 선택/해제합니다. 행을 클릭해도 선택 토글됩니다. |
| **박스번호** | 포장 공정에서 발행된 박스 고유번호(BX... 형식). |
| **품목명** | 박스 안에 담긴 완제품 품목명. |
| **수량** | 박스 내 완제품 수량. |

## 버튼별 기능

| 버튼 | 언제 사용 | 동작 | 결과 / 주의 |
|------|------|------|------|
| **새로 고침** (헤더) | 입고 이력을 최신 상태로 갱신할 때 | 현재 기간 조건으로 이력 재조회 | 좌측 그리드만 갱신됨 |
| **새로 고침** (패널 아이콘) | 입고 가능 박스 목록이 최신인지 확인할 때 | `/shipping/box-stock` 재조회 | 우측 목록만 갱신됨 |
| **입고** | 선택 박스를 창고에 입고할 때 | 선택 박스를 순차 처리해 재고 반영 | 성공 박스는 목록에서 사라지고 이력 그리드에 나타남. 실패 박스는 빨간색 오류 메시지로 표시됨 |

## 업무 흐름

```
포장 완료(포장관리 화면)
    ↓  박스번호 생성
제품입고 화면 진입
    ↓
(방법 A) 바코드 스캐너로 박스번호 스캔 → Enter
    ↓  또는
(방법 B) 우측 목록에서 박스 행 클릭으로 체크
    ↓
입고창고 선택
    ↓
[입고] 버튼 클릭
    ↓
박스별 순차 입고 처리
    ↓
성공: 좌측 이력에 FG_IN 거래 생성 / 실패: 오류 목록 표시
```

1. 포장 공정에서 포장이 완료된 박스는 자동으로 우측 목록에 나타납니다.
2. 바코드 스캐너로 스캔하거나 목록에서 직접 클릭해 박스를 선택합니다.
3. 여러 박스를 동시에 선택한 뒤 입고창고를 지정하고 **입고** 버튼을 누릅니다.
4. 입고된 박스는 목록에서 사라지고, 좌측 이력에 `FG_IN` 거래가 기록됩니다.

## 입력 규칙 / 검증

- 입고창고를 선택하지 않으면 **입고** 버튼이 비활성 상태입니다.
- 박스를 한 개도 선택하지 않으면 **입고** 버튼이 비활성 상태입니다.
- 스캔한 박스번호가 목록에 없으면 다음 두 가지 중 하나로 안내합니다:
  - 박스가 존재하지만 입고 불가 상태(`PACKED_WAITING`이 아님, 이미 입고됐거나 취소됨) → "입고 불가능한 박스"
  - 박스번호 자체가 없음 → "박스 없음" 메시지
- 이미 입고된 박스는 목록에 나타나지 않으며, 이중입고 시도 시 서버에서 자동 차단됩니다.
- 여러 박스를 선택해 입고할 때, 일부 박스가 실패해도 나머지 박스는 계속 처리됩니다.

## 자주 묻는 질문

- **Q. 입고 가능 박스 목록이 비어 있습니다.**
  **A.** 포장관리 화면에서 포장이 완료된 박스가 있어야 나타납니다. 패널 오른쪽 위 새로 고침 아이콘을 눌러 최신 상태를 확인하세요.

- **Q. 스캔했는데 "입고 불가능한 박스"라고 나옵니다.**
  **A.** 이미 입고되었거나 아직 포장 완료 상태가 아닌 박스입니다. 포장관리 화면에서 해당 박스의 상태를 확인하세요.

- **Q. 입고 후 재고가 반영되지 않은 것 같습니다.**
  **A.** 제품현재고 화면에서 기간 조건과 창고를 확인하세요. 입고 처리가 완료되면 바로 반영됩니다.

- **Q. 일부 박스만 실패했습니다.**
  **A.** 패널 하단의 오류 목록에서 실패한 박스번호와 사유를 확인한 뒤, 해당 박스만 다시 선택해 재처리하세요.

- **Q. 입고 이력에서 FG_IN_CANCEL(취소)이 보입니다.**
  **A.** 입고취소 처리된 거래입니다. 취소는 별도의 입고취소 화면에서 수행됩니다.
