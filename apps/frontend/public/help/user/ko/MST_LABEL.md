# 라벨관리

## 화면 목적
바코드 라벨의 **디자인을 시각적으로 설계하고 템플릿으로 저장**하는 화면입니다. 캔버스 위에 바코드·텍스트·박스·선·원·이미지 객체를 배치하고, 각 객체를 데이터 소스(설비코드·자재 UID 등)에 연결합니다. 여기서 만든 디자인은 각 업무 화면(입고라벨발행·소모품라벨발행·포장 등)에서 실제 발행 시 사용됩니다. **이 화면은 디자인·템플릿만 관리하며, 실제 라벨 출력은 하지 않습니다.**

## 화면 구성
- **왼쪽 패널(소스와 객체)**: 라벨이 사용할 **소스테이블** 선택, 객체 추가 도구(글자·1D·2D·박스·선·원·이미지), 소스 필드 목록(클릭하면 글자 객체로 추가).
- **가운데(디자인 캔버스)**: 라벨 실물 비율 캔버스. 객체를 끌어 이동하고 모서리 앵커로 크기를 조절합니다. 상단에서 라벨 가로/세로(mm)를 조정합니다.
- **오른쪽(객체 속성)**: 선택한 객체의 위치(X/Y)·크기(W/H)·소스 필드·글자/바코드/선 속성·색상을 편집. 복제·삭제 버튼 포함.
- **우측 사이드(템플릿/미리보기)**: 템플릿 목록(저장·불러오기·덮어쓰기·삭제), 최종 라벨 미리보기와 크기·객체 수 표시.

라벨 디자인은 내부적으로 **DESIGN_DATA(JSON)** 형태로 저장됩니다(화면 우상단 표기).

```
소스테이블(설비/소모품/자재LOT 등) ──연결──▶ 객체(바코드·글자)의 소스 필드
디자인(객체 배치 + 라벨 크기) ──저장──▶ 템플릿(이름 + 카테고리)
```

---

## ① 라벨 전체 설정

| 컬럼 | 역할 / 의미 |
|------|------|
| **소스테이블(sourceTable)** | 이 라벨이 데이터를 가져올 대상입니다. 설비(equipment)·소모품(consumable)·작업자(worker)·자재 LOT(mat_lot)·제품포장/박스(box)·팔레트(pallet)·반제품 SG(sg_label) 중 선택. 소스를 바꾸면 객체의 소스 필드가 새 소스 기준으로 재매핑됩니다. |
| **라벨 가로(labelWidth)** | 라벨 용지의 가로 길이(mm)입니다. 캔버스 상단 `W`로 조정. |
| **라벨 세로(labelHeight)** | 라벨 용지의 세로 길이(mm)입니다. 캔버스 상단 `H`로 조정. |

## ② 객체 공통 속성

| 컬럼 | 역할 / 의미 |
|------|------|
| **객체 유형(type)** | 캔버스에 배치하는 객체 종류입니다. 글자(text)·바코드(barcode)·박스(box)·선(line)·원(circle)·이미지(image). |
| **X 위치(x)** | 객체 좌측 상단의 가로 위치(mm)입니다. 끌어서 이동하거나 직접 입력. |
| **Y 위치(y)** | 객체 좌측 상단의 세로 위치(mm)입니다. |
| **너비(width)** | 객체 가로 크기(mm)입니다. 모서리 앵커로 조절 가능. |
| **높이(height)** | 객체 세로 크기(mm)입니다. |
| **소스 필드(sourceField)** | 글자·바코드·이미지 객체가 어떤 데이터를 표시할지 지정합니다. 예: 설비 라벨의 바코드를 `equipCode`(설비코드)에 연결. 발행 시 이 필드의 실제 값으로 치환됩니다. |
| **선/글자 색(strokeColor/textColor)** | 글자 색 또는 도형 테두리 색입니다. |
| **겹침 순서(zIndex)** | 객체가 겹칠 때 위에 그려지는 순서입니다(값이 클수록 위). |

## ③ 글자(text) 객체 속성

| 컬럼 | 역할 / 의미 |
|------|------|
| **고정 문구(text)** | 소스 필드 대신 항상 같은 글자를 출력할 때 입력합니다(예: 회사명, 단위). |
| **글자 크기(fontSize, mm)** | 글자 높이(mm)입니다. |
| **글꼴(fontFamily)** | 고딕(sans-serif)·명조(serif)·고정폭(monospace) 중 선택. 코드·시리얼은 고정폭이 가독성에 유리합니다. |
| **굵기(fontWeight)** | normal(보통) / bold(굵게). |
| **정렬(align)** | left(좌) / center(가운데) / right(우). |

## ④ 바코드(barcode) 객체 속성

| 컬럼 | 역할 / 의미 |
|------|------|
| **바코드 유형(barcodeFormat)** | 바코드 종류입니다. QR Code(qrcode)·DataMatrix(datamatrix)는 2D, Code 128(code128)·Code 39(code39)는 1D입니다. 스캐너·인쇄 면적에 맞춰 선택합니다. |
| **소스 필드(sourceField)** | 바코드로 인코딩할 값입니다(예: `conUid` 소모품 UID, `boxNo` 박스번호). |

## ⑤ 도형(박스·선·원) 객체 속성

| 컬럼 | 역할 / 의미 |
|------|------|
| **선 두께(lineWidth, mm)** | 박스 테두리·선·원의 굵기(mm)입니다. |
| **선/테두리 색(strokeColor)** | 도형 외곽선 색입니다. |
| **채움 색(fillColor)** | 박스·원의 내부 채움 색입니다. `transparent`면 투명(외곽선만). |

## ⑥ 이미지(image) 객체 속성

| 컬럼 | 역할 / 의미 |
|------|------|
| **기본 이미지 URL(imageUrl)** | 로고·고정 이미지 경로입니다. 소스 필드(예: 소모품 사진 URL)에 연결해 동적으로 바꿀 수도 있습니다. |

## ⑦ 템플릿

| 컬럼 | 역할 / 의미 |
|------|------|
| **템플릿 이름(templateName)** | 저장하는 디자인의 이름입니다. 같은 카테고리 안에서 중복 불가(이름+카테고리가 식별 키). |
| **카테고리(category)** | 현재 선택한 소스테이블에 대응하는 라벨 분류입니다. 업무 화면이 이 카테고리로 디자인을 찾습니다. |
| **기본 여부(isDefault)** | ⭐ 표시. 카테고리별 기본 템플릿입니다(목록의 별 아이콘). |

## 사용 순서
1. 왼쪽에서 **소스테이블**을 고릅니다(예: 소모품).
2. 객체 도구로 **바코드·글자**를 추가하거나, 필드 목록에서 필드를 클릭해 글자 객체를 바로 추가합니다.
3. 캔버스에서 객체를 끌어 배치하고 모서리 앵커로 크기를 맞춥니다.
4. 오른쪽 **객체 속성**에서 소스 필드·글꼴·바코드 유형·색상을 다듬습니다.
5. 상단 `W`/`H`로 라벨 크기(mm)를 맞춥니다.
6. 우측 **새 저장**으로 템플릿 이름을 입력해 저장합니다. 기존 템플릿은 💾(덮어쓰기)로 갱신합니다.

## 입력 규칙 / 검증
- 템플릿 **이름이 비어 있으면 저장 버튼이 비활성화**됩니다.
- 같은 **카테고리 안에서 이름이 중복**되면 저장이 거부됩니다(중복 시 덮어쓰기 사용).
- 미저장 변경이 있는 상태에서 다른 템플릿을 불러오면 **"저장하지 않은 변경사항" 경고 모달**이 뜹니다. 계속하면 현재 작업이 사라집니다.
- 객체는 라벨 영역(가로·세로 mm) 밖으로 나가지 않도록 위치·크기가 자동 보정됩니다.

## 자주 묻는 질문
- **Q.** 이 화면에서 라벨을 바로 출력하나요?
  **A.** 아니요. 디자인·템플릿만 만듭니다. 실제 발행은 입고라벨발행·소모품라벨발행·포장 등 각 업무 화면에서 이 템플릿을 사용합니다.
- **Q.** 소스테이블을 바꾸면 기존 객체는 어떻게 되나요?
  **A.** 객체는 유지되고, 각 객체의 소스 필드가 새 소스에 같은 키가 있으면 유지, 없으면 첫 번째 필드로 자동 변경됩니다.
- **Q.** 1D와 2D 바코드의 차이는?
  **A.** 1D(Code 128/39)는 가로 줄무늬형, 2D(QR/DataMatrix)는 정사각형이며 작은 면적에 더 많은 정보를 담습니다.
- **Q.** 같은 이름으로 저장이 안 됩니다.
  **A.** 같은 카테고리에 동일 이름이 이미 있습니다. 목록의 💾(덮어쓰기)로 갱신하세요.
