# 소모품출고

## 화면 목적
입고된 소모품의 **개별 UID(conUid) 바코드를 스캔해 현장으로 출고**하거나, 잘못 출고한 건을 **출고취소(반납)**하는 화면입니다. 처리한 출고·출고취소 내역은 아래 이력 표에서 기간·유형으로 조회할 수 있습니다.

> 소모품은 입고 시 1개마다 고유 UID(conUid)가 부여되어 **개별 추적**됩니다. 출고도 UID 1건 단위로 처리되며, 한 번 스캔할 때마다 수량 1이 차감됩니다(개별 묶음 출고).

## 화면 구성
- **상단(스캔 패널)**: 출고 / 출고취소 모드를 고르고 UID 바코드를 스캔하는 영역입니다.
- **하단(이력 표)**: 출고·출고취소 이력 목록입니다. 위쪽 도구줄에서 기간·검색·유형 필터를 사용합니다.
- 우상단 **새로고침** 버튼으로 목록을 다시 불러옵니다.

---

## 출고/출고취소 흐름
같은 UID가 상태를 따라 오갑니다.

```
입고완료(ACTIVE)  ──[출고]──▶  출고됨(ISSUED)
       ▲                              │
       └──────[출고취소(반납)]─────────┘
```

- **출고**: 입고완료(ACTIVE) 상태의 UID만 출고할 수 있습니다. 출고하면 상태가 **출고됨(ISSUED)**으로 바뀌고 마스터 재고수량이 1 줄어듭니다.
- **출고취소(반납)**: 출고됨(ISSUED) 상태의 UID만 취소할 수 있습니다. 취소하면 다시 **입고완료(ACTIVE)**로 돌아오고 재고수량이 1 복원됩니다.

---

## ① 스캔 패널

| 항목 | 역할 / 의미 |
|------|------|
| **출고(모드 버튼)** | 스캔한 UID를 출고 처리하는 모드입니다. 기본 모드입니다. |
| **출고반품(모드 버튼)** | 스캔한 UID의 출고를 취소(반납)하는 모드입니다. 선택 시 "출고취소 모드" 안내가 표시됩니다. |
| **UID 입력창** | 소모품 UID 바코드를 스캔하거나 직접 입력합니다. Enter 또는 확정 버튼으로 처리합니다. 처리 후 자동으로 다시 포커스되어 연속 스캔이 가능합니다. |
| **출고확정 / 취소확정(버튼)** | 현재 모드에 따라 라벨이 바뀝니다. 출고 모드면 **출고확정**, 출고반품 모드면 **취소확정**입니다. 입력값이 없으면 비활성화됩니다. |

## ② 출고 이력 표 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **일시(createdAt)** | 출고·출고취소가 처리된 일시입니다. |
| **소모품코드(consumableCode)** | 출고된 소모품의 마스터 코드입니다. |
| **소모품명(consumableName)** | 출고된 소모품의 이름입니다. |
| **UID(conUid)** | 출고된 개별 인스턴스의 고유 UID입니다. 어떤 물리 개체가 나갔는지 식별합니다. |
| **유형(logType)** | **출고(OUT)** 또는 **출고반품(OUT_RETURN)**입니다. 출고는 파란색, 출고반품은 보라색 배지로 표시됩니다. |
| **수량(qty)** | 처리 수량입니다. 출고는 빨간색 `-1`, 출고취소(반품)는 초록색 `+1`로 표시됩니다(부호로 가감 방향 구분). |
| **라인(lineCode)** | 출고 대상 라인입니다(기록된 경우). |
| **설비(equipCode)** | 출고 대상 설비입니다(기록된 경우). |
| **비고(remark)** | 출고·취소 시 입력된 메모 또는 자동 메시지입니다. |

> 표 상단 도구줄: **시작일·종료일**(기본값 오늘 하루), **검색**(소모품코드·소모품명), **유형 필터**(전체 유형 / 출고 / 출고반품). 우상단 **내보내기**로 표를 파일로 저장할 수 있습니다.

## 사용 순서
1. 상단 스캔 패널에서 **출고**(또는 잘못 나간 건을 되돌릴 때는 **출고반품**) 모드를 선택합니다.
2. UID 입력창에 소모품 **UID 바코드를 스캔**(또는 직접 입력)합니다.
3. Enter를 누르거나 **출고확정 / 취소확정** 버튼을 클릭합니다.
4. 처리되면 입력창이 비워지고 다시 포커스되어 바로 다음 UID를 스캔할 수 있습니다.
5. 하단 이력 표에서 처리 결과를 확인합니다(기간·유형 필터로 조회).

## 입력 규칙 / 검증
- **출고**는 입고완료(ACTIVE) 상태의 UID만 가능합니다. 미입고(PENDING)·이미 출고됨(ISSUED)·반납(RETURNED) 상태면 거부됩니다.
- **출고취소(출고반품)**는 출고됨(ISSUED) 상태의 UID만 가능합니다.
- 존재하지 않는 UID를 스캔하면 "UID를 찾을 수 없습니다" 오류가 표시됩니다.
- 한 번 스캔에 수량 1이 처리됩니다. 여러 개를 출고하려면 각 UID를 차례로 스캔합니다.

## 자주 묻는 질문
- **Q.** 스캔했더니 "ACTIVE 상태만 가능합니다" 오류가 나요.
  **A.** 해당 UID가 아직 입고확정되지 않았거나(미입고), 이미 출고된 상태입니다. [소모품입고](/consumables/receiving)에서 입고확정 여부를 확인하세요.
- **Q.** 출고를 잘못했어요. 되돌리려면요?
  **A.** 상단에서 **출고반품** 모드로 바꾼 뒤 같은 UID를 스캔해 **취소확정**하면 입고완료(ACTIVE)로 복원됩니다.
- **Q.** 이력 표에 방금 출고한 건이 안 보여요.
  **A.** 기본 조회 기간이 오늘 하루입니다. 날짜 범위와 유형 필터를 확인하고 **새로고침**하세요.
- **Q.** 수량을 2개 이상으로 한 번에 출고할 수 없나요?
  **A.** 소모품은 UID 1건 단위로 추적·출고됩니다. 여러 개는 UID를 하나씩 스캔합니다.
