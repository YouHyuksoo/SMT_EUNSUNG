# 원자재공정수불

## 화면 목적
설비(공정) 단위로 관리되는 **원자재 공정재고(WIP)**의 거래 이력을 조회합니다. 원자재가 공정으로 투입(WIP_IN)되거나 생산에서 소비(PROD_CONSUME)된 내역, 그리고 그 취소 내역을 설비별·일자별로 추적할 수 있습니다.

## 화면 구성
- **DataGrid 목록**: 거래일시, 유형, 설비, 품목, LOT, 수량, 참조, 비고 등 8개 컬럼 표시
- **상단 필터**: 설비 선택 + 거래유형 선택 + 일자 범위 + 검색어(품목코드/품목명/LOT/설비명)

---

## ① DataGrid 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **일시(createdAt)** | 거래가 처리된 일시입니다. |
| **거래유형(transType)** | 공정 수불 유형입니다. **WIP_IN**(공정입고, 파랑) / **WIP_IN_CANCEL**(입고취소, 빨강) / **PROD_CONSUME**(생산소비, 주황) / **PROD_CONSUME_CANCEL**(소비취소, 빨강). |
| **설비(equipName)** | 거래가 발생한 설비명과 코드입니다. |
| **품목(itemCode)** | 거래 대상 원자재 품목코드와 품목명입니다. |
| **LOT(matUid)** | 거래된 LOT/시리얼 번호입니다. |
| **수량(qty)** | 변동 수량입니다. **양수(파랑)** = 공정입고(재고 증가), **음수(빨강)** = 생산소비(재고 감소). |
| **참조(refType)** | 원본 거래의 참조 유형과 ID입니다(예: 작업지시번호). |
| **비고(remark)** | 거래에 대한 추가 메모입니다. |

## 거래유형 설명
| 유형 | 의미 | 수량 |
|------|------|------|
| WIP_IN | 원자재가 자재창고에서 설비(공정)로 투입됨 | + (양수) |
| WIP_IN_CANCEL | 공정입고 취소 — 투입된 원자재가 다시 창고로 회수됨 | - (음수) |
| PROD_CONSUME | 설비에서 원자재가 실제 생산에 소비됨 | - (음수) |
| PROD_CONSUME_CANCEL | 생산소비 취소 — 소비된 원자재가 공정재고로 환원됨 | + (양수) |

## 사용 순서
1. 상단 필터에서 **설비**와 **거래유형**을 선택하고, 필요 시 **일자 범위**와 **검색어**를 입력합니다.
2. 조회된 목록에서 개별 거래 내역을 확인합니다.
3. 컬럼 필터·정렬 기능으로 원하는 데이터를 찾습니다.
4. **내보내기**로 결과를 엑셀로 다운로드할 수 있습니다.
