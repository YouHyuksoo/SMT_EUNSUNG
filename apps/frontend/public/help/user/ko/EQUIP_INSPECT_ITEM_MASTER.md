# 점검항목 마스터

## 화면 목적
설비 점검에 사용할 **점검항목**(Inspection Item)을 기준정보로 등록·관리합니다. 각 항목은 점검유형(일상/정기/PM/작업자), 판정구분(판정형/측정형), 주기, 판정기준(상한/하한) 등의 속성을 가집니다. 등록된 항목은 [설비별 점검항목] 화면에서 개별 설비에 연결하여 사용합니다.

## 화면 구성
- **DataGrid 목록**: 등록된 점검항목 목록을 사진·항목코드·설비유형·점검항목명·점검유형·판정구분·판정기준·주기·사용여부와 함께 표시합니다.
- **우측 패널 (등록/수정)**: 항목 추가 또는 편집 시 오른쪽에서 슬라이드 패널이 열립니다.
- **상단 필터**: 검색어 + 설비유형 + 점검유형 선택

## ① DataGrid 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **사진(imageUrl)** | 점검항목과 관련된 참조 이미지입니다. 썸네일로 표시되며 클릭 시 확대됩니다. |
| **항목코드(itemCode)** | 점검항목을 식별하는 고유 코드입니다. 등록 후 수정할 수 없습니다. |
| **설비유형(equipType)** | 해당 점검항목이 적용되는 설비의 유형입니다(공통코드 `EQUIP_TYPE`). |
| **점검항목명(itemName)** | 점검항목의 이름입니다. |
| **점검유형(inspectType)** | 점검의 주체/시기에 따른 구분입니다. **DAILY**(일상점검, 파랑) / **PERIODIC**(정기점검, 주황) / **PM**(예방보전, 보라) / **WORKER**(작업자점검, 초록). |
| **판정구분(itemType)** | 점검 결과 판정 방식입니다. **VISUAL**(판정형, 회색) = 육안OK/NG / **MEASURE**(측정형, 청록) = 수치 측정. |
| **판정기준(criteria)** | 점검 결과의 합불 기준입니다. 측정형은 **하한(LSL) ~ 상한(USL)** 값과 단위로 표시됩니다. |
| **주기(cycle)** | 점검 수행 주기입니다. DAILY / WEEKLY / MONTHLY / QUARTERLY / SEMI_ANNUAL / ANNUAL. |
| **사용(useYn)** | `Y`(초록) = 사용 중, `N`(빨강) = 미사용. |

## ② 등록/수정 패널 필드

| 필드 | 역할 / 의미 |
|------|------|
| **항목코드(itemCode)** | 점검항목의 고유 식별 코드입니다. 등록 후 수정 불가. |
| **설비유형(equipType)** | 점검항목이 적용될 설비 유형입니다. |
| **점검항목명(itemName)** | 점검항목의 명칭입니다. |
| **점검유형(inspectType)** | 점검 유형(DAILY/PERIODIC/PM/WORKER)을 선택합니다. |
| **판정구분(itemType)** | **VISUAL**(판정형): 육안으로 OK/NG 판정 / **MEASURE**(측정형): 수치를 측정하여 기준 범위 내인지 확인. |
| **주기(cycle)** | 점검 수행 주기를 선택합니다. |
| **사용(useYn)** | 사용 여부(Y/N). |
| **판정기준(criteria)** | VISUAL: OK/NG 판단 기준 텍스트 입력. / MEASURE: 단위 선택 + **하한(LSL)** + **상한(USL)** 수치 입력. |
| **사진(imageUrl)** | 점검 참조 이미지를 업로드합니다(JPEG/PNG/GIF/WebP). 클릭 또는 드래그&드롭으로 업로드 가능. |
| **비고(remark)** | 추가 관리 메모입니다. |

## 사용 순서
1. **추가** 버튼으로 새 점검항목을 등록합니다(우측 패널).
2. 점검유형·판정구분·주기 등 속성을 입력하고, 측정형일 경우 상한/하한 기준값을 설정합니다.
3. 필요 시 점검 참조 사진을 업로드합니다.
4. 목록에서 편집 아이콘을 클릭하여 기존 항목을 수정합니다.
5. 등록된 항목은 [설비별 점검항목] 화면에서 개별 설비에 연결하여 사용합니다.

## 입력 규칙 / 검증
- 항목코드·점검항목명은 **필수 입력**입니다.
- 측정형(MEASURE)인 경우 하한(LSL)과 상한(USL)을 모두 입력해야 합니다.
- 사진은 이미지 파일(JPEG/PNG/GIF/WebP)만 업로드 가능하며, 최대 5MB로 제한됩니다.

## 자주 묻는 질문
- **Q.** 점검유형별 차이는 무엇인가요? **A.** **DAILY**=매일 작업 전 점검, **PERIODIC**=정기(월/분기/반기/연), **PM**=예방보전 설비정비 시, **WORKER**=작업자가 자체 수행하는 점검입니다.
- **Q.** 판정형(VISUAL)과 측정형(MEASURE)의 차이는? **A.** 판정형은 OK/NG 같은 정성 판단, 측정형은 구체적인 수치를 측정하여 기준 범위(하한~상한) 내인지 확인합니다.
- **Q.** 등록 후 항목코드를 변경할 수 있나요? **A.** PK이므로 변경 불가합니다. 삭제 후 새로 등록해야 합니다.
