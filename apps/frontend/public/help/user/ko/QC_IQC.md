# 수입검사(IQC)

## 화면 목적
입하된 자재 LOT를 품질 검사하여 **합격(PASS) / 불합격(FAIL)**을 판정하는 화면입니다.

품목마스터에서 `IQC 여부 = Y`로 설정된 품목은 입하 후 **검사대기(PENDING)** 상태가 됩니다. 이 화면에서 검사를 완료해야 입고 처리로 이어집니다. 불합격 판정 시 해당 입하건의 모든 시리얼은 **불용창고로 자동 이동**됩니다.

> 검사 대상은 **입하번호 + 품목** 단위로 묶여 표시됩니다. 한 입하건에 같은 품목이 여러 시리얼로 들어온 경우 1행으로 집계됩니다.

---

## 화면 구성
- **상단 필터 바**: 검색어(입하번호·품목코드·품목명), 검사구분 필터(FULL/SKIP), 검사상태 필터
- **목록 그리드**: 검사 대기·진행 중·완료 건을 표 형식으로 표시. 행마다 **검사** 버튼
- **검사 모달**: 검사 버튼 클릭 시 열림 — **3단 레이아웃** (입력 패널 / 스캔 시리얼 목록 / 측정 패널)

---

## ① 목록 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **검사(버튼)** | 검사 모달을 엽니다. 상태가 `검사대기` 또는 `검사중`인 건만 활성화됩니다. |
| **입하번호** | 입하를 식별하는 번호입니다. |
| **입하일** | 자재가 입하된 날짜입니다. |
| **공급업체** | 납품 협력사 이름입니다. |
| **품목코드** | 검사 대상 품목의 MES 내부 코드입니다. |
| **품목명** | 검사 대상 품목의 이름입니다. |
| **검사구분** | 품목마스터의 검사 방식입니다. `검사(FULL)` = 빨간 배지, `무검사(SKIP)` = 회색 배지. SKIP인 경우에도 이 목록에 표시될 수 있습니다. |
| **시리얼수** | 입하건에 포함된 자재 시리얼 수입니다. 스캔해야 할 대상 수를 나타냅니다. |
| **총수량** | 모든 시리얼의 수량 합계입니다(단위 포함). |
| **상태** | 검사 진행 상태입니다. 아래 상태값 참조. |
| **검사자** | 검사를 수행한 담당자 이름입니다. 완료 후 표시됩니다. |

### 검사상태 값

| 상태 코드 | 표시 | 의미 |
|------|------|------|
| `PENDING` | 검사대기 | 입하 완료 후 검사 전 상태. 검사 시작 가능. |
| `IQC_IN_PROGRESS` | 검사중 | 검사가 시작되었으나 아직 완료되지 않은 상태. 검사 버튼 활성화. |
| `PASSED` | 합격 | 검사 판정 결과 합격. 입고 처리로 이어집니다. |
| `FAILED` | 불합격 | 검사 판정 결과 불합격. 시리얼이 불용창고로 자동 이동됩니다. |

### 필터 및 기타 기능

| 기능 | 설명 |
|------|------|
| **검색어 입력** | 입하번호, 품목코드, 품목명을 한꺼번에 검색합니다. |
| **검사구분 필터** | FULL(검사) / SKIP(무검사)로 필터링합니다. |
| **검사상태 필터** | PENDING / IQC_IN_PROGRESS / PASSED / FAILED로 필터링합니다. |
| **새로고침** | 목록을 최신 상태로 갱신합니다. |
| **내보내기** | 현재 목록을 Excel 파일로 저장합니다(파일명: iqc_inspection). |

## ② 검사 모달 — 상단 입하정보

모달 상단에 현재 검사 중인 입하건의 기본 정보가 고정 표시됩니다.

| 항목 | 역할 / 의미 |
|------|------|
| **입하번호** | 현재 검사 중인 입하건 번호입니다. |
| **공급업체** | 납품 협력사 이름입니다. |
| **시리얼수** | 이 입하건에 속한 전체 시리얼 수입니다. |
| **총수량** | 전체 시리얼의 수량 합계(단위 포함)입니다. |
| **품목** | 검사 대상 품목명과 품목코드입니다. |

## ③ 검사 모달 — 왼쪽 입력 패널

### 시리얼 스캔

| 항목 | 역할 / 의미 |
|------|------|
| **시리얼 스캔 입력란** | 바코드를 스캔하거나 직접 입력 후 Enter 합니다. 검사대기 목록에 없는 시리얼은 "검사대기 시리얼이 아닙니다" 오류를 표시합니다. |
| **스캔 진행 카운터** | `스캔수 / 전체 대기수` 형식으로 진행률을 표시합니다. |
| **검사대기 시리얼 조회 버튼** | 클릭하면 PENDING 시리얼 목록 팝업이 열립니다. 각 행에는 시리얼과 수량이 표시되며, 개별 클릭으로 한 건씩 추가하거나 **전체 추가** 버튼으로 모두 한 번에 등록할 수 있습니다. |

### 기본 입력 항목

| 항목 | 역할 / 의미 |
|------|------|
| **검사자** | 품질팀 부서에 등록된 작업자 목록에서 선택합니다(드롭다운). |
| **비고** | 검사 관련 메모를 자유롭게 입력합니다. |
| **기본시료수** | 품목마스터에 등록된 기본 시료 수량입니다. 검사자가 수정할 수 있습니다. |

### AQL 샘플수량(자동 조회)

품목에 연결된 AQL 정책을 기준으로 자동 산출된 권장 시료 수와 검사수준, 검사모드가 표시됩니다.

```
예시: 50 / II / NORMAL
     → 시료 50개, 검사수준 II, 정상검사 모드
```

검사항목별 Ac/Re도 항목명과 함께 최대 3건 미리 보여 줍니다.

### 검사성적서

파일 선택 버튼으로 검사 관련 서류를 첨부합니다.
- 지원 형식: `.pdf`, `.jpg`, `.jpeg`, `.png`, `.xlsx`, `.xls`
- 결과 등록 시 서버에 함께 업로드됩니다.

### 파괴/전수 검사

품목에 파괴검사(DESTRUCTIVE) 또는 전수검사(FULL) 유형의 검사항목이 등록된 경우 이 영역이 나타납니다.
- 항목명 옆에 기준 시료수가 표시됩니다.
- **검사수량**과 **불량수** 2개 입력란을 채웁니다.
- 불량수가 1개 이상이면 해당 항목은 자동으로 FAIL입니다.

### 불량코드

**FAIL 판정(시리얼 또는 파괴검사)이 발생했을 때만** 이 영역이 표시됩니다.

- 불량 유형을 선택하면 해당 불량코드의 **등급(Critical/Major/Minor) 배지**가 함께 나타납니다.
- **추가** 버튼으로 불량 유형을 여러 건 등록할 수 있습니다.
- FAIL 판정이 있는데 불량코드를 입력하지 않으면 등록 버튼이 비활성화됩니다.
- PASS 판정인데 불량코드를 입력하면 "PASS 판정에는 불량코드를 입력할 수 없습니다." 오류가 표시되고 등록이 차단됩니다.

## ④ 검사 모달 — 중간 패널 (스캔한 시리얼 목록)

스캔 또는 추가한 시리얼이 순서대로 나열됩니다. 각 항목에는 순번, 시리얼(matUid), 판정 상태 배지가 표시됩니다.

| 판정 배지 | 의미 |
|------|------|
| 초록 **PASS** | 모든 검사항목 판정 완료, 전부 합격 |
| 빨간 **FAIL** | 검사항목 중 하나 이상 불합격 |
| 회색 **대기** | 아직 모든 항목 판정이 완료되지 않음 |

행 우측의 **×** 버튼으로 잘못 스캔한 시리얼을 제거합니다.

---

## ⑤ 검사 모달 — 오른쪽 측정 패널

중간 패널에서 시리얼을 선택하면 해당 시리얼의 검사항목 표가 표시됩니다.

| 컬럼 | 역할 / 의미 |
|------|------|
| **#** | 검사항목 순번입니다. |
| **검사항목** | 항목 이름 + 불량등급 배지(Critical/Major/Minor). 항목명 하단에 `검사수준 II · AQL 1.0` 형식으로 적용 기준이 표시됩니다. |
| **규격** | 해당 항목의 규격 설명입니다. |
| **하한(LSL)** | 계측값의 합격 하한입니다. 입력값이 이 값 미만이면 자동 FAIL입니다. |
| **상한(USL)** | 계측값의 합격 상한입니다. 입력값이 이 값 초과이면 자동 FAIL입니다. |
| **판정기준** | 육안 검사 등 수치 이외의 합격 기준 설명입니다. |
| **측정값** | LSL/USL이 있는 항목: 숫자 입력 → 범위 내이면 자동 PASS. LSL/USL이 없는 항목: PASS / FAIL 버튼으로 직접 선택. |
| **판정** | ✓(합격)/✗(불합격)/-(미입력) 아이콘입니다. |

> **검사항목이 없는 품목**: 노란 경고 박스가 표시되고, 시리얼 단위로 PASS / FAIL 버튼을 직접 클릭해 판정합니다.

## 검사결과 등록 버튼 조건

모달 하단의 **검사결과 등록** 버튼은 아래 조건을 모두 충족해야 활성화됩니다.

| 조건 | 설명 |
|------|------|
| 시리얼 스캔 | 스캔한 시리얼이 1개 이상 있어야 합니다(파괴/전수 전용 품목은 예외). |
| 판정 완료 | 스캔한 모든 시리얼의 모든 검사항목 판정이 끝나야 합니다. |
| 불량코드 일치 | FAIL이 있으면 불량코드 필수, PASS이면 불량코드 입력 불가. |

하단 왼쪽에는 현재 판정 상태 요약이 표시됩니다.
- 스캔 시리얼 없음 → "스캔한 시리얼이 없습니다."
- 미완료 → "판정이 끝나지 않은 시리얼이 있습니다."
- FAIL 포함 → 빨간 글씨 "FAIL N / PASS N"
- 전부 PASS → 초록 글씨 "PASS N"

---

## 사용 순서

1. 목록에서 검사할 입하건 행의 **검사** 버튼을 클릭합니다.
2. 모달 왼쪽 스캔 입력란에 자재 시리얼을 바코드 스캔 또는 직접 입력 후 Enter 합니다(또는 **검사대기 시리얼 조회 → 전체 추가**).
3. 중간 패널에서 시리얼을 클릭하면 오른쪽에 검사항목 표가 나타납니다.
4. 항목별로 측정값을 입력하거나 PASS/FAIL을 선택합니다(LSL/USL이 있으면 자동 판정).
5. 파괴/전수 검사 항목이 있으면 왼쪽 패널 하단에서 검사수량과 불량수를 입력합니다.
6. FAIL이 발생하면 나타나는 **불량코드** 영역에서 불량 유형을 선택하고 수량을 입력합니다.
7. 필요하면 **검사성적서** 파일을 첨부합니다.
8. 모달 하단의 **검사결과 등록** 버튼을 클릭합니다.

---

## 입력 규칙 / 검증

- 스캔한 시리얼이 PENDING 상태가 아니거나 다른 입하번호에 속하면 오류가 표시됩니다.
- 스캔한 시리얼 중 판정이 완료되지 않은 것이 하나라도 있으면 등록 버튼이 비활성화됩니다.
- **FAIL 판정이 있으면 불량코드를 최소 1건 이상** 입력해야 등록됩니다.
- **PASS 판정인데 불량코드를 입력하면** 등록이 차단됩니다.
- 검사항목이 없는 경우 시리얼 단위 수동 판정(PASS/FAIL 버튼)으로 대체합니다.

## 자주 묻는 질문

- **Q.** 검사 버튼이 비활성화됩니다.
  **A.** 해당 입하건의 상태가 `PASSED` 또는 `FAILED`로 이미 판정이 완료된 경우입니다. 재검사가 필요하면 운영자에게 판정 취소를 요청하세요.
- **Q.** AQL 샘플수량이 표시되지 않습니다.
  **A.** 해당 품목에 AQL 정책이 연결되어 있지 않습니다. 품목마스터에서 AQL 정책을 설정해 주세요.
- **Q.** 시리얼 스캔 시 "검사대기 시리얼이 아닙니다" 오류가 납니다.
  **A.** 스캔한 시리얼이 PENDING 상태가 아니거나 다른 입하건에 속합니다. **검사대기 시리얼 조회**로 올바른 시리얼 목록을 확인하세요.
- **Q.** 불합격 처리 후 재고가 사라졌습니다.
  **A.** IQC 불합격 시 해당 시리얼은 자동으로 불용창고로 이동됩니다. 재검사가 필요하면 운영자에게 판정 취소를 요청하세요.
- **Q.** 검사항목이 없을 때 어떻게 판정하나요?
  **A.** 품목에 IQC 검사항목이 등록되지 않은 경우, 모달 오른쪽 영역에서 시리얼별로 PASS / FAIL 버튼을 직접 클릭해 판정합니다.
- **Q.** 파괴검사 항목만 있고 시리얼 스캔이 없어도 됩니까?
  **A.** 네. AQL 시리얼 검사항목이 없고 파괴/전수 검사 항목만 있는 경우, 시리얼 스캔 없이 파괴검사 수량만 입력하고 결과를 등록할 수 있습니다.
- **Q.** 검사자 목록에 제 이름이 없습니다.
  **A.** 검사자 목록은 품질팀 부서에 등록된 작업자만 표시됩니다. 운영자에게 작업자 등록을 요청하세요.
- **Q.** 불량코드 입력 후 등록이 차단됩니다.
  **A.** 두 가지 경우가 있습니다. (1) FAIL 판정이 없는데 불량코드를 입력한 경우 — 판정을 다시 확인하세요. (2) 불량코드는 선택했으나 수량을 0으로 입력한 경우 — 수량을 1 이상으로 입력하세요.
