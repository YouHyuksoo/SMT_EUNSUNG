# 라우팅관리

## 화면 목적
완제품·반제품이 **어떤 공정을 어떤 순서로 거치는지(라우팅)**를 정의합니다. 라우팅은 생산 작업지시·실적·설비투입·검사의 기준이 됩니다. 공정마다 **양품조건(합격 기준)**, **투입자재(BOM 연결)**, **자주검사 항목**을 함께 등록합니다.

## 화면 구성
- **왼쪽 영역(8칸)**
  - **라우팅 목록**: 완제품 라우팅 그룹 목록. 라우팅코드/라우팅명/품목으로 검색, **라우팅 추가**.
  - **BOM 구조**: 선택한 라우팅 완제품의 BOM 트리(원자재·소모품 제외). 완제품·반제품 노드를 클릭하면 그 품목의 공정순서를 봅니다.
  - **라우팅 공정 순서**: 선택 품목(완제품/반제품)의 공정 목록. 순서·공정명·샘플검사·라벨발행 표시, **공정추가**·수정·삭제.
- **오른쪽 영역(4칸, 탭)**: 공정 행을 선택하면 활성화됩니다.
  - **선택 공정 양품조건** 탭 — 공정 통과 기준값
  - **투입자재** 탭 — 공정에서 투입할 BOM 자재
  - **자주검사 설정** 탭 — 공정별 자주검사(공정샘플검사) 항목
- 상단 **새로고침** 버튼으로 전체를 다시 불러옵니다.

> 작업 흐름: **라우팅(완제품) 선택 → BOM 트리에서 품목 클릭 → 공정 추가 → 우측 탭에서 양품조건·투입자재·자주검사 등록**.

### 데이터 구조
```
라우팅 그룹 (ROUTING_GROUPS, 완제품 기준)
  └─ 공정순서 (ROUTING_PROCESSES, SEQ 단위)
       ├─ 양품조건 (PROCESS_QUALITY_CONDITIONS)
       ├─ 투입자재 (ROUTING_MATERIALS, BOM 자재 연결)
       └─ 자주검사 항목 (SELF_INSPECT_ITEMS, 공정코드 기준)
```

## ① 라우팅 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **라우팅코드(routingCode)** | 라우팅 그룹을 식별하는 고유 코드입니다. 한 번 등록하면 변경하지 않는 것을 권장합니다(공정·조건·실적이 이 코드로 연결됨). |
| **라우팅명(routingName)** | 라우팅을 식별할 이름입니다. |
| **연결 품목(itemCode / itemName)** | 이 라우팅을 적용할 완제품입니다. 목록은 **완제품(FINISHED)** 라우팅만 표시합니다. |

## ② BOM 구조 트리

| 항목 | 역할 / 의미 |
|------|------|
| **품목 노드(itemName / itemCode)** | 라우팅 완제품의 BOM 전개 트리입니다. **원자재·소모품(RAW/RM/CONSUMABLE)은 자동 제외**되어, 라우팅을 둘 수 있는 완제품·반제품만 보입니다. |
| **유형 배지(itemType)** | 노드 품목의 유형입니다(FINISHED/SEMI_PRODUCT 등). 완제품·반제품(FINISHED/SEMI_PRODUCT/FG/WIP)만 공정 추가가 가능합니다. |

> BOM 트리에서 **반제품**을 클릭하면 그 반제품의 라우팅을 따로 등록할 수 있습니다(중간 조립품도 자체 공정순서를 가짐).

## ③ 라우팅 공정 순서 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **순서(seq)** | 공정 진행 순서입니다. 보통 10·20·30…으로 매겨 중간 삽입 여유를 둡니다. 신규 추가 시 마지막 순서 +10이 기본값. |
| **공정명(processName)** | 공정의 이름입니다. 공정코드를 선택하면 공정마스터 이름이 자동으로 채워집니다(직접 입력 불가). |
| **공정코드(processCode)** | 공정마스터(PROCESS_MASTERS)에서 선택한 공정 코드입니다. |
| **공정유형(processType)** | 선택한 공정의 유형(공통코드 PROCESS_TYPE)입니다. 공정코드 선택 시 자동 표시(읽기 전용). |
| **설비유형(equipType)** | 이 공정에서 사용할 설비 유형(공통코드 EQUIP_TYPE)입니다. |
| **표준시간(stdTime)** | 1개 처리에 걸리는 표준 작업시간(초)입니다. CAPA·계획 산출에 사용. |
| **셋업시간(setupTime)** | 공정 준비/교체에 걸리는 셋업 시간(초)입니다. |
| **공정샘플검사(sampleInspectYn)** | `Y`면 이 공정에서 샘플검사(공정생품검사)가 필요함을 표시합니다. 목록에 "필요" 배지. |
| **라벨발행 — SG(issueSgLabelYn)** | `Y`면 이 공정 완료 시 **반제품 라벨(SG)** 발행 대상입니다. 목록에 `SG` 배지. |
| **라벨발행 — FG(issueFgLabelYn)** | `Y`면 이 공정 완료 시 **완제품 라벨(FG)** 발행 대상입니다. 목록에 `FG` 배지. |

## ④ 선택 공정 양품조건 (우측 탭)

| 컬럼 | 역할 / 의미 |
|------|------|
| **양품조건(conditionCode)** | 이 공정 통과를 위한 검사 항목입니다(공통코드 QUALITY_CONDITION). 예: 압착높이, 융착온도 등. |
| **수치(MIN)(minValue)** | 합격으로 인정하는 최소값입니다. 비워두면 하한 제한 없음. |
| **수치(MAX)(maxValue)** | 합격으로 인정하는 최대값입니다. 비워두면 상한 제한 없음. |
| **단위(unit)** | 기준값의 단위입니다(공통코드 UNIT_TYPE). |
| **설비 I/F(equipInterfaceYn)** | `Y`면 이 측정값을 **설비에서 자동 수신**합니다(수기 입력이 아닌 설비 인터페이스 대상). |

> 양품조건은 **일괄 저장** 방식입니다. 행을 추가/삭제/수정한 뒤 **저장** 버튼을 눌러야 반영되며, 저장 시 조건 순번이 1부터 다시 매겨집니다.

## ⑤ 투입자재 (우측 탭)

| 컬럼 | 역할 / 의미 |
|------|------|
| **선택(selected)** | 이 공정에 실제로 투입할 자재만 체크합니다. 체크된 자재만 저장됩니다. |
| **자재(childItemCode / childItemName)** | 라우팅 품목의 BOM 자재 목록입니다. 자재 코드/품번을 함께 표시합니다. |
| **BOM수량(qtyPer)** | BOM에 정의된 단위 소요량입니다(참고용, 읽기 전용). |
| **회로사양(circuitId / circuitNo)** | 전선류 자재일 때 연결할 회로를 선택합니다. 미연결도 가능. |
| **길이/Strip(lengthMm / stripA / stripB)** | 선택한 회로의 전선 길이(mm)와 양끝 탈피(Strip A/B) 사양입니다. 회로 선택 시 자동 표시. |
| **투입수량(allocQty)** | 이 공정에서 실제 투입(차감)할 수량입니다. 기본값은 BOM수량. |
| **투입방식(issueMethod)** | **백플러시(BACKFLUSH)** = 실적 보고 시 자동 차감 / **선투입(PRE_ISSUE)** = 작업 전 미리 불출. |

> 투입자재도 **일괄 저장**입니다. 체크 → 회로/수량/방식 설정 → **저장**.

## ⑥ 자주검사 설정 (우측 탭)

| 컬럼 | 역할 / 의미 |
|------|------|
| **항목명(itemName)** | 자주검사(공정샘플검사) 항목 이름입니다. 필수. |
| **유형(itemType)** | **판정형(VISUAL)** = OK/NG 판정 / **측정형(MEASURE)** = 수치 측정. |
| **기준/규격(standard)** | 검사 기준·규격 설명입니다(선택). |
| **단위·LSL·USL(unit / lslValue / uslValue)** | 측정형일 때만 입력합니다. LSL=하한규격, USL=상한규격. |
| **검사방법(inspectMethod)** | **직접검사(DIRECT)** / **의뢰검사(DELEGATE)**. |
| **샘플수(sampleCount)** | 검사 시료 수입니다(초물 시점에 주로 적용). |
| **파괴(isDestructive)** | 파괴검사 항목인지 여부입니다. |
| **시점(timing)** | 검사 시점입니다. **초물(FIRST)·중물(MID)·종물(LAST)** 복수 선택 가능. |
| **순서(sortOrder)** | 항목 정렬 순서입니다. |
| **사용(useYn)** | `Y`일 때만 사용합니다. |

> 자주검사 항목은 **행 단위 저장**입니다. 항목을 추가/수정하면 해당 행의 **저장** 버튼으로 즉시 저장하고, 삭제 아이콘으로 바로 삭제합니다. 이 항목은 공정코드(processCode) 기준이라 같은 공정을 쓰는 다른 라우팅에도 공유됩니다.

## 사용 순서
1. 왼쪽 **라우팅 목록**에서 라우팅(완제품)을 선택합니다. 없으면 **라우팅 추가**로 완제품과 라우팅코드·라우팅명을 등록합니다.
2. **BOM 구조** 트리에서 공정을 등록할 품목(완제품 또는 반제품)을 클릭합니다.
3. **라우팅 공정 순서**에서 **공정추가**로 공정코드를 선택하고 표준시간·라벨발행·샘플검사 등을 입력해 저장합니다. (선택 품목에 라우팅이 없으면 자동 생성됩니다.)
4. 공정 행을 클릭한 뒤 우측 탭에서 **양품조건 / 투입자재 / 자주검사**를 등록합니다.

## 입력 규칙 · 검증
- 라우팅 추가 시 **라우팅코드·라우팅명·연결 품목**이 모두 있어야 저장됩니다.
- 공정 추가 시 **공정코드**(공정명 자동 채움)가 있어야 저장됩니다. 순서(seq)는 수정 시 변경 불가.
- 공정 추가 버튼은 선택 품목이 **완제품/반제품(FINISHED/SEMI_PRODUCT/FG/WIP)**일 때만 활성화됩니다.
- 양품조건·투입자재는 **변경 후 저장** 버튼을 눌러야 반영됩니다(미저장 변경사항 표시).
- 자주검사 항목은 **항목명**이 비어 있으면 저장되지 않습니다.

## 자주 묻는 질문
- **Q.** 공정명을 직접 입력할 수 있나요?
  **A.** 아니요. 공정코드를 선택하면 공정마스터의 공정명·공정유형이 자동으로 채워집니다.
- **Q.** 반제품도 라우팅을 따로 등록하나요?
  **A.** 네. BOM 트리에서 반제품을 클릭하면 그 반제품 전용 공정순서를 등록할 수 있습니다.
- **Q.** 양품조건의 "설비 I/F"는 무슨 뜻인가요?
  **A.** 그 측정값을 작업자가 수기 입력하지 않고 설비에서 자동으로 받아오는 항목이라는 표시입니다.
- **Q.** SG·FG 라벨 발행 체크는 무슨 의미인가요?
  **A.** 해당 공정 완료 시 반제품 라벨(SG) 또는 완제품 라벨(FG)을 발행하는 지점이라는 뜻입니다.
- **Q.** 투입방식 백플러시와 선투입의 차이는?
  **A.** 백플러시는 실적 보고 시 자동 차감, 선투입은 작업 전에 미리 자재를 불출하는 방식입니다.
