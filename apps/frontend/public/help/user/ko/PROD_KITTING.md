# 실적입력(서브공정)

## 화면 목적
반제품 작업지시의 서브공정에서, 이전 공정 반제품(SFG) 라벨과 설비에 장착된 원자재를 스캔으로 소모하여 회로 단위로 새로운 반제품(SFG) 라벨을 발행·확정합니다. 완제품을 만드는 [실적입력(조립)](/production/input-assembly)의 한 단계 아래 버전이며, 설비를 선택하면 그 설비에서 진행 중이던 작업지시가 서버에서 자동으로 복원됩니다.

## 화면 구성
```
[헤더] 실적입력(서브공정)                          [초기화]
[상단 고정 바] 설비 선택  |  작업지시 스캔/검색  |  회로 선택
┌───────────────┬─────────────────────────┬───────────────┐
│ 좌: 설비 자재  │  중앙: 작업지도서        │ 우: 이전 공정 │
│ 장착(BOM 검증) │  (선택 품목+공정 기준)   │ SFG 스캔      │
├───────────────┴─────────────────────────┴───────────────┤
│ 하단 액션바: [키팅 실행 → SFG 라벨 발행] | 발행 라벨(자동 인쇄) + 실물 스캔 확인 │
└────────────────────────────────────────────────────────┘
```
- 설비를 선택하면 그 설비에 마지막으로 배정된 작업지시가 있는지 서버에 물어보고, 있으면 자동으로 선택해 줍니다.
- 회로 선택은 선택한 품목에 회로 정보가 있을 때만 활성화되며, 있으면 발행·확정 모두 필수입니다.

## ① 설비 자재 장착 목록 컬럼 (좌측 패널)
| 컬럼 | 역할 / 의미 |
|------|------|
| **품목명 · 품목코드** | 설비에 장착된 원자재의 품목 정보입니다. |
| **LOT(matUid)** | 장착된 자재 LOT/UID입니다. |
| **잔량** | 이 LOT에서 아직 소모되지 않은 가용 수량입니다. 0이 되면 "보충 스캔" 표시가 뜹니다. |
| **작업지시 BOM 원자재 체크리스트** | 선택된 작업지시 BOM의 **원자재(RAW_MATERIAL)** 항목만 이 설비에 장착됐는지(✓ 장착 / ⚠ 미장착)와 단위당 수량을 보여줍니다. |
| **장착 대기(공정재고) 칩** | 공정재고에는 있지만 아직 설비에 장착 안 된 LOT 목록입니다. 클릭하면 스캔 없이 바로 장착됩니다. |

## ② 이전 공정 SFG 스캔 목록 컬럼 (우측 패널)
| 컬럼 | 역할 / 의미 |
|------|------|
| **#** | 스캔한 순서 번호입니다. |
| **SFG 바코드** | 스캔한 이전 공정 반제품 라벨 바코드입니다. |
| **품번** | 해당 SFG 라벨의 품목코드입니다. |
| **잔량** | SFG 라벨에 남은 수량입니다. |
| **상태** | SFG 라벨의 현재 상태(IN_STOCK/MOUNTED 등)입니다. |
| **삭제(휴지통)** | 잘못 스캔한 SFG를 목록에서 제거합니다(화면에서만 제거, 서버 미반영). |

## 버튼별 기능
| 버튼 | 언제 사용 | 동작 | 결과 / 주의 |
|------|------|------|------|
| **설비 선택** (상단 콤보 버튼) | 화면 진입 후 가장 먼저 클릭 | 설비 선택 모달을 엽니다 | 설비를 고르면 그 설비에 서버가 기억하고 있던 작업지시(반제품인 경우만)를 자동으로 복원하고, 아니면 스캔 입력에 포커스를 이동합니다 |
| **초기화** (헤더 우측) | 처음부터 다시 시작할 때 | 설비·작업지시·회로·스캔 목록·발행 상태를 전부 리셋 | 설비에 저장된 현재 작업지시도 함께 해제됩니다 |
| **작업지시번호 스캔/입력** | 설비 선택 후, 자동 복원이 안 됐을 때 | 입력값으로 작업지시를 조회해 자동 선택 | `WAITING`/`RUNNING` 상태의 **반제품(SEMI_PRODUCT)** 작업지시만 조회됩니다. 선택 즉시 해당 설비의 "현재 작업지시"로 서버에 저장됩니다 |
| **검색** (작업지시 옆) | 목록에서 고르고 싶을 때 | 작업지시 검색 모달을 엽니다(선택 설비 공정 + 반제품 + WAITING/RUNNING 조건) | |
| **변경** (작업지시 선택 후) | 다른 작업지시로 바꿀 때 | 작업지시·회로·SFG 스캔 목록·발행 상태를 초기화 | 설비의 "현재 작업지시" 저장값도 해제됩니다 |
| **회로 선택** | 선택 품목에 회로 정보가 있을 때 | 드롭다운에서 회로 선택 | 회로가 있는 품목은 발행·확정 모두 회로를 고르지 않으면 진행할 수 없습니다 |
| **자재 바코드 스캔** (좌측 패널) | 원자재 LOT을 설비에 장착할 때 | 작업지시 BOM과 매칭해 장착(오장착 검증 포함) | BOM 원자재 목록에 없는 품목이거나 공정재고가 없으면 거부됩니다 |
| **해제** (장착 자재 행) | 잘못 장착했거나 교체할 때 | 설비 재고를 공정재고(장착 대기)로 되돌립니다 | 예약 수량이 있으면 해제할 수 없습니다 |
| **SFG 바코드 스캔** (우측 패널) | 이전 공정 반제품 라벨을 투입할 때 | 라벨을 검증 후 스캔 목록에 추가 | 이미 스캔한 라벨, 완제품(FG) 바코드, BOM에 없는 품목(오투입), 잔량 0, 사용 불가 상태는 모두 거부됩니다 |
| **삭제** (스캔 목록 행) | 잘못 스캔한 SFG를 뺄 때 | 화면 목록에서만 제거 | |
| **키팅 실행 → SFG 라벨 발행** (하단, 크게 강조) | 작업지시·설비 선택 + SFG 1건 이상 스캔 + (회로 있으면) 회로 선택 완료 시 활성화 | 새 SFG 바코드를 채번해 `ISSUED` 상태로 저장 | 발행 즉시 Print Agent로 **자동 인쇄**됩니다(확정 전에 먼저 출력). 이 단계에서는 아직 입력 SFG 소비·자재 차감·재고·실적이 반영되지 않습니다 |
| **실물 SFG 라벨 스캔** (발행 후 노출) | 자동 출력된 실물 라벨을 부착한 뒤 | 스캔값이 발행된 바코드와 일치하면 키팅을 확정 | 값이 다르면 확정을 거부합니다. 회로가 있는 품목은 확정 시에도 회로 선택이 필요합니다 |
| **확인** (실물 스캔 옆) | 스캔 대신 버튼으로 확정을 제출할 때 | 입력창의 값으로 확정 요청 | |
| **발행 취소** (되돌리기 아이콘) | 잘못 발행했을 때 | 화면의 발행 상태만 초기화 | ⚠ 서버의 SFG 라벨 레코드(`ISSUED`)는 취소되지 않고 그대로 남습니다. 이미 자동 인쇄된 실물 라벨은 폐기하고 새로 키팅을 실행해 라벨을 다시 발행하세요 |

## 업무 흐름
1. **설비 선택** — 설비가 공정을 자동으로 결정하고, 그 설비에서 진행 중이던 작업지시가 있으면 자동 복원됩니다.
2. **작업지시 확인/선택** — 자동 복원되지 않았다면 스캔·입력 또는 검색으로 반제품 작업지시를 선택합니다.
3. **회로 선택** — 선택 품목에 회로 정보가 있으면 반드시 고릅니다.
4. **자재 장착 확인** — 좌측 "작업지시 BOM 원자재" 체크리스트로 원자재가 다 장착됐는지 확인하고, 부족하면 스캔하거나 "장착 대기" 목록에서 장착합니다.
5. **작업지도서 확인** — 중앙 패널에서 선택 품목·공정 기준 작업지도서를 확인합니다.
6. **이전 공정 SFG 스캔** — 우측 패널에서 BOM의 반제품 구성만큼 SFG 라벨을 스캔합니다.
7. **키팅 실행 → SFG 라벨 발행** 클릭 — 새 SFG 바코드가 발행되고 즉시 프린터로 자동 출력됩니다.
8. 출력된 실물 라벨을 부착 후 **스캔하여 확정** — 입력 SFG 소비, 설비 자재 차감, genealogy 생성, 생산실적 1건 등록, 반제품 WIP 재고 적재가 한 번에 처리됩니다.
9. 확정 완료 후 SFG 스캔 목록과 발행 상태가 초기화되며, 같은 작업지시로 다음 키팅을 이어서 진행합니다.

## 입력 규칙 / 검증
- 설비를 먼저 선택해야 작업지시 조회와 자재 장착이 가능합니다(설비 → 공정 자동 결정).
- 완료(`DONE`)·취소(`CANCELED`)·홀딩(`HOLD`) 상태의 작업지시에는 SFG 라벨을 발행하거나 확정할 수 없습니다.
- 선택 품목에 회로 정보가 있으면 발행·확정 모두 회로 선택이 필수입니다.
- SFG 라벨은 상태가 `IN_STOCK` 또는 `MOUNTED`이고 잔량이 1 이상이어야 스캔할 수 있습니다.
- 스캔한 SFG의 품목이 새로 만들 반제품의 BOM 반제품 구성에 없으면 "오투입"으로 거부됩니다.
- 자재 스캔은 작업지시 BOM의 원자재 목록과 매칭해 검증됩니다.
- 실물 SFG 라벨 스캔값은 직전에 발행한 바코드와 정확히 일치해야 확정됩니다.

## 자주 묻는 질문
- **Q. 설비를 선택했더니 작업지시가 자동으로 채워졌습니다.**
  **A.** 서버가 그 설비의 마지막 작업지시를 기억하고 있다가 복원한 것입니다. 다른 작업지시로 바꾸려면 "변경"을 누르세요.
- **Q. "회로를 선택하세요" 오류가 뜹니다.**
  **A.** 선택한 품목에 도면상 회로 정보가 있는 경우 회로 선택이 필수입니다. 상단 회로 드롭다운에서 고른 뒤 다시 시도하세요.
- **Q. SFG 스캔이 "완제품(FG) 바코드입니다"로 거부됩니다.**
  **A.** FG 라벨을 잘못 스캔했습니다. 이전 공정에서 부착된 SFG 라벨을 스캔하세요.
- **Q. 발행하자마자 라벨이 자동으로 출력되는데, 확정 전에 다시 실행 버튼을 눌러도 되나요?**
  **A.** 아니요. 발행 후에는 실행 버튼이 비활성화됩니다. 라벨을 다시 받으려면 "발행 취소"로 화면 상태를 초기화한 뒤 새로 실행해야 하며, 이 경우 먼저 출력된 실물 라벨은 폐기해야 합니다(서버의 이전 발행 건은 자동 취소되지 않습니다).
