# 환경설정

## 화면 목적

`/system/config` 화면은 MES 전체 동작에 영향을 주는 운영 설정을 그룹별로 관리하는 화면입니다. 자재관리, 생산관리, 품질관리, 시스템 설정은 목록에서 직접 값을 바꾸고 저장합니다. AI, Embedding, AI 카탈로그는 전용 패널에서 모델, API 키, 문서 인덱스, 테이블 설명을 관리합니다.

> 설정값은 다른 화면의 조회 조건, 자동 처리, AI 응답, 도움말 검색 품질에 영향을 줄 수 있습니다. 의미를 모르는 설정은 변경 전 담당자에게 확인하세요.

## 화면 구성

| 영역 | 설명 |
|------|------|
| 제목 영역 | `환경설정` 제목과 화면 설명이 표시됩니다. 일반 설정 탭에서는 `설정 추가`, `설정 저장` 버튼이 보입니다. |
| 그룹 탭 | `전체`, `자재관리`, `생산관리`, `품질관리`, `시스템`, `AI`, `Embedding`, `AI 카탈로그` 탭을 전환합니다. |
| 일반 설정 목록 | `전체/자재관리/생산관리/품질관리/시스템` 탭에서 설정 행이 카드 형태로 표시됩니다. |
| AI 탭 | LLM 채팅 제공자, 모델, 사용 여부, API 키, 연결 테스트를 관리합니다. |
| Embedding 탭 | 도움말/문서 RAG 검색용 임베딩 제공자, 모델, 차원, 청킹 대상, 재색인, 검색 테스트, 인덱스 상태를 관리합니다. |
| AI 카탈로그 탭 | AI SQL 생성에 참고할 테이블 설명, 동의어, JOIN 관계를 관리합니다. |

## ① 일반 설정 목록 항목

| 항목 | 역할 / 의미 |
|------|------|
| 표시 라벨 | 운영자가 읽는 설정 이름입니다. |
| 설명 | 설정을 켜거나 값을 바꿨을 때의 의미를 설명합니다. |
| 설정 키 | 시스템에서 사용하는 영문 키입니다. 예: `OQC_ENABLED`, `MAX_OPEN_TABS` |
| 입력 영역 | 설정 타입에 따라 토글, 선택상자, 숫자 입력, 텍스트 입력으로 표시됩니다. |
| 삭제 아이콘 | 해당 설정을 삭제합니다. 삭제 전 확인 창이 열립니다. |

설정 타입별 입력 방식은 다음과 같습니다.

| 타입 | 화면 입력 방식 | 값 예시 |
|------|------|------|
| `BOOLEAN` | 토글 스위치 | `Y`, `N` |
| `SELECT` | 드롭다운 | options JSON에 등록된 값 |
| `NUMBER` | 숫자 입력 | `10`, `30` |
| `TEXT` | 텍스트 입력 | URL, 코드, 경로, 설명 문자열 |

## ② 설정 추가 모달

`설정 추가` 버튼을 누르면 신규 설정 등록 모달이 열립니다.

| 입력 항목 | 설명 |
|------|------|
| 그룹 | `자재관리`, `생산관리`, `품질관리`, `시스템` 중 선택합니다. |
| 타입 | `사용여부(Y/N)`, `선택`, `숫자`, `텍스트` 중 선택합니다. |
| 설정 키(영문) | 시스템에서 사용할 고유 키입니다. 영문 대문자와 언더스코어 형식을 권장합니다. |
| 표시 라벨 | 화면에 보이는 설정 이름입니다. |
| 설명 | 운영자가 의미를 알 수 있도록 작성합니다. |
| 기본값 | 최초 저장할 설정값입니다. |
| 선택 옵션(JSON) | 타입이 `선택`일 때만 표시됩니다. 예: `[{"value":"V1","label":"옵션1"}]` |

## ③ AI 탭

| 항목 | 역할 / 의미 |
|------|------|
| AI 제공자 | 채팅과 SQL 생성에 사용할 LLM 제공자입니다. `Mistral`, `OpenAI`, `OpenRouter` 중 선택합니다. |
| 모델 | 선택한 제공자의 모델을 선택합니다. |
| AI 채팅 활성화 | AI 채팅 기능 사용 여부입니다. |
| API 키 | 새 키를 입력하면 저장됩니다. 이미 키가 있으면 원문 대신 설정 여부만 표시됩니다. |
| 연결 테스트 | 현재 제공자, 모델, 입력한 API 키로 연결 가능 여부를 확인합니다. |
| 저장 | 제공자, 모델, 활성화 여부, API 키를 저장합니다. |

## ④ Embedding 탭

| 항목 | 역할 / 의미 |
|------|------|
| Embedding 제공자 | 도움말/문서 검색 벡터를 만들 제공자입니다. |
| Embedding 모델 | 문서 chunk와 질문을 벡터화할 모델입니다. |
| 차원 | 선택한 모델의 벡터 차원입니다. 모델 변경 후 재색인이 필요합니다. |
| API 키 | 임베딩 제공자 키입니다. 비우면 저장된 provider 키 또는 서버 환경변수를 사용합니다. |
| 연결 테스트 | 임베딩 API 호출 가능 여부를 확인합니다. |
| 설정 저장 | 임베딩 제공자, 모델, 차원을 저장합니다. |
| 청킹 대상 | 도움말, 표준 문서, 계획 문서, 테이블 카탈로그 등 재색인 대상 경로입니다. |
| 추가 | 프로젝트 상대경로를 청킹 대상에 추가합니다. 절대경로나 `..` 경로는 사용할 수 없습니다. |
| 기본 대상 복원 | 기본 청킹 대상 목록으로 되돌립니다. |
| 청킹 + 임베딩 재생성 | 선택한 문서를 다시 읽고 chunk와 embedding 인덱스를 재생성합니다. |
| 검색 테스트 | 입력 문장으로 RAG 인덱스 검색 결과를 확인합니다. |
| 인덱스 상태 | vector DB 사용 여부, chunk 수, provider, 모델, DB 경로, 마지막 재생성 시간을 표시합니다. |

## ⑤ AI 테이블 카탈로그 탭

| 항목 | 역할 / 의미 |
|------|------|
| 테이블 검색 | 좌측 테이블 목록을 테이블명, 설명, 동의어 기준으로 필터링합니다. |
| 설명 | 선택한 DB 테이블이 어떤 업무 데이터를 의미하는지 작성합니다. |
| 동의어 | 사용자가 질문할 때 쓰는 업무 용어를 쉼표로 입력합니다. |
| 관계(JOIN 키) | 현재 테이블 컬럼과 대상 테이블/컬럼 관계를 선택합니다. |
| 백링크 | 다른 테이블이 현재 테이블을 참조하는 관계를 보여줍니다. |
| 고급(원문) | 카탈로그 Markdown 원문을 직접 편집합니다. |
| DB와 동기화 | 실제 DB 테이블 목록과 카탈로그를 동기화합니다. |
| 변경 저장 | 구조화 편집 내용을 저장합니다. |

## 버튼별 기능

| 버튼 | 언제 사용 | 동작 | 결과 / 주의 |
|------|------|------|------|
| 설정 추가 | 일반 설정을 새로 만들 때 | 신규 설정 모달을 엽니다. | `설정 키`, `표시 라벨`이 없으면 등록할 수 없습니다. |
| 설정 저장 / N개 항목 변경됨 | 일반 설정 값을 바꾼 뒤 | 변경된 항목만 일괄 저장합니다. | 저장 후 전역 설정 캐시가 갱신됩니다. |
| 삭제 | 일반 설정 행을 제거할 때 | 확인 후 설정을 삭제합니다. | 삭제된 설정을 참조하는 기능은 기본값이나 오류 상태가 될 수 있습니다. |
| 연결 테스트 | AI 또는 Embedding 설정 검증 시 | 현재 provider/model/key로 API 호출을 시험합니다. | 성공 후에도 실제 사용 전 모델/권한을 확인하세요. |
| 설정 저장 | AI 또는 Embedding 설정 저장 시 | 전용 설정 키를 생성 또는 수정합니다. | API 키를 비우면 기존 키를 유지합니다. |
| 청킹 + 임베딩 재생성 | 도움말/문서 검색 결과를 최신화할 때 | 선택한 Markdown 문서를 chunk로 나누고 embedding을 생성합니다. | 문서가 많으면 오래 걸릴 수 있습니다. |
| 검색 | Embedding 검색 테스트 시 | RAG 인덱스에서 관련 chunk를 조회합니다. | 결과가 비어 있으면 재색인 또는 대상 경로를 확인하세요. |
| DB와 동기화 | AI 카탈로그에 새 DB 테이블을 반영할 때 | 실제 DB 테이블 목록을 읽어 카탈로그와 맞춥니다. | 기존 설명과 관계는 보존됩니다. |
| 고급(원문) | 구조화 편집으로 표현하기 어려운 경우 | 카탈로그 원문을 textarea로 편집합니다. | 형식 오류를 만들지 않도록 주의하세요. |

## 업무 흐름

1. 일반 설정을 바꿀 때는 그룹 탭을 선택합니다.
2. 원하는 설정의 토글, 선택상자, 숫자, 텍스트 값을 수정합니다.
3. 변경된 행은 강조 표시되고, 상단 저장 버튼이 `N개 항목 변경됨`으로 바뀝니다.
4. `설정 저장`을 누르면 변경값이 저장되고 다른 화면에서 사용하는 전역 설정도 다시 불러옵니다.
5. 새 설정이 필요하면 `설정 추가`를 누르고 그룹, 타입, 설정 키, 라벨, 기본값을 입력한 뒤 등록합니다.
6. AI 답변 모델을 바꿀 때는 `AI` 탭에서 제공자/모델/API 키를 저장하고 `연결 테스트`로 확인합니다.
7. 도움말 또는 설계문서를 추가한 뒤 AI 검색에 반영하려면 `Embedding` 탭에서 대상을 선택하고 `청킹 + 임베딩 재생성`을 실행합니다.
8. AI가 SQL을 만들 때 테이블 의미를 잘못 이해하면 `AI 카탈로그` 탭에서 테이블 설명, 동의어, 관계를 보강합니다.

## 입력 규칙 / 검증

- `설정 키`는 시스템에서 PK처럼 쓰이므로 중복될 수 없습니다.
- 일반 설정의 `SELECT` 타입은 선택 옵션 JSON이 올바른 배열이어야 화면에서 선택지가 표시됩니다.
- `BOOLEAN` 설정은 `Y` 또는 `N`을 사용합니다.
- 일반 설정 저장은 변경된 값만 보냅니다. 저장하지 않고 탭을 이동하면 변경값은 화면 상태에 남아 있지만 DB에는 반영되지 않습니다.
- Embedding 청킹 대상은 프로젝트 상대경로만 허용합니다. 절대경로, `..` 포함 경로, 빈 경로는 추가되지 않습니다.
- API 키는 조회 시 원문을 보여주지 않습니다. 변경할 때만 새 키를 입력합니다.

## 자주 묻는 질문

| 질문 | 답변 |
|------|------|
| 전체 탭에서 AI 설정이 안 보입니다. | AI 관련 설정은 `AI`, `Embedding`, `AI 카탈로그` 전용 탭에서 관리합니다. |
| 값을 바꿨는데 다른 화면에 바로 반영되나요? | 일반 설정 저장 후 전역 설정을 다시 불러옵니다. 이미 열린 화면은 자체 캐시나 조회 시점에 따라 새로고침이 필요할 수 있습니다. |
| Embedding 검색 결과가 오래된 것 같습니다. | 도움말이나 문서를 수정한 뒤 `청킹 + 임베딩 재생성`을 실행해야 RAG 검색 인덱스가 최신화됩니다. |
| API 키가 설정되어 있는지 어떻게 확인하나요? | AI 탭의 키 상태 표시와 `연결 테스트` 결과를 확인합니다. |
| AI 카탈로그 관계는 직접 입력하나요? | 구조화 편집에서는 DB 컬럼 드롭다운을 사용합니다. 필요한 경우에만 `고급(원문)`으로 직접 편집합니다. |

## 관련 화면

| 화면 | 관련 이유 |
|------|------|
| 통신설정관리 | 장비/프린터/통신 설정은 별도 통신설정 화면에서 관리합니다. |
| 사용자관리 | 설정 변경 권한과 사용자 계정을 관리합니다. |
| 역할관리 | 메뉴 접근 권한을 관리합니다. |
| 전체 도움말 | 작성된 도움말 문서를 전체 목록에서 확인합니다. |
