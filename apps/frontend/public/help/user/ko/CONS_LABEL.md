# 소모품 라벨발행

## 화면 목적
소모품(금형·지그·공구 등)을 **개별 인스턴스로 추적**하기 위해 각 개체에 고유한 **소모품 UID(conUid)**를 발행하고, 그 UID가 박힌 라벨(바코드)을 인쇄하는 화면입니다. 같은 소모품 코드라도 입고된 낱개마다 UID가 따로 생기므로, 라벨을 붙여 두면 이후 입고·출고·장착·교체를 개별로 관리할 수 있습니다.

> UID 발행은 곧 **미입고(PENDING) 인스턴스 생성**입니다. 라벨을 인쇄해 실물에 붙인 뒤, 소모품입고 화면에서 바코드를 스캔해 입고확정하면 사용가능(ACTIVE) 상태가 되고 재고수량이 올라갑니다.

## 화면 구성
- **왼쪽(목록)**: 라벨 발행이 가능한 소모품 마스터 그리드. 상단에서 **소모품코드/소모품명 검색**, **카테고리 필터**, **새로고침**을 합니다. 각 행에 **선택 체크박스**와 **발행수량** 입력칸이 있습니다.
- **상단 우측(발행 도구)**: **상태 메시지**(발행/인쇄 진행 안내), **새로고침**, **라벨 템플릿 선택**, **UID 발행** 버튼.
- **오른쪽(상세 패널)**: 목록에서 행을 클릭하면 그 소모품의 **미입고(PENDING) 인스턴스 목록**이 열립니다. 각 인스턴스를 **미리보기**하거나 **재발행**(라벨 다시 인쇄)할 수 있습니다.

## 발행에서 인쇄까지 흐름
```
소모품 선택 + 발행수량 입력
        │
        ▼
[UID 발행] 클릭 → 수량만큼 conUid 채번 (미입고 인스턴스 생성)
        │
        ▼
선택한 라벨 템플릿으로 라벨 그려서 브라우저 인쇄창 호출
        │
        ▼
실물 소모품에 라벨 부착 → 소모품입고 화면에서 스캔·입고확정
```

## ① 목록 컬럼 (왼쪽 그리드)

| 컬럼 | 역할 / 의미 |
|------|------|
| **선택(체크박스)** | 라벨을 발행할 소모품을 고릅니다. 헤더 체크박스로 **전체 선택/해제**가 됩니다. 1건 이상 선택해야 UID 발행이 활성화됩니다. |
| **이미지** | 소모품 사진 썸네일입니다. 클릭하면 크게 볼 수 있고, 인쇄 라벨에도 함께 들어갑니다(템플릿 설정에 따라). 사진이 없으면 `-`로 표시됩니다. |
| **소모품코드(consumableCode)** | 발행 대상 소모품의 고유 코드입니다. 발행되는 모든 UID가 이 코드에 묶입니다. |
| **소모품명(consumableName)** | 소모품 이름입니다. 검색·라벨 표기에 쓰입니다. |
| **카테고리(category)** | 금형(MOLD)·지그(JIG)·공구(TOOL) 구분입니다(공통코드 CONSUMABLE_CATEGORY). 상단 카테고리 필터의 기준입니다. |
| **현재고(stockQty)** | 소모품 마스터에 누적된 보유 재고 수량입니다. 입고확정될 때마다 늘어납니다. |
| **인스턴스(instanceCount)** | 이 소모품에 대해 **지금까지 발행된 UID(개체) 수**입니다. 옆에 노란색으로 **(미입고: n)**이 함께 표시되면, 그중 아직 입고확정되지 않은 PENDING 건수를 뜻합니다. |
| **발행수량(qtyInput)** | 이번에 발행할 UID 개수입니다(1~99). 예: `5`를 넣으면 conUid 5개가 채번됩니다. 기본값은 1입니다. |

## ② 라벨 발행 도구 (상단 우측)

| 항목 | 역할 / 의미 |
|------|------|
| **상태 메시지** | 발행·인쇄 진행 상황을 한 줄로 보여줍니다(예: "{{count}}건 발행 완료", 오류 시 빨간색). 선택 건수가 있고 진행 중이 아니면 "{{count}}건 선택됨"이 표시됩니다. |
| **새로고침** | 목록과 발행수·미입고수를 최신 상태로 다시 불러옵니다. |
| **라벨 템플릿 선택** | 인쇄에 쓸 라벨 디자인을 고릅니다. **기본 디자인** 또는 등록된 템플릿(소모품/지그 카테고리)을 선택합니다. 템플릿마다 라벨 크기·바코드·표시 항목·인쇄 방식이 다릅니다. |
| **UID 발행** | 선택한 소모품을 발행수량만큼 채번하고, 곧바로 라벨 인쇄창을 띄웁니다. 발행 중에는 "발행 중...", 인쇄 준비 중에는 "출력중"으로 바뀝니다. |

## ③ 미입고 인스턴스 패널 (오른쪽)
목록에서 소모품 행을 클릭하면 열립니다. 그 소모품의 **미입고(PENDING) UID**만 보여줍니다(이미 입고된 건은 제외).

| 항목 | 역할 / 의미 |
|------|------|
| **요약(현재고·미입고·보관위치)** | 패널 상단에 선택 소모품의 현재고, 미입고 건수, 기본 보관위치를 요약합니다. |
| **소모품 UID(conUid)** | 발행된 개별 UID입니다. 라벨·바코드에 박히는 추적 키입니다. |
| **상태** | 인스턴스 상태 배지입니다. 이 패널은 **미입고(PENDING)**만 표시합니다(공통코드 CON_STOCK_STATUS). |
| **사용횟수(currentCount / expectedLife)** | 누적 타수와 기대수명입니다. 미입고 단계에서는 보통 `0`입니다. |
| **입고일(recvDate)** | 입고확정된 날짜입니다. 미입고 건은 비어 있습니다(`-`). |
| **미리보기** | 출력 전에 그 UID 라벨의 바코드·배치를 모달로 확인합니다(인쇄하지 않음). |
| **재발행** | 그 UID 라벨을 **다시 인쇄**합니다. 라벨이 훼손·분실됐을 때 사용하며, 라벨 프린터(agent)로 전송됩니다. |

## 사용 순서
1. 왼쪽 목록에서 라벨을 발행할 소모품을 **체크**합니다(검색·카테고리 필터로 빠르게 찾기).
2. 각 행의 **발행수량**을 입력합니다(낱개 수만큼).
3. 상단의 **라벨 템플릿**을 확인·선택합니다.
4. **UID 발행**을 누르면 UID가 채번되고 인쇄창이 뜹니다. 인쇄 다이얼로그에서 출력합니다.
5. 출력한 라벨을 실물 소모품에 붙입니다.
6. (재인쇄가 필요하면) 목록에서 그 소모품을 클릭해 오른쪽 패널에서 해당 UID를 **미리보기/재발행**합니다.
7. 라벨 부착 후 **소모품입고** 화면에서 바코드를 스캔해 입고확정합니다.

## 입력 규칙 / 검증
- **소모품을 1건 이상 선택**해야 UID 발행 버튼이 활성화됩니다.
- **발행수량**은 1~99 범위입니다(범위를 벗어나면 자동 보정).
- 인쇄는 **새 창(팝업)**으로 열립니다. 브라우저가 팝업을 차단하면 발행이 중단되니, 이 사이트의 팝업을 허용하세요.
- 발행수가 많거나 바코드 생성이 끝나기 전에는 미리보기에서 라벨을 먼저 확인한 뒤 출력하세요(바코드 미완료 시 출력이 막힙니다).

## 자주 묻는 질문
- **Q.** UID를 발행하면 바로 재고가 늘어나나요?
  **A.** 아닙니다. 발행은 **미입고(PENDING)** 인스턴스를 만들 뿐입니다. 라벨을 붙이고 **소모품입고**에서 스캔·입고확정해야 사용가능(ACTIVE)이 되고 현재고가 늘어납니다.
- **Q.** 라벨을 잃어버렸어요. 다시 뽑으려면?
  **A.** 목록에서 그 소모품을 클릭하고, 오른쪽 미입고 패널에서 해당 UID의 **재발행**을 누르세요. 같은 UID로 다시 인쇄됩니다.
- **Q.** 인쇄창이 안 떠요.
  **A.** 브라우저 팝업 차단 때문입니다. 주소창의 팝업 허용을 켜고 다시 발행하세요.
- **Q.** 발행수량을 100 이상 넣을 수 없어요.
  **A.** 한 번에 1~99까지만 발행됩니다. 많으면 나눠서 발행하세요.
- **Q.** 인스턴스 옆의 "(미입고: n)"은 무슨 뜻인가요?
  **A.** 발행은 됐지만 아직 입고확정되지 않은 UID 수입니다. 라벨 부착 후 입고 처리가 남았다는 신호입니다.
