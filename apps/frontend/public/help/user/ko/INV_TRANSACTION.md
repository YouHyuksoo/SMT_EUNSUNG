# 자재수불이력조회

## 화면 목적
모든 자재 수불(입고·출고·이동·보정·폐기 등) 트랜잭션 이력을 조회합니다. 재고가 변동된 모든 내역을 일자별·유형별로 추적할 수 있습니다.

## 화면 구성
- **DataGrid 목록**: 트랜잭션 번호, 유형, 일자, 창고, 품목, 수량, 상태 등 13개 컬럼 표시
- **상단 필터**: 거래유형 선택 + 일자 범위 + LOT번호 검색

---

## ① DataGrid 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **트랜잭션번호(transNo)** | 각 재고 변동 내역을 식별하는 고유 번호입니다. |
| **유형(transType)** | 재고 변동 유형입니다. **입고/출고/이동/보정/폐기** 등이 있으며 컬러칩으로 구분됩니다. |
| **처리일(transDate)** | 트랜잭션이 처리된 일시입니다. |
| **출고창고(fromWarehouse)** | 재고가 나간 창고(출고/이동 시). |
| **입고창고(toWarehouse)** | 재고가 들어온 창고(입고/이동 시). |
| **품목코드(itemCode)** | 변동된 품목의 코드입니다. |
| **품목명(itemName)** | 변동된 품목의 이름입니다. |
| **LOT번호(matUid)** | 해당 트랜잭션과 연관된 LOT/시리얼 번호입니다. |
| **수량(qty)** | 변동된 수량입니다. **양수(+) = 재고 증가, 음수(-) = 재고 감소**. |
| **상태(status)** | 트랜잭션 상태입니다. `DONE`(정상 처리, 초록) / `CANCELED`(취소, 빨강). |
| **원본거래(original)** | 취소 시 원본 트랜잭션 번호입니다(취소 건만 표시). |
| **비고(remark)** | 트랜잭션에 대한 추가 메모입니다. |

## 사용 순서
1. 상단 필터에서 거래유형, 조회 기간, LOT번호를 입력합니다.
2. 조회 결과에서 특정 트랜잭션의 상세 정보를 확인합니다.
3. 컬럼 필터·정렬 기능으로 원하는 데이터를 찾을 수 있습니다.
4. **내보내기**로 결과를 엑셀로 다운로드할 수 있습니다.

## 주요 거래유형
| 유형 | 의미 | 수량 부호 |
|------|------|----------|
| RECEIVE | 자재입고 | + |
| MAT_OUT | 자재출고 | - |
| ADJUST_IN | 보정증가 | + |
| ADJUST_OUT | 보정감소 | - |
| TRANSFER | 창고간 이동 | 출고창고 -, 입고창고 + |
| LOT_SPLIT_IN/OUT | LOT 분할 | 분할됨 -, 생성됨 + |
| SCRAP | 폐기 | - |
| MISC_IN | 기타입고 | + |
| PROD_CONSUME | 생산소비 | - |
