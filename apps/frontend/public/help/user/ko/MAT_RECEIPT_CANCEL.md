# 자재입고취소

## 화면 목적
**이미 입고 완료(DONE)된 자재 거래**를 선택하여 취소하고 그 내역을 관리합니다. 취소 시 원본 거래의 상태가 `CANCELED`로 변경되고, 반대 부호의 역트랜잭션(Reverse Transaction)이 `STOCK_TRANSACTIONS`에 자동 생성되며 `MatStock` 재고가 차감됩니다.

> 잘못 입고되었거나 품질 이상이 발견된 자재의 입고를 되돌리는 데 사용합니다.

## 화면 구성
- **상단 — 필터 영역**: **기간 필터(DateRangeFilter)**로 조회할 입고 기간을 지정하고, **검색어 입력**으로 품목코드·품목명·거래처·거래번호를 검색할 수 있습니다. **새로고침(Refresh)** 버튼으로 목록을 갱신합니다.
- **하단 — DataGrid 목록**: 취소 가능한 입고(DONE) 건과 이미 취소된(CANCELED) 건을 표시합니다. 상태 필터로 **DONE(정상)** / **CANCELED(취소)**를 구분하여 볼 수 있습니다.
- **모달 — 취소 처리**: 목록에서 **취소** 버튼을 클릭하면 취소 확인 모달이 열립니다. 거래 정보를 확인하고 **취소 사유**를 입력하여 취소를 확정합니다.

## ① DataGrid 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **거래일(transDate)** | 입고 거래가 발생한 일자입니다. 기간 필터로 조회 범위를 지정할 수 있습니다. |
| **거래번호(transNo)** | 입고 거래의 고유 식별 번호(PK)입니다. |
| **거래유형(transType)** | 거래 유형입니다. 입고취소 화면에서는 `RECEIVE` 유형만 표시됩니다. |
| **품목코드(itemCode)** | 입고된 품목의 코드입니다. |
| **품목명(itemName)** | 입고된 품목의 이름입니다. |
| **자재시리얼(matUid)** | 입고된 자재 LOT의 고유 시리얼입니다. |
| **공급처(vendorName)** | 자재를 납품한 거래처명입니다. |
| **창고(warehouseName)** | 자재가 입고된 창고명입니다. |
| **수량(qty)** | 입고된 수량입니다. |
| **상태(status)** | `DONE`(정상, 초록색) / `CANCELED`(취소, 빨간색)로 표시됩니다. |
| **액션(actions)** | **취소** 버튼이 있습니다. `CANCELED` 상태인 건은 버튼이 비활성화됩니다. |

## ② 취소 처리 모달

| 항목 | 역할 / 의미 |
|------|------|
| **거래번호(transNo)** | 취소 대상 원본 거래번호가 표시됩니다. |
| **거래일(transDate)** | 원본 거래일이 표시됩니다. |
| **품목코드 / 품목명** | 취소 대상 품목이 표시됩니다. |
| **자재시리얼(matUid)** | 취소 대상 LOT의 시리얼이 표시됩니다. |
| **공급처 / 창고** | 거래처와 창고 정보가 표시됩니다. |
| **취소 수량** | 원본 입고 수량이 표시됩니다. |
| **취소 사유(reason)** | **필수 입력**입니다. 취소하는 이유를 구체적으로 입력합니다(예: "품질 불량으로 반품", "입고 수량 오류 정정"). |
| **취소 확정** | 사유 입력 후 **취소** 버튼을 눌러 확정합니다. |

## 사용 순서
1. 상단 **기간 필터**로 조회할 입고 기간을 선택하고 **검색** 버튼을 누릅니다.
2. 목록에서 취소할 입고 건을 찾아 **취소** 버튼을 클릭합니다.
3. 취소 모달에서 거래 정보를 확인하고 **취소 사유**를 입력합니다.
4. **취소 확정** 버튼을 누르면 역트랜잭션이 생성되고 해당 건의 상태가 `CANCELED`로 변경됩니다.
5. 목록에서 상태 필터로 취소 내역을 확인할 수 있습니다.

## 입력 규칙 / 검증
- 취소 사유는 **필수 입력**입니다. 비어 있으면 취소 버튼이 비활성화됩니다.
- **이미 취소된 건**은 다시 취소할 수 없습니다(버튼 비활성화).
- 입고 유형(`RECEIVE`)이 아닌 거래는 이 화면에 나타나지 않으며 취소 대상이 아닙니다.
- 취소 대상 입고 건과 연결된 **후행 작업(progress)이 있는 경우** 취소가 차단될 수 있습니다(예: 해당 LOT이 이미 생산에 투입된 경우).

## 자주 묻는 질문
- **Q.** 취소한 내역을 되돌릴 수 있나요? **A.** 취소 후에는 원복할 수 없습니다. 재입고가 필요하면 정규 입고 절차(MAT_RECEIVE)로 다시 입고 처리해야 합니다.
- **Q.** 목록에 입고 건이 보이지 않아요. **A.** 기간 필터가 적절한지 확인하세요. 기본은 당일 기준이며, 지난 입고는 기간을 넓혀 조회합니다. 또한 `RECEIVE` 유형만 표시됩니다.
- **Q.** 취소 버튼이 비활성화되어 있어요. **A.** 해당 건이 이미 `CANCELED` 상태이거나, 후행 작업(생산 투입 등)이 진행되어 취소가 불가능한 상태입니다.
- **Q.** 취소 시 재고는 어떻게 되나요? **A.** 원본 입고 수량만큼 `MatStock`에서 차감되며, `STOCK_TRANSACTIONS`에 역부호 트랜잭션이 기록됩니다.
