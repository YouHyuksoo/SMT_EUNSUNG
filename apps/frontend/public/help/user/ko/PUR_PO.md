# PO관리

## 화면 목적
공급업체(SUPPLIER)에 원자재 등을 발주하는 **구매발주서(PO)**를 등록·조회·수정·삭제하는 화면입니다. PO가 확정(CONFIRMED)되어야 이후 입하·수입검사 단계로 진행되며, 모든 품목의 입고가 완료되면 마감(CLOSED)합니다.

## 화면 구성
- **왼쪽(PO 목록 그리드)**: 등록된 PO를 표 형태로 조회합니다. 상단에서 PO번호·거래처·품목을 통합 검색하거나, 발주일 구간·상태로 필터링할 수 있습니다.
- **오른쪽(등록/수정 패널)**: **등록** 버튼 또는 행의 ✏️을 클릭하면 우측에서 슬라이드되어 열립니다. PO 헤더 정보와 품목 목록을 함께 입력합니다.

---

## ① PO 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **PO No.(poNo)** | 발주를 식별하는 고유 번호입니다. 등록 시 자동 채번되며(`PO-YYYYMMDD-NNN` 형식), 한 번 생성하면 변경할 수 없습니다. |
| **거래처(partnerName)** | 발주 대상 공급업체 이름입니다. 거래처 마스터의 SUPPLIER 유형에서 선택합니다. |
| **발주일(orderDate)** | PO를 발행한 날짜입니다. 기본값은 오늘(등록 시). |
| **납기일(dueDate)** | 납품 요청 기한입니다. 입하 관리에서 납기 준수율을 판단하는 기준이 됩니다. |
| **품목수(itemCount)** | 이 PO에 포함된 발주 품목의 라인 수입니다. |
| **총금액(totalAmount)** | 품목별 (발주수량 × 단가)의 합계입니다. 단가를 입력하지 않으면 0으로 계산됩니다. |
| **상태(status)** | PO의 현재 진행 상태입니다(아래 상태 흐름 참조). |

### 상태 흐름

```
DRAFT(임시등록)
    ↓ 확정
CONFIRMED(발주 확정)
    ↓ 일부 입고
PARTIAL(부분 입고)
    ↓ 전체 입고
RECEIVED(입고 완료)
    ↓ 마감
CLOSED(마감)
```

> **DRAFT 상태일 때만 수정·삭제가 가능합니다.** CONFIRMED 이후에는 입하 과정에서 상태가 자동으로 변경됩니다.

---

## ② PO 헤더 입력 필드

| 컬럼 | 역할 / 의미 |
|------|------|
| **PO No.(poNo)** | 등록 시 자동으로 채번됩니다(`PO-YYYYMMDD-NNN`). 직접 수정도 가능하나 중복될 수 없습니다. 수정 시에는 변경할 수 없습니다(잠금). |
| **거래처(partnerName)** | 발주할 공급업체를 선택합니다. SUPPLIER 유형 거래처만 선택 목록에 표시됩니다. |
| **발주일(orderDate)** | PO 발행일. 기본값은 오늘. |
| **납기일(dueDate)** | 납품 요청일. 비워둘 수 있으나, 입하 일정 관리를 위해 입력을 권장합니다. |
| **비고(remark)** | PO 전체에 대한 메모입니다(특이사항, 참조 계약 번호 등). |

---

## ③ 발주 품목 입력 필드

품목 추가 버튼을 누르면 품목 검색 모달이 열립니다. 여러 품목을 한 번에 선택할 수 있습니다(다중 선택 지원). 이미 추가된 품목코드는 중복 추가되지 않습니다.

| 컬럼 | 역할 / 의미 |
|------|------|
| **품목코드(itemCode)** | 발주할 품목의 MES 코드입니다. 품목 검색 모달에서 선택하면 자동으로 채워집니다. |
| **품목명(itemName)** | 선택한 품목의 이름입니다(자동 표시). |
| **라인번호(lineNo)** | 이 PO 내에서의 품목 행 순번입니다. 기본값은 추가 순서대로 자동 부여됩니다. |
| **릴리즈번호(revNo)** | 동일 품목의 발주 차수입니다. 같은 PO에서 동일 품목을 다른 납기로 분할할 때 구분합니다. 기본값은 1. |
| **발주수량(orderQty)** | 이 라인에서 발주하는 수량입니다. **1 이상의 정수**만 입력할 수 있습니다. |
| **비고(remark)** | 해당 품목 라인에 대한 메모입니다. |

> 단가(unitPrice)는 현재 화면에서 직접 입력하는 UI는 표시되지 않습니다. 필요 시 운영자에게 문의하세요. 총금액은 단가 정보가 있을 때 자동 계산됩니다.

---

## 검색·필터

| 필터 | 동작 |
|------|------|
| **통합 검색** | PO 번호 또는 거래처명을 입력하면 대소문자 구분 없이 부분 일치 검색합니다. |
| **발주일 구간** | 발주일 기준 조회 기간을 설정합니다. 기본값은 오늘(당일). |
| **상태** | 선택한 상태의 PO만 표시합니다. 비우면 전체 조회. |

---

## 사용 순서

1. 우측 상단 **등록** 버튼을 클릭해 PO 등록 패널을 엽니다.
2. **거래처**를 선택합니다(SUPPLIER 유형).
3. **발주일**과 **납기일**을 입력합니다.
4. **품목 추가** 버튼으로 발주할 품목을 검색·선택합니다(다중 선택 가능).
5. 각 품목의 **발주수량**을 입력합니다(1 이상 정수 필수).
6. **등록** 버튼으로 저장합니다. PO 번호가 자동 채번되어 DRAFT 상태로 생성됩니다.
7. 발주 내용이 확정되면 그리드에서 해당 PO를 열어 상태를 **확정(CONFIRMED)**으로 전환합니다.
8. 입고 처리는 **입하관리** 화면에서 진행합니다.

---

## 입력 규칙 / 검증

- **PO No.**: 필수. 중복 불가.
- **거래처**: 필수.
- **발주 품목**: 1개 이상 필수.
- **발주수량**: 1 이상의 정수 필수. 소수점·0·음수 입력 시 저장이 차단됩니다.
- **삭제 가능 조건**: DRAFT 상태이고, 입하 이력이 없어야 합니다.

---

## 자주 묻는 질문

- **Q.** PO 번호를 직접 지정하고 싶어요.
  **A.** 자동 채번된 번호를 수정해 직접 입력할 수 있습니다. 단, 중복되면 저장이 차단됩니다.

- **Q.** 확정한 PO를 다시 수정하고 싶어요.
  **A.** CONFIRMED 이후 상태는 화면에서 직접 수정할 수 없습니다. 운영자에게 DRAFT로 되돌리는 절차를 문의하세요.

- **Q.** PO를 삭제하려는데 삭제가 안 돼요.
  **A.** DRAFT 상태이고, 입하 이력이 없어야 삭제할 수 있습니다. CONFIRMED 이후이거나 입하가 일부라도 진행됐다면 삭제가 차단됩니다.

- **Q.** 총금액이 0으로 표시돼요.
  **A.** 단가가 입력되지 않으면 총금액은 0으로 계산됩니다. 단가 등록은 운영자 설정이 필요합니다.

- **Q.** 동일 품목을 같은 PO에 두 번 추가할 수 있나요?
  **A.** 동일 품목코드는 한 번만 추가됩니다. 납기·차수를 달리 해야 한다면 릴리즈번호를 구분하거나 별도 PO를 생성하세요.
