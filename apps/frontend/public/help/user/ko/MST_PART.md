# 품목마스터

## 화면 목적
MES에서 다루는 **모든 품목(원자재·반제품·완제품·소모품)의 기본정보**를 등록·관리합니다. 여기 등록된 품목이 BOM·생산실적·자재수불·수입검사(IQC)·재고의 기준이 됩니다.

## 화면 구성
- **좌측 메인 영역**: 품목 DataGrid. 상단 툴바에 검색·필터·액션 버튼 배치.
- **우측 슬라이드 패널**: 행 ✏(수정) 또는 **품목 추가** 버튼 클릭 시 열리는 등록/수정 폼.

### 툴바 항목
| 영역 | 구성 | 설명 |
|------|------|------|
| 검색 | 입력창 | 품목코드 또는 품목명 검색 (300ms 디바운스) |
| 필터 | 품목유형(ITEM_TYPE), 사용여부(Y/N), IQC대상(Y/N), 검사구분(IQC_INSPECT_METHOD), AQL 정책 | 좌측 필터 변경 시 목록 자동 갱신 |
| 액션 | ERP 동기화, 새로고침, 품목 추가 | 우측 상단 버튼 그룹 |

## ① 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **품번(itemNo)** | 도면·ERP·고객 문서에서 쓰는 품번. 현장에서 가장 많이 부르는 번호. |
| **사진(imageUrl)** | 품목 사진 썸네일. 로드 실패 시 아이콘 fallback. |
| **품목코드(itemCode)** | MES 내부 식별 고유 코드. 등록 후 변경 불가. |
| **품목명(itemName)** | 현장 식별용 품목명. |
| **품목유형(itemType)** | 뱃지 표시: 원자재(파랑)/반제품(노랑)/완제품(초록)/소모품(보라). 자재수불·생산 처리 방식 분기. |
| **품목그룹(productType)** | 공통코드 PRODUCT_TYPE 기반 품목군. |
| **차종(modelName)** | 차량 모델 또는 차종 관리 특성. |
| **모델구분(defectModelGroup)** | 불량코드 적용 범위를 모델군으로 구분. 공통코드 DEFECT_MODEL_GROUP. |
| **규격(spec)** | 품목 규격·사양·치수. |
| **Rev** | 도면/사양 리비전. |
| **마킹문구(markingText)** | 라벨·마킹 설비 전달 문구. |
| **고객품번(custPartNo)** | 고객사 사용 품번(출하·라벨 매칭용). |
| **단위(unit)** | 공통코드 UNIT_TYPE. 재고·불출 기준 단위. 코드-명칭 표시. |
| **색상(color)** | 전선색 등 품목 색상 정보. |
| **박스장입수량(boxQty)** | 박스 하나에 장입하는 기준 수량. |
| **최소불출단위수량(자재)(minPackQty)** | 자재 불출 최소 단위 수량. |
| **묶음단위수량(생산공정품)(lotUnitQty)** | 생산 공정품 묶음 처리 기준 수량. |
| **검사구분(inspectMethod)** | FULL(검사/빨강) 또는 SKIP(무검사/회색) 뱃지. |
| **기본시료수(sampleQty)** | IQC 검사 기본 시료 수량(AQL 샘플수와 별개). |
| **AQL 정책(iqcAqlPolicyCode)** | 연결된 AQL 정책 코드. |
| **유효기간(expiryDate)** | 입고/제조 기준 유효기간 일수. "N일" 표시. |
| **연장기간(expiryExtDays)** | 유효기간 연장 가능 최대 일수. "N일" 표시. |
| **팔레트구성단위(packUnit)** | 팔레트당 박스 수. |
| **품목고정 적재로케이션(storageLocation)** | 품목별 기본 적재 로케이션. |
| **사용여부(useYn)** | Y(초록)/N(빨강) 뱃지. N이면 행 전체 빨간색. |

- 클릭하여 이미지 파일(jpeg/png/gif/webp) 업로드.

## 업무 흐름
1. **품목 추가**(또는 행 ✏)로 슬라이드 패널을 엽니다.
2. **기본정보** 탭에서 품목코드·품번·품목명·품목유형·품목그룹 등 필수 항목 입력.
3. **수량** 탭에서 박스장입수량·안전재고·유효기간 등 필요값 입력.
4. 품목에 사진이 필요하면 사진 영역 클릭하여 업로드.
5. **저장** 버튼으로 등록/수정 완료.
6. IQC 대상(`iqcYn=Y`)이면 **검사구분**과 **AQL 정책**을 연결.
7. ERP 품목을 대량 가져오려면 **ERP 동기화** 실행.

## 입력 규칙 / 검증
- **품목코드**: 등록 후 수정 불가(불변 키). BOM·재고·실적이 이 코드로 연결됨.
- **품번·품목명·품목그룹**: 필수 입력(저장 버튼 비활성화).
- **AQL 정책**: `IQC대상=Y` + `검사구분=FULL`인 경우 **필수**. 미설정 시 저장 불가.
- **패널 열린 상태**: 다른 행 클릭 시 변경사항 유실 방지 확인 모달.
- **사용여부=N**: 목록에서 빨간색 행 표시. 조회·선택·사용 제외.

## 자주 묻는 질문
- **Q.** 품목코드와 품번은 무엇이 다른가요?
  **A.** 품목코드는 MES 내부 식별자(불변), 품번은 도면/ERP/고객 문서에서 쓰는 번호입니다.
- **Q.** AQL 정책을 비워두면?
  **A.** `IQC대상=N` 또는 `검사구분=SKIP`일 때는 필수가 아닙니다. `IQC대상=Y`+`검사=FULL`이면 AQL 자동 판정을 위해 반드시 설정해야 합니다.
- **Q.** ERP 동기화로 잘못 추가됐어요.
  **A.** ERP-IF로 들어온 품목은 `CREATED_BY='ERP-IF'`로 식별 가능합니다. 운영자가 정리할 수 있습니다.
- **Q.** 사진 업로드 후 목록에서 안 보여요.
  **A.** 저장 후 목록을 새로고침 해보세요. 사진 업로드는 저장과 별도 API로 처리됩니다.
