# 품목마스터

## 화면 목적
품목마스터는 은성전장 MES에서 사용하는 품목 기준정보를 등록하고 관리하는 화면입니다. 여기서 관리한 품목은 BOM, 생산, 자재수불, 입고검사, 재고 조회에서 공통 기준으로 사용됩니다.

주 데이터는 `ID_ITEM` 테이블에 저장되며, 조직별 `ORGANIZATION_ID`와 `ITEM_CODE` 조합으로 품목을 식별합니다.

## 화면 구성

| 영역 | 설명 |
|------|------|
| 상단 필터 | 검색어, 품목유형, 사용여부, IQC대상, 검사구분, AQL 정책으로 목록을 좁힙니다. |
| 품목 목록 | 품목코드, 품번, 품목명, 분류, 규격, IQC 조건, 수량 기준, 적재 위치, 사진을 조회합니다. |
| 우측 패널 | **품목 추가** 또는 행의 수정 버튼을 눌렀을 때 등록/수정 폼이 열립니다. |
| 확인 모달 | 삭제, ERP 동기화, 변경사항 취소처럼 영향이 있는 작업 전에 확인합니다. |

## 조회와 필터

| 항목 | 사용 방법 |
|------|----------|
| 검색어 | 품목코드, 품번, 품목명, 고객품번, 차종, 규격, 마킹문구를 검색합니다. |
| 품목유형 | 원자재, 반제품, 완제품, 소모품 등 공통코드 `ITEM_TYPE` 기준으로 필터링합니다. |
| 사용여부 | 사용 품목과 미사용 품목을 구분합니다. 미사용 품목은 행 색상으로도 구분됩니다. |
| IQC대상 | 수입검사 대상 여부를 `Y/N`으로 필터링합니다. |
| 검사구분 | 공통코드 `IQC_INSPECT_METHOD` 기준으로 검사 또는 무검사 품목을 찾습니다. |
| AQL 정책 | 특정 AQL 정책이 연결된 품목만 조회합니다. |

## 주요 컬럼

| 컬럼 | 의미 |
|------|------|
| 품목코드 | MES 내부 품목 식별자입니다. 등록 후에는 수정할 수 없습니다. |
| 품번 | 도면, ERP, 고객 문서에서 사용하는 품번입니다. |
| 품목명 | 현장에서 식별할 품목명입니다. |
| 품목유형 | 원자재, 반제품, 완제품, 소모품 등 품목 흐름 분류입니다. |
| 제품유형 | 공통코드 `PRODUCT_TYPE` 기반 품목군입니다. |
| 차종 / 모델구분 | 차량 모델과 불량 모델군을 구분하는 관리 항목입니다. |
| 규격 / Rev / 마킹문구 | 도면, 사양, 라벨, 마킹 설비에 필요한 보조 정보입니다. |
| 단위 / 색상 | 재고 수량 단위와 품목 색상 정보입니다. |
| IQC대상 / 검사구분 / AQL 정책 | 수입검사 적용 여부와 판정 기준입니다. |
| 박스장입수량 / 최소불출단위 / 묶음단위 | 포장, 불출, 생산 공정품 처리 기준 수량입니다. |
| 유효기간 / 연장기간 | 입고 또는 제조 기준 유효기간과 연장 가능 일수입니다. |
| 적재로케이션 | 품목별 기본 적재 위치입니다. |
| 사용여부 | 품목을 계속 사용할지 여부입니다. |

## 등록과 수정

1. 신규 품목은 **품목 추가**를 누릅니다.
2. 기존 품목은 목록 행의 수정 버튼을 누릅니다.
3. 우측 패널에서 필수 항목을 입력합니다.
4. 사진이 필요하면 사진 영역을 눌러 업로드합니다.
5. **저장**을 눌러 반영합니다.

필수 항목은 품목코드, 품번, 품목명, 품목유형, 제품유형입니다. 수정 화면에서는 품목코드가 비활성화됩니다.

## 입력 항목

| 구분 | 항목 |
|------|------|
| 기본정보 | 품목코드, 품번, 품목명, 고객품번, Rev, 마킹문구, 품목유형, 제품유형, 차종, 모델구분, 규격, 색상, 단위, 사용여부 |
| IQC | IQC대상, 검사구분, 기본시료수, AQL 정책 |
| 수량 | 박스장입수량, 최소불출단위수량, 묶음단위수량, 안전재고, 유효기간, 유효기간 연장, 팔레트구성단위 |
| 위치 | 품목고정 적재로케이션 |
| 사진 | 품목 사진 업로드, 변경, 삭제 |
| 비고 | 품목 관리 참고사항 |

## IQC와 AQL 규칙

- `IQC대상=Y`이고 `검사구분=검사(FULL)`이면 AQL 정책을 반드시 선택해야 합니다.
- `IQC대상=N`이거나 `검사구분=무검사(SKIP)`이면 AQL 정책은 필수가 아닙니다.
- 기본시료수는 품목에 저장하는 기준 수량이며, AQL 정책의 LOT별 자동 샘플수와는 별도입니다.

## 버튼별 기능

| 버튼 | 기능 |
|------|------|
| ERP 동기화 | ERP 품목 마스터를 MES로 가져옵니다. 실행 전 확인 모달이 표시되고, 완료 후 신규/변경 건수를 보여줍니다. |
| 새로고침 | 현재 필터 조건으로 품목 목록을 다시 조회합니다. |
| 품목 추가 | 우측 등록 패널을 엽니다. |
| 수정 | 선택한 품목을 우측 패널에서 수정합니다. |
| 삭제 | 품목을 삭제합니다. 재고, BOM, 생산계획 등 참조 데이터가 있으면 삭제되지 않을 수 있습니다. |
| 사진 업로드/삭제 | 품목 사진을 등록하거나 제거합니다. |

## 주의사항

- 품목코드는 BOM, 재고, 생산 실적과 연결되는 기준값이므로 등록 후 변경하지 않습니다.
- 이미 사용 이력이 있는 품목은 삭제보다 `사용여부=N` 처리가 안전합니다.
- ERP 동기화는 대량 변경을 만들 수 있으므로 실행 전 현재 필터가 아니라 전체 ERP 품목 기준으로 반영된다는 점을 확인합니다.
- 우측 패널에서 변경한 뒤 다른 행을 선택하거나 닫을 때는 변경사항 유실 방지 확인이 표시됩니다.
