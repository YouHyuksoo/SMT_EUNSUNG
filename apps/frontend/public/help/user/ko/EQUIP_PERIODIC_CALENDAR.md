# 정기점검 캘린더

## 화면 목적
설비의 **정기점검**(주기적으로 수행하는 예방점검)을 월별 캘린더로 관리하는 화면입니다.
설비별 점검항목 마스터에 정의된 **주기(cycle)**에 따라 "어느 날 어떤 설비를 점검해야 하는지"가 자동으로 계산되어 캘린더에 표시되고, 날짜를 선택해 설비별로 점검 결과(합격/불합격)를 입력합니다.

> 일상점검 캘린더와 화면 구조·조작이 동일하며, **점검 유형만 정기점검(PERIODIC)**입니다. 점검항목과 결과는 일상점검과 별도로 관리됩니다.

## 화면 구성
- **상단 헤더**: 공정 필터, 당월/차월 생성 버튼, 새로고침
- **월간 통계 카드 4종**: 총 예정 / 완료 / 불합격 / 지연
- **왼쪽 — 캘린더**: 날짜별 점검 진행상황을 색·진행바로 표시
- **오른쪽 — 일자별 패널**: 선택한 날짜에 점검해야 할 설비 목록 + 점검 실행/개별 추가
- **점검 실행 모달**: 항목별 합·불 입력 및 저장

## 개념 관계
```
설비별 점검항목(주기 cycle) ──자동 스케줄──▶ 캘린더(날짜별 점검대상) ──날짜 선택──▶ 설비별 점검 실행(항목 합·불) ──▶ 종합판정(PASS/FAIL)
```

---

## ① 상단 헤더

| 항목 | 역할 / 의미 |
|------|------|
| **공정 필터** | 특정 공정의 설비만 캘린더·일자패널에 표시합니다. 비우면 전체 공정 대상입니다. |
| **당월 생성** | 오늘이 속한 달로 이동해 그 달의 점검 스케줄을 다시 계산해 표시합니다. |
| **차월 생성** | 다음 달로 이동해 다음 달 점검 스케줄을 미리 확인합니다. |
| **새로고침** | 현재 보고 있는 달의 데이터를 다시 불러옵니다. |

## ② 월간 통계 카드

| 카드 | 역할 / 의미 |
|------|------|
| **총 예정** | 이번 달에 점검 예정인 (설비×날짜) 건수 합계입니다. 주기(cycle)에 따라 자동 산정됩니다. |
| **완료** | 점검 결과가 입력되어 완료된 건수입니다. |
| **불합격** | 종합판정이 **불합격(FAIL)**인 건수입니다. |
| **지연** | 점검 예정일이 지났는데 아직 미점검인 **지연(OVERDUE)** 날짜 수입니다. |

## ③ 캘린더

각 날짜 셀은 그 날 점검 진행상황을 한눈에 보여줍니다.

| 표시 | 역할 / 의미 |
|------|------|
| **비율 텍스트 (N/M)** | 완료 건수 / 전체 예정 건수입니다. |
| **진행바** | 완료 비율을 막대로 표시합니다. |
| **상태 색** | 날짜별 상태를 색으로 구분합니다(아래). |

> 날짜 상태(status)
> - **전체완료(ALL_PASS)**: 예정 점검을 모두 완료했고 불합격이 없음 — 초록
> - **불합격 있음(HAS_FAIL)**: 불합격 건이 있음 — 빨강
> - **진행중(IN_PROGRESS)**: 일부만 완료 — 노랑
> - **지연(OVERDUE)**: 예정일이 지났는데 미시작 — 빨간 테두리
> - **예정(NOT_STARTED)**: 아직 오지 않은 미래 예정일
> - **없음(NONE)**: 그 날 점검 스케줄 없음

## ④ 일자별 패널 — 설비 카드

선택한 날짜에 점검해야 할 설비가 카드로 나열됩니다.

| 항목 | 역할 / 의미 |
|------|------|
| **설비코드 / 설비명** | 점검 대상 설비입니다. |
| **라인 / 설비유형** | 설비의 라인코드·유형(참고 정보)입니다. |
| **상태 아이콘** | 합격(초록 체크)·불합격(빨강 X)·미점검(회색 시계)을 나타냅니다. |
| **종합 판정** | 점검 완료 시 합격(PASS)/불합격(FAIL) 배지로 표시됩니다. |
| **점검항목 태그** | 그 설비의 점검 항목들입니다. 항목별로 ✅합격 / ❌불합격 / ⏳미점검이 표시됩니다. |
| **점검자** | 점검을 수행한 작업자명입니다. |
| **실행 / 편집 버튼** | 미점검이면 **점검 실행**, 완료면 **편집**으로 결과를 수정합니다. |
| **개별 점검 추가** | 패널 우측 상단 버튼. 그 날 자동 스케줄에 없는 설비라도 직접 골라 점검을 추가합니다. |

## ⑤ 점검 실행 모달

| 항목 | 역할 / 의미 |
|------|------|
| **점검일** | 선택한 날짜로 자동 지정됩니다(수정 불가). |
| **점검자** | 점검을 수행한 작업자를 선택합니다. |
| **점검 항목(순서·항목명·기준·기준이미지)** | 마스터에 정의된 항목과 판정 기준이 표시됩니다. 기준이미지가 있으면 함께 보여줍니다. |
| **결과(합격/불합격)** | 항목별로 합격(PASS)·불합격(FAIL)을 선택합니다. |
| **불합격 사유(비고)** | 항목을 불합격으로 선택하면 사유를 입력합니다. |
| **종합 판정** | 입력 결과로 자동 계산됩니다. **불합격 항목이 1개라도 있으면 종합 불합격(FAIL)**, 모두 합격이면 합격(PASS)입니다. |
| **전체 비고** | 점검 전체에 대한 메모입니다(선택). |

> ⚠️ 종합판정이 **불합격(FAIL)**으로 저장되면 해당 설비는 자동으로 **인터락(INTERLOCK, 사용잠금)** 상태가 됩니다. 조치 후 설비 상태를 정상으로 되돌려야 합니다.

## ⑥ 개별 점검 추가 모달

| 항목 | 역할 / 의미 |
|------|------|
| **설비 선택** | 점검할 설비를 고릅니다. 선택하면 그 설비의 **정기점검 항목**이 마스터에서 자동으로 불러와집니다. |
| **다음** | 불러온 항목으로 점검 실행 모달을 엽니다. |

---

## 사용 순서
1. **공정 필터**(필요 시)와 **당월/차월 생성**으로 점검할 달을 표시합니다.
2. 캘린더에서 점검할 **날짜를 클릭**하면 오른쪽에 그 날 점검 대상 설비가 나옵니다.
3. 설비 카드의 **점검 실행**을 눌러 항목별 **합격/불합격**을 입력합니다(불합격은 사유 입력).
4. **저장**하면 종합판정이 자동 계산되고 캘린더 진행상황이 갱신됩니다.
5. 자동 스케줄에 없는 설비를 추가로 점검하려면 **개별 점검 추가**를 사용합니다.

## 입력 규칙 / 검증
- 점검 항목은 **모두 합격/불합격을 선택**해야 종합판정이 확정됩니다(미입력 항목이 있으면 저장 전 확인).
- **불합격 항목**은 사유(비고)를 입력하는 것을 원칙으로 합니다.
- 점검일은 캘린더에서 선택한 날짜로 고정됩니다.
- 같은 설비·같은 날짜의 정기점검은 1건으로 관리되며, 다시 점검하면 **편집**으로 수정됩니다.

## 자주 묻는 질문
- **Q.** 캘린더에 점검 대상이 안 보입니다.
  **A.** 설비별 점검항목 마스터에 **정기점검(PERIODIC) 항목**과 **주기(cycle)**가 등록돼 있어야 합니다. 주기에 따라 해당 날짜에만 표시됩니다(주간=월요일, 월간=매월 1일).
- **Q.** 일상점검과 무엇이 다른가요?
  **A.** 화면 조작은 같지만 점검 유형이 정기점검이며, 항목·결과가 일상점검과 별도로 집계됩니다.
- **Q.** 불합격으로 저장했더니 설비가 잠겼습니다.
  **A.** 불합격 시 설비가 인터락(사용잠금)됩니다. 원인 조치 후 설비 상태를 정상으로 전환하세요.
- **Q.** 지연(OVERDUE)은 언제 표시되나요?
  **A.** 점검 예정일이 지났는데 점검 기록이 하나도 없는 날짜에 표시됩니다.
