# 문서관리

## 화면 목적
품질·공정 운영에 쓰는 **문서화된 정보(절차서·지침서·양식 등)를 등록하고 개정·승인·만료를 관리**하는 화면입니다. IATF 16949 7.5(문서화된 정보)에 대응합니다. 등록한 문서는 **초안(DRAFT) → 승인(APPROVED) → 개정(OBSOLETE 처리 후 새 초안)** 흐름으로 관리되며, 만료가 가까운 문서는 목록에서 강조 표시됩니다.

## 화면 구성
- **상단 통계 카드 4개**: 전체 문서 수 / 초안 / 승인됨 / 만료임박(30일 이내) 건수를 보여줍니다.
- **상단 도구막대**: 문서번호·문서명 검색, **문서유형** 필터, **상태** 필터, **새로고침**, **추가** 버튼.
- **목록(DataGrid)**: 문서 그리드. 만료 30일 이내 문서는 행이 노란색 배경으로 강조되고 만료일에 경고 아이콘이 붙습니다. 행의 ✏️(연필) 또는 행 클릭으로 우측 패널이 열립니다.
- **우측 슬라이드 패널**: 등록/수정 폼. 수정 시 상단에 문서번호·개정번호·상태·승인자 등 읽기전용 정보가 표시되고, 상태에 따라 **승인**·**개정** 버튼이 나타납니다.

### 상태 흐름
```
초안(DRAFT) ──승인──▶ 승인됨(APPROVED) ──개정──▶ 기존 문서는 폐기(OBSOLETE)
                                              └▶ 새 초안(DRAFT, 개정번호 +1)
```

---

## ① 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **수정(actions)** | 행의 ✏️ 버튼입니다. 클릭하면 우측 등록/수정 패널이 열립니다. |
| **문서번호(docNo)** | 문서를 식별하는 고유 번호입니다. 등록 시 `DOC-YYYYMMDD-NNN` 형식으로 **자동 채번**됩니다(예: DOC-20260624-001). 직접 입력하지 않습니다. |
| **문서명(docTitle)** | 문서의 제목입니다. 검색 대상(문서번호와 함께)입니다. |
| **문서유형(docType)** | 문서의 종류입니다. 공통코드 `DOC_TYPE`로 관리되며 색상 배지로 표시됩니다(절차서·지침서·양식 등). |
| **분류(category)** | 문서를 추가로 묶는 사용자 분류값입니다. 부서·공정 등 자유롭게 입력합니다. |
| **개정번호(revisionNo)** | 문서의 개정 차수입니다. 최초 등록은 Rev.1이며, 개정할 때마다 1씩 증가합니다. |
| **개정일(revisionDate)** | 가장 최근 개정(승인) 시점입니다. 승인 시 자동으로 갱신됩니다. |
| **상태(status)** | 문서의 진행 상태입니다. 공통코드 `DOC_STATUS` 배지로 표시됩니다. 초안(DRAFT)·검토(REVIEW)·승인됨(APPROVED)·폐기(OBSOLETE). |
| **승인자(approvedBy)** | 문서를 승인한 사용자입니다. 승인 시 자동 기록됩니다. |
| **만료일(expiresAt)** | 문서 유효성이 만료되는 날짜입니다. 오늘부터 30일 이내면 행이 노란색으로 강조되고 경고 아이콘이 붙습니다. 비어 있으면 `-`로 표시됩니다. |

## ② 등록/수정 폼 필드

| 필드 | 역할 / 의미 |
|------|------|
| **문서명(docTitle)** | (필수) 문서 제목입니다. 최대 200자. 비우면 저장할 수 없습니다. |
| **문서유형(docType)** | (필수) 공통코드 `DOC_TYPE`에서 선택합니다. 비우면 저장할 수 없습니다. |
| **분류(category)** | (선택) 사용자 분류값입니다. 최대 50자. |
| **파일경로(filePath)** | (선택) 실제 문서 파일이 보관된 경로/링크입니다. 최대 500자. |
| **보존기간(월)(retentionPeriod)** | (선택) 문서를 보존해야 하는 기간을 **개월** 단위로 입력합니다. |
| **설명(description)** | (선택) 문서에 대한 부가 설명입니다. 최대 1000자. |
| **문서번호(docNo)** | (읽기전용) 수정 시 상단에 표시됩니다. 채번된 값으로 변경 불가. |
| **개정번호(revisionNo)** | (읽기전용) 현재 개정 차수(Rev.N)입니다. |
| **상태(status)** | (읽기전용) 현재 상태 배지입니다. |
| **승인자(approvedBy) / 승인일(approvedAt)** | (읽기전용) 승인 처리 후 채워집니다. |

> 문서유형은 코드성 데이터이므로 직접 입력하지 않고 **셀렉트로 선택**합니다. 목록에 없는 유형은 공통코드 `DOC_TYPE`에 먼저 등록되어야 합니다.

## 사용 순서
1. 상단 **추가** 버튼으로 우측 등록 패널을 엽니다.
2. **문서명**과 **문서유형**(필수)을 입력하고, 필요 시 분류·파일경로·보존기간·설명을 채웁니다.
3. **저장**하면 문서번호가 자동 채번되고 **초안(DRAFT)** 상태로 생성됩니다.
4. 내용이 확정되면 문서를 열어 **승인**을 누릅니다. 상태가 **승인됨(APPROVED)**으로 바뀌고 승인자·승인일·개정일이 기록됩니다.
5. 승인된 문서를 변경해야 하면 **개정**을 누릅니다. 기존 문서는 폐기되고, **개정번호가 1 증가한 새 초안**이 만들어집니다. 새 초안을 수정 후 다시 승인합니다.

## 입력 규칙 / 검증
- **문서명·문서유형**이 비어 있으면 저장 버튼이 비활성화됩니다(둘 다 필수).
- 수정은 **초안(DRAFT) 상태에서만** 가능합니다. 승인됨/폐기 상태의 문서는 필드가 읽기전용이 되며 **개정**을 통해서만 바꿀 수 있습니다.
- **승인**은 초안 또는 검토 상태에서만 가능합니다.
- **개정**은 승인됨 상태에서만 가능합니다.
- 보존기간은 1 이상의 정수(개월)입니다.

## 자주 묻는 질문
- **Q.** 문서번호는 어떻게 정하나요?
  **A.** 직접 입력하지 않습니다. 등록 시 `DOC-오늘날짜-순번` 형식으로 자동 채번됩니다.
- **Q.** 승인된 문서를 고치고 싶어요.
  **A.** 승인 후에는 직접 수정할 수 없습니다. **개정** 버튼으로 새 초안을 만들어 수정한 뒤 다시 승인하세요. 이전 버전은 폐기(OBSOLETE)로 남습니다.
- **Q.** 만료임박은 며칠 기준인가요?
  **A.** 만료일이 **오늘부터 30일 이내**인 문서가 만료임박으로 집계되고 목록에서 강조됩니다.
- **Q.** 문서를 삭제할 수 있나요?
  **A.** 초안(DRAFT) 상태에서만 삭제할 수 있습니다(운영 가이드 참조).
