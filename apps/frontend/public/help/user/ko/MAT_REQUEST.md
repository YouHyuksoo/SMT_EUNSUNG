# 출고요청관리

## 화면 목적

작업지시에 필요한 원자재를 **창고 담당자에게 출고 요청**하는 화면입니다. BOM 기준으로 소요량이 자동 계산되어 수량 입력 부담을 줄이고, 요청·승인·출고까지 이어지는 흐름을 한 화면에서 추적합니다.

> 흐름: 출고요청(REQUESTED) → 승인(APPROVED) → 실출고(COMPLETED) / 반려(REJECTED)

작업지시와 연결하는 방식이 기본이지만, 작업지시 없이 품목을 직접 검색해 **수동 출고요청**도 가능합니다.

## 화면 구성

- **상단 헤더**: 화면 제목 + [수동 출고요청] 버튼.
- **필터 바**: 작업지시번호, 모델(품목코드/품목명), 작업지시 상태를 입력해 좌측 목록을 줄입니다.
- **좌측 — 작업지시 목록**: 현재 존재하는 작업지시를 카드 형태로 나열합니다. 클릭하면 우측에 해당 작업지시의 내역이 표시됩니다.
- **우측 — 출고요청 내역 / 신규 작성**: 작업지시를 선택하면 기존 출고요청 내역이 요청건별로 그룹 표시되고, [신규 작성]을 누르면 BOM 기준 품목 그리드로 전환됩니다. 작업지시를 선택하지 않으면 최근 출고요청 목록이 기본 표시됩니다.

---

## ① 필터 바 항목

| 항목 | 역할 / 의미 |
|------|------|
| **작업지시번호** | 특정 작업지시번호를 입력해 좌측 목록을 필터링합니다. 입력 즉시 300ms 뒤 검색됩니다. |
| **모델 (품목코드/품목명)** | 작업지시에 연결된 제품의 품목코드 또는 품목명으로 필터링합니다. |
| **작업지시 상태** | 공통코드 `JOB_ORDER_STATUS` 기준 상태 선택(WAITING/RUNNING/HOLD/DONE 등). 진행 중인 작업지시만 보고 싶을 때 사용합니다. |
| **초기화** | 입력된 필터를 모두 지웁니다. 필터가 하나라도 입력됐을 때만 표시됩니다. |

---

## ② 작업지시 목록 (좌측) 컬럼

| 항목 | 역할 / 의미 |
|------|------|
| **작업지시번호(orderNo)** | 작업지시 식별 번호입니다. 클릭하면 해당 작업지시를 선택해 우측에 내역을 불러옵니다. |
| **품목명(itemName)** | 생산할 제품의 품목명입니다. |
| **품목코드(itemCode)** | 생산할 제품의 품목코드입니다(품목명 아래 소문자로 표시). |
| **계획수량(planQty)** | 작업지시의 생산 목표 수량입니다. BOM 소요량 계산의 기준이 됩니다. |
| **상태(status)** | 공통코드 `JOB_ORDER_STATUS` 기준 배지(예: 대기/진행/완료)로 표시됩니다. |

---

## ③ 출고요청 내역 (우측 — 내역 모드)

작업지시를 선택하면 해당 작업지시에 연결된 출고요청 건이 그룹별로 표시됩니다.

### 요청 그룹 헤더

| 항목 | 역할 / 의미 |
|------|------|
| **요청번호(requestNo)** | 출고요청 식별 번호입니다. 클릭하면 상세 모달이 열립니다. |
| **상태(status)** | 요청의 현재 상태 배지입니다(아래 상태 정의 참조). |
| **요청일(requestDate)** | 출고요청이 등록된 날짜·시각입니다. |
| **품목 수** | 이 출고요청에 포함된 품목 건수입니다. |
| **요청수량 합계(totalRequestQty)** | 이 요청에 포함된 모든 품목의 요청수량 합산값입니다. |
| **요청자(requester)** | 출고를 요청한 담당자입니다. |

### 요청 그룹 품목 테이블

| 컬럼 | 역할 / 의미 |
|------|------|
| **품목코드(itemCode)** | 출고 요청 품목의 코드입니다. |
| **품목명(itemName)** | 출고 요청 품목의 이름입니다. |
| **단위(unit)** | 수량 단위(EA, M, KG 등)입니다. |
| **요청수량(requestQty)** | 창고에 요청한 출고 수량입니다. |
| **기출고(issuedQty)** | 이 요청에서 실제로 출고된 누적 수량입니다. 요청수량과 같으면 완료입니다. |

---

## ④ 신규 작성 — BOM 품목 그리드 (우측 — 신규 작성 모드)

[신규 작성] 버튼을 클릭하면 작업지시의 BOM을 기반으로 소요 원자재 목록이 자동으로 계산되어 표시됩니다.

| 컬럼 | 역할 / 의미 |
|------|------|
| **품목코드(itemCode)** | 소요 원자재의 코드입니다. |
| **품목명(itemName)** | 소요 원자재의 이름입니다. |
| **단위(unit)** | 수량 단위입니다. |
| **BOM필요(bomReqQty)** | 이 작업지시 계획수량 기준 BOM 소요량입니다(= BOM 단위 소요량 × 계획수량). |
| **기불출(prevIssueQty)** | 이 작업지시에서 이미 불출된 누적 수량입니다. BOM필요에서 이 값을 빼면 실제 더 필요한 양을 알 수 있습니다. |
| **현장재고(floorStockQty)** | 현장(공정 라인)에 이미 보유 중인 수량입니다. 현장재고가 충분하면 출고요청 수량을 줄일 수 있습니다. |
| **현재고(currentStock)** | 창고 현재 재고 수량입니다. 요청수량이 현재고를 초과하면 빨간 경고가 표시됩니다. |
| **요청수량(requestQty)** | 직접 입력하는 출고 요청 수량입니다. 0인 품목은 요청에서 제외됩니다. |

요청수량 입력 후 상단의 **[요청사유]** 셀렉트(생산투입/시작품/샘플/기타)를 선택하고 **[요청등록]**을 누릅니다.

## ⑤ 출고요청 상세 모달

요청번호를 클릭하면 상세 모달이 열립니다.

| 항목 | 역할 / 의미 |
|------|------|
| **요청번호(requestNo)** | 출고요청 고유 식별 번호입니다. |
| **작업지시(workOrderNo)** | 연결된 작업지시 번호입니다. 없으면 '-'. |
| **상태(status)** | 현재 요청 상태 배지입니다. |
| **출고계정(issueType)** | 공통코드 `ISSUE_TYPE` 기준 출고 계정 구분입니다(예: 생산 투입, 샘플 등). |
| **요청일시(requestDate)** | 출고요청이 등록된 날짜·시각입니다. |
| **요청자(requester)** | 요청을 등록한 담당자입니다. |
| **승인자(approver)** | 요청을 승인한 담당자입니다. 미승인이면 '-'. |
| **승인일시(approvedAt)** | 승인이 처리된 날짜·시각입니다. |
| **요청수량 / 출고수량 / 잔여수량** | 요청 전체의 집계값입니다. 잔여 = 요청 - 출고. |
| **비고(remark)** | 요청 시 입력한 사유 또는 메모입니다. |
| **반려사유(rejectReason)** | 반려된 경우 사유가 표시됩니다. |

### 상세 모달 품목 테이블

| 컬럼 | 역할 / 의미 |
|------|------|
| **품목코드/품목명** | 요청 품목 식별 정보입니다. |
| **요청(requestQty)** | 요청한 수량입니다. |
| **출고(issuedQty)** | 실제 출고된 수량입니다. |
| **잔여** | 아직 출고되지 않은 수량입니다(= 요청 - 출고). |
| **현재고(currentStock)** | 요청 당시 창고 재고 수량입니다. |
| **BOM소요(bomReqQty)** | BOM 기준 소요량입니다. |
| **기불출(prevIssueQty)** | 이 작업지시에서 이미 불출된 기존 수량입니다. |
| **현장재고(floorStockQty)** | 현장에 보유 중인 수량입니다. |
| **단위(unit)** | 수량 단위입니다. |

## ⑥ 수동 출고요청 모달

작업지시 없이 원자재가 필요할 때는 상단 [수동 출고요청] 버튼을 사용합니다.

1. **작업지시**: 선택하면 BOM 기준 품목이 자동으로 채워집니다. 선택하지 않아도 됩니다.
2. **요청사유**: 생산투입 / 시작품 / 샘플 / 기타 중 선택합니다.
3. **품목 검색**: 품목코드 또는 품목명을 입력하고 검색합니다. 검색된 품목의 현재고를 확인 후 [+]로 요청 목록에 추가합니다.
4. **요청수량 입력**: 각 품목의 수량을 입력합니다. 현재고 초과 시 경고가 표시됩니다.
5. **요청등록** 버튼을 클릭합니다.

---

## 출고요청 상태값

| 상태 코드 | 화면 표시 | 의미 |
|------|------|------|
| **REQUESTED** | 대기(노란색) | 출고요청이 등록되어 창고 담당자의 승인을 기다리는 상태입니다. |
| **APPROVED** | 승인(파란색) | 담당자가 승인하여 실출고 처리가 가능한 상태입니다. |
| **REJECTED** | 반려(빨간색) | 요청이 반려된 상태입니다. 반려사유를 확인해 재요청하세요. |
| **COMPLETED** | 출고완료(초록색) | 승인 후 실출고가 완료된 상태입니다. |

---

## 사용 순서 (작업지시 기준 요청)

1. 필터 바에서 작업지시번호 또는 모델명을 입력해 대상 작업지시를 찾습니다.
2. 좌측에서 작업지시를 클릭해 선택합니다.
3. 우측 내역 모드에서 기존 요청 여부를 확인합니다.
4. [신규 작성]을 클릭해 BOM 기준 품목 그리드로 전환합니다.
5. 각 품목의 BOM필요·기불출·현장재고·현재고를 참고해 요청수량을 입력합니다.
6. 요청사유를 선택하고 [요청등록]을 클릭합니다.
7. 우측 내역 모드에서 등록된 요청건과 상태를 확인합니다.

> 요청수량이 0인 품목은 자동으로 제외됩니다. 최소 1개 품목에 수량이 1 이상이어야 등록됩니다.

---

## 입력 규칙

- 요청수량은 양의 정수(1 이상)여야 합니다.
- 수동 출고요청 모달에서는 모든 품목에 수량을 입력해야 등록됩니다(0인 품목이 있으면 불가).
- BOM 신규 작성 모드에서는 수량이 0인 품목을 제외하고 등록합니다.
- 요청수량이 현재고를 초과해도 등록 자체는 가능하지만 경고가 표시됩니다(차단하지 않음).

---

## 자주 묻는 질문

- **Q.** BOM 기준 수량이 0으로 표시됩니다.
  **A.** 품목마스터에 BOM이 등록되지 않았거나, 작업지시 계획수량이 0인 경우입니다. 수동으로 수량을 입력하거나 BOM을 점검하세요.

- **Q.** 현재고보다 더 많은 수량을 요청할 수 있나요?
  **A.** 경고가 표시되지만 등록은 가능합니다. 실제 출고는 창고 재고 범위 내에서만 가능합니다.

- **Q.** 출고요청이 반려됐습니다.
  **A.** 상세 모달에서 반려사유를 확인하고 사유를 해소한 뒤 신규 작성으로 재요청하세요.

- **Q.** APPROVED 상태인데 재고에 반영이 안 됩니다.
  **A.** 승인은 출고요청을 허가한 것이며, 실제 재고 변동은 자재출고관리 화면에서 실출고 처리 후 발생합니다.

- **Q.** 수동 출고요청에서 원자재 외 품목이 검색되지 않습니다.
  **A.** 출고요청 품목 검색은 원자재(RAW_MATERIAL) 품목만 대상입니다.
