# 자재출고관리(양산)

## 화면 목적
생산계획에 따라 현장에서 요청한 **자재 출고요청을 검토·승인·출고**하거나, 창고 담당자가 **LOT 바코드를 스캔해 즉시 전량 출고**하는 화면입니다. 출고계정은 **양산(PRODUCTION)** 고정이며, 양산 외 계정(불량·샘플·외주·폐기 등)은 [기타출고관리](/material/issue-other)에서 처리합니다.

> 흐름: 출고요청 생성(현장) → 승인(창고/관리자) → 출고처리 → 재고 차감(MatStock)

## 화면 구성
화면은 세 개의 탭으로 구성됩니다.

| 탭 | 역할 |
|---|---|
| **출고요청처리** | 현장에서 접수된 출고요청 목록 조회. 승인·반려·출고처리 액션 수행 |
| **바코드스캔** | LOT 바코드를 직접 스캔해 즉시 전량 출고 처리 |
| **출고이력** | 완료·취소 건 포함 전체 출고 이력 조회 및 출고 취소 |

## ① 출고요청처리 탭

### 통계 카드

| 카드 | 의미 |
|---|---|
| **요청** | 아직 승인 대기 중인 출고요청 건수 |
| **승인** | 승인되어 출고처리를 기다리는 건수 |
| **완료** | 출고처리가 완료된 건수 |
| **반려** | 반려 처리된 건수 |

### 요청 목록 컬럼

| 컬럼 | 역할 / 의미 |
|---|---|
| **요청번호(requestNo)** | 출고요청의 고유 식별번호입니다. 채번 형식은 `REQ-YYYYMMDD-NNN`. |
| **요청일(requestDate)** | 출고요청이 등록된 일자입니다. |
| **작업지시(workOrderNo)** | 이 출고요청과 연결된 생산 작업지시 번호입니다. 작업지시 기반 요청이면 파란색으로 강조 표시됩니다. |
| **품목수(itemCount)** | 이 요청에 포함된 품목 라인 수입니다. |
| **총수량(totalQty)** | 요청된 전체 품목의 수량 합계입니다. |
| **출고계정(issueType)** | 출고 유형입니다. 이 화면에서는 **양산(PRODUCTION)** 고정입니다. |
| **상태(status)** | 현재 처리 상태입니다(아래 상태값 참조). |
| **요청자(requester)** | 출고를 요청한 담당자입니다. |
| **처리(actions)** | 상태에 따라 승인·반려·출고처리 버튼이 표시됩니다. |

### 출고요청 상태값

| 상태 코드 | 표시 | 의미 |
|---|---|---|
| `REQUESTED` | 요청 | 출고요청이 접수되어 승인 대기 중입니다. |
| `APPROVED` | 승인 | 승인이 완료되어 출고처리 대기 중입니다. |
| `COMPLETED` | 완료 | 모든 품목이 출고 처리되었습니다. |
| `REJECTED` | 반려 | 요청이 반려되었습니다. |

### 출고처리 모달 컬럼 (승인 후 출고처리 시)

| 컬럼 | 역할 / 의미 |
|---|---|
| **품목코드(itemCode)** | 출고할 품목의 코드입니다(품목마스터 기준). |
| **품목명(itemName)** | 출고할 품목의 이름입니다. |
| **요청수량(requestQty)** | 현장에서 요청한 수량입니다. |
| **기출고(issuedQty)** | 이미 출고 처리된 수량입니다(부분 출고 누계). |
| **잔여(remainQty)** | 아직 출고되지 않은 잔여 수량입니다(= 요청수량 − 기출고). |
| **출고 LOT(matUidSelect)** | 출고할 자재 LOT를 선택합니다. 선택 시 `LOT시리얼 / 창고명 / 가용수량` 형식으로 표시됩니다. |
| **출고수량(issueQty)** | 실제로 출고할 수량을 입력합니다. 잔여 수량을 초과할 수 없습니다. |
| **단위(unit)** | 수량의 단위입니다(품목마스터 기준). |

## ② 바코드스캔 탭

LOT 바코드를 스캔하면 해당 LOT의 **전체 잔여 재고를 즉시 출고**합니다. 개별 수량 조정 없이 신속한 출고가 필요할 때 사용합니다.

### 스캔 결과 정보

| 항목 | 역할 / 의미 |
|---|---|
| **품목코드** | 스캔된 LOT의 품목코드입니다. |
| **품목명** | 스캔된 LOT의 품목명입니다. |
| **자재시리얼(matUid)** | 스캔된 LOT의 고유 시리얼(LOT 식별값)입니다. |
| **현재수량** | 해당 LOT의 현재 재고 수량입니다. 전량출고 시 이 수량이 차감됩니다. |
| **IQC** | 수입검사 결과입니다. PASS 상태가 아니거나 보류(HOLD) LOT는 출고가 차단됩니다. |
| **공급업체** | 자재를 납품한 공급업체명입니다. |

### 금일 스캔 출고 이력

| 컬럼 | 역할 / 의미 |
|---|---|
| **시간(issuedAt)** | 스캔 출고 처리 시각입니다. |
| **품목코드** | 출고된 품목의 코드입니다. |
| **품목명** | 출고된 품목의 이름입니다. |
| **자재시리얼(matUid)** | 출고된 LOT의 시리얼입니다. |
| **출고수량(issueQty)** | 출고된 수량입니다. |
| **단위(unit)** | 수량 단위입니다. |

## ③ 출고이력 탭

### 필터

| 필터 | 역할 / 의미 |
|---|---|
| **출고일(기간)** | 조회할 출고일 범위입니다. 기본값은 오늘 하루입니다. |
| **검색(searchText)** | 출고번호·품목코드·품목명·자재시리얼로 텍스트 검색합니다. |
| **상태(statusFilter)** | 완료/취소 필터입니다. |
| **출고계정(issueTypeFilter)** | 출고 유형으로 필터합니다(공통코드 ISSUE_TYPE). |

### 이력 컬럼

| 컬럼 | 역할 / 의미 |
|---|---|
| **출고번호(issueNo)** | 출고 처리 건의 고유 식별번호입니다. |
| **출고일(issueDate)** | 출고가 처리된 일자입니다. |
| **품목코드(itemCode)** | 출고된 품목의 코드입니다. |
| **품목명(itemName)** | 출고된 품목의 이름입니다. |
| **자재시리얼(matUid)** | 출고된 LOT의 시리얼입니다. |
| **수량(issueQty)** | 출고 수량입니다. 단위 포함 표시됩니다. |
| **출고계정(issueType)** | 출고 유형입니다(공통코드 ISSUE_TYPE). |
| **작업지시(jobOrderNo)** | 출고와 연결된 작업지시 번호입니다(있는 경우). |
| **상태(status)** | 완료(DONE) 또는 취소(CANCELED). |
| **(취소 버튼)** | 완료(DONE) 상태일 때만 표시됩니다. 클릭 시 취소 사유 입력 후 출고 취소 처리됩니다. 취소 시 재고가 복원됩니다. |

## 사용 순서

### 출고요청처리 방식
1. **출고요청처리** 탭을 클릭합니다.
2. 요청 목록에서 대상을 확인합니다. 상태 필터나 텍스트 검색을 활용할 수 있습니다.
3. 상태가 **요청(REQUESTED)**인 건에서 체크(승인) 아이콘을 클릭해 승인합니다. 반려가 필요하면 X 아이콘을 클릭하고 사유를 입력합니다.
4. 상태가 **승인(APPROVED)**인 건에서 재생(출고처리) 아이콘을 클릭합니다.
5. 출고처리 모달에서 각 품목의 **출고 LOT**를 선택하고 **출고수량**을 확인(필요시 수정)한 뒤 **출고** 버튼을 클릭합니다.
6. 모든 품목 수량이 완전히 출고되면 요청 상태가 **완료(COMPLETED)**로 변경됩니다.

### 바코드스캔 방식
1. **바코드스캔** 탭을 클릭합니다.
2. 입력창에 LOT 바코드를 스캔하거나 자재시리얼을 직접 입력하고 Enter를 누릅니다.
3. LOT 정보(품목명, 현재수량, IQC 상태 등)를 확인합니다.
4. **전량출고** 버튼을 클릭합니다.
5. 하단 '금일 스캔 출고 이력'에 처리 건이 추가되며, 입력창에 자동으로 포커스가 돌아와 다음 스캔을 이어갈 수 있습니다.

---

## 입력 규칙 / 검증

- 출고처리 시 **출고 LOT를 선택**하지 않으면 출고가 차단됩니다.
- 출고수량은 **잔여수량을 초과**할 수 없습니다.
- 바코드 스캔 출고는 **IQC 합격(PASS)** 상태이고 **보류(HOLD)가 아닌** LOT만 가능합니다.
- 바코드 스캔 시 재고가 이미 소진된 LOT는 출고가 거부됩니다.
- 출고 취소는 **완료(DONE)** 상태에서만 가능하며, 취소 사유 입력이 필수입니다.
- 뒤 공정(생산실적)이 이미 진행된 출고는 취소가 차단됩니다.

## 자주 묻는 질문

- **Q.** 출고요청 목록에 건이 안 보여요.
  **A.** 상태 필터가 특정 상태로 설정되어 있는지 확인하세요. '전체 상태'로 변경하거나 검색어를 지워보세요.

- **Q.** 출고처리 모달에서 LOT 목록이 비어 있어요.
  **A.** 해당 품목에 출고 가능한 재고(IQC 합격, 가용수량 > 0)가 없는 경우입니다. 입고 현황을 확인하세요.

- **Q.** 바코드 스캔 시 "IQC 미합격 LOT" 오류가 납니다.
  **A.** 수입검사(IQC)가 합격 처리되지 않은 LOT입니다. IQC 담당자에게 검사 완료를 요청하세요.

- **Q.** 출고 취소가 안 됩니다.
  **A.** 뒤 공정(생산실적)이 이미 진행된 경우 취소가 차단됩니다. 생산실적 → FG 라벨 → 박스/OQC → 팔레트 → 출하 순서로 역처리 후 다시 시도하세요.

- **Q.** 양산 이외 계정(샘플, 불량 등)으로 출고하려면요?
  **A.** [기타출고관리](/material/issue-other) 화면을 사용하세요.
