# BOM관리

## 화면 목적
완제품·반제품이 어떤 **하위 품목(자품목)으로 구성되는지**를 정의하는 **자재명세서(BOM)**를 관리합니다. 여기 등록된 구성이 생산 시 자재 소요량 계산과 투입의 기준이 됩니다. 같은 상위-하위 품목 조합이라도 **적용일**을 다르게 하면 새 버전(개정)으로 등록할 수 있습니다.

## 화면 구성
- **왼쪽 — 제품/반제품 목록**: 검색창(품목코드/품목명), 유형 필터 pill(전체/완제품/반제품 — 기본은 **완제품**), 목록에서 행을 클릭해 상위품목을 선택합니다.
- **오른쪽 — BOM 자재 구조**: 선택한 상위품목의 **BOM 트리**입니다. 자품목이 반제품이면 그 아래 다시 하위 BOM이 **레벨(Lv)**로 펼쳐집니다(재귀 구조). 최상위 행(Lv 0)은 상위품목 자신입니다.
- **상단 툴바(공통)**: 폼 다운로드, 엑셀 업로드, **기준일/전체이력** 토글, 기준일자, 새로고침.
- **트리 영역 툴바**: 내보내기, 엑셀 업로드(축소 화면에서는 숨김), BOM 추가, 전체 펼치기/전체 접기.
- **오른쪽 라우팅 패널**(트리 행의 라우팅 아이콘 클릭 시 표시): 선택한 품목의 공정순서(라우팅) 목록과, 공정을 선택하면 아래에 품질조건/투입자재 편집 탭이 나타납니다.

## 개념 관계
```
상위품목(완제품) ─┬─ 자품목 A (소요량, 적용일 2026-01-01)
                 ├─ 자품목 A (소요량, 적용일 2026-06-01)  ← 같은 조합, 적용일 다른 버전
                 └─ 반제품 C (소요량) ─┬─ 자품목 C1
                                       └─ 자품목 C2
```
> 같은 **상위품목+자품목** 조합은 **적용일**이 다르면 여러 행으로 공존할 수 있습니다(버전 관리). 단, 상위+자품목+적용일 조합까지 같으면 중복으로 막힙니다.

## ① BOM 자재 구조 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **Lv** | BOM 트리에서의 깊이입니다. 상위품목 자신이 0, 그 자품목이 1, 자품목이 다시 부모가 되면 2…로 펼쳐집니다(저장값이 아니라 화면에서 계산). |
| **자품목코드** | 상위품목을 구성하는 **하위(자) 품목의 코드**입니다. 품목마스터(ITEM_MASTERS)에 등록된 품목이어야 합니다. |
| **자품목명** | 자품목의 이름(품목마스터에서 가져와 표시). |
| **품목유형** | 자품목의 유형 배지입니다(완제품/반제품/원자재/소모품). 반제품이면 그 아래로 하위 BOM이 더 전개됩니다. |
| **투입공정** | 이 자품목이 투입되는 공정코드·공정명입니다. 미지정이면 "-". |
| **소요량** | 상위품목 1개를 만드는 데 필요한 이 자품목의 **수량**입니다. 총 필요량 = 생산수량 × 소요량. |
| **리비전** | BOM 구성의 리비전 표시값입니다(기본 `A`). 사양 구분용 라벨이며, **행을 구분하는 키는 아닙니다**(아래 "입력 규칙" 참고). |
| **사이드** | 적용 위치 구분입니다. **좌(Left) / 우(Right) / N/A** 중 하나. |
| **적용일** | 이 BOM 행이 적용되기 시작하는 날짜입니다. **같은 상위+자품목 조합을 구분하는 키**입니다. |
| **완료일** | 이 BOM 행의 적용이 끝나는 날짜입니다(기간 외에는 조회에서 제외됨). |
| **관리** | 행별 아이콘: 라우팅이동(보라 GitBranch) · 수정(연필) · 삭제(휴지통). 최상위(Lv 0) 행은 라우팅이동만 가능합니다. |

### BOM 추가/수정 폼에서 함께 입력하는 항목
| 항목 | 역할 / 의미 |
|------|------|
| **모품목** | 이 행이 속하는 상위품목입니다(읽기전용, 트리에서 선택한 행 기준으로 자동 채워짐). |
| **자품목코드** | 등록 시 품목코드/품목명으로 검색해 선택합니다. **수정 시에는 자품목을 바꿀 수 없습니다**(비활성화 — 다른 자품목으로 바꾸려면 삭제 후 새로 등록). |
| **순서** | 같은 상위품목 안에서 자품목 표시/처리 순서입니다. |
| **공정코드** | 라우팅관리에 등록된 공정 중에서 선택합니다. |
| **비고** | 메모입니다. |

## 버튼별 기능

| 버튼 | 언제 사용 | 동작 | 결과 / 주의 |
|------|------|------|------|
| **폼 다운로드** | 엑셀 업로드용 빈 양식이 필요할 때 | 헤더만 있는 `BOM_template.xlsx` 다운로드 | 상위품목코드/하위품목코드/소요량/리비전/순서/BOM그룹/공정코드/사이드/ECO번호/유효시작일/유효종료일/비고 12개 열 |
| **엑셀 업로드** | 여러 BOM 행을 한 번에 등록할 때 | 업로드 모달을 엽니다 | 아래 "엑셀 업로드 흐름" 참고 |
| **기준일 / 전체이력** | 특정 시점 기준으로 볼지, 전체 이력을 다 볼지 전환 | **기준일**: 아래 기준일자 하나만 적용해 유효한 행만 표시. **전체이력**: 기준일 필터 없이 모든 적용기간의 행을 표시 | 전체이력 선택 시 기준일자 입력칸은 비활성화 |
| **기준일자** | 기준일 모드에서 특정 날짜 기준 조회 | 날짜 선택 → 그 날짜에 적용중인(적용일 ≤ 날짜 ≤ 완료일) 행만 목록/트리에 표시 | 기본값은 **오늘(로컬 기준)** |
| **새로고침** | 목록을 다시 불러올 때 | 왼쪽 제품/반제품 목록을 재조회 | |
| **내보내기** | 현재 선택한 상위품목의 BOM을 엑셀로 받고 싶을 때 | `BOM_{상위품목코드}_{오늘날짜}.xlsx` 다운로드 | |
| **BOM 추가** | 자품목을 새로 등록할 때 | BOM 추가 폼 모달 오픈 | 저장 시 같은 상위+자품목+적용일 조합이 이미 있으면 저장 거부(중복 안내) |
| **전체 펼치기 / 전체 접기** | 트리가 깊을 때 | 하위가 있는 모든 행을 펼치거나 접음 | |
| **라우팅이동(보라 아이콘)** | 이 자품목의 공정순서를 보고 싶을 때 | 오른쪽에 라우팅 패널 오픈(공정목록 · 품질조건 · 투입자재) | "라우팅관리로 이동" 버튼으로 `/master/routing` 화면으로도 이동 가능 |
| **수정(연필 아이콘)** | 기존 행의 소요량/공정/적용일 등을 바꿀 때 | BOM 수정 폼 모달 오픈(자품목코드는 비활성) | 적용일을 바꾸면 같은 상위+자품목의 다른 행과 적용일이 겹치는지 다시 검사 |
| **삭제(휴지통 아이콘)** | 행을 없앨 때 | 확인 모달("이 BOM 항목을 삭제하시겠습니까?") 후 삭제 | 최상위(Lv 0) 행에는 없음 |

## 업무 흐름
1. 왼쪽에서 유형 필터·검색으로 **상위품목**을 찾아 선택합니다.
2. 오른쪽 트리에서 **BOM 추가**로 자품목코드·소요량·공정·사이드·적용일/완료일을 입력해 등록합니다.
3. 등록한 자품목이 반제품이면, 왼쪽 목록에서 그 반제품을 다시 상위품목으로 선택해 하위 BOM을 계속 구성합니다.
4. 특정 시점의 구성만 확인하고 싶으면 **기준일**을 선택하고 날짜를 지정합니다. 변경 이력을 전부 보려면 **전체이력**으로 전환합니다.
5. 여러 행을 한 번에 등록해야 하면 **폼 다운로드**로 양식을 받아 채운 뒤 **엑셀 업로드**로 올립니다.
6. 자품목의 실제 공정순서가 궁금하면 행의 라우팅이동 아이콘을 눌러 우측 패널에서 확인합니다.

## 엑셀 업로드 흐름
1. **엑셀 업로드** 버튼 → 업로드 모달 오픈.
2. **파일 선택**(.xlsx만 허용) → 파일을 고르면 **자동으로 미리보기**가 실행됩니다(파일 분석 중... 표시).
3. 미리보기 결과로 **신규 / 중복 / 오류** 건수 배지와 행별 상태표(행, 상위품목, 하위품목, 적용일자, 완료일자, 수량, 상태)가 나타납니다.
   - **신규**: 문제없이 등록 가능한 행.
   - **DB중복 / 파일중복**: 동일한 **상위품목+하위품목+적용일자** 조합이 이미 DB에 있거나 파일 안에서 반복됨.
   - **오류**: 필수값 누락(소요량/유효일자), 상위=하위 품목, 파일/DB 내 상위+하위+리비전이 겹치는 등.
4. **중복이 하나라도 있으면 업로드 버튼 자체가 나타나지 않습니다.** 안내 문구를 보고 파일을 수정해 다시 선택해야 합니다.
5. 중복이 없으면 **업로드 (N건)** 버튼으로 실제 등록을 진행합니다.
6. 업로드가 끝나면 **추가 / 스킵 / 에러** 건수 카드와, 에러가 있으면 행별 에러 상세(행 번호 + 사유)가 표시됩니다. 내용을 확인한 뒤 **닫기**를 누르면 화면 목록이 새로고침됩니다.

## 입력 규칙 / 검증
- 자품목코드는 품목마스터에 존재해야 하며, **상위품목과 같을 수 없습니다.**
- 소요량·유효시작일·유효종료일은 필수입니다.
- **행을 구분하는 조합은 상위품목 + 자품목 + 적용일(유효시작일)입니다.** 같은 상위+자품목이라도 적용일만 다르면 새 행(새 버전)으로 등록됩니다. 리비전은 표시용 라벨일 뿐, 같은 적용일에 리비전만 다르게 해서 중복 등록할 수는 없습니다.
- 수정 화면에서 적용일을 다른 날짜로 바꾸면, 바뀐 적용일이 같은 상위+자품목의 다른 행과 겹치는지 다시 검사합니다.

## 자주 묻는 질문
- **Q.** 레벨(Lv)은 어떻게 정해지나요?
  **A.** 직접 입력이 아니라, 자품목이 다시 BOM을 가지면 화면에서 자동으로 한 단계 깊어집니다(재귀 전개).
- **Q.** 같은 자품목을 리비전만 바꿔서 두 번 등록할 수 있나요?
  **A.** 아니요. 적용일까지 같으면 리비전이 달라도 중복으로 거부됩니다. 새 버전을 추가하려면 **적용일을 다르게** 등록하세요.
- **Q.** 소요량은 무엇 기준인가요?
  **A.** 상위품목 1개를 만들 때 필요한 자품목 수량입니다. 총 필요량 = 생산수량 × 소요량.
- **Q.** 엑셀 업로드에서 왜 업로드 버튼이 안 보이나요?
  **A.** 미리보기에서 **중복(1건 이상)**이 발견되면 업로드 버튼 자체가 숨겨집니다. 중복 행을 파일에서 수정한 뒤 파일을 다시 선택하세요.
- **Q.** 기준일자를 오늘로 뒀는데 방금 등록한 행이 안 보여요.
  **A.** 방금 등록한 행의 **적용일이 오늘보다 미래**일 수 있습니다. 기준일자를 그 적용일로 바꾸거나 **전체이력**으로 전환해 확인하세요.
