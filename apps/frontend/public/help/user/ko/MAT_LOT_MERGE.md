# 자재병합

## 화면 목적
같은 품목이면서 **같은 입하건**으로 들어온 자재 LOT(시리얼) 여러 개를 **하나의 신규 통합 시리얼로 합치는** 화면입니다. 잘게 쪼개진 소량 LOT을 묶어 관리·불출을 단순화할 때 사용합니다. 병합하면 원본 LOT은 모두 폐기(MERGED)되고, 합산 수량을 가진 **신규 자재 시리얼 1개**가 새로 발번되며 라벨을 출력할 수 있습니다.

> 자재병합은 자재분할(LOT 분할)과 대칭 개념입니다. 분할은 1개를 여러 개로 나누고, 병합은 여러 개를 1개로 합칩니다.

## 화면 구성
- **바코드 스캔 영역(상단)**: 병합할 시리얼을 바코드로 스캔하거나 직접 입력해 누적합니다. 누적된 시리얼이 태그로 나열되고, 선택 LOT 수·합산 수량 통계 카드와 [선택 병합] 버튼이 함께 표시됩니다.
- **병합 가능 LOT 목록(하단 그리드)**: 현재 병합 가능한 모든 LOT을 보여줍니다. 각 행의 **＋** 버튼으로 스캔 영역에 누적할 수 있습니다. 상단 검색으로 LOT번호·품목코드를 필터링합니다.
- **병합 확인 모달**: 병합 대상과 합산 수량을 최종 확인합니다.
- **라벨 미리보기 모달**: 병합 성공 후 신규 시리얼의 자재 라벨을 미리 보고 출력합니다.

---

## ① 병합 가능 LOT 목록 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **(추가)** | 행을 스캔 영역에 누적하는 **＋** 버튼입니다. 이미 추가된 시리얼이면 비활성화됩니다. |
| **자재시리얼(matUid)** | 자재 LOT의 고유 식별자(라벨 바코드)입니다. 병합 대상 식별 기준입니다. |
| **품목코드(itemCode)** | 자재의 품목코드입니다. 병합은 **같은 품목코드**끼리만 가능합니다. |
| **품목명(itemName)** | 자재 품목 이름입니다. |
| **수량(qty)** | 해당 LOT의 현재 재고 수량입니다. 병합 시 이 수량들이 모두 합산됩니다. |
| **입하번호(arrivalNo)** | LOT이 입하된 입하번호입니다. 병합은 **같은 입하번호**끼리만 허용됩니다(입하 라벨 계승 목적). |
| **공급처(vendor)** | LOT을 납품한 거래처입니다. 거래처명이 있으면 이름으로, 없으면 코드로 표시됩니다. |

> 이 목록에는 **정상(NORMAL)·재고 보유·입고완료·예약 없음·출고 이력 없음** 조건을 모두 만족하는 LOT만 나타납니다.

---

## ② 바코드 스캔 영역

| 항목 | 역할 / 의미 |
|------|------|
| **스캔 입력창** | 자재 바코드를 스캔하거나 시리얼을 직접 입력하고 Enter를 누르면 누적됩니다. 스캔 후 자동으로 입력창에 다시 포커스됩니다. |
| **추가 버튼** | 입력창의 값을 누적 목록에 추가합니다. |
| **누적 시리얼 태그** | 현재까지 누적된 시리얼이 수량과 함께 태그로 표시됩니다. 각 태그의 **✕**로 개별 제거할 수 있습니다. |
| **선택 LOT(카드)** | 누적된 시리얼 개수입니다. |
| **합산 수량(카드)** | 누적된 시리얼 수량의 합계입니다. 병합 후 신규 시리얼의 수량이 됩니다. |
| **선택 해제 버튼** | 누적한 시리얼을 모두 비웁니다. |
| **선택 병합 버튼** | 병합 확인 모달을 엽니다. 누적이 2개 미만이면 비활성화됩니다. |

---

## ③ 병합 확인 모달

| 항목 | 역할 / 의미 |
|------|------|
| **병합 LOT** | 병합 대상 시리얼 목록을 보여줍니다(읽기 전용). |
| **합산 수량** | 신규 통합 시리얼에 들어갈 총 수량입니다. |
| **병합 실행 버튼** | 실제 병합을 수행합니다. 원본은 모두 폐기되고 신규 시리얼 1개가 발번됩니다. |

> 병합하면 원본 LOT은 모두 폐기(MERGED)되고, 합산 수량의 **신규 시리얼 1개**가 새로 만들어집니다. 되돌릴 수 없으니 신중히 확인하세요.

---

## ④ 라벨 미리보기 모달

병합 성공 후 신규 시리얼의 자재 라벨이 미리보기로 열립니다. 라벨에는 신규 자재 UID, 품목코드·품목명, 합산 수량, 입하번호, 제조사 등이 인쇄됩니다. [라벨 출력]으로 라벨 프린터에 전송해 신규 시리얼 박스에 부착합니다.

---

## 사용 순서

1. 하단 목록에서 병합할 LOT을 찾거나 상단 검색으로 필터링합니다.
2. 병합할 자재 바코드를 스캔하거나, 목록 행의 **＋** 버튼으로 시리얼을 누적합니다(2개 이상).
3. 누적 시리얼·합산 수량을 통계 카드에서 확인합니다.
4. **선택 병합** 버튼을 클릭합니다.
5. 확인 모달에서 병합 LOT과 합산 수량을 확인하고 **병합 실행**을 클릭합니다.
6. 라벨 미리보기에서 신규 시리얼 라벨을 확인하고 **라벨 출력** 후 신규 박스에 부착합니다.

---

## 입력 규칙 / 검증

- **2개 이상**의 서로 다른 시리얼을 누적해야 병합할 수 있습니다.
- **같은 품목코드**의 LOT만 함께 병합할 수 있습니다(다른 품목 추가 시 오류 안내).
- **같은 입하번호**의 LOT만 함께 병합할 수 있습니다. 입하번호가 없는 LOT은 병합할 수 없습니다.
- 다음 LOT은 병합할 수 없습니다: HOLD 상태, 정상(NORMAL)이 아닌 상태, 재고 0, 예약 수량 보유, 입고 미완료(입고확정 필요), 자재출고 이력 있음.
- 이미 추가된 시리얼은 중복 추가되지 않습니다.

---

## 자주 묻는 질문

- **Q.** 다른 품목을 함께 스캔했더니 오류가 나요.
  **A.** 병합은 같은 품목코드끼리만 가능합니다. 먼저 추가한 LOT과 다른 품목은 추가되지 않습니다.

- **Q.** 입하번호가 다른 LOT은 왜 못 합치나요?
  **A.** 신규 시리얼이 원본의 입하번호·입하정보를 계승해야 라벨·이력 추적이 일관되기 때문에, 같은 입하건의 LOT만 병합을 허용합니다.

- **Q.** 병합하면 원래 시리얼은 어떻게 되나요?
  **A.** 원본 LOT은 모두 폐기(MERGED)되고 재고가 0이 됩니다. 합산 수량의 신규 시리얼 1개가 새로 발번됩니다. 수불 이력은 남으므로 추적은 가능합니다.

- **Q.** 잘못 병합했어요. 되돌릴 수 있나요?
  **A.** 병합은 되돌리는 화면이 없습니다. 필요하면 [자재분할](/material/lot-split)로 다시 나누어 사용하세요.

- **Q.** 목록에 LOT이 안 보여요.
  **A.** 입고 미완료, 재고 0, 예약 수량 보유, 출고 이력, HOLD/비정상 상태인 LOT은 목록에서 제외됩니다. 입고확정·예약 해제 등을 먼저 확인하세요.
