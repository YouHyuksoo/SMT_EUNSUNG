# 의뢰검사 입력

## 화면 목적
생산 현장의 키오스크에서 작업자가 **직접 판정하기 어려운 검사 항목**(예: 측정기·시험기가 필요한 항목, 파괴검사)은 작업자가 결과를 입력하지 않고 **품질담당에게 의뢰(DELEGATE)** 합니다. 이렇게 넘어온 검사 건이 "대기(PENDING)" 상태로 이 화면에 쌓이며, 품질담당이 실제로 측정·확인한 뒤 **합격(PASS)/불합격(FAIL)** 을 판정하는 곳입니다.

> 자주검사(自主檢査): 생산 공정에서 작업자 또는 담당이 직접 수행하는 공정검사입니다. 검사 방법이 **직접검사(DIRECT)** 면 키오스크에서 바로 판정하고, **의뢰검사(DELEGATE)** 면 이 화면으로 넘어옵니다.

> 키오스크에서 의뢰한 검사가 **대기(PENDING) 상태로 남아 있으면 해당 작업지시의 생산실적 입력이 차단**됩니다. 이 화면에서 판정을 마쳐야 작업자가 실적을 이어갈 수 있습니다.

## 화면 구성
- **왼쪽 — 의뢰검사 대기 목록**: 아직 판정되지 않은 의뢰검사(PENDING) 항목을 요청 순서(오래된 것부터)로 보여줍니다. 행을 클릭하면 오른쪽에 입력 영역이 열립니다.
- **오른쪽 — 결과 입력**: 선택한 항목의 검사 기준(LSL/USL·단위·기준)을 확인하고, 측정값·비고를 입력한 뒤 합격/불합격을 누릅니다. 선택 전에는 "좌측에서 항목을 선택하세요" 안내만 표시됩니다.

## 처리 흐름
```
키오스크 자주검사(의뢰검사 항목) ──의뢰──▶ 대기(PENDING) 목록 ──품질담당 판정──▶ 합격(PASS)/불합격(FAIL)
                                                                          │
                                                                  대기 해소 → 키오스크 실적입력 가능
```

---

## ① 대기 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **작업지시(orderNo)** | 이 검사가 발생한 생산 작업지시 번호입니다. 어느 지시의 생산 중에 나온 검사인지 식별합니다. |
| **공정(processCode)** | 검사가 일어난 공정 코드입니다. 어느 공정 단계의 검사인지 구분합니다. |
| **항목명(itemName)** | 검사 항목 이름입니다(예: 외관, 치수, 압착력 등). 무엇을 검사하는지 나타냅니다. |
| **시점(timing)** | 생산 중 검사 시점입니다. **초물(FIRST)** = 작업 시작 후 첫 생산품, **중물(MID)** = 작업 중간, **종물(LAST)** = 작업 종료 시점. 같은 작업이라도 시점마다 검사가 따로 발생할 수 있습니다. |
| **요청일시(createdAt)** | 키오스크에서 이 검사가 의뢰된 시각입니다. 오래된 요청부터 위에 표시되므로 먼저 처리하면 됩니다. |

---

## ② 결과 입력 영역

행을 선택하면 항목 정보와 함께 아래 항목이 표시·입력됩니다.

| 항목 | 역할 / 의미 |
|------|------|
| **검사 기준 — LSL** | 규격 하한값(Lower Spec Limit). 측정값이 이 값보다 **작으면 규격 미달**입니다. 측정형 항목에만 표시됩니다. |
| **검사 기준 — USL** | 규격 상한값(Upper Spec Limit). 측정값이 이 값보다 **크면 규격 초과**입니다. LSL~USL 사이가 정상 범위입니다. |
| **단위(unit)** | 측정값의 단위입니다(예: mm, N, kgf). 어떤 단위로 측정·입력하는지 알려줍니다. |
| **기준/규격(standard)** | 항목에 설정된 검사 기준 설명 텍스트입니다(측정 규격이 숫자로 없을 때의 판단 기준 등). |
| **측정값(measureValue)** | 실제 측정한 수치를 입력합니다. 선택 입력이며, 판정형(VISUAL) 항목이면 비워 둘 수 있습니다. 입력하면 검사 결과에 함께 기록됩니다. |
| **비고(remark)** | 특이사항·판정 사유 등을 적는 메모입니다. 선택 입력입니다. |
| **합격 (PASS)** | 검사 결과를 **합격**으로 확정합니다. 입력한 측정값·비고와 함께 저장되고 대기 목록에서 사라집니다. |
| **불합격 (FAIL)** | 검사 결과를 **불합격**으로 확정합니다. 불합격 처리 후 후속 품질 조치(불량 처리 등)는 별도 절차를 따릅니다. |

> 판정 기준이 LSL/USL로 설정된 항목은 측정값이 **LSL 이상, USL 이하**일 때 합격으로 보는 것이 일반적입니다. 다만 이 화면의 합격/불합격은 담당자가 직접 누르는 **수동 판정**이므로, 규격을 참고하되 최종 판단은 담당자가 합니다.

---

## 사용 순서
1. 왼쪽 **대기 목록**에서 처리할 항목을 클릭합니다(오래된 요청이 위에 있습니다).
2. 오른쪽에서 **검사 기준(LSL/USL·단위)** 을 확인합니다.
3. 실제로 측정·확인한 뒤 **측정값**을 입력하고(필요 시 **비고** 작성), 결과에 따라 **합격(PASS)** 또는 **불합격(FAIL)** 을 누릅니다.
4. 처리된 항목은 대기 목록에서 빠지고, 해당 작업지시의 키오스크 생산실적 입력 차단이 풀립니다.
5. 목록이 갱신되지 않으면 오른쪽 위 **새로고침**을 누릅니다.

## 입력 규칙 / 검증
- 항목을 선택해야 결과 입력·판정 버튼이 활성화됩니다.
- 측정값과 비고는 모두 **선택 입력**입니다. 판정형(외관 등) 항목은 측정값 없이 합격/불합격만 누를 수 있습니다.
- 측정값은 숫자만 입력합니다.

## 자주 묻는 질문
- **Q.** 직접검사와 의뢰검사는 무엇이 다른가요?
  **A.** 직접검사(DIRECT)는 키오스크에서 작업자가 바로 판정합니다. 의뢰검사(DELEGATE)는 작업자가 판정하지 않고 품질담당에게 넘겨, 이 화면에서 판정합니다.
- **Q.** 여기서 처리하지 않으면 어떻게 되나요?
  **A.** 해당 작업지시에 대기(PENDING) 의뢰검사가 남아 있으면 키오스크 생산실적 입력이 막힙니다. 빨리 판정해야 생산이 이어집니다.
- **Q.** LSL/USL이 안 보입니다.
  **A.** 그 항목이 판정형(VISUAL)이거나 규격(LSL/USL)이 설정되지 않은 경우입니다. 이때는 "판정형 항목 (규격 없음)" 또는 "설정된 규격이 없습니다"로 표시되며, 담당자가 육안·기준으로 판정합니다.
