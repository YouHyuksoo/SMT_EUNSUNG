# AQL 기준관리

## 화면 목적
수입검사(IQC)에서 입고 LOT을 **샘플링검사**로 합격/불합격 판정하기 위한 모든 기준을 등록·관리합니다.

LOT 전체를 검사하지 않고 일부 표본만 검사하는 방식이 샘플링검사입니다. 이를 위해 "표본을 몇 개 뽑고(샘플수), 그중 불량이 몇 개까지면 합격(Ac)·몇 개부터 불합격(Re)인가"를 미리 정해 두어야 합니다. 이 화면은 그 기준 전체를 국제표준(ISO 2859-1)에 맞춰 관리합니다.

> **AQL(Acceptable Quality Limit, 합격품질한계)**: LOT를 합격으로 인정할 수 있는 최대 허용 불량 수준입니다. **값이 작을수록 더 엄격**합니다(예: 1.0이 2.5보다 엄격). 주요 AQL 값: 0.065 / 0.10 / 0.15 / 0.25 / 0.40 / 0.65 / **1.0** / 1.5 / **2.5** / 4.0 / 6.5

---

## AQL 표준 핵심 개념

AQL 샘플링 판정은 **3단계 순서**로 이루어집니다.

### 1단계: 검사수준 선택
LOT 크기에서 얼마나 많은 표본을 뽑을지 강도를 정합니다.

| 검사수준 | 설명 | 용도 |
|------|------|------|
| **I** | 표본을 적게 뽑음(완화) | 공급자 신뢰도가 높거나 검사비용이 비쌀 때 |
| **II** | 표준(기본값) | 대부분의 경우 사용 |
| **III** | 표본을 많이 뽑음(엄격) | 신규 공급자·불량 이력이 있을 때 |
| **S-1~S-4** | 특별검사수준(소량 추출) | 파괴검사처럼 표본을 최소화해야 할 때 |

### 2단계: Code Letter 결정
LOT 크기 + 검사수준의 교차점에서 **Code Letter**(알파벳 A~R)가 결정됩니다. 이 화면의 **"Code Letter 표" 탭**에서 확인할 수 있습니다.

```
예) LOT 크기 151~280개 + 검사수준 II → Code Letter = H
    Code Letter H 의 표준 샘플수 = 50개
```

### 3단계: Ac/Re 결정
Code Letter + AQL 값의 교차점에서 **Ac(합격판정개수)**와 **Re(불합격판정개수)**가 정해집니다. 이 화면의 **"Sampling Plan 표" 탭**에서 확인할 수 있습니다.

```
예) Code Letter H + AQL 1.0 → 샘플수 50, Ac=1, Re=2
    → 50개 중 불량 1개까지 합격, 2개부터 불합격
```

### 화살표(↑/↓)의 의미
Sampling Plan 표에서 특정 셀에 화살표(`↑B`, `↓D` 등)가 표시될 수 있습니다.

| 기호 | 의미 |
|------|------|
| `↑B` | 해당 Code Letter에서는 이 AQL로 판정이 불가능하므로, **더 작은(위쪽) Code Letter(더 많은 샘플)** 기준을 적용 |
| `↓D` | 해당 Code Letter에서는 이 AQL로 판정이 불가능하므로, **더 큰(아래쪽) Code Letter(더 적은 샘플)** 기준을 적용 |

### 결함 등급(Defect Grade)
| 등급 | 의미 | 판정 기준 |
|------|------|------|
| **Critical(치명결함)** | 안전·법규에 영향을 미치는 결함 | 1건이라도 발생하면 **즉시 LOT 불합격** (Ac/Re 무관) |
| **Major(중결함)** | 기능·성능에 영향을 주는 결함 | Major AQL 기준의 Ac/Re로 판정 |
| **Minor(경결함)** | 외관 등 경미한 결함 | Minor AQL 기준의 Ac/Re로 판정 |

> 판정 우선순위: **Critical 먼저 → Major → Minor**. Critical 불량이 1건이라도 있으면 Major/Minor 결과와 무관하게 LOT 불합격입니다.

## 화면 구성 (4개 탭)

이 화면은 4개 탭으로 구성됩니다.

| 탭 | 역할 |
|------|------|
| **AQL 정책관리** | 품목에 직접 연결하는 단위. Major/Minor AQL 기준 조합을 묶어 정책으로 만듭니다. |
| **AQL 기준** | AQL 기준 헤더(코드·명칭·AQL 값)를 관리합니다. |
| **Code Letter 표** | ISO 2859-1 Table I — LOT 크기 × 검사수준 → Code Letter 대조표 |
| **Sampling Plan 표** | ISO 2859-1 Table II-A — Code Letter × AQL 값 → 샘플수·Ac·Re 대조표 |

### 데이터 흐름
```
품목마스터
  └─ AQL 정책 코드 ──▶ AQL 정책 (정책관리 탭)
                              │
                   ┌──────────┴──────────┐
                   ▼                     ▼
             Major AQL 기준         Minor AQL 기준
             (AQL 기준 탭)          (AQL 기준 탭)
                   │                     │
                   ▼                     ▼
           Code Letter 표 ──▶ Sampling Plan 표
           (LOT×검사수준)      (Code Letter×AQL → Ac/Re)
```

## ① AQL 정책관리 탭

품목에 연결하는 최종 판정 조합입니다. 행을 클릭하면 상단 폼에 내용이 불러와져 수정할 수 있습니다.

### 정책 폼 입력 항목

| 항목 | 역할 / 의미 |
|------|------|
| **정책 코드(policyCode)** | 정책을 식별하는 고유 코드입니다. **품목마스터의 'AQL 정책' 필드가 이 코드를 참조**합니다. 등록 후 변경 불가. 예: `AQLP-II-1.0-2.5` |
| **정책명(policyName)** | 사람이 알아보기 위한 이름입니다. 예: `일반검사 Major 1.0 Minor 2.5` |
| **검사수준(inspectionLevel)** | 표본 크기를 결정하는 수준입니다(I / II / III / S1~S4). 기본값 II. |
| **Major AQL(majorAqlCode)** | 중결함에 적용할 AQL 기준을 선택합니다. 목록에는 사용중인(useYn=Y) AQL 기준만 표시됩니다. |
| **Minor AQL(minorAqlCode)** | 경결함에 적용할 AQL 기준을 선택합니다. 보통 Major보다 느슨한(값이 큰) 기준을 씁니다. |

> 치명결함(Critical) 처리는 정책에 관계없이 **항상 IMMEDIATE_FAIL**이 적용됩니다. Critical 불량 1건이라도 발생하면 즉시 LOT 불합격입니다.

### 정책 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **정책 코드** | 고유 식별 코드 |
| **정책명** | 표시용 이름 |
| **검사수준** | I / II / III / S1~S4 |
| **Major** | Major AQL 기준 코드 |
| **Minor** | Minor AQL 기준 코드 |
| **사용** | Y=활성(품목 연결 가능), N=사용중지 |

### 정책 탭 버튼

| 버튼 | 동작 | 주의 |
|------|------|------|
| **정책 추가** | 폼을 초기화해 신규 입력 상태로 전환 | |
| **정책 저장** | 폼 내용을 저장(신규 or 수정) | 정책코드·정책명·Major·Minor 모두 필수 |
| **사용중지** | 선택된 정책을 비활성(useYn=N)으로 처리 | **품목에 연결된 정책은 사용중지 불가** |

## ② AQL 기준 탭

AQL 값별로 기준 헤더를 만들고, 정책에서 Major/Minor로 참조합니다.

### AQL 기준 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **AQL 코드(aqlCode)** | 기준 고유 코드. 정책의 Major/Minor가 참조. 등록 후 변경 불가. 예: `AQL-1.0` |
| **AQL 명칭(aqlName)** | 표시용 이름. 예: `일반검사 AQL 1.0` |
| **검사수준(inspectionLevel)** | 이 기준이 전제하는 검사수준(보통 II). |
| **AQL 값(aqlValue)** | 합격품질한계 수치(0.065~6.5). **작을수록 엄격**. |
| **사용(useYn)** | Y=활성(정책에서 선택 가능), N=사용중지 |

### AQL 기준 폼 입력 항목

| 항목 | 역할 / 의미 |
|------|------|
| **AQL 코드** | 대문자 자동 변환. 등록 후 수정 불가. |
| **AQL 명칭** | 필수. 검사원이 알아볼 수 있게 명확하게 작성. |
| **검사수준** | 공통코드 선택. |
| **AQL 값** | 공통코드 선택(0.065/0.10/0.15/0.25/0.40/0.65/1.0/1.5/2.5/4.0/6.5). |
| **사용여부** | Y/N. |
| **비고** | 적용 근거, 고객 요구사항, 개정 이력 등. |

> **ISO 판정표 관리 안내**: LOT 수량별 샘플수·Ac·Re는 이 탭에서 직접 입력하지 않습니다. "Code Letter 표" 탭과 "Sampling Plan 표" 탭에서 ISO 2859-1 기준 데이터를 관리합니다.

### AQL 기준 탭 버튼

| 버튼 | 동작 | 주의 |
|------|------|------|
| **AQL 기준 추가** | 폼 초기화 | |
| **저장** | 신규 등록 또는 수정 | 코드·명칭 필수 |
| **사용중지** | 선택 기준을 비활성 | **활성 정책에서 참조 중인 기준은 사용중지 불가** |
| **내보내기** | 목록을 Excel로 저장 | |

## ③ Code Letter 표 탭 (ISO 2859-1 Table I)

**LOT 크기 × 검사수준 → Code Letter** 대조표입니다. 화면에서 바로 확인하고 검사에 활용합니다.

읽는 방법:
1. 검사할 LOT 수량이 속하는 행(Lot Size)을 찾습니다.
2. 적용할 검사수준 열(I / II / III 또는 S1~S4)을 찾습니다.
3. 교차점의 알파벳이 **Code Letter**입니다(A~R).

| Lot Size | Level I | Level II | Level III |
|------|------|------|------|
| 2~8 | A | A | B |
| 9~15 | A | B | C |
| 16~25 | B | C | D |
| 26~50 | C | D | E |
| 51~90 | C | E | F |
| 91~150 | D | F | G |
| 151~280 | E | G | H |
| 281~500 | F | H | J |
| 501~1200 | G | J | K |
| 1201~3200 | H | K | L |
| 3201~10000 | J | L | M |
| ... | ... | ... | ... |

> 표의 전체 내용은 화면에서 확인하세요. 위 표는 일반검사수준(I/II/III) 일부입니다.

---

## ④ Sampling Plan 표 탭 (ISO 2859-1 Table II-A)

**Code Letter × AQL 값 → 샘플수·Ac·Re** 대조표입니다. 정상검사(Normal Inspection) 기준입니다.

읽는 방법:
1. Code Letter 표에서 결정한 알파벳 행을 찾습니다.
2. 적용할 AQL 값 열을 찾습니다.
3. 교차점의 **Ac / Re 숫자**가 합격·불합격 판정 기준입니다.

| Code Letter | 샘플수 | AQL 0.65 Ac/Re | AQL 1.0 Ac/Re | AQL 2.5 Ac/Re |
|------|------|------|------|------|
| A | 2 | — | — | 0/1 |
| B | 3 | — | 0/1 | 0/1 |
| C | 5 | — | 0/1 | 0/1 |
| D | 8 | 0/1 | 0/1 | 1/2 |
| E | 13 | 0/1 | 0/1 | 1/2 |
| F | 20 | 0/1 | 1/2 | 2/3 |
| G | 32 | 1/2 | 1/2 | 3/4 |
| H | 50 | 1/2 | 1/2 (↑) | 5/6 |
| J | 80 | 1/2 | 2/3 | 7/8 |
| K | 125 | 2/3 | 3/4 | 10/11 |

> 표의 전체 내용(A~R, AQL 0.065~6.5)은 화면에서 확인하세요. `↑`/`↓` 화살표가 있으면 해당 행의 실제 Ac/Re 대신 화살표가 가리키는 Code Letter의 값을 사용합니다.

## 실무 사용 예시

### 예시 1 — LOT 200개, 검사수준 II, AQL 1.0

1. Code Letter 표에서 → LOT 151~280, Level II → **Code Letter G**
2. Sampling Plan 표에서 → Code Letter G, AQL 1.0 → 샘플수 **32개**, Ac=**1**, Re=**2**
3. **32개 표본 검사 → 불량 1개이면 합격 / 2개 이상이면 불합격**

### 예시 2 — LOT 1000개, 검사수준 II, Major AQL 0.65 / Minor AQL 2.5

1. Code Letter 표 → LOT 501~1200, Level II → **Code Letter J**
2. Sampling Plan 표 → Code Letter J, AQL 0.65 → Major: 샘플 **80개**, Ac=1, Re=2
3. Sampling Plan 표 → Code Letter J, AQL 2.5 → Minor: 샘플 **80개**, Ac=7, Re=8
4. **80개 표본 검사:**
   - Critical 불량이 1건이라도 → 즉시 불합격
   - Major 불량 2개 이상 → 불합격
   - Minor 불량 8개 이상 → 불합격
   - 위 어느 것도 해당 없으면 → **합격**

## 등록 순서 (처음 세팅할 때)

1. **AQL 기준 탭** → AQL 기준 추가: AQL 코드·명칭·검사수준·AQL 값 입력 후 저장
   - 예) `AQL-0.65` (AQL 0.65), `AQL-1.0` (AQL 1.0), `AQL-2.5` (AQL 2.5)
2. **Code Letter 표 탭** → ISO 표 데이터 확인 (사전 입력되어 있음)
3. **Sampling Plan 표 탭** → ISO 표 데이터 확인 (사전 입력되어 있음)
4. **AQL 정책관리 탭** → 정책 추가: 정책 코드·정책명 입력, Major/Minor AQL 기준 선택 후 저장
   - 예) `AQLP-II-1.0-2.5` — 검사수준 II, Major AQL-1.0, Minor AQL-2.5
5. **[품목마스터]** → 해당 품목의 **AQL 정책** 필드에 위 정책 코드를 지정
6. 이후 수입검사(IQC)에서 해당 품목 입고 LOT 검사 시 자동 판정

## 입력 규칙 / 검증
- 정책 코드·정책명·Major AQL·Minor AQL은 모두 필수입니다.
- AQL 코드는 저장 시 **자동으로 대문자로 변환**됩니다.
- 정책 코드·AQL 코드는 등록 후 변경할 수 없습니다.
- 사용중인 AQL 기준만 정책의 Major/Minor 선택 목록에 나옵니다.

## 자주 묻는 질문

- **Q.** AQL 값은 어떻게 골라야 하나요?
  **A.** 일반적으로 중결함(Major)에는 **1.0**, 경결함(Minor)에는 **2.5**를 많이 씁니다. 안전·기능 부품은 0.65 이하의 엄격한 기준을 씁니다. 고객 도면이나 규격서에 지정된 값이 있으면 그것을 우선합니다.
- **Q.** 검사수준 II가 기본인데 왜 I이나 III도 있나요?
  **A.** 공급자 신뢰도나 검사 비용에 따라 강도를 조절합니다. 신규 공급자는 III(엄격), 오랫동안 합격률이 높은 공급자는 I(완화)로 전환할 수 있습니다.
- **Q.** Code Letter 표는 직접 수정할 수 있나요?
  **A.** Code Letter 표와 Sampling Plan 표는 ISO 2859-1 표준 데이터로 사전 입력되어 있습니다. 표준과 다르게 운영하려면 운영자에게 문의하세요.
- **Q.** Sampling Plan 표에 화살표(↑/↓)가 있는 셀은 어떻게 읽나요?
  **A.** 해당 Code Letter·AQL 조합에서는 직접 판정이 불가능해 다른 Code Letter로 이동한다는 표시입니다. 예를 들어 `↑B`이면 Code Letter B의 기준을 사용합니다.
- **Q.** Major와 Minor를 왜 따로 두나요?
  **A.** 결함의 심각도에 따라 합격 기준을 다르게 적용하기 위해서입니다. 보통 중결함(Major)은 엄격하게(작은 AQL 값), 경결함(Minor)은 느슨하게(큰 AQL 값) 설정합니다.
- **Q.** 정책을 삭제하고 싶은데 사용중지가 안 돼요.
  **A.** 품목마스터에서 해당 정책을 참조 중인 품목이 있으면 사용중지할 수 없습니다. 먼저 품목마스터에서 정책 연결을 해제하세요.
- **Q.** AQL 기준 사용중지가 안 돼요.
  **A.** 활성 상태인 AQL 정책에서 Major 또는 Minor로 참조 중인 기준은 사용중지할 수 없습니다. 먼저 해당 정책의 Major/Minor 연결을 다른 기준으로 변경하거나 정책을 비활성화하세요.
- **Q.** 정책을 품목에 연결하지 않으면 어떻게 되나요?
  **A.** AQL 자동 판정이 적용되지 않습니다. 수입검사에서 수동으로 합·불을 판정해야 합니다.
