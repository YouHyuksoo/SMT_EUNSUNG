# 거래처관리

## 화면 목적
회사가 거래하는 **모든 거래처(공급사·고객사·제조사)의 기본정보**를 등록·관리합니다. 여기 등록된 거래처가 구매발주·자재입하·출하·수입검사(IQC)에서 매입처·매출처·제조사로 참조됩니다.

## 화면 구성
- **왼쪽(목록)**: 거래처 그리드. 상단 툴바에서 **검색**(거래처코드·거래처명·사업자번호·담당자), **거래처 유형 필터**, **사용여부 필터**를 제공하고, 우측 상단에 **새로고침**·**거래처 추가** 버튼이 있습니다.
- **오른쪽(슬라이드 패널)**: 행의 ✏️(수정) 또는 "거래처 추가" 클릭 시 오른쪽에서 열리는 등록/수정 폼입니다. 폼은 **기본정보**와 **연락처** 두 영역으로 나뉩니다.
- 사용여부가 `N`인 거래처는 목록에서 붉은색으로 표시됩니다.

---

## ① 목록 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **작업(actions)** | 행별 **수정(✏️)**·**삭제(🗑️)** 버튼입니다. 수정은 우측 폼을 열고, 삭제는 확인 모달을 거쳐 거래처를 삭제합니다. |
| **거래처코드(partnerCode)** | MES 내부에서 거래처를 식별하는 **고유 코드**입니다. 한 번 등록하면 변경하지 않습니다(발주·입하·출하 데이터가 이 코드로 연결됨). |
| **거래처명(partnerName)** | 거래처(회사) 이름입니다. 현장·문서에서 식별하는 이름. |
| **거래처 유형(partnerType)** | 거래처의 분류입니다. 공급사(SUPPLIER)·고객사(CUSTOMER)·제조사(MFG). 구매/판매/제조 흐름에서 어디에 쓰이는지를 결정하며, 색상 배지로 표시됩니다. |
| **사업자번호(bizNo)** | 거래처의 사업자등록번호입니다(세금계산서·법인 식별용). |
| **대표자(ceoName)** | 거래처 대표자 이름입니다. |
| **전화번호(tel)** | 거래처 대표 전화번호입니다. |
| **담당자(contactPerson)** | 거래처 측 연락 담당자 이름입니다. |
| **이메일(email)** | 거래처 대표 이메일 주소입니다(발주·통지 연락용). |
| **사용여부(useYn)** | `Y`일 때만 조회·선택·사용 대상입니다. 거래 종료 시 `N`으로 비활성화합니다. |

## ② 등록/수정 폼 — 기본정보

| 컬럼 | 역할 / 의미 |
|------|------|
| **거래처코드(partnerCode)** | 거래처 고유 코드입니다(**필수**). 등록(추가) 시에만 입력할 수 있고, 수정 시에는 잠겨서 변경할 수 없습니다(불변 키). |
| **거래처 유형(partnerType)** | 공급사·고객사·제조사 중 선택합니다(**필수**, 공통코드 PARTNER_TYPE). 신규 등록 시 기본값은 공급사(SUPPLIER)입니다. |
| **거래처명(partnerName)** | 거래처 이름입니다(**필수**). |
| **사업자번호(bizNo)** | 사업자등록번호입니다. |
| **대표자(ceoName)** | 거래처 대표자 이름입니다. |

## ③ 등록/수정 폼 — 연락처

| 컬럼 | 역할 / 의미 |
|------|------|
| **주소(address)** | 거래처 주소입니다. |
| **전화번호(tel)** | 거래처 대표 전화번호입니다. |
| **팩스번호(fax)** | 거래처 팩스번호입니다. |
| **이메일(email)** | 거래처 대표 이메일 주소입니다. |
| **담당자(contactPerson)** | 거래처 측 연락 담당자 이름입니다. |
| **비고(remark)** | 거래처 관리 참고사항입니다. |
| **사용여부(useYn)** | `Y`/`N` 라디오로 선택합니다. `Y`만 활성(조회·선택 대상), `N`은 사용 중지. 신규 기본값은 `Y`입니다. |

## 사용 순서
1. 상단 **거래처 추가**(또는 행 ✏️)로 오른쪽 슬라이드 폼을 엽니다.
2. **기본정보**에서 거래처코드·거래처 유형·거래처명을 입력합니다(필수 3개).
3. 필요 시 사업자번호·대표자, **연락처**(주소·전화·팩스·이메일·담당자)와 비고를 채웁니다.
4. **저장**을 누릅니다. 거래종료 등으로 더 쓰지 않을 거래처는 **사용여부**를 `N`으로 바꿔 비활성화합니다.
5. 목록에서 **검색어**·**거래처 유형**·**사용여부** 필터로 원하는 거래처를 빠르게 찾습니다.

## 입력 규칙·검증
- **거래처코드**와 **거래처명**은 필수입니다. 둘 중 하나라도 비어 있으면 저장 버튼이 비활성화됩니다.
- 거래처코드는 **중복될 수 없습니다.** 이미 존재하는 코드로 추가하면 저장에 실패합니다(중복 오류).
- 거래처코드는 **수정 시 변경 불가**입니다(등록 시 한 번만 입력).
- 거래처 유형은 공급사·고객사·제조사 중에서만 선택할 수 있습니다.

## 자주 묻는 질문
- **Q.** 공급사·고객사·제조사는 어떻게 구분하나요?
  **A.** 공급사(SUPPLIER)는 자재를 납품받는 매입처, 고객사(CUSTOMER)는 제품을 출하하는 매출처, 제조사(MFG)는 자재의 제조원입니다. 한 거래처가 여러 역할을 하면 유형별로 각각 등록합니다.
- **Q.** 거래처코드를 잘못 입력했어요.
  **A.** 거래처코드는 등록 후 변경할 수 없습니다. 코드를 바꾸려면 새 코드로 다시 등록하고 기존 거래처는 사용여부를 `N`으로 내립니다.
- **Q.** 삭제와 사용여부 `N`은 무엇이 다른가요?
  **A.** 사용여부 `N`은 기록을 남긴 채 선택 목록에서만 빼는 것이고, 삭제는 거래처 데이터를 제거합니다. 발주·입하 이력이 있는 거래처는 사용여부 `N`을 권장합니다.
