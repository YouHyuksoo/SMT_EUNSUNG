# 실적입력(검사장비)

## 화면 목적
검사장비 측정값과 판정을 작업지시·설비 기준 생산실적으로 남깁니다. 화면의 값은 작업지시, 생산실적, 재고, 라벨 추적성 중 하나 이상과 연결되므로 조회 조건과 처리 상태를 함께 확인해야 합니다.

## 화면 구성
- **조회 조건**: 기간, 상태, 품목, 작업지시번호, 바코드 등 화면별 조건으로 대상을 좁힙니다.
- **목록 그리드**: 선택한 조건에 맞는 생산 데이터를 표시합니다.
- **상세/처리 영역**: 선택 행의 상세 라벨, 공정, 수량, 승인, 실행 결과를 확인하거나 처리합니다.

## 사용 순서
1. 조회 조건을 실제 작업 기준으로 설정합니다.
2. 목록에서 작업지시번호, 품목코드, 바코드/UID, 상태를 확인합니다.
3. 수량 항목은 계획수량, 양품수량, 불량수량, 가용수량처럼 의미가 다르므로 컬럼명을 기준으로 구분합니다.
4. 처리 버튼이 있는 화면은 대상 상태가 처리 가능한지 확인한 뒤 실행합니다.
5. 처리 후 목록을 다시 조회해 상태와 수량 변경을 확인합니다.

## 입력 규칙·검증
- 작업지시, 품목, 설비, 작업자, 바코드/UID 중 화면에서 요구하는 기준값이 없으면 저장 또는 실행할 수 없습니다.
- 취소·삭제·승인·완료 처리는 재고와 추적성에 영향을 주므로 실제 업무 사유가 있을 때만 수행합니다.
- 집계 화면의 수량과 라벨 상세 수량은 기준 단위가 다를 수 있습니다. 품목 집계와 바코드 상세를 분리해 확인하세요.

## 자주 묻는 질문
- **Q. 목록 수량과 상세 라벨 수량이 다릅니다.**
  **A.** 목록은 품목/창고/작업지시 단위 집계이고 상세는 바코드/UID 단위일 수 있습니다. 같은 기준으로 비교해야 합니다.
- **Q. 처리 버튼이 동작하지 않습니다.**
  **A.** 대상 상태가 이미 완료, 취소, 출하, 소비 상태인지 확인하세요. 상태가 맞지 않으면 서버가 처리를 차단합니다.
