# 입하관리

## 화면 목적
발주(PO)에 등록된 품목이 실제로 납품되었을 때 **입하를 등록**하는 화면입니다. 라인 단위로 입하수량을 기록하면 자재 LOT 시리얼이 자동 채번되고, LOT 라벨을 출력해 현장에서 붙일 수 있습니다. 입하된 자재는 수입검사(IQC) 대기 상태로 전환됩니다.

> 입하 = 납품 확인 단계. 입고(재고 반영)는 IQC 통과 후 별도 처리됩니다.

## 흐름도
```
발주(PO) 확정 → 입하관리(이 화면)에서 납품 확인
  → 자재 LOT 시리얼 자동 발급 + 라벨 출력
  → IQC 수입검사 대기(PENDING)
  → IQC 합격 → 입하재고 → 입고(자재 현재고)
```

## 화면 구성
- **메인 그리드(PO 라인 목록)**: 입하 가능한 PO 라인을 전부 표시합니다. 상단에 상태 필터·품목코드·PO번호 검색이 있습니다.
- **자재입하 버튼(셀 또는 행 클릭)**: 해당 라인의 입하 등록 모달을 엽니다.
- **수동 입하 버튼(우상단)**: PO 없이 품목/수량을 직접 지정해 입하합니다.
- **라벨 템플릿 셀렉트(우상단)**: 라벨 출력에 사용할 템플릿을 미리 선택합니다.

---

## ① PO 라인 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **(액션)** | **자재입하** 버튼. 잔량이 0이거나 CLOSE 상태이면 비활성화됩니다. |
| **발주번호(poNo)** | 구매발주(PO) 번호입니다. 발주관리에서 CONFIRMED 상태가 된 PO만 목록에 표시됩니다. |
| **L/N(lineNo)** | 발주서 라인 번호입니다. 한 PO에 여러 품목이 있을 때 각 품목을 구분합니다. |
| **R/N(revNo)** | 라인의 개정번호(Revision Number)입니다. 동일 라인이 수정된 이력을 구분합니다. |
| **품목코드(itemCode)** | 자재의 MES 내부 품목코드입니다. |
| **품목명(itemName)** | 자재 품목 이름입니다. |
| **발주수량(orderQty)** | 발주서에 기재된 총 발주 수량입니다. |
| **누적입하(receivedQty)** | 지금까지 이 라인에 등록된 입하 수량의 합계입니다. |
| **잔량(remainingQty)** | 아직 입하되지 않은 수량(발주수량 − 누적입하)입니다. 파란 굵은 글씨로 표시됩니다. |
| **발주일(orderDate)** | 발주서가 작성된 날짜입니다. |
| **거래처(partnerName)** | 납품 공급업체(거래처) 이름입니다. |
| **사용구분(useType)** | 발주의 사용 목적 구분입니다(예: 생산용 PROD, 비생산 등). 공통코드 `PO_USE_TYPE` 기준. |
| **상태(lineStatus)** | 라인의 입하 진행 상태입니다. `OPEN`(미입하) / `PARTIAL`(일부입하, 노란 배경) / `CLOSE`(완료, 회색 배경·비활성). |

> 행 배경색 의미: 흰색=미입하, 노란색=일부입하, 파란색=잔량 0(CLOSE 전), 회색=CLOSE(입하 완료).

## ② 입하 등록 모달(자재입하 클릭 시)

선택한 PO 라인에 대한 입하 정보를 입력합니다.

| 입력 항목 | 역할 / 의미 |
|------|------|
| **발주정보(상단 요약)** | 선택한 발주번호 / L번호 / R번호와 품목코드·품목명·거래처를 요약 표시합니다(읽기 전용). |
| **누적입하 / 발주수량 / 잔량** | 현재까지의 입하 현황을 요약합니다(읽기 전용). 잔량이 입하 가능한 최대값입니다. |
| **입하수량 \*** | 이번에 실제로 납품받은 수량입니다. 잔량을 초과할 수 없습니다. |
| **입하일 \*** | 자재가 실제로 들어온 날짜입니다. 기본값은 오늘이며, 미래 날짜는 입력할 수 없습니다. |
| **제조사 \*** | 자재를 실제로 제조한 회사입니다. `PARTNER_MASTERS` 중 제조사(MFG) 유형에서 선택합니다. 라벨에 인쇄됩니다. |
| **창고 \*** | 입하 자재를 적치할 원자재 창고입니다. 원자재(RAW) 유형 창고만 선택 가능합니다. |
| **시리얼 구성단위(lotUnitQty)** | 품목마스터에 등록된 LOT 구성단위(한 시리얼에 담을 수량)입니다. 읽기 전용. 미설정 시 "1 LOT"으로 표시. |
| **예상 시리얼 수** | 입하수량 ÷ 시리얼 구성단위로 자동 계산된 발급될 시리얼 개수입니다. 확인용이며 수정 불가입니다. |
| **비고** | 입하 메모를 자유롭게 입력합니다(선택). |

저장 후에는 **시리얼 발급 최종 확인 모달**이 뜹니다. 확인하면 서버에 등록되고 **라벨 미리보기 모달**이 열립니다.

---

## ③ 라벨 미리보기 모달(입하 등록 성공 후)

발급된 자재 LOT 시리얼 라벨을 미리 볼 수 있습니다.

| 항목 | 역할 / 의미 |
|------|------|
| **라벨 템플릿 선택** | 출력할 라벨 디자인을 고릅니다. 우상단에서 미리 선택한 템플릿이 기본 적용됩니다. |
| **라벨 미리보기 영역** | 발급된 시리얼 수만큼 라벨이 나열됩니다. 각 라벨에는 자재 UID, 품목코드, 품목명, 수량, 제조사, 입하번호, 입하일 등이 인쇄됩니다. |
| **라벨 출력(프린트 버튼)** | 로컬 Print Agent를 통해 라벨 프린터로 전송합니다. |

---

## ④ 수동 입하 모달(PO 없이 입하)

PO가 없는 긴급 납품·재고 조정 등 예외 상황에서 사용합니다.

| 입력 항목 | 역할 / 의미 |
|------|------|
| **품목코드 \*** | 입하할 자재를 품목 검색으로 선택합니다. |
| **창고 \*** | 입하 자재를 적치할 창고입니다. |
| **수량 \*** | 입하 수량입니다. |
| **공급업체 UID(supUid)** | 납품업체가 자체 부여한 자재 시리얼 번호입니다(선택). |
| **제조일(manufactureDate)** | 자재의 제조(생산) 날짜입니다(선택). |
| **거래처(vendor)** | 납품한 공급업체입니다(선택). `SUPPLIER` 유형 거래처에서 선택. |
| **비고** | 입하 사유·메모입니다(선택). |

---

## 사용 순서

**PO 기반 입하 (일반)**
1. 상단 검색에서 품목코드 또는 발주번호로 원하는 라인을 찾습니다.
2. 상태 필터에서 `OPEN` / `PARTIAL`이 체크되어 있는지 확인합니다.
3. 해당 라인의 **자재입하** 버튼을 클릭합니다.
4. 입하수량, 입하일, 제조사, 창고를 입력하고 **저장**을 클릭합니다.
5. 예상 시리얼 수를 확인하고 **확인**을 클릭합니다.
6. 라벨 미리보기에서 라벨 내용을 확인하고 **라벨 출력**합니다.
7. 출력한 라벨을 자재 박스에 부착합니다.

**PO 없이 긴급 입하**
1. 우상단 **수동 입하** 버튼을 클릭합니다.
2. 품목, 창고, 수량 (필수)을 입력하고 **등록**합니다.

---

## 입력 규칙 / 검증

- 입하수량은 1 이상이어야 하며 잔량을 초과할 수 없습니다.
- 입하일은 오늘 이전 날짜(과거 포함 오늘)만 허용합니다.
- 제조사와 창고는 필수 항목입니다.
- CLOSE 상태이거나 잔량이 0인 라인은 **자재입하** 버튼이 비활성화됩니다.

---

## 자주 묻는 질문

- **Q.** 잔량이 0인데 CLOSE가 아닌 라인이 있어요.
  **A.** 잔량이 0이더라도 PO가 아직 CLOSE 처리되지 않은 경우입니다. 자재입하 버튼은 비활성 상태이므로 추가 입하는 불가합니다.

- **Q.** 시리얼 구성단위가 비어 있어요(1 LOT).
  **A.** 품목마스터에 LOT 구성단위(lotUnitQty)가 설정되지 않은 경우입니다. 이 경우 입하 수량 전체가 1개의 시리얼로 발급됩니다.

- **Q.** 입하 후 IQC 검사는 어디서 하나요?
  **A.** 입하관리에서 등록 완료 후 [수입검사(IQC)] 화면에서 LOT별로 검사를 진행합니다.

- **Q.** 잘못 입하했어요. 취소할 수 있나요?
  **A.** 운영자 권한이 있으면 [입하내역/수불원장] 화면에서 DONE 상태의 입하 건을 취소할 수 있습니다. 취소는 삭제가 아닌 역분개 방식으로 처리됩니다.

- **Q.** 라벨이 출력되지 않아요.
  **A.** PC에 Print Agent가 실행 중인지 확인하세요. Agent가 없으면 라벨 미리보기 화면에서 라벨을 수동으로 저장해 출력할 수 있습니다.
