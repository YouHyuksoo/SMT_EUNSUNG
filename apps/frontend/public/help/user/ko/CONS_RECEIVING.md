# 소모품 입고

## 화면 목적
입고할 **소모품을 창고 재고로 확정**하는 화면입니다. 라벨 발행으로 채번된 소모품 UID 바코드를 스캔해 한 건씩 입고하거나, UID 없이 수량 단위로 수동 입고할 수 있습니다. 입고 처리하면 소모품 마스터의 보유 재고가 늘어나고, 입고 이력이 남습니다.

> 소모품 입고에는 두 가지 경로가 있습니다. **① 바코드 스캔 입고**는 [소모품 라벨발행](/consumables/label)에서 미리 발행한 개별 UID를 스캔해 확정하는 방식이고, **② 수동 입고**는 UID 없이 소모품·수량만으로 재고를 올리는 방식입니다.

## 화면 구성
- **상단(바코드 스캔 패널)**: 입고/반납 모드를 토글하고 UID 바코드를 스캔합니다. 입고 모드에서는 아직 입고되지 않은 **미입고 대기 목록**도 함께 보여줍니다.
- **하단(입고 이력)**: 입고·반납 이력 그리드입니다. 위쪽 도구막대에서 **기간**, **검색어**, **유형 필터**로 조회 범위를 좁힙니다.
- **우측(슬라이드 패널)**: 상단 **입고등록** 버튼을 누르면 열리는 수동 입고 폼입니다.

---

## ① 바코드 스캔 패널

| 항목 | 역할 / 의미 |
|------|------|
| **입고 / 반납 모드** | 좌측 토글로 처리 종류를 고릅니다. **입고**는 미입고 UID를 입고 확정하고, **반납(반납입고)**은 이미 입고된 UID를 되돌려 재고를 차감합니다. |
| **UID 바코드 입력** | 소모품 UID 바코드를 스캔(또는 입력 후 Enter)하면 즉시 처리됩니다. 스캔 후 입력칸은 자동으로 비워지고 다음 스캔을 받습니다. |
| **보관위치(입고 모드)** | 입고할 소모품을 둘 보관위치를 선택합니다(선택 항목). 입고 확정 시 그 UID의 보관위치로 기록됩니다. |
| **반품사유(반납 모드)** | 반납하는 이유를 입력합니다(선택 항목). 반납 이력의 사유로 남습니다. |
| **입고확정 / 반납확정 버튼** | 입력한 UID를 처리합니다. 바코드 스캐너로 Enter가 입력되면 버튼을 누르지 않아도 자동 처리됩니다. |

### 미입고 대기 목록 (입고 모드에서만 표시)
라벨만 발행되고 아직 입고되지 않은(대기 중인) UID들을 보여줍니다. 여기 있는 UID를 스캔하면 입고 확정됩니다.

| 컬럼 | 역할 / 의미 |
|------|------|
| **UID(conUid)** | 입고 대기 중인 소모품 개별 식별번호입니다. |
| **소모품코드(consumableCode)** | 해당 UID의 소모품 코드입니다. |
| **소모품명(consumableName)** | 소모품 이름입니다. |
| **카테고리(category)** | 금형·지그·공구 등 소모품 분류입니다. |
| **라벨인쇄일(labelPrintedAt)** | 그 UID 라벨이 발행된 시각입니다. |
| **공급업체명(vendorName)** | 라벨 발행 시 입력된 공급업체입니다. |

## ② 입고 이력 그리드

| 컬럼 | 역할 / 의미 |
|------|------|
| **일시(createdAt)** | 입고·반납이 처리된 일시입니다. |
| **소모품코드(consumableCode)** | 입고된 소모품 코드입니다. |
| **소모품명(consumableName)** | 소모품 이름입니다. |
| **UID(conUid)** | 바코드 스캔 입고일 때의 개별 UID입니다. 수동 입고는 UID가 없어 `-`로 표시됩니다. |
| **유형(logType)** | **입고(IN)** 또는 **입고반품(IN_RETURN)**입니다. 색상 배지로 구분됩니다. |
| **수량(qty)** | 입고/반납 수량입니다. 입고는 `+`, 반납은 `-`로 표시됩니다. |
| **공급업체코드(vendorCode)** | 공급업체 코드입니다. |
| **공급업체명(vendorName)** | 공급업체 이름입니다. |
| **단가(unitPrice)** | 입고 단가입니다(원). |
| **입고구분(incomingType)** | **신규(NEW)** 또는 **교체(REPLACEMENT)**입니다. |
| **비고(remark)** | 입고/반납 시 남긴 메모입니다. |

### 도구막대 필터
| 항목 | 역할 / 의미 |
|------|------|
| **시작일 / 종료일** | 조회할 입고 이력 기간입니다. |
| **검색어** | 소모품코드·소모품명으로 이력을 검색합니다. |
| **유형 필터** | 전체 / 입고 / 입고반품 중에서 표시할 유형을 고릅니다. |

## ③ 수동 입고 폼 (우측 패널)

상단 **입고등록** 버튼으로 엽니다. UID 없이 소모품·수량 단위로 재고를 올립니다.

| 항목 | 역할 / 의미 |
|------|------|
| **소모품** | 돋보기 버튼으로 소모품을 검색·선택합니다. 입고 대상이며 필수입니다. |
| **수량(qty)** | 입고할 수량입니다. 이 수량만큼 보유 재고가 증가합니다(기본 1). |
| **입고구분(incomingType)** | **신규** 또는 **교체** 중 선택합니다. |
| **공급업체코드 / 공급업체명** | 입고 소모품의 공급업체 정보입니다. |
| **단가(unitPrice)** | 소모품 1개의 입고 단가입니다(선택). |
| **비고(remark)** | 입고 메모입니다(선택). |

## 사용 순서
1. **바코드 스캔 입고**: 상단에서 **입고** 모드 확인 → (필요 시) 보관위치 선택 → UID 바코드 스캔 → 즉시 입고 확정. 미입고 대기 목록에서 대상 UID를 확인할 수 있습니다.
2. **반납입고**: 모드를 **반납**으로 전환 → (필요 시) 반품사유 입력 → 입고된 UID 스캔 → 반납 확정(재고 차감).
3. **수동 입고**: 상단 **입고등록** → 소모품 검색·선택 → 수량·입고구분·공급업체·단가 입력 → **등록**.
4. 처리 결과는 하단 **입고 이력** 그리드에서 확인합니다.

## 입력 규칙 / 검증
- 수동 입고는 **소모품을 선택해야** 등록 버튼이 활성화됩니다.
- 스캔 입고는 **미입고(대기) 상태의 UID**만 확정할 수 있습니다. 이미 입고된 UID를 다시 스캔하면 오류가 납니다.
- 반납입고는 **현재 입고(활성) 상태인 UID**만 처리할 수 있습니다.

## 자주 묻는 질문
- **Q.** 미입고 대기 목록에 UID가 없어요.
  **A.** 입고하려면 먼저 [소모품 라벨발행](/consumables/label)에서 UID를 발행해야 합니다. 라벨이 발행되면 대기 목록에 나타납니다.
- **Q.** 스캔했더니 "이미 입고된 상태"라고 나와요.
  **A.** 그 UID는 이미 입고 확정된 건입니다. 입고 이력에서 확인하세요.
- **Q.** 수동 입고와 스캔 입고는 무엇이 다른가요?
  **A.** 스캔 입고는 UID 단위로 개별 추적되는 1개 입고이고, 수동 입고는 UID 없이 수량만큼 재고를 올리는 방식입니다. 둘 다 보유 재고를 증가시킵니다.
- **Q.** 입고 수량이 재고에 어떻게 반영되나요?
  **A.** 입고(IN)는 소모품 보유 재고를 늘리고, 반납(IN_RETURN)은 줄입니다.
