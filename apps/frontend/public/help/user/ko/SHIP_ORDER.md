# 출하지시관리

## 화면 목적

고객사에 출하할 완제품 품목과 수량을 지정하는 **출하지시서**를 작성·확정·출력하는 화면입니다.

- 출하지시 작성(DRAFT) → 확정(CONFIRMED) → 포장·팔레트 작업 → 출하완료(SHIPPED) 순서로 진행됩니다.
- 확정 전(DRAFT)에만 수정·삭제할 수 있습니다.
- 출하지시서는 A4 QR코드 포함 양식으로 인쇄할 수 있습니다.

## 화면 구성

```
┌─────────────────────────────────────────────────────────────┐
│  헤더: 제목 + [출력] [확정] [확정취소] [삭제] [새로고침] [등록] │
├─────────────────────────────────────────────────────────────┤
│  ① 출하지시 목록 그리드                                       │
│  (출하일 기간 필터 + 검색 + 상태 필터 + 내보내기)              │
└──────────────────────────────────┬──────────────────────────┘
                                   │ 행 클릭 후 [등록/수정] 클릭 시 열림
                              ② 등록·수정 패널 (우측 슬라이드)
```

> **기간 밖 미완료 알림**: 조회 기간 밖이지만 아직 출하가 끝나지 않은 지시(DRAFT/CONFIRMED/SHIPPING)는 자동으로 목록에 포함되고, 왼쪽에 주황색 테두리로 표시됩니다.

## ① 출하지시 목록 그리드 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **(편집 아이콘)** | 클릭 시 우측 수정 패널을 엽니다. DRAFT 상태만 내용을 수정할 수 있습니다. |
| **출하지시번호** | 출하지시 고유번호. 자동 채번됩니다. |
| **고객사** | 출하 대상 고객사 이름. |
| **고객 PO번호** | 고객이 발행한 구매오더(PO) 번호. 없으면 `-`로 표시됩니다. |
| **납기일** | 고객이 요구한 납품 기한 일자. |
| **출하일** | 실제 출하 예정일. 등록 시 필수 항목입니다. |
| **품목수** | 출하지시에 포함된 품목 종류 수. |
| **총수량** | 출하지시 전체 품목의 합산 수량. |
| **상태** | 출하지시의 현재 진행 상태(DRAFT/CONFIRMED/SHIPPING/SHIPPED/CLOSED). |

### 그리드 도구 모음

| 도구 | 역할 |
|------|------|
| **출하일 기간** | 기본값은 당일. 기간을 변경해 과거 또는 미래 지시를 조회합니다. |
| **검색** | 출하지시번호, 고객사명 등 텍스트로 빠른 필터. |
| **상태** | 특정 상태(DRAFT/CONFIRMED/SHIPPED 등)만 골라 봅니다. |
| **내보내기** | 조회된 목록을 엑셀로 내려받습니다. |

## ② 등록·수정 패널 항목

| 항목 | 필수 | 역할 / 의미 |
|------|:----:|------|
| **출하지시번호** | — | 등록 시 자동 생성. 수정 시 기존 번호 표시(변경 불가). |
| **거래처(고객사)** | | 출하 대상 고객사를 선택합니다. |
| **고객 PO번호** | | 고객의 구매오더 번호를 입력합니다(최대 100자). |
| **납기일** | | 고객 납품 기한일을 선택합니다. |
| **출하일** | ✓ | 출하 예정일을 선택합니다. 저장을 위해 반드시 입력해야 합니다. |
| **비고** | | 참고 사항을 자유롭게 입력합니다. |

### 품목 목록 항목

| 항목 | 필수 | 역할 / 의미 |
|------|:----:|------|
| **품목 추가(+)** | — | 품목검색 팝업에서 완제품(FINISHED) 품목을 선택해 추가합니다. 동일 품목 중복 추가 불가. |
| **품목코드 / 품목명 / 단위** | — | 선택된 품목 정보. 읽기 전용. |
| **수량** | ✓ | 출하 지시 수량. 1 이상의 정수만 입력 가능합니다. |
| **비고** | | 해당 품목에 대한 메모. |
| **삭제(X)** | — | 해당 품목을 목록에서 제거합니다. |

### 패널 버튼

| 버튼 | 활성 조건 | 동작 |
|------|------|------|
| **취소** | 항상 | 패널을 닫습니다. 저장하지 않은 내용은 사라집니다. |
| **저장 후 확정** | DRAFT + 출하일 + 품목 ≥ 1개 + 모든 수량 > 0 | 저장 완료 후 바로 CONFIRMED로 확정합니다. |
| **확정취소** | 수정 패널에서 CONFIRMED 상태인 경우만 표시 | CONFIRMED → DRAFT로 되돌립니다. |
| **저장** | 출하일 + 품목 ≥ 1개 + 모든 수량 > 0 | DRAFT 상태로 저장합니다. |

## 버튼별 기능

| 버튼 | 언제 사용 | 활성 조건 | 동작 |
|------|------|------|------|
| **출력** | 출하지시서를 인쇄할 때 | 행 선택 시 | 선택 행 출하지시서를 A4·QR코드 포함 인쇄 |
| **확정** | DRAFT 지시를 확정할 때 | 행 선택 + DRAFT + 품목 ≥ 1 | 확인 팝업 후 CONFIRMED로 전환 |
| **확정취소** | 실수로 확정한 경우 되돌릴 때 | 행 선택 + CONFIRMED | 확인 팝업 후 DRAFT로 복귀 |
| **삭제** | 잘못 등록한 지시를 제거할 때 | 행 선택 + DRAFT | 확인 팝업 후 영구 삭제 |
| **새로 고침** | 목록을 최신 상태로 갱신할 때 | 항상 | 현재 필터 조건으로 재조회 |
| **등록** | 새 출하지시를 만들 때 | 항상 | 우측 등록 패널 열기 |

## 업무 흐름

```
[등록] 버튼 클릭
    ↓
우측 패널: 거래처 · 출하일 입력 → 품목 추가 → 수량 입력
    ↓
[저장] → DRAFT 상태 저장   또는   [저장 후 확정] → CONFIRMED 바로 전환
    ↓
(DRAFT 수정 필요 시) 편집 아이콘 클릭 → 패널에서 수정 → [저장]
    ↓
(확정) 행 클릭 → 헤더 [확정] 버튼 → 확인 팝업 → CONFIRMED
    ↓
이후 포장관리(/shipping/pack)에서 박스 할당 → 팔레트적재 → 출하
```

## 입력 규칙 / 검증

- **출하일**은 반드시 입력해야 저장·확정이 가능합니다.
- **품목이 없으면** 저장 및 확정이 모두 불가능합니다.
- 품목 수량은 **1 이상의 정수**여야 합니다. 0이거나 소수이면 저장이 막힙니다.
- **CONFIRMED 이후**에는 수정·삭제가 불가능합니다. 수정이 필요하면 먼저 확정취소를 해야 합니다.
- 확정취소는 **출하수량 또는 배정된 박스·팔레트가 없을 때만** 가능합니다.
- 동일한 품목 코드는 중복 추가할 수 없습니다.

## 자주 묻는 질문

- **Q. 조회 기간을 당일로 설정했는데 오래된 지시가 목록에 보입니다.**
  **A.** 기간 밖이지만 아직 출하가 끝나지 않은(DRAFT/CONFIRMED/SHIPPING) 지시는 자동으로 포함됩니다. 주황색 왼쪽 테두리로 구분됩니다. 상태 필터에서 원하는 상태를 선택하면 기간 밖 자동 포함이 적용되지 않습니다.

- **Q. 확정 버튼이 비활성화되어 클릭되지 않습니다.**
  **A.** 행을 먼저 클릭해 선택해야 합니다. 선택한 지시의 상태가 DRAFT이고 품목이 1개 이상이어야 확정 버튼이 활성화됩니다.

- **Q. CONFIRMED 상태에서 수정이 안 됩니다.**
  **A.** 확정된 지시는 수정 불가입니다. 수정하려면 먼저 [확정취소] 버튼으로 DRAFT 상태로 되돌린 뒤 수정하세요.

- **Q. 확정취소가 실패합니다.**
  **A.** 이미 포장이나 팔레트 작업에 연결된 박스가 있으면 확정취소가 차단됩니다. 포장관리·팔레트 화면에서 해당 박스·팔레트를 먼저 해제한 뒤 확정취소를 진행하세요.

- **Q. 품목 추가 팝업에서 원하는 품목이 검색되지 않습니다.**
  **A.** 출하지시에는 완제품(FINISHED) 품목만 추가할 수 있습니다. 해당 품목이 품목마스터에 FINISHED 유형으로 등록되어 있는지 확인하세요.

- **Q. 출하지시서 인쇄가 되지 않습니다.**
  **A.** 브라우저의 팝업 차단 설정을 확인하세요. 또는 브라우저 인쇄 대화상자가 나타나지 않는 경우 다시 시도해 보세요.
