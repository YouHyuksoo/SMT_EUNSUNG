# 공정 CAPA 관리

## 화면 목적
**공정과 품목의 조합별 생산능력(CAPA)**을 등록·관리하는 화면입니다. 한 공정에서 특정 품목을 1시간/하루에 얼마나 만들 수 있는지를 표준 택트타임·UPH·일일 CAPA로 정의합니다. 여기 등록한 값이 **생산계획·생산 시뮬레이션(MPS)**에서 공정별 소요 능력·일정을 산출하는 기준이 됩니다.

## 화면 구성
- **상단(통계 카드)**: 전체 / 설비종속 / 인력종속 / 비활성 건수를 한눈에 표시합니다.
  - 설비종속 = 설비 대수가 1 이상인 CAPA, 인력종속 = 작업자 수가 1 이상인 CAPA, 비활성 = 사용여부 `N`.
- **왼쪽(목록)**: 공정 CAPA 그리드. 상단에서 **공정 필터**와 검색(공정/품목 코드·이름), **CAPA 추가**.
- **오른쪽(슬라이드 패널)**: 행 클릭 또는 "CAPA 추가" 시 열리는 등록/수정 폼. 택트타임 입력 시 **UPH가 자동 계산**되고, 하단에 **일일 CAPA 미리보기**가 실시간으로 표시됩니다.

> CAPA의 키는 **공정 + 품목**입니다. 같은 공정이라도 품목이 다르면 별도로 등록합니다.

---

## ① 전체 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **공정(processCode)** | 이 CAPA가 적용되는 공정입니다(공정관리에 등록된 공정). 목록에는 공정명으로 표시됩니다. 공정+품목 조합으로 생산능력을 관리합니다. |
| **품목(itemCode)** | 이 공정에서 생산하는 품목입니다(품목마스터에 등록된 품목). 공정+품목 조합이 CAPA의 키가 됩니다. |
| **품목명(itemName)** | 선택한 품목의 이름입니다(품목마스터 참조, 읽기 전용 표시). |
| **표준 택트타임(stdTactTime)** | 1개를 생산하는 데 걸리는 표준 시간(**초**)입니다. 입력하면 UPH가 `3600 ÷ 택트타임`으로 자동 계산됩니다. |
| **표준 UPH(stdUph)** | 시간당 생산량(Units Per Hour)입니다. 택트타임으로부터 자동 계산되며 직접 수정하지 않습니다. 예: 택트타임 30초 → UPH 120. |
| **작업자 수(workerCnt)** | 이 공정에 투입하는 표준 작업자 수입니다. **설비 대수가 없을 때** 일일 CAPA 계산의 배수로 사용됩니다. |
| **보드/지그 수(boardCnt)** | 이 공정에서 사용하는 작업 보드(지그·판) 수입니다. 능력 참고용 정보입니다. |
| **설비 대수(equipCnt)** | 이 공정에 투입하는 설비 수입니다. **값이 있으면** 일일 CAPA 계산에서 작업자 수 대신 우선 적용됩니다(설비종속). |
| **전환시간(setupTime)** | 모델 전환 등 작업 준비에 걸리는 시간(**분**)입니다. 셋업/체인지오버 시간 참고용입니다. |
| **밸런싱 효율(balanceEff)** | 라인 밸런싱 효율(**%**)입니다. 일일 CAPA에 곱해지는 가동 효율로, 기본값은 85%입니다. 100%면 손실 없이 가동하는 이상치입니다. |
| **일일 CAPA(dailyCapa)** | 하루 생산능력입니다. `UPH × 8시간 × (설비 대수 또는 작업자 수) × 밸런싱 효율`로 자동 계산됩니다. 목록·미리보기에 강조 표시됩니다. |
| **사용여부(useYn)** | `Y`일 때만 조회·선택·계산 대상입니다. 사용 중지는 `N`. |
| **비고(remark)** | 공정 CAPA 관리 참고사항입니다. |

> **일일 CAPA 산식**: `UPH × 8h × 배수 × (밸런싱효율 ÷ 100)`. 배수는 **설비 대수가 1 이상이면 설비 대수**, 아니면 작업자 수, 둘 다 0이면 1을 사용합니다. 결과는 소수점 버림(내림).

## 사용 순서
1. 상단 **CAPA 추가**(또는 행 클릭)로 슬라이드 폼을 엽니다.
2. **공정**을 선택하고, **품목** 검색 버튼으로 품목을 고릅니다(신규일 때만 변경 가능).
3. **표준 택트타임(초)**을 입력하면 **UPH가 자동 계산**됩니다.
4. **작업자 수** 또는 **설비 대수**, **밸런싱 효율**을 입력합니다. 하단 **일일 CAPA 미리보기**가 실시간으로 갱신됩니다.
5. 필요 시 보드/지그 수·전환시간·비고를 채우고 **저장**합니다.

## 입력 규칙·검증
- **공정**과 **품목**은 필수입니다. 둘 다 선택해야 저장 버튼이 활성화됩니다.
- 같은 **공정 + 품목** 조합은 중복 등록할 수 없습니다(이미 있으면 오류).
- 등록하려는 공정은 **공정관리**에, 품목은 **품목마스터**에 미리 등록되어 있어야 합니다.
- **표준 택트타임**은 0.01초 이상이어야 합니다.
- **밸런싱 효율**은 0~100% 범위로 입력합니다.
- 수정 시 **공정·품목은 변경할 수 없습니다**(키 항목). 다른 조합이 필요하면 새로 등록하세요.
- UPH·일일 CAPA는 자동 계산 값이므로 직접 입력하지 않습니다.

## 자주 묻는 질문
- **Q.** 택트타임만 넣었는데 UPH가 자동으로 바뀌어요.
  **A.** 정상입니다. UPH는 `3600 ÷ 택트타임(초)`으로 자동 계산됩니다. 택트타임 30초면 UPH는 120입니다.
- **Q.** 작업자 수와 설비 대수를 모두 넣으면 일일 CAPA는 어떻게 계산되나요?
  **A.** 설비 대수가 1 이상이면 **설비 대수**를 배수로 우선 사용하고, 작업자 수는 계산에 쓰이지 않습니다(설비종속).
- **Q.** 일일 CAPA는 가동시간을 몇 시간으로 보나요?
  **A.** 하루 8시간(1교대) 기준입니다. 더 긴 가동시간은 밸런싱 효율이나 별도 계획에서 반영합니다.
- **Q.** 같은 공정에 품목이 여러 개면요?
  **A.** 품목마다 한 건씩 등록합니다. CAPA는 공정+품목 단위로 관리됩니다.
