# 제품생산계획

## 화면 목적
완제품·반제품을 **언제(어느 달) 얼마나 생산할지**를 계획으로 등록·관리하는 화면입니다.
계획은 **월(YYYY-MM) 단위**로 만들며, 한 건이 "이 달에 이 품목을 몇 개 생산한다"를 뜻합니다. 계획을 **확정**한 뒤에는 그 계획을 근거로 **작업지시**를 발행해 실제 생산을 지시합니다.

> 생산계획은 일자가 아니라 **달 단위**입니다. 같은 달·같은 품목이라도 계획은 여러 건 만들 수 있고, 각 건마다 계획번호(PLAN_NO)가 부여됩니다.

## 화면 구성
- **상단 — 도구 버튼**: 새로고침 / 템플릿 다운로드 / 엑셀 업로드 / ERP 인터페이스 / 자동편성 / 계획 추가
- **상단 필터(그리드 위)**: 시작일·종료일(계획월 범위), 검색어, 품목유형, 상태
- **가운데 — 계획 목록(그리드)**: 등록된 생산계획을 한 줄씩 표시. DRAFT 행에는 좌측에 수정·삭제 아이콘이 나타납니다.
- **우측 — 등록/수정 패널**: '계획 추가' 또는 수정 아이콘을 누르면 오른쪽에서 슬라이드로 열립니다.

## 상태 흐름
```
DRAFT(초안) ──확정──▶ CONFIRMED(확정) ──마감──▶ CLOSED(마감)
                 ◀─확정취소─┘
DRAFT에서만 수정·삭제 가능 / CONFIRMED에서만 작업지시 발행 가능
```

## ① 계획 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **(수정/삭제)** | **초안(DRAFT)** 행에만 표시되는 좌측 아이콘입니다. 연필=수정 패널 열기, 휴지통=삭제. 확정 이후 상태에서는 보이지 않습니다(수정·삭제 불가). |
| **계획번호(planNo)** | 계획을 식별하는 고유 번호입니다. 형식은 `PP-YYYYMM-NNN`(예: `PP-202606-001`). 등록 시 자동 채번되며 변경할 수 없습니다. |
| **계획월(planMonth)** | 이 계획이 적용되는 **생산 대상 월**입니다(`YYYY-MM`). 일자는 없으며, 작업지시 발행 시점에 구체적인 계획일을 지정합니다. |
| **작업(actions)** | 상태에 따라 버튼이 달라집니다. DRAFT면 **확정**, CONFIRMED면 **작업지시 발행**·**확정취소** 버튼이 나옵니다. 잔여수량이 0이면 발행 버튼은 비활성화됩니다. |
| **품목유형(itemType)** | `FINISHED`(완제품) 또는 `SEMI_PRODUCT`(반제품)입니다. 무엇을 생산하는 계획인지 구분합니다. |
| **품목코드(partCode)** | 생산 대상 품목의 코드입니다. 품목마스터에서 선택한 값입니다. |
| **품목명(partName)** | 품목코드에 연결된 품목의 이름입니다(품목마스터 기준 표시). |
| **계획수량(planQty)** | 이 달에 생산하려는 **목표 수량**입니다. 작업지시 발행의 상한 기준이 됩니다. |
| **발행수량(orderQty)** | 이 계획에서 **지금까지 작업지시로 발행된 누계 수량**입니다. 발행할 때마다 자동으로 늘어납니다. |
| **발행률(issueRate)** | `발행수량 ÷ 계획수량`을 막대그래프와 %로 표시합니다. 100%면 계획수량을 모두 발행했다는 뜻이며 막대가 녹색으로 바뀝니다. |
| **라인(lineCode)** | 생산을 진행할 기본 생산라인입니다. 작업지시 발행 시 기본값으로 상속되며 변경할 수 있습니다. |
| **고객사(customer)** | 이 계획과 연결된 고객사입니다(수주 대응 등 참고용). |
| **우선순위(priority)** | 계획 처리 우선순위입니다(1~10, 기본 5). **작을수록 먼저** 처리되며, 목록도 우선순위 오름차순으로 정렬됩니다. |
| **상태(status)** | 계획의 진행 단계입니다. **DRAFT(초안)**=수정·삭제 가능, **CONFIRMED(확정)**=수정·삭제 불가·작업지시 발행 가능, **CLOSED(마감)**=종료. 헤더의 물음표(?)에 전이 설명이 있습니다. |

> 상단 필터의 **시작일·종료일**은 계획월 범위 필터입니다. 입력한 날짜의 **월(앞 7자리)**까지만 비교하므로, 같은 달 안의 어느 날을 넣어도 해당 월이 포함됩니다.

## ② 등록/수정 패널 입력 필드

'계획 추가' 또는 수정 아이콘으로 열리는 오른쪽 패널의 입력 항목입니다. **수정 모드에서는 계획월·품목코드·품목유형을 바꿀 수 없습니다**(잠금).

### 기본 정보
| 필드 | 역할 / 의미 |
|------|------|
| **계획월(planMonth)** | 생산 대상 월(`YYYY-MM`)을 선택합니다. 계획번호 채번 기준이 됩니다. 수정 시 잠깁니다. |
| **품목유형(itemType)** | 완제품/반제품을 선택합니다. 품목 검색의 필터로도 쓰입니다. 수정 시 잠깁니다. |
| **품목코드(itemCode)** | 돋보기 버튼으로 품목마스터에서 생산 품목을 선택합니다. 직접 입력은 불가(검색 선택만). 필수 항목입니다. |
| **계획수량(planQty)** | 목표 생산 수량입니다. **1 이상**이어야 저장됩니다(0 이하면 저장 버튼 비활성). |
| **우선순위(priority)** | 1~10 사이의 값입니다(기본 5). 작을수록 우선 처리됩니다. |

### 상세 정보
| 필드 | 역할 / 의미 |
|------|------|
| **고객사(customer)** | 거래처(고객) 목록에서 선택합니다(선택 입력). |
| **라인(lineCode)** | 기본 생산라인을 선택합니다(선택 입력). 작업지시 발행 시 기본값으로 사용됩니다. |
| **비고(remark)** | 자유 메모입니다(선택 입력). |

---

## ③ 작업지시 발행(IssueJobOrder) 입력 필드

목록에서 **CONFIRMED** 계획의 **작업지시 발행** 버튼을 누르면 열리는 모달입니다. 상단에 계획번호·품목·계획수량·발행수량·**잔여수량**이 요약 표시됩니다.

| 필드 | 역할 / 의미 |
|------|------|
| **발행수량(issueQty)** | 이번에 작업지시로 내보낼 수량입니다. **잔여수량(계획수량−발행수량) 이하**여야 하며, 초과하면 경고가 뜨고 발행이 차단됩니다. 필수 항목입니다. |
| **계획일(planDate)** | 작업지시의 구체적인 생산 예정일입니다(선택). 계획은 월 단위지만 작업지시는 일자를 가집니다. |
| **라인(lineCode)** | 작업지시를 진행할 라인입니다. 계획의 라인을 기본값으로 상속하며 변경할 수 있습니다. |
| **우선순위(priority)** | 작업지시 우선순위(1~10)입니다. 계획값을 상속합니다. |
| **BOM 반제품 자동생성(autoCreateChildren)** | 체크하면 이 품목의 BOM 하위 **반제품(SEMI_PRODUCT)** 작업지시도 전 계층 자동 생성합니다. 반제품을 함께 생산해야 할 때 사용합니다. |
| **비고(remark)** | 작업지시 메모입니다(미입력 시 `(계획번호)에서 발행`으로 기록). |

> 발행하면 작업지시(상태 WAITING)가 생성되고, 계획의 **발행수량이 발행수량만큼 자동 증가**합니다. 잔여수량을 모두 발행하면 발행률이 100%가 됩니다.

---

## ④ 자동편성(MPS) 입력 항목

상단 **자동편성** 버튼으로 여는 모달입니다. 수주(고객 주문)를 기반으로 계획을 한꺼번에 DRAFT로 만들어 줍니다.

| 항목 | 역할 / 의미 |
|------|------|
| **대상월(month)** | 생성할 계획의 월입니다(기본값=다음 달). 월을 바꾸면 납기 기간이 그 달 1일~말일로 자동 설정됩니다. |
| **납기 시작일·종료일** | 대상으로 삼을 수주의 납기 범위입니다. |
| **고객(customer)** | 특정 고객의 수주만 대상으로 좁힐 수 있습니다(미선택=전체). |
| **미리보기(preview)** | 조건에 맞는 수주를 표로 조회합니다. 행마다 체크박스가 있어 편성할 항목만 선택할 수 있습니다(주문번호·납기·품목·고객·수주량 표시). |
| **편성 실행** | 선택한 수주 항목을 생산계획 DRAFT로 생성합니다. 실행 시 해당 월의 **기존 DRAFT는 삭제 후 재편성**되므로 확인 모달이 뜹니다. |

---

## 사용 순서
1. **계획 추가**(또는 자동편성/엑셀 업로드)로 계획을 등록합니다 → 상태 **DRAFT**.
2. 내용이 맞으면 목록의 **확정** 버튼을 눌러 **CONFIRMED**로 전이합니다(이후 수정·삭제 불가).
3. CONFIRMED 계획에서 **작업지시 발행** 버튼으로 잔여수량 범위 내에서 작업지시를 발행합니다.
4. 발행할 때마다 발행수량·발행률이 갱신됩니다. 더 처리하지 않을 계획은 마감(CLOSED)합니다.
5. 잘못 확정했다면 **확정취소**로 DRAFT로 되돌린 뒤 수정합니다.

## 입력 규칙 / 검증
- 품목코드는 필수이며, 계획수량은 **1 이상**이어야 저장됩니다.
- **DRAFT 상태만** 수정·삭제할 수 있습니다(CONFIRMED·CLOSED는 불가).
- 작업지시는 **CONFIRMED 상태에서만**, **잔여수량 이하**로만 발행됩니다.
- 우선순위는 1~10 범위입니다.
- 엑셀 업로드는 모든 행이 검증을 통과해야(오류 0건) 업로드 버튼이 활성화됩니다.

## 자주 묻는 질문
- **Q.** 계획을 확정했는데 수량을 잘못 넣었습니다.
  **A.** 아직 작업지시를 발행하지 않았다면 **확정취소**로 DRAFT로 되돌려 수정한 뒤 다시 확정하세요.
- **Q.** 발행 버튼이 비활성화되어 있습니다.
  **A.** 잔여수량(계획수량−발행수량)이 0이거나, 계획이 CONFIRMED 상태가 아닐 때입니다.
- **Q.** 한 계획에서 작업지시를 여러 번 나눠 발행할 수 있나요?
  **A.** 네. 잔여수량이 남아 있는 한 여러 번 나눠 발행할 수 있습니다(발행수량이 누계로 합산).
- **Q.** 자동편성을 실행하면 기존 계획이 사라지나요?
  **A.** 같은 달의 **DRAFT 계획만** 재편성으로 대체됩니다. CONFIRMED·CLOSED 계획은 영향받지 않습니다.
