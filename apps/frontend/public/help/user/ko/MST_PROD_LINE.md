# 생산라인관리

## 화면 목적
공정에서 실제 작업이 일어나는 **물리적 생산라인**을 등록·관리하는 기준정보 화면입니다. 여기 등록된 라인이 작업지시·생산실적·설비 배치·창고 연계의 기준이 됩니다.

## 화면 구성
- **목록(그리드)**: 등록된 생산라인 목록. 상단 검색창에서 라인코드·라인명·공정코드로 검색하고, 헤더의 **새로고침**·**라인 추가** 버튼으로 갱신/등록합니다.
- **각 행의 작업 버튼**: ✏️(수정), 🗑️(삭제).
- **등록/수정 모달**: "라인 추가" 또는 행 ✏️ 클릭 시 열리는 입력 폼. 각 입력 라벨 옆 도움말(?) 아이콘에 항목 설명이 있습니다.

---

## ① 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **라인코드(lineCode)** | MES 내부에서 생산라인을 식별하는 **고유 코드**입니다(예: `P2001`). 작업지시·실적·설비배치가 이 코드로 연결되므로, 한 번 등록하면 변경하지 않는 것이 원칙입니다(수정 모달에서 비활성화됨). |
| **라인명(lineName)** | 현장에서 식별할 생산라인 이름입니다(예: 압착1생산라인). |
| **공정코드(oper)** | 이 라인이 매핑되는 **ERP 공정코드**입니다(예: `#0100`). 어느 공정에 속한 라인인지를 나타냅니다. |
| **라인유형(lineType)** | 조립·포장 등 라인의 성격을 구분하는 분류 값입니다(예: `PACKING`). 목록에서 배지로 표시됩니다. |
| **창고위치(whLoc)** | 이 라인에 연계되는 **기본 창고 로케이션**입니다(예: `LOC002`). 자재·완제품 입출고 위치 연계에 사용됩니다. |
| **ERP코드(erpCode)** | ERP 시스템에서 사용하는 라인 연계 코드입니다. ERP와 라인을 매칭할 때 사용합니다. |
| **비고(remark)** | 생산라인 관리 참고사항(메모)입니다. |
| **사용(useYn)** | 라인 사용 여부입니다. `Y`(초록 점)일 때만 활성 대상이며, 사용 중지는 `N`(회색 점)입니다. |

## 사용 순서
1. 헤더의 **라인 추가** 버튼(또는 행 ✏️)으로 입력 모달을 엽니다.
2. **라인코드**와 **라인명**(필수)을 입력합니다.
3. 필요 항목(공정코드·라인유형·창고위치·ERP코드·비고)을 채웁니다.
4. **추가/수정** 버튼으로 저장합니다.
5. 더 이상 쓰지 않는 라인은 삭제하거나, 이력 보존이 필요하면 수정에서 사용여부를 `N`으로 둡니다.

## 입력 규칙·검증
- **라인코드**와 **라인명**은 필수입니다. 둘 중 하나라도 비어 있으면 저장되지 않습니다.
- **라인코드는 등록 후 변경 불가**입니다(수정 모달에서 입력칸이 잠김). 이미 존재하는 코드로 추가하면 중복 오류가 발생합니다.
- 라인코드는 최대 20자, 라인명 100자, 공정코드 10자, 라인유형 30자, 창고위치·ERP코드 20자, 비고 500자까지 입력할 수 있습니다.
- 검색창은 **라인코드·라인명·공정코드**를 대상으로 합니다.

## 자주 묻는 질문
- **Q.** 라인코드를 잘못 입력했는데 수정이 안 됩니다.
  **A.** 라인코드는 식별 키라 변경할 수 없습니다. 잘못된 행을 삭제하고 새 코드로 다시 등록하세요(작업지시·실적이 연결돼 있으면 운영자와 상의).
- **Q.** 공정코드와 ERP코드는 무엇이 다른가요?
  **A.** 공정코드(oper)는 라인이 속한 ERP 공정을 가리키고, ERP코드는 ERP에서 라인 자체를 가리키는 연계 코드입니다.
- **Q.** 목록에서 라인이 사라졌어요.
  **A.** 검색어가 남아 있거나 사용여부가 `N`인 경우가 많습니다. 검색창을 비우고 새로고침해 보세요.
