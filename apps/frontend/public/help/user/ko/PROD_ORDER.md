# 작업지시 관리

## 화면 목적
완제품·반제품의 **생산 명령(작업지시)**을 만들고, 생산 진행 상태를 관리하는 화면입니다.
작업지시 1건은 "어떤 품목을, 언제(계획일), 몇 개(계획수량) 만들라"는 지시이며, 생성 후 **시작 → 완료**로 진행됩니다. 필요 시 **홀딩(보류)**으로 중단하거나 **취소**할 수 있습니다.

> 작업지시는 생산실적·검사·출하의 출발점입니다. 작업지시가 없으면 그 품목의 실적을 등록할 수 없습니다.

## 화면 구성
- **상단 — 제목·버튼**: 도구보기(AI 채팅), 새로고침, 목록뷰/트리뷰 전환, 작업지시 생성.
- **액션바**: 행을 선택하면 그 작업지시의 번호·상태가 표시되고, 현재 상태에서 가능한 동작 버튼(시작/완료/홀딩/홀딩해제/취소)만 활성화됩니다. 작업지시서 출력 버튼도 여기 있습니다.
- **목록(그리드)**: 작업지시 목록. 상단에 검색·상태·설비·품목유형·계획일 필터가 있습니다. 우측 상단 내보내기 버튼으로 Excel 저장이 가능합니다.
- **우측 슬라이드 패널**: 생성/수정 입력 폼(오른쪽에서 열림).

## 작업지시 계층 구조
완제품 작업지시를 저장하면 두 가지 하위 작업지시가 자동으로 함께 생성됩니다.

```
완제품 작업지시 (ITEM)
├─ 공정별 작업지시 (OPERATION) — 라우팅 공정 수만큼 자동 생성
│     예) 공정 1 · 공정 2 · 공정 3
└─ 반제품 작업지시 (ITEM) — BOM의 반제품(SEMI_PRODUCT) 품목마다 자동 생성
      └─ 반제품의 공정별 작업지시 (OPERATION)
```

- **트리뷰 전환**: 목록 우상단 버튼으로 이 계층 관계를 들여쓰기 트리로 시각화합니다.
- **구분(orderKind)** 컬럼의 `품목` 배지(파란색)는 ITEM, `공정` 배지(초록색)는 OPERATION을 나타냅니다.

## 상태 흐름
작업지시는 아래 5개 상태를 가지며, 정해진 순서로만 전이됩니다.

```
WAITING(대기) ──시작──▶ RUNNING(진행중) ──완료──▶ DONE(완료)
   │                        │
   │                        └──홀딩──┐
   └──홀딩───────────────────────────┤
   │                                 ▼
   │                            HOLD(홀딩) ──홀딩해제──▶ 이전 상태로 복귀
   │
   └──취소──▶ CANCELED(취소)   (HOLD 상태에서도 취소 가능)
```

| 상태 | 의미 |
|------|------|
| **대기(WAITING)** | 생성 직후 기본 상태. 아직 작업을 시작하지 않은 상태입니다. 시작·홀딩·취소가 가능합니다. |
| **진행중(RUNNING)** | 작업을 시작한 상태. 이때 생산실적을 등록합니다. 완료·홀딩이 가능합니다. |
| **홀딩(HOLD)** | 일시 보류 상태. **실적등록·출하가 전부 차단**됩니다. 홀딩해제로 이전 상태(대기/진행중)로 복귀합니다. |
| **완료(DONE)** | 작업이 끝난 상태. 실적이 자동 집계됩니다. 더 이상 수정·상태변경이 불가합니다. |
| **취소(CANCELED)** | 취소된 상태. 실적이 1건이라도 있으면 취소할 수 없습니다. |

## ① 목록 컬럼

| 컬럼 | 역할 / 의미 |
|------|------|
| **작업(수정/삭제)** | 행별 빠른 동작 버튼입니다. 연필=수정 패널 열기, 휴지통=삭제. (진행중 작업지시는 삭제 불가) |
| **계획일(planDate)** | 이 작업지시의 **생산 예정일**입니다. 목록은 기본적으로 계획일 범위로 조회되며, 계획일 미지정 건은 항상 함께 표시됩니다. |
| **우선순위(priority)** | 작업 처리 순서입니다. **1이 가장 높고 10이 가장 낮습니다**(기본 5). 1~3은 강조(빨강 테두리), 7~10은 흐리게 표시되어 급한 지시를 빠르게 식별합니다. 목록은 우선순위 오름차순으로 정렬됩니다. |
| **작업지시번호(orderNo)** | 작업지시를 식별하는 고유 번호입니다(예: `W260519-001`). 트리뷰에서는 들여쓰기와 화살표로 상·하위(완제품-반제품) 관계를 보여줍니다. |
| **구분(orderKind)** | 작업지시 유형 배지입니다. **품목(파란색)=ITEM**은 완제품·반제품 단위 작업지시이고, **공정(초록색)=OPERATION**은 라우팅의 각 공정별 세부 작업지시입니다. `공정 1`, `공정 2` 처럼 공정 순서(routingSeq)도 함께 표시됩니다. |
| **품목코드(partCode)** | 생산 대상 품목의 코드입니다. |
| **품목명(partName)** | 생산 대상 품목의 이름입니다. |
| **품목유형(partType)** | 품목 종류입니다. **완제품(파란색 배지)=FINISHED**, **반제품(보라색 배지)=SEMI_PRODUCT**으로 색 배지로 구분합니다. |
| **라인(lineCode)** | 작업을 수행할 생산 라인입니다(선택값). |
| **공정(processCode)** | 이 작업지시의 **대표 공정**입니다. 품목 라우팅의 첫 공정에서 자동 상속되며, 직접 지정할 수도 있습니다. |
| **설비(equipCode)** | 작업 설비 코드입니다. 생성 시 비워두고 이후 배정할 수 있습니다. |
| **고객PO번호(custPoNo)** | 이 지시가 대응하는 고객 발주(PO) 번호입니다(선택값). |
| **계획수량(planQty)** | 생산하기로 계획한 수량입니다. |
| **실적수량(goodQty)** | 실제 생산된 **양품 수량**입니다. 생산실적이 집계되어 표시됩니다. |
| **진행률(progress)** | 계획수량 대비 양품수량 비율(%)입니다. 막대 그래프로 표시됩니다. |
| **상태(status)** | 현재 작업지시 상태(대기/진행중/홀딩/완료/취소)입니다. 색 배지로 표시됩니다. |

## ② 목록 필터

| 필터 | 역할 |
|------|------|
| **검색** | 작업지시번호·품목코드·품목명으로 텍스트 검색합니다. |
| **상태** | 대기/진행중/홀딩/완료/취소 중 하나를 선택해 필터링합니다. |

| **계획일(시작~종료)** | 계획일 범위로 필터링합니다. 기본값은 오늘입니다. 계획일이 없는 건은 범위와 무관하게 항상 표시됩니다. |

## ③ 생성/수정 패널 입력 항목

| 항목 | 역할 / 의미 |
|------|------|
| **작업지시번호(orderNo)** | 직접 입력하지 않습니다. **저장 시 자동 생성**됩니다(`W+날짜+순번`). |
| **품목명(itemCode)** | 돋보기 버튼으로 품목 검색 모달을 열어 선택합니다. **완제품·반제품만** 선택할 수 있습니다. 선택하면 라우팅 정보가 자동 조회됩니다. |
| **계획수량(planQty)** | 생산할 수량입니다. **필수**이며 1 이상이어야 합니다. |
| **계획일(planDate)** | 생산 예정일입니다. **필수**입니다. |
| **우선순위(priority)** | 1~10 사이 값(기본 5). 작을수록 먼저 처리합니다. |
| **고객PO번호(custPoNo)** | 대응 고객 PO 번호입니다(선택값, 예: `PO-2026-0001`). |
| **라우팅 정보** | 입력값이 아니라 **자동 표시**입니다. 선택한 품목의 라우팅 코드·이름과 공정 순서(화살표)를 보여줍니다. 라우팅이 없으면 "라우팅 미등록 품목입니다" 안내가 나옵니다. **라우팅이 없으면 저장할 수 없습니다.** |
| **라인(lineCode)** | 작업 라인을 선택합니다(선택값). |
| **공정(processCode)** | 대표 공정을 선택합니다. 미지정 시 라우팅 첫 공정이 자동 적용됩니다. 공정을 바꾸면 설비 선택이 초기화됩니다. |
| **설비(equipCode)** | 선택한 공정에 등록된 설비 목록에서 버튼 형태로 고릅니다. **미지정**을 선택하면 설비 없이 저장됩니다. 해당 공정에 설비가 없으면 "설비가 없습니다" 안내가 표시됩니다. |
| **비고(remark)** | 관리용 메모입니다. |

> **완제품 저장 시 자동 생성**: 저장 즉시 라우팅 공정 수만큼 공정별(OPERATION) 작업지시와 BOM의 반제품별 ITEM 작업지시가 함께 자동 생성됩니다. 별도의 체크박스 조작 없이 항상 자동입니다.

## ⑤ 작업지시서 출력 (A4 양식)

액션바의 **작업지시서 출력** 버튼을 누르면 A4 크기 출력 양식이 모달로 열립니다.

| 항목 | 설명 |
|------|------|
| **QR코드** | 해당 작업지시번호로 이 화면에 바로 접근하는 딥링크. 모바일 카메라로 스캔하면 자동 조회됩니다. |
| **작업지시 정보** | 작업지시번호, 상태, 품목코드, 품목명, 계획수량(단위 포함), 계획일, 라인, 고객PO, 비고 |
| **자재요청 내역** | BOM에서 **원자재(RAW_MATERIAL)** 품목만 추려 표시합니다. No·품목코드·품목명·규격·공정·BOM수량·소요량(BOM수량 × 계획수량)·단위·확인란 |
| **서명란** | 작성자 / 창고확인 / 수령자 서명 공간 |
| **인쇄 버튼** | 모달 우상단의 인쇄 버튼으로 프린터 출력합니다. 프린트 시 모달 버튼 영역은 자동으로 숨겨집니다. |

## 사용 순서
1. **[작업지시 생성]**을 눌러 우측 패널을 엽니다.
2. **품목**(돋보기), **계획수량**, **계획일**을 입력합니다(필수 3종). 필요 시 라인·공정·설비·우선순위·고객PO를 지정합니다.
3. **저장**을 누르면 공정별 OPERATION 작업지시와 BOM 반제품 ITEM 작업지시가 자동 생성됩니다.
4. 목록에서 작업지시 행을 클릭해 선택하고, **시작** 버튼으로 작업을 착수합니다.
5. 생산이 진행되면 [생산실적]에서 실적을 등록합니다.
6. 작업이 끝나면 **완료** 버튼으로 종료합니다(실적 자동 집계).

**작업지시서가 필요한 경우**: 목록에서 행을 선택 → 액션바 **작업지시서 출력** → 모달에서 인쇄.

## 입력 규칙 / 검증
- 품목·계획수량·계획일이 모두 있어야 저장됩니다(계획수량은 1 이상).
- **품목에 라우팅이 등록되어 있어야** 저장됩니다. 라우팅이 없으면 저장 시 오류가 발생합니다.
- 작업지시번호는 직접 입력하지 않으며 저장 시 자동 채번됩니다.
- 상태는 액션 버튼으로만 바꿀 수 있고, 임의로 건너뛸 수 없습니다(예: 대기에서 바로 완료 불가).
- 완료·취소된 작업지시는 수정할 수 없습니다.
- 진행중 작업지시는 삭제할 수 없습니다.
- 취소는 실적이 한 건도 없을 때만 가능합니다.

## 자주 묻는 질문
- **Q.** 작업지시번호를 직접 정할 수 있나요?
  **A.** 아니요. 저장 시 `W+YYMMDD+순번` 형식으로 자동 생성됩니다.
- **Q.** 저장했더니 작업지시가 여러 개 생겼어요.
  **A.** 정상입니다. 완제품 1건을 저장하면 라우팅 공정 수만큼 공정별(OPERATION) 작업지시와, BOM의 반제품마다 ITEM 작업지시가 함께 자동 생성됩니다. 트리뷰에서 계층을 확인할 수 있습니다.
- **Q.** 구분 배지에 "공정 1", "공정 2" 같은 것이 보여요.
  **A.** OPERATION 타입 작업지시(라우팅의 각 공정별 세부 작업)입니다. 공정 순서 번호가 함께 표시됩니다.
- **Q.** 품목을 선택했는데 라우팅 정보가 안 나와요.
  **A.** 해당 품목에 라우팅이 등록되지 않았습니다. [라우팅관리]에서 먼저 등록하세요. 라우팅이 없으면 작업지시를 생성할 수 없습니다.
- **Q.** 홀딩과 취소는 무엇이 다른가요?
  **A.** 홀딩은 **잠시 멈춤**(나중에 해제해 재개)이고, 취소는 **종료 처리**(되돌릴 수 없음)입니다.
- **Q.** 완료를 눌렀는데 계획수량을 다 못 채웠어요.
  **A.** 잔량이 있어도 완료됩니다. 완료 시점까지의 실적이 집계됩니다.
- **Q.** 목록에 작업지시가 안 보여요.
  **A.** 상단 계획일 필터 범위를 확인하세요. 기본은 오늘 날짜입니다. 계획일을 넓게 잡으면 더 많이 표시됩니다.
- **Q.** QR코드를 스캔하면 어떻게 되나요?
  **A.** 모바일 또는 바코드 리더로 스캔하면 이 화면이 열리면서 해당 작업지시가 자동 검색됩니다.
- **Q.** 작업지시서 출력 모달에 자재가 안 나와요.
  **A.** BOM에 원자재(RAW_MATERIAL) 품목이 등록되지 않았거나, 반제품·소모품만 있는 경우입니다. BOM에 원자재를 등록하면 표시됩니다.
