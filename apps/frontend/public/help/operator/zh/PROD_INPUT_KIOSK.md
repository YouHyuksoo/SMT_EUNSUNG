# 实绩输入( Kiosk) — 操作指南

## 系统目的·角色
在现场设备旁终端上登记生产实绩的综合界面。在一个界面中处理设备·工单·操作员选择，准备检查(互锁)，材料/耗材投入，生产数量·不良输入，自检(首件/中件/末件)。以全屏(chromeless, `view=work`)模式运行，状态通过`kioskStore`(zustand persist)保存。

## 界面组成 (4区域)
- 上方2行头: ①设备ID·条码·设备日常检查 ②工单·操作员·生产实绩·操作员设备检查
- 左侧: BOM材料列表 + 消耗性设备部件
- 中央: 作业指导书 + 下方3块(自检 | 不良 | 实绩输入)
- 右侧: 良品条件 + 操作记录

## 互锁(准备检查)逻辑
- 设备日常检查·操作员设备检查等必需检查完成前，实绩保存被阻止(`isAllInterlockDone`)。
- 未选择工单·操作员时也无法保存。

## 自检触发器
| 时机 | 条件 |
|------|------|
| 首件 FIRST | `savedResultCount === 0` 后首次保存 → 自动打开 |
| 中件 MID | 面板按钮点击，或**进行率60%时阻止**(未执行则无法继续) |
| 末件 LAST | 面板按钮点击 |

## 材料/耗材2种区分
- **BOM投入材料**: 左侧材料列表 = 基于工单产品BOM的投入对象(材料扫描)。
- **消耗性设备部件**: 下方"消耗性设备部件"部分 = 安装在设备上的消耗品，与产品BOM无关。(两种区分显示，避免混淆)

## 实绩/不良保存
| 输入 | 含义 / 处理 |
|------|------|
| 良品数量(goodQty) | 良品生产数量。 |
| 不良数量(defectQty) | 不良数量。**有等级别详细(pendingDefects)合计时优先使用该值**(防止重复统计)。 |
| 生产序列号(prdUid) | 产品序列号(如有)。 |
| 不良等级 | 致命/重/轻等级别详细与生产实绩**在同一事务**中保存(解决了单独defect-logs双重计数缺陷)。 |

- 实绩保存时更新工单进行率·生产实绩，并反映到WIP/产品库存流程(取决于设备分配·工序)。

## 权限
现场操作员(实绩登记)。终端以Kiosk模式运行。

## 故障排除
| 症状 | 原因 | 措施 |
|------|------|------|
| 实绩无法保存 | 互锁未完成(设备/操作员检查)或工单·操作员未选择 | 确认准备检查·选择完成 |
| 60%时进行受阻 | 中件自检未执行 | 执行中件自检 |
| 怀疑不良数量重复统计 | 等级别详细 + 直接输入混用 | 等级别合计优先应用(设计上无重复统计) |
| 材料扫描不匹配 | 非BOM投入材料/与耗材混淆 | 区分左侧BOM材料与下方消耗性设备部件 |
| 作业指导书不显示 | 物料/工序作业指导书未登记或文件路径问题 | 登记作业指导书·确认路径 |

## 数据·关联
- 状态: `kioskStore`(zustand persist)
- 关联: 工单, 物料/BOM, 作业指导书, 设备/工序, 自检, 生产实绩·不良, 库存(WIP/产品)
- 范围: `COMPANY='40'`, `PLANT_CD='1000'`
