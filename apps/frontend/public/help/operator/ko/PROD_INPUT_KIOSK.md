# 실적입력(키오스크) — 운영 가이드

## 시스템 목적·역할
현장 설비 옆 단말에서 생산실적을 등록하는 통합 화면입니다. 설비·작업지시·작업자 선택, 준비 점검(인터록), 자재/소모품 투입, 생산수량·불량 입력, 자주검사(초물/중물/종물)를 한 화면에서 처리합니다. 전체화면(chromeless, `view=work`) 모드로 동작하며 상태는 `kioskStore`(zustand persist)로 보존됩니다.

## 화면 구성 (4영역)
- 상단 헤더 2단: ①설비ID·바코드·설비일일검사 ②작업지시·작업자·생산실적·작업자설비검사
- 좌측: BOM 자재리스트 + 소모성 설비부품
- 중앙: 작업지도서 + 하단 3칸(자주검사 | 불량 | 실적입력)
- 우측: 양품조건 + 작업이력

## 인터록(준비 점검) 로직
- 설비 일일검사·작업자 설비검사 등 필수 점검 완료 전에는 실적 저장이 차단됩니다(`isAllInterlockDone`).
- 작업지시·작업자 미선택 시에도 저장 불가.

## 자주검사 트리거
| 시점 | 조건 |
|------|------|
| 초물 FIRST | `savedResultCount === 0` 이후 첫 저장 → 자동 오픈 |
| 중물 MID | 패널 버튼 클릭, 또는 **진행률 60%에서 차단**(미수행 시 진행 불가) |
| 종물 LAST | 패널 버튼 클릭 |

## 자재/소모품 2종 구분
- **BOM 투입 자재**: 좌측 자재리스트 = 작업지시 제품의 BOM 기준 투입 대상(자재 스캔).
- **소모성 설비부품**: 하단 "소모성 설비부품" 섹션 = 설비에 장착되는 소모품으로 제품 BOM과 별개. (두 종류를 혼동하지 않도록 분리 표시)

## 실적/불량 저장
| 입력 | 의미 / 처리 |
|------|------|
| 양품수량(goodQty) | 양품 생산수량. |
| 불량수량(defectQty) | 불량수량. **등급별 상세(pendingDefects) 합계가 있으면 그 값 우선** 적용(이중 집계 방지). |
| 생산 시리얼(prdUid) | 생산품 시리얼(있을 때). |
| 불량 등급 | 치명/중/경 등급별 상세를 생산실적과 **같은 트랜잭션**으로 저장(별도 defect-logs 이중 카운트 결함 해소됨). |

- 실적 저장 시 작업지시 진행률·생산실적이 갱신되고, WIP/제품 재고 흐름에 반영됩니다(설비 배정·공정에 따름).

## 권한
현장 작업자(실적 등록). 단말은 키오스크 모드로 운용.

## 문제 해결 (트러블슈팅)
| 증상 | 원인 | 조치 |
|------|------|------|
| 실적 저장 불가 | 인터록 미완(설비/작업자 검사) 또는 작업지시·작업자 미선택 | 준비 점검·선택 완료 확인 |
| 60%에서 진행 차단 | 중물 자주검사 미수행 | 중물 자주검사 수행 |
| 불량수량 이중 집계 의심 | 등급별 상세 + 직접 입력 혼용 | 등급별 합계가 우선 적용됨(설계상 이중 집계 없음) |
| 자재 스캔이 안 맞음 | BOM 투입 자재 아님/소모품 혼동 | 좌측 BOM 자재 vs 하단 소모성 설비부품 구분 |
| 작업지도서 안 보임 | 품목/공정 작업지도서 미등록 또는 파일 경로 문제 | 작업지도서 등록·경로 확인 |

## 데이터·연계
- 상태: `kioskStore`(zustand persist)
- 연계: 작업지시, 품목/BOM, 작업지도서, 설비/공정, 자주검사, 생산실적·불량, 재고(WIP/제품)
- 스코프: `COMPANY='40'`, `PLANT_CD='1000'`
