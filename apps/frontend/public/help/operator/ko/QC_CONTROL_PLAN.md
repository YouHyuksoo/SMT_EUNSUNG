# 관리계획서 — 운영 가이드

## 시스템 목적·역할
품목/공정별 관리 항목, 검사 방법, 주기, 반응계획을 관리합니다. 운영자는 이 화면의 데이터가 기준정보, 실적, 재고, 검사, 출하 추적성과 일치하는지 확인합니다.

## 데이터 구조
```
CONTROL_PLANS
  -> 화면별 API/서비스 조인 테이블
스코프: COMPANY='40', PLANT_CD='1000'
```

## 사전 설정
- 품목, 거래처, 라인, 공정, 설비, 작업자, 검사 기준, 출하 기준정보가 먼저 정합해야 합니다.
- 상태/판정/사유 코드는 공통코드 또는 전용 기준정보에서 관리합니다.

## 운영 절차
1. 조회 조건과 회사/공장 스코프를 확인합니다.
2. 대상 행의 상태, 수량, 바코드/문서번호를 확인합니다.
3. 처리 전후 주요 테이블의 상태와 수량 변화를 대조합니다.
4. 오류가 있으면 화면 API와 CONTROL_PLANS 기준 데이터를 함께 확인합니다.

## 권한
- 일반 사용자는 조회와 현장 입력을 수행합니다.
- 관리자만 승인, 취소, 삭제, 상태 되돌림, 기준 확정 작업을 수행합니다.

## 문제 해결
| 증상 | 원인 | 조치 |
|------|------|------|
| 대상이 조회되지 않음 | 기간, 상태, 권한, 회사/공장 스코프 불일치 | 조건 초기화 후 문서번호나 바코드로 재조회합니다. |
| 저장 또는 실행 실패 | 필수값 누락 또는 현재 상태에서 허용되지 않는 처리 | 대상 상태와 필수 입력값을 확인합니다. |
| 수량 불일치 | 집계 단위와 시리얼/상세 단위 혼동 | 집계 테이블과 상세 테이블을 같은 기준으로 대조합니다. |
