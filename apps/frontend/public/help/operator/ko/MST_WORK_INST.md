# 작업지도서 관리 — 운영 가이드

## 시스템 목적·역할
품목별·공정별로 현장 작업자가 참조할 **작업 지침서(Work Instruction)**를 문서 단위로 관리합니다. 품목마스터의 품목코드를 기준으로 연결되며, 동일 품목이라도 공정·리비전에 따라 여러 버전의 문서를 동시에 운영할 수 있습니다. 생산 현장에서는 이 문서를 기준으로 작업 방법·순서·주의사항을 확인합니다.

## 데이터 구조
```
ITEM_MASTERS.ITEM_CODE
        │ (참조·연계 — 논리적 FK, DB 제약 없음)
        ▼
WORK_INSTRUCTIONS (COMPOSITE PK: ITEM_CODE + PROCESS_CODE + REVISION)
        │
        ├── 기본정보: TITLE, CONTENT(CLOB), IMAGE_URL, USE_YN
        ├── 멀티테넌시: COMPANY, PLANT_CD
        └── 이력: CREATED_AT, CREATED_BY, UPDATED_AT, UPDATED_BY
```

## ① 작업지도서 — WORK_INSTRUCTIONS (전체 컬럼)

| 화면 항목 | DB 컬럼 | 역할 / 의미 · 운영 포인트 |
|------|------|------|
| 품목코드 | `ITEM_CODE` | **PK (1/3)**. 작업지도서의 대상 품목. `ITEM_MASTERS.ITEM_CODE`와 논리적 연결(DB FK 제약 없음). 등록 후 변경 불가. |
| 공정코드 | `PROCESS_CODE` | **PK (2/3)**. 해당 품목 내 공정. 등록 후 변경 불가. |
| 리비전 | `REVISION` | **PK (3/3)**. 문서 개정 버전. 기본값 `A`. 등록 후 변경 불가. 신규 개정 시 새 행으로 등록. |
| 문서제목 | `TITLE` | 작업지도서 제목. 목록·미리보기에 표시. 현장 식별용. |
| 내용 | `CONTENT` | **CLOB 타입**. 작업 순서·주의사항 등 본문. 긴 텍스트 저장 가능. |
| 첨부파일 URL | `IMAGE_URL` | 업로드된 파일 경로 또는 외부 URL. `varchar2(500)`. |
| 사용여부 | `USE_YN` | `Y` = 활성(목록 표시), `N` = 비활성(소프트 삭제 개념). 기본값 `Y`. |
| 멀티테넌시 | `COMPANY`, `PLANT_CD` | 회사코드(`40`) / 플랜트코드(`1000`) 스코프. 모든 쿼리에 자동 적용. |
| 생성자 | `CREATED_BY` | 최초 등록 사용자. |
| 수정자 | `UPDATED_BY` | 최종 수정 사용자. |
| 생성일시 | `CREATED_AT` | 최초 등록 시각(자동). |
| 최종수정일 | `UPDATED_AT` | 최종 수정 시각(자동). 목록에 표시됨. |

## 파일 업로드 구조

- **업로드 엔드포인트**: `POST /master/work-instructions/upload` (multipart/form-data)
- **저장 경로**: 서버 `./uploads/work-instructions/` 디렉토리 아래에 타임스탬프 기반 파일명으로 저장
- **허용 형식**: 이미지(`image/*`), PDF, Office 문서(`.doc/.docx/.xls/.xlsx`), 텍스트(`.txt`)
- **최대 크기**: 10MB
- **URL 직접 입력**: 업로드 대신 외부 URL도 입력 가능(이미지·PDF 등)
- **반환값**: `{ url, originalName, size }` — url은 화면에서 `IMAGE_URL`에 저장됨

> `IMAGE_URL`이 업로드된 파일 경로인지 외부 URL인지는 자동 판별되지 않습니다. 운영 시 경로 정합성에 주의하세요.

## 리비전 관리

- 동일 `ITEM_CODE + PROCESS_CODE`에 여러 `REVISION` 행을 둘 수 있습니다.
- 리비전은 순차적으로 운영하되 시스템이 자동 증가시키지는 않습니다. 운영자가 명시적으로 지정합니다.
- 모든 조회는 `REVISION DESC` 정렬이므로 최신 리비전이 먼저 보입니다.
- 불필요한 구 리비전은 `USE_YN='N'`으로 비활성 처리할 수 있습니다(실제 DELETE는 물리 삭제).

---

## 운영 절차
1. **품목마스터**에서 작업지도서가 필요한 품목을 확인합니다.
2. **작업지도서 관리** 화면에서 해당 품목코드·공정코드로 문서를 등록합니다.
3. 첨부파일(작업표준 이미지/PDF)을 업로드하고 본문에 작업 순서를 작성합니다.
4. 문서 개정이 필요하면 기존 문서와 동일한 PK에 새 리비전(예: B)으로 추가 등록합니다.
5. 현장에서 문서 참조 시 최신 리비전이 우선 표시됩니다.

## 권한
마스터 데이터 관리 권한이 있는 사용자(기준정보 관리자). 일반 사용자는 조회만 가능합니다(화면 접근 권한 정책에 따름).

## 문제 해결 (트러블슈팅)

| 증상 | 원인 | 조치 |
|------|------|------|
| 저장 버튼 비활성화됨 | 품목코드·공정코드·제목 중 빈 값 있음 | 필수 필드를 모두 채움 |
| "중복 문서" 오류 | 동일 PK(품목+공정+리비전)가 이미 존재 | 리비전을 변경하거나 기존 문서 수정 |
| 파일 업로드 실패 | 10MB 초과 또는 지원하지 않는 형식 | 파일 크기·형식 확인 |
| 미리보기에 이미지 안 보임 | `IMAGE_URL` 경로가 올바르지 않거나 파일 삭제됨 | URL/파일 경로 확인 |
| 이미지인데 PDF로 표시됨 | URL 확장자가 이미지 형식이 아님 | URL이 `.jpg/.png/.gif` 등으로 끝나는지 확인 |

## 데이터·연계
- **테이블**: `WORK_INSTRUCTIONS`
- **연계**: `ITEM_MASTERS.ITEM_CODE` (논리적 참조), 파일 업로드 스토리지
- **스코프**: `COMPANY='40'`, `PLANT_CD='1000'`
- **파일 저장**: 서버 로컬 `./uploads/work-instructions/` (별도 파일 스토리지 연동 필요 시 확장 가능)
