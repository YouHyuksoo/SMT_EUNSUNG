# Continuity Inspection — Operator Guide

## System Purpose & Role
This screen performs **continuity inspection** for finished goods (FG). Select a tester, browse job orders, and issue/inspect FG barcodes. FG labels are generated together with PASS/FAIL judgment. Consumable parts must be mounted before inspection.

```
FG Issue → Tester Connection → Continuity Inspect(PASS/FAIL) → FG_LABELS (ISSUED) → Visual Inspect
```

## Data Structure
```
JOB_ORDERS (PK: ORDER_NO)
   ├─ PROD_RESULTS → FG_LABELS (barcode issuance)
   └─ CONSUMABLES (consumable parts mapping)

FG_LABELS (PK: FG_BARCODE)
   State transition: ISSUED → VISUAL_PASS/FAIL → PACKED → SHIPPED
   Continuity PASS creates ISSUED status → transitions to VISUAL_PASS via visual inspect

INSPECT_RESULTS (PK: RESULT_NO)
   Continuity inspection results (INSPECT_TYPE='CONTINUITY')
```

## Screen Layout
Left 4 / Right 8 column grid:

### Left Panel (4/12)
- **Tester Selector**: `GET /equipment/equips/type/TESTER` — persists in localStorage
- **Job Order List**: `GET /quality/continuity-inspect/job-orders`
  - Order no·item code·item name·plan qty/good/defect
  - Status badge (`JOB_ORDER_STATUS` common code)
  - Search: order no·item code·item name (300ms debounce)
- **Consumable Panel** (`ConsumablePanel`): Parts mapped to selected order+tester combo
  - Scan consumable UID barcode → mount
  - Inspection blocked if not all mounted

### Right Panel (8/12)
- `InspectPanel` — inspection execution interface
- **FG Barcode History** DataGrid: `GET /quality/continuity-inspect/fg-labels/{orderNo}`
  - FG barcode·issue time·status·reprint count·inspectPassYn
  - Label recovery: re-inspect(FAIL only) / reprint / void
- **PASS/FAIL buttons** (large toggle)
- **Scan mode** (configurable): scan product barcode + circuit label before PASS

## Inspection Flow

### ① Select Tester
`GET /equipment/equips/type/TESTER`
- All inspection blocked if no tester selected
- Selection persists in `localStorage` (`hanes:inspection:equip:CONTINUITY`)

### ② Select Job Order
`GET /quality/continuity-inspect/job-orders?finishedOnly=true`
- Only finished goods orders shown (`finishedOnly=true`)
- Consumable mount status checked on selection

### ③ Mount Consumable Parts
`POST /production/job-orders/{orderNo}/consumables/scan { conUid, equipCode }`
- Scan consumable UID barcode → auto-mount
- Unmount: `DELETE /production/job-orders/{orderNo}/consumables/{mountedConUid}`
- Inspection blocked if consumables not mounted

### ④ PASS/FAIL Inspection
`POST /quality/continuity-inspect/inspect { orderNo, itemCode, equipCode, passYn, ... }`
- **PASS**: FG barcode auto-issued + FG_LABELS created with `ISSUED` status
- **FAIL**: `FailModal` → defect code(`CONTINUITY_DEFECT`) + detail reason → recorded in `INSPECT_RESULTS` (no FG_LABELS)
- Scan mode (optional): product barcode + circuit label (`CIRCUIT_LABEL`) must be scanned before PASS

### ⑤ Label Actions
- **Re-inspect**: `POST /quality/continuity-inspect/re-inspect/{barcode}` — FAIL→PASS conversion
- **Reprint**: `POST /quality/continuity-inspect/reprint/{barcode}` — increment reprint count
- **Void**: `POST /quality/continuity-inspect/void/{barcode}` — FG_LABELS → `VOIDED`

## Interlock (Inspection Blocking Conditions)
All conditions must be met for PASS/FAIL buttons to activate:

| Condition | Message | Resolution |
|------|--------|-----------|
| Tester selected | Select a tester first | Choose from tester dropdown |
| Consumables mounted | N consumable(s) not mounted | Scan consumable UIDs to mount |
| Barcode scanned (scan mode) | Scan the barcode first | Scan product barcode |
| Circuit label scanned (scan mode PASS) | Scan circuit label for PASS | Scan CIRCUIT_LABEL |

## Consumable Part Status

| Indicator | Meaning | Action |
|------|---------|--------|
| Green border + CheckCircle | Mounted OK | Inspection can proceed |
| Orange border + AlertTriangle | Warning count reached | Consider replacement |
| Red border + AlertCircle | Expected life exceeded | Replace immediately |
| Red border + AlertCircle (unmounted) | Not mounted | Scan UID to mount |

## Prerequisites (Master·Common Code)
- Common codes: `CONTINUITY_DEFECT` (continuity defect codes), `JOB_ORDER_STATUS`, `TESTER` (equipment type)
- Equipment master: `EQUIPMENTS.equipType='TESTER'` must be registered
- Consumable mapping: Pre-mapped to job order item + tester combination
- FG_LABELS: `SEQ_FG_BARCODE` sequence required

## Permissions
Quality inspector (continuity inspection execution/label management). Administrators can modify/delete inspection history.

## Troubleshooting
| Symptom | Cause | Action |
|------|------|------|
| Tester list empty | No TESTER type equipment in `EQUIPMENTS` | Register TESTER equipment in master |
| Job order list empty | No finished goods orders | Check production plan and orders |
| PASS button disabled | Consumables not mounted or equipment not selected | Check interlock conditions |
| Consumable scan fails | Invalid UID or not mapped | Verify UID and mapping |
| FG barcode not issued | `SEQ_FG_BARCODE` sequence issue | Check sequence status |
| Reprint fails to print | Printer connection or bwip-js error | Check printer and browser print settings |
| Void button disabled | Already PACKED/SHIPPED/VOIDED | Check current status |
| Re-inspect button disabled | FG is not FAIL status | PASS status doesn't need re-inspect |
